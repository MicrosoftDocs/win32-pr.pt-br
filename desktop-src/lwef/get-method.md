## <a name="remarks"></a>Comentários

Se você carregar um caractere usando o protocolo HTTP (um . Arquivo ACF), você deve usar o **método Get** para recuperar dados de animação antes de poder reproduzir a animação. Você não usará esse método se tiver carregado o caractere usando o protocolo UNC (um . Arquivo ACS). Você também não poderá recuperar dados HTTP para um caractere **usando Get** se tiver carregado esse caractere usando o protocolo UNC (. Arquivo de caractere aCS).

Se você declarar uma referência de objeto e defini-la para esse método, ela retornará um [**objeto Request.**](/windows/desktop/lwef/the-request-object) Se a animação associada não for  carregada, o servidor define a propriedade [**Status**](status-property.md) do objeto Request como "failed" com um número de erro apropriado. Você pode usar o [**evento RequestComplete**](requestcomplete-event.md) para verificar o status e determinar qual ação deve ser tomada.

Os dados de animação ou som recuperados com **o método Get** são armazenados no cache do navegador. As chamadas subsequentes verificarão o cache e, se os dados de animação já estão lá, o controle carrega os dados diretamente do cache. Depois de carregados, a animação ou os dados de som podem ser reproduzir com os métodos [**Reproduzir**](play-method.md) [**ou**](speak-method.md) Falar.
