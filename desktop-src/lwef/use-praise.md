# <a name="use-praise"></a>Usar elogios

\[O Microsoft Agent foi preterido a partir do Windows 7 e pode não estar disponível nas versões subsequentes do Windows.\]

Os seres humanos respondem melhor ao elogios do que críticas, mesmo quando o elogio pode não ser garantido. Embora a maioria das interfaces de software se esforçam para evitar comentários avaliaçãos, o elogio é demonstrável mais eficaz do que não fornecer nenhuma avaliação. Além disso, muitas interfaces criadas como neutras geralmente são percebidas como críticas pelos usuários, pois raramente recebem comentários positivos quando as coisas estão funcionando normalmente e mensagens de erro quando as coisas dão errado. Da mesma forma, palavras que dizem ao usuário que há uma maneira melhor de executar uma tarefa implicitamente criticizes o usuário.

Como os caracteres criam um contexto social, o uso cuidadoso de elogios e críticas é ainda mais importante do que em uma interface de usuário tradicional. Embora poucas pessoas desfrutem do comportamento sycophantic, os limites do uso de elogios de liberal ainda devem ser demonstrados. Os elogios são especialmente eficazes em situações em que os usuários estão menos confiantes em relação ao desempenho de uma tarefa. Por outro lado, críticas deve ser usado com moderação. Mesmo quando você acredita que críticas é apropriado, lembre-se de que os humanos têm uma tendência para descartá-lo ou redirecioná-lo para sua fonte.

No entanto, evite o autoelogio, a menos que seja uma parte bem-humorada da personalidade dos projetos de caracteres. Tendemos a avaliar conceit com ceticismo. Se você quiser direcionar approbation para um caractere, faça com que ele seja direcionado de outra fonte, como outro caractere ou explicação descritiva.
