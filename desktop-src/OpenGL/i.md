# <a name="i-opengl"></a>I (OpenGL)

<dl> <dt>

<span id="opengl_image"></span><span id="OPENGL_IMAGE"></span>**imagem**
</dt> <dd>

Uma matriz retangular de pixels, seja na memória do cliente ou no framebuffer.

</dd> <dt>

<span id="opengl_image_primitive"></span><span id="OPENGL_IMAGE_PRIMITIVE"></span>**primitiva de imagem** 
</dt> <dd>

Um bitmap ou uma imagem.

</dd> <dt>

<span id="opengl_immediate_mode"></span><span id="OPENGL_IMMEDIATE_MODE"></span>**modo imediato**
</dt> <dd>

Modo no qual um comando OpenGL é chamado diretamente, em vez de uma lista de exibição. Não existe nenhum bit de modo imediato; o modo no modo imediato refere-se ao uso do OpenGL, em vez de um bit específico de estado OpenGL.

</dd> <dt>

<span id="opengl_index"></span><span id="OPENGL_INDEX"></span>**index**
</dt> <dd>

Um único valor que é interpretado como um valor absoluto, em vez de um valor normalizado em um intervalo especificado (como é um componente). Os índices de cores são os nomes das cores, que são desreferenciadas pelo hardware de vídeo usando o mapa de cores. Normalmente, os índices são mascarados, em vez de clamped, quando estão fora do intervalo. Por exemplo, o índice 0xf7 é mascarado para 0x7 quando gravado em um buffer de 4 bits (cor ou estêncil). Os índices de cor e os índices de estêncil são sempre tratados como índices, nunca como componentes.

</dd> <dt>

<span id="opengl_iris_gl"></span><span id="OPENGL_IRIS_GL"></span>**ÍRIS GL**
</dt> <dd>

Biblioteca gráfica proprietária do Silicon Graphics, desenvolvida de 1982 até 1992. O OpenGL foi projetado com o íris GL como um ponto de partida.

</dd> </dl>
