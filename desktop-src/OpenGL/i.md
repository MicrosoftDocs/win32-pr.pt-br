# <a name="i-opengl"></a>I (OpenGL)

<dl> <dt>

<span id="opengl_image"></span><span id="OPENGL_IMAGE"></span>**Imagem**
</dt> <dd>

Uma matriz retangular de pixels, na memória do cliente ou no framebuffer.

</dd> <dt>

<span id="opengl_image_primitive"></span><span id="OPENGL_IMAGE_PRIMITIVE"></span>**primitivo de imagem** 
</dt> <dd>

Um bitmap ou uma imagem.

</dd> <dt>

<span id="opengl_immediate_mode"></span><span id="OPENGL_IMMEDIATE_MODE"></span>**modo imediato**
</dt> <dd>

Modo no qual um comando OpenGL é chamado diretamente, em vez de em uma lista de exibição. Não existe nenhum bit de modo imediato; o modo no modo imediato refere-se ao uso do OpenGL, em vez de a um bit específico de estado OpenGL.

</dd> <dt>

<span id="opengl_index"></span><span id="OPENGL_INDEX"></span>**Índice**
</dt> <dd>

Um único valor que é interpretado como um valor absoluto, em vez de como um valor normalizado em um intervalo especificado (como é um componente). Índices de cores são os nomes de cores, que são desreferenciados pelo hardware de exibição usando o mapa de cores. Os índices normalmente são mascarados, em vez de fixados, quando estão fora do intervalo. Por exemplo, a 0xf7 é mascarada para 0x7 quando gravado em um buffer de 4 bits (cor ou estêncil). Índices de cores e índices de estêncil são sempre tratados como índices, nunca como componentes.

</dd> <dt>

<span id="opengl_iris_gl"></span><span id="OPENGL_IRIS_GL"></span>**IRIS GL**
</dt> <dd>

Biblioteca gráfica proprietária da Silicon Graphics, desenvolvida de 1982 a 1992. O OpenGL foi projetado com IRIS GL como um ponto de partida.

</dd> </dl>
