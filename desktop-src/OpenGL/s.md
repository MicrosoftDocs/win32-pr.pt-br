# <a name="s-opengl"></a>S (OpenGL)

[A](a.md) [B](b.md) [C](c.md) [D](d.md) [E](e.md) [F](f.md) [G](g.md) [H](h.md) [i](i.md) [J K](jk.md) [L](l.md) [M](m.md) [N](n.md) [O](o.md) [P](p.md) [Q](q.md) [R](r.md) S [T](t.md) [U V](u-v.md) [W](w.md) [X Y Z](x-y-z.md)

<dl> <dt>

<span id="opengl_server"></span><span id="OPENGL_SERVER"></span>**servidor**
</dt> <dd>

O computador no qual os comandos OpenGL são executados. Isso pode ser diferente do computador do qual os comandos são emitidos. Consulte também cliente.

</dd> <dt>

<span id="opengl_shading"></span><span id="OPENGL_SHADING"></span>**sombreamento**
</dt> <dd>

O processo de interpolação da cor dentro do interior de um polígono ou entre os vértices de uma linha, durante a rasterização.

</dd> <dt>

<span id="opengl_single_buffering"></span><span id="OPENGL_SINGLE_BUFFERING"></span>**buffer único**
</dt> <dd>

Um contexto OpenGL sem um buffer de cor de fundo.

</dd> <dt>

<span id="opengl_stipple"></span><span id="OPENGL_STIPPLE"></span>**stipple**
</dt> <dd>

Um padrão binário único ou bidimensional que derrota a geração de fragmentos em que seu valor é zero. A linha stipples é unidimensional e é aplicada em relação ao início de uma linha. Polígono stipples são bidimensionais e são aplicados com uma orientação fixa para a janela.

</dd> </dl>
