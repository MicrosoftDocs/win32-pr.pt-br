-   <span data-ttu-id="0f5c0-125">Para obter a rasterização de duas dimensões exata, especifique cuidadosamente a projeção ortográfica e os vértices dos primitivos que devem ser rasterizados.</span><span class="sxs-lookup"><span data-stu-id="0f5c0-125">To obtain exact two-dimensional rasterization, carefully specify both the orthographic projection and the vertices of the primitives that are to be rasterized.</span></span> <span data-ttu-id="0f5c0-126">Especifique a projeção ortográfica com coordenadas de inteiro, conforme mostrado no exemplo a seguir.</span><span class="sxs-lookup"><span data-stu-id="0f5c0-126">Specify the orthographic projection with integer coordinates, as shown in the following example.</span></span>

    <span data-ttu-id="0f5c0-127">A *largura* e a *altura* dos parâmetros são as dimensões do visor.</span><span class="sxs-lookup"><span data-stu-id="0f5c0-127">The parameters *width* and *height*     <span data-ttu-id="0f5c0-132">Um comprometimento ideal que permite que todos os primitivos sejam especificados em posições de inteiros, enquanto ainda garante a rasterização previsível, é converter *x* e *y* por 0,375, conforme mostrado no exemplo de código a seguir.</span><span class="sxs-lookup"><span data-stu-id="0f5c0-132">An optimum compromise that allows all primitives to be specified at integer positions, while still ensuring predictable rasterization, is to translate *x* and *y* by 0.375, as shown in the following code sample.</span></span> <span data-ttu-id="0f5c0-133">Tal tradução mantém as bordas do polígono e da imagem de pixel com segurança, longe dos centros de pixels, ao mesmo tempo em que os vértices de linha são fechados o suficiente para os centros de pixel.</span><span class="sxs-lookup"><span data-stu-id="0f5c0-133">Such a translation keeps polygon and pixel image edges safely away from the centers of pixels, while moving line vertices close enough to the pixel centers.</span></span>
