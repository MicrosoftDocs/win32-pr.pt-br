# <a name="parental-controls-problem-statement"></a>Declaração de problema de controles dos pais

O uso do computador para atividades online e offline abre um novo mundo para coleta de informações, comunicação, comércio, produtividade e entretenimento. Ele também apresenta novos riscos para compartilhamentos, divulgação de informações e fácil acesso a conteúdo inadequado em sites, mensagens, downloads de arquivos e multimídia de jogos e áudio/vídeo. Para adultos, os riscos podem ser controlados por tecnologias de filtragem configuradas por um adulto para sua própria conta ou para um computador como um todo. Gerenciamento de cookies, bloqueadores pop-up, filtros anti-spam, configurações de site confiáveis e bloqueadas e gerenciamento de lista de amigos de mensagens instantâneas são exemplos comuns de esforços de filtragem por auto-filtragem.

A imposição de controles dos pais é diferente da auto-filtragem. As políticas definidas por um pai ou guardião devem ser impostas aos dependentes sem que os dependentes sejam capazes de alterar facilmente essas políticas. O mesmo vale para adulteração de informações de log de atividades que podem ser essenciais para um pai ou guardião ter informações suficientes para definir e manter políticas efetivas. O bastião para controle e status se torna uma identidade privilegiada, associada a uma identidade de direitos reduzida cujos processos normalmente implementam restrições de monitoramento e atividade sem poder modificar trivialmente as políticas ou registrar dados em log.

Para identidades puramente online, é relativamente fácil configurar identidades privilegiadas e protegidas e suas associações expostas por nomes de usuário online e senhas. A extensão para atividades de computador offline, como jogos de pc ou DVDs, ou o uso de clientes online que não os navegadores da Web se torna muito mais difícil. A implementação de controles dos pais hoje em dia, portanto, normalmente resulta em soluções singulares de ISVs individuais. Alcançar uma cobertura abrangente sobre a maior parte das áreas de risco pode exigir o uso de vários produtos, com distribuição resultante de dados de monitoramento de atividade e configurações de política entre várias interfaces do usuário. Além disso, as soluções atualmente devem ser implementadas em camadas de software que podem não ter contexto suficiente para filtragem ou direitos legais para protocolos e expor preocupações sobre o acesso a dados confidenciais. Essas soluções também podem ser prejudicadas pela criptografia de link.
