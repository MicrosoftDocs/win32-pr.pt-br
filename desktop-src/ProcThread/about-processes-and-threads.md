# <a name="about-processes-and-threads"></a><span data-ttu-id="ea3c4-103">Sobre Processos e Threads</span><span class="sxs-lookup"><span data-stu-id="ea3c4-103">About Processes and Threads</span></span>

<span data-ttu-id="ea3c4-104">Cada *processo* fornece os recursos necessários para executar um programa.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-104">Each *process* provides the resources needed to execute a program.</span></span> <span data-ttu-id="ea3c4-105">Um processo tem um espaço de endereço virtual, código executável, identificadores abertos para objetos do sistema, um contexto de segurança, um identificador de processo exclusivo, variáveis de ambiente, uma classe de prioridade, tamanhos mínimo e máximo de conjunto de trabalho e pelo menos um thread de execução.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-105">A process has a virtual address space, executable code, open handles to system objects, a security context, a unique process identifier, environment variables, a priority class, minimum and maximum working set sizes, and at least one thread of execution.</span></span> <span data-ttu-id="ea3c4-106">Cada processo é iniciado com um único thread, geralmente chamado de *thread principal*, mas pode criar threads adicionais de qualquer um de seus threads.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-106">Each process is started with a single thread, often called the *primary thread*, but can create additional threads from any of its threads.</span></span>

<span data-ttu-id="ea3c4-107">Um *thread* é a entidade em um processo que pode ser agendada para execução.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-107">A *thread* is the entity within a process that can be scheduled for execution.</span></span> <span data-ttu-id="ea3c4-108">Todos os threads de um processo compartilham seu espaço de endereço virtual e os recursos do sistema.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-108">All threads of a process share its virtual address space and system resources.</span></span> <span data-ttu-id="ea3c4-109">Além disso, cada thread mantém manipuladores de exceção, uma prioridade de agendamento, um armazenamento local de thread, um identificador de thread exclusivo e um conjunto de estruturas que o sistema usará para salvar o contexto de thread até que ele seja agendado.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-109">In addition, each thread maintains exception handlers, a scheduling priority, thread local storage, a unique thread identifier, and a set of structures the system will use to save the thread context until it is scheduled.</span></span> <span data-ttu-id="ea3c4-110">O *contexto de thread* inclui o conjunto de registros de computador do thread, a pilha de kernel, um bloco de ambiente de thread e uma pilha de usuário no espaço de endereço do processo do thread.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-110">The *thread context* includes the thread's set of machine registers, the kernel stack, a thread environment block, and a user stack in the address space of the thread's process.</span></span> <span data-ttu-id="ea3c4-111">Os threads também podem ter seu próprio contexto de segurança, que pode ser usado para representar clientes.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-111">Threads can also have their own security context, which can be used for impersonating clients.</span></span>

<span data-ttu-id="ea3c4-112">O Microsoft Windows dá suporte à *Preemptive multitarefa*, que cria o efeito de execução simultânea de vários threads a partir de vários processos.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-112">Microsoft Windows supports *preemptive multitasking*, which creates the effect of simultaneous execution of multiple threads from multiple processes.</span></span> <span data-ttu-id="ea3c4-113">Em um computador multiprocessador, o sistema pode executar simultaneamente tantos segmentos quantos houver processadores no computador.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-113">On a multiprocessor computer, the system can simultaneously execute as many threads as there are processors on the computer.</span></span>

<span data-ttu-id="ea3c4-114">Um *objeto de trabalho* permite que grupos de processos sejam gerenciados como uma unidade.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-114">A *job object* allows groups of processes to be managed as a unit.</span></span> <span data-ttu-id="ea3c4-115">Os objetos de trabalho são objetos namable, protegíveis e compartilháveis que controlam os atributos dos processos associados a eles.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-115">Job objects are namable, securable, sharable objects that control attributes of the processes associated with them.</span></span> <span data-ttu-id="ea3c4-116">As operações executadas no objeto de trabalho afetam todos os processos associados ao objeto de trabalho.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-116">Operations performed on the job object affect all processes associated with the job object.</span></span>

<span data-ttu-id="ea3c4-117">Um aplicativo pode usar o *pool de threads* para reduzir o número de threads do aplicativo e fornecer gerenciamento dos threads de trabalho.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-117">An application can use the *thread pool* to reduce the number of application threads and provide management of the worker threads.</span></span> <span data-ttu-id="ea3c4-118">Os aplicativos podem enfileirar itens de trabalho, associar o trabalho a identificadores de espera, enfileirar automaticamente com base em um temporizador e associar a e/s.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-118">Applications can queue work items, associate work with waitable handles, automatically queue based on a timer, and bind with I/O.</span></span>

<span data-ttu-id="ea3c4-119">O ums ( *agendamento de modo de usuário* ) é um mecanismo leve que os aplicativos podem usar para agendar seus próprios threads.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-119">*User-mode scheduling* (UMS) is a lightweight mechanism that applications can use to schedule their own threads.</span></span> <span data-ttu-id="ea3c4-120">Um aplicativo pode alternar entre threads UMS no modo de usuário sem envolver o [Agendador do sistema](scheduling.md) e reobter o controle do processador se um thread UMS for bloqueado no kernel.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-120">An application can switch between UMS threads in user mode without involving the [system scheduler](scheduling.md) and regain control of the processor if a UMS thread blocks in the kernel.</span></span> <span data-ttu-id="ea3c4-121">Cada thread UMS tem seu próprio contexto de thread em vez de compartilhar o contexto de thread de um único thread.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-121">Each UMS thread has its own thread context instead of sharing the thread context of a single thread.</span></span> <span data-ttu-id="ea3c4-122">A capacidade de alternar entre threads no modo de usuário torna o UMS mais eficiente do que os pools de threads para itens de trabalho de curta duração que exigem poucas chamadas do sistema.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-122">The ability to switch between threads in user mode makes UMS more efficient than thread pools for short-duration work items that require few system calls.</span></span>

<span data-ttu-id="ea3c4-123">Uma *fibra* é uma unidade de execução que deve ser agendada manualmente pelo aplicativo.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-123">A *fiber* is a unit of execution that must be manually scheduled by the application.</span></span> <span data-ttu-id="ea3c4-124">Os fibras são executados no contexto dos threads que os agendam.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-124">Fibers run in the context of the threads that schedule them.</span></span> <span data-ttu-id="ea3c4-125">Cada thread pode agendar várias fibras.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-125">Each thread can schedule multiple fibers.</span></span> <span data-ttu-id="ea3c4-126">Em geral, os fibras não fornecem vantagens em relação a um aplicativo multithread bem projetado.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-126">In general, fibers do not provide advantages over a well-designed multithreaded application.</span></span> <span data-ttu-id="ea3c4-127">No entanto, o uso de fibras pode facilitar a porta de aplicativos que foram projetados para agendar seus próprios threads.</span><span class="sxs-lookup"><span data-stu-id="ea3c4-127">However, using fibers can make it easier to port applications that were designed to schedule their own threads.</span></span>
