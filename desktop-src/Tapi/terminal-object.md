# <a name="terminal-object"></a><span data-ttu-id="67eaf-103">Objeto terminal</span><span class="sxs-lookup"><span data-stu-id="67eaf-103">Terminal Object</span></span>

<span data-ttu-id="67eaf-104">Na TAPI versão 3,0 e posterior, o modelo de objeto TAPI usa objetos de terminal para representar a origem ou o coletor de um fluxo de mídia associado a uma sessão de chamada ou de comunicação.</span><span class="sxs-lookup"><span data-stu-id="67eaf-104">In TAPI version 3.0 and later, the TAPI object model uses terminal objects to represent the source or sink of a media stream associated with a call or communications session.</span></span> <span data-ttu-id="67eaf-105">Esse modelo de objeto permite que um aplicativo especifique, em um nível detalhado, como a mídia é processada em uma chamada.</span><span class="sxs-lookup"><span data-stu-id="67eaf-105">This object model enables an application to specify, at a detailed level, how media are processed on a call.</span></span> <span data-ttu-id="67eaf-106">Esse modelo também permite que vários terminais sejam selecionados simultaneamente, por exemplo, uma chamada pode ser saída para um alto-falante de áudio e gravada simultaneamente.</span><span class="sxs-lookup"><span data-stu-id="67eaf-106">This model also enables multiple terminals to be selected simultaneously, so for example, a call can be output to an audio speaker and recorded simultaneously.</span></span>

<span data-ttu-id="67eaf-110">Os terminais são normalmente implementados por um provedor de serviços de mídia (MSP) e os objetos de terminal não estarão disponíveis se não houver um MSP associado a uma sessão de comunicações.</span><span class="sxs-lookup"><span data-stu-id="67eaf-110">Terminals are normally implemented by a media service provider (MSP) and terminal objects will not be available if there is no MSP associated with a communications session.</span></span> <span data-ttu-id="67eaf-111">Uma exceção é que, com o Windows 2000 SP1 e posterior, um aplicativo pode implementar uma forma de [terminal conectável](pluggable-terminals.md).</span><span class="sxs-lookup"><span data-stu-id="67eaf-111">One exception is that with Windows 2000 SP1 and later, an application can implement a form of [pluggable terminal](pluggable-terminals.md).</span></span> <span data-ttu-id="67eaf-112">Isso permite que um servidor de conferência crie terminais de ponte para que os clientes que não são do Windows 2000 SP1 ou do H323 não multicast possam ser adicionados a conferências de multicast/IP de várias partes da TAPI 3.</span><span class="sxs-lookup"><span data-stu-id="67eaf-112">This enables a conference server to create bridging terminals so that non-Windows 2000 SP1 or non-multicast H323 clients can be added to TAPI 3 multi-party SDP/IP multicast conferences.</span></span>

<span data-ttu-id="67eaf-113">Cada terminal pertence a uma [classe terminal](terminal-class.md).</span><span class="sxs-lookup"><span data-stu-id="67eaf-113">Each terminal belongs to a [terminal class](terminal-class.md).</span></span> <span data-ttu-id="67eaf-114">Uma classe de terminal representa um conjunto de recursos de origem ou de renderização.</span><span class="sxs-lookup"><span data-stu-id="67eaf-114">A terminal class represents a set of source or render features.</span></span> <span data-ttu-id="67eaf-115">Por exemplo, um terminal que mapeia para um conjunto de alto-falantes de áudio seria identificado como CLSID \_ SpeakersTerminal, e o provedor de serviços deve ser a implementação do controle de volume.</span><span class="sxs-lookup"><span data-stu-id="67eaf-115">For example, a terminal that maps to a set of audio speakers would be identified as CLSID\_SpeakersTerminal, and the service provider would be expected to implement volume control.</span></span> <span data-ttu-id="67eaf-116">A TAPI 3 define um conjunto de classes de terminal, um MSP pode definir classes adicionais e um aplicativo pode registrar novas classes de terminal.</span><span class="sxs-lookup"><span data-stu-id="67eaf-116">TAPI 3 defines a set of terminal classes, an MSP can define additional classes, and an application can register new terminal classes.</span></span> <span data-ttu-id="67eaf-117">Cada classe de terminal é atribuída a um GUID (identificador global exclusivo).</span><span class="sxs-lookup"><span data-stu-id="67eaf-117">Each terminal class is assigned a globally unique identifier (GUID).</span></span>

<span data-ttu-id="67eaf-118">Do ponto de vista de um aplicativo, um terminal é descrito por seu tipo e [direção](/windows/desktop/api/Tapi3if/ne-tapi3if-terminal_direction)de [terminal](/windows/desktop/api/Tapi3if/ne-tapi3if-terminal_type) .</span><span class="sxs-lookup"><span data-stu-id="67eaf-118">From the point of view of an application, a terminal is described by its [terminal type](/windows/desktop/api/Tapi3if/ne-tapi3if-terminal_type) and [direction](/windows/desktop/api/Tapi3if/ne-tapi3if-terminal_direction).</span></span> <span data-ttu-id="67eaf-119">O tipo pode ser estático ou dinâmico.</span><span class="sxs-lookup"><span data-stu-id="67eaf-119">The type may be either static or dynamic.</span></span> <span data-ttu-id="67eaf-120">Um terminal estático é mapeado para hardware, como um telefone ou microfone.</span><span class="sxs-lookup"><span data-stu-id="67eaf-120">A static terminal maps to hardware, such as a telephone or microphone.</span></span> <span data-ttu-id="67eaf-121">Um terminal dinâmico é mapeado para um objeto transitório, como um arquivo ou uma janela de vídeo.</span><span class="sxs-lookup"><span data-stu-id="67eaf-121">A dynamic terminal maps to a transient object, such as a file or a video window.</span></span> <span data-ttu-id="67eaf-122">A direção descreve se um determinado terminal é uma origem ou um processador.</span><span class="sxs-lookup"><span data-stu-id="67eaf-122">Direction describes whether a given terminal is a source or a renderer.</span></span>
