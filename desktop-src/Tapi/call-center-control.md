# <a name="call-center-control"></a>Controle do Call Center

A TAPI 2,2 básica pode ser usada para gerenciar os centros de chamadas e outros elementos da infraestrutura de rede de telefonia (como servidores de IVR e de correio de voz) por meio de controle de chamada de terceiros. Para ajudar a criar proxies AD que serão executados em servidores, as funções foram adicionadas à TAPI 2,2 para fornecer assistência adicional aos desenvolvedores de aplicativos.

Os tópicos a seguir descrevem os recursos TAPI 2,2 que você pode usar para implementar controles de Call Center:

-   [Modelagem de um Call Center](modeling-of-a-call-center.md)
-   [Rádio](stations.md)
-   [Discagem preditiva](predictive-dialing.md)
-   [Filas de chamadas e pontos de rota](call-queues-and-route-points.md)
-   [Monitoramento e controle do AD Agent](acd-agent-monitoring-and-control.md)
-   [Controle de status de estação](station-status-control.md)
-   [Controle de status de estação](station-status-control.md)
-   [Temporizador de estado de chamada](call-state-timer.md)
-   [Temporizadores de eventos de mídia](media-event-timers.md)

A TAPI 3. x é a API preferida de escrever aplicativos de agente AD. Consulte a seção [controles do Call Center](./about-call-center-controls.md) para obter informações sobre como usar essas interfaces e métodos. No entanto, a TAPI 2,2 permanece utilizável para conjuntos de aplicativos ad completos, se desejado.
