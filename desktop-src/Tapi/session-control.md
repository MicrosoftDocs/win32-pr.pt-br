# <a name="session-control"></a>Controle de sessão

Uma sessão ou chamada representa uma conexão entre dois ou mais endereços. Essa conexão é dinâmica e os objetos de programação associados devem ser criados, manipulados e liberados conforme necessário. No caso mais simples, isso significa fazer e desconectar uma chamada telefônica. Para aplicativos avançados, o controle de sessão pode envolver o gerenciamento de conferências de multimídia em uma rede IP até o nível de participantes individuais.

O controle de sessão envolve duas áreas básicas:

-   [Operações de sessão](session-operations-ovr.md) são controles que iniciam, mantêm e encerram uma sessão de comunicações.
-   [Informações de sessão](session-information-ovr.md) descrevem dados que podem ser obtidos em uma sessão de comunicações.

Dois tipos especializados de controle de sessão não são abordados nesta seção. Para obter informações sobre os centros de distribuição de chamada automática, consulte [controle do Call Center](./call-center-control.md) (TAPI 2. x) ou [sobre controles do Call Center](about-call-center-controls.md) (TAPI 3. x). Para obter informações sobre a conferência por IP, consulte [sobre a reunião de teleconferência de telefonia IP](about-rendezvous-ip-telephony-conferencing.md).
