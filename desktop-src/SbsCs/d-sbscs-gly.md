# <a name="d-isolated-applications-and-side-by-side-assemblies"></a>D (Aplicativos isolados e assemblies lado a lado)

<dl> <dt>

<span id="_win32_def_context_assemblyidentity_gly"></span><span id="_WIN32_DEF_CONTEXT_ASSEMBLYIDENTITY_GLY"></span>**Assembly de contexto de DEFIdentity**
</dt> <dd>

Um **subelemento assemblyIdentity** que define o assembly lado a lado descrito pelo manifesto. Um subelemento **assemblyIdentity** de contexto de DEF é sempre o primeiro subelemento de um **elemento de** assembly.

</dd> <dt>

<span id="_win32_dll_versioning_conflict_gly"></span><span id="_WIN32_DLL_VERSIONING_CONFLICT_GLY"></span>**Conflito de versão de DLL**
</dt> <dd>

Problema de compatibilidade. Problemas de compartilhamento de assembly ocorrem quando um aplicativo instala uma versão de um assembly compartilhado que não é compatível com versões anteriores com a versão instalada anteriormente. Uma solução para conflitos de versão de DLL é usar o compartilhamento de assembly lado a lado e aplicativos isolados. Com o compartilhamento de assembly lado a lado, várias versões do mesmo assembly Windows podem ser executados simultaneamente. Os desenvolvedores podem escolher qual assembly lado a lado usar.

</dd> </dl>
