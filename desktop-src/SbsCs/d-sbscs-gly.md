# <a name="d-isolated-applications-and-side-by-side-assemblies"></a>D (aplicativos isolados e assemblies lado a lado)

<dl> <dt>

<span id="_win32_def_context_assemblyidentity_gly"></span><span id="_WIN32_DEF_CONTEXT_ASSEMBLYIDENTITY_GLY"></span>**AssemblyIdentity de contexto de DEF**
</dt> <dd>

Um subelemento **AssemblyIdentity** que define o assembly lado a lado descrito pelo manifesto. Um subelemento **AssemblyIdentity** de contexto def é sempre o primeiro subelemento de um elemento de **assembly** .

</dd> <dt>

<span id="_win32_dll_versioning_conflict_gly"></span><span id="_WIN32_DLL_VERSIONING_CONFLICT_GLY"></span>**Conflito de controle de versão de DLL**
</dt> <dd>

Problema de compatibilidade. Problemas de compartilhamento de assembly ocorrem quando um aplicativo instala uma versão de um assembly compartilhado que não é compatível com versões anteriores com a versão instalada anteriormente. Uma solução para conflitos de controle de versão de DLL é usar o compartilhamento de assembly lado a lado e aplicativos isolados. Com o compartilhamento de assembly lado a lado, várias versões do mesmo assembly do Windows podem ser executadas simultaneamente. Os desenvolvedores podem escolher qual assembly lado a lado usar.

</dd> </dl>
