# <a name="persistent-object-state"></a>Estado do objeto persistente

Alguns objetos COM podem salvar seu estado interno quando solicitado por um cliente.

O COM define padrões por meio dos quais os clientes podem solicitar que os objetos sejam inicializados, carregados e salvos em um armazenamento de dados (por exemplo, arquivo simples, armazenamento estruturado ou memória). É responsabilidade do cliente gerenciar o local em que os dados persistentes do objeto são armazenados, mas não o formato dos dados. Objetos COM que aderem a esses padrões são chamados *de objetos persistentes.*

Para obter mais informações, consulte estes tópicos:

-   [Interfaces de objeto persistente](persistent-object-interfaces.md)
-   [Inicializando objetos persistentes](initializing-persistent-objects.md)
