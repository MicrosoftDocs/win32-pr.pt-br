<span data-ttu-id="f4b8a-111">Como os monikers de classes diferentes podem ser compostos um com o outro, os monikers fornecem a capacidade de unir vários namespaces.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-111">Because monikers of different classes can be composed with one another, monikers provide the ability to join multiple namespaces.</span></span> <span data-ttu-id="f4b8a-117">Uma chamada para [**IMoniker:: GetDisplayName**](/windows/desktop/api/ObjIdl/nf-objidl-imoniker-getdisplayname) em uma composição cria um nome usando a concatenação de todos os [**IMoniker::GetDisplayName**](/windows/desktop/api/ObjIdl/nf-objidl-imoniker-getdisplayname) on a composite creates a name using the concatenation of all the individual monikers' display names.</span></span>

<span data-ttu-id="f4b8a-118">Além disso, como você pode escrever sua própria classe de moniker, a composição do moniker permite que você adicione extensões personalizadas ao namespace para objetos.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-118">Furthermore, because you can write your own moniker class, moniker composition allows you to add customized extensions to the namespace for objects.</span></span>

<span data-ttu-id="f4b8a-123">A *composição genérica*, por outro lado, permite a conexão de quaisquer dois monikers, não importa quais são suas classes.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-123">*Generic composition*, on the other hand, permits the connection of any two monikers, no matter what their classes.</span></span> <span data-ttu-id="f4b8a-124">Por exemplo, você pode compor um moniker de item em um moniker de arquivo, embora não, é claro, o contrário.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-124">For example, you could compose an item moniker onto a file moniker, although not, of course, the other way around.</span></span>

<span data-ttu-id="f4b8a-125">Como uma composição não genérica depende da classe dos monikers envolvidos, seus detalhes são definidos pela implementação de uma classe de moniker específica.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-125">Because a nongeneric composition depends on the class of the monikers involved, its details are defined by the implementation of a particular moniker class.</span></span> <span data-ttu-id="f4b8a-126">Você pode definir novos tipos de composições não genéricas se escrever uma nova classe de moniker.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-126">You can define new types of nongeneric compositions if you write a new moniker class.</span></span> <span data-ttu-id="f4b8a-127">Por outro lado, as composições genéricas são definidas pelo OLE.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-127">By contrast, generic compositions are defined by OLE.</span></span> <span data-ttu-id="f4b8a-128">Os monikers criados como resultado da composição genérica são chamados de monikers compostos genéricos.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-128">Monikers created as a result of generic composition are called generic composite monikers.</span></span>

<span data-ttu-id="f4b8a-129">Essas três classes, os monikers de arquivo, os monikers de item e os monikers compostos genéricos, todos trabalham juntos e são as classes mais usadas de monikers.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-129">These three classes, file monikers, item monikers, and generic composite monikers, all work together, and they are the most commonly used classes of monikers.</span></span>

<span data-ttu-id="f4b8a-130">Os clientes do moniker devem chamar [**IMoniker:: ComposeWith**](/windows/desktop/api/ObjIdl/nf-objidl-imoniker-composewith) para criar uma composição no moniker com outra.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-130">Moniker clients should call [**IMoniker::ComposeWith**](/windows/desktop/api/ObjIdl/nf-objidl-imoniker-composewith) to create a composite on moniker with another.</span></span> <span data-ttu-id="f4b8a-131">O moniker no qual ele é chamado internamente decide se ele pode fazer uma composição genérica ou não genérica.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-131">The moniker it is called on internally decides whether it can do a generic or nongeneric composition.</span></span> <span data-ttu-id="f4b8a-132">Se a implementação do moniker determinar que uma composição genérica é utilizável, o OLE fornecerá a função [**CreateGenericComposite**](/windows/desktop/api/Objbase/nf-objbase-creategenericcomposite) para facilitar isso.</span><span class="sxs-lookup"><span data-stu-id="f4b8a-132">If the moniker implementation determines that a generic composition is usable, OLE provides the [**CreateGenericComposite**](/windows/desktop/api/Objbase/nf-objbase-creategenericcomposite) function to facilitate this.</span></span>
