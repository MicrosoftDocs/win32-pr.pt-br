# <a name="registering-a-class-at-installation"></a><span data-ttu-id="10298-103">Registrando uma classe na instalação</span><span class="sxs-lookup"><span data-stu-id="10298-103">Registering a Class at Installation</span></span>

<span data-ttu-id="10298-104">Se uma classe for destinada a estar disponível para os clientes a qualquer momento, como a maioria dos aplicativos, você normalmente o registrará por meio de um programa de instalação e instalação.</span><span class="sxs-lookup"><span data-stu-id="10298-104">If a class is intended to be available to clients at any time, as most applications are, you usually register it through an installation and setup program.</span></span> <span data-ttu-id="10298-105">Isso significa colocar informações sobre o aplicativo no registro, incluindo como e onde seus objetos devem ser instanciados.</span><span class="sxs-lookup"><span data-stu-id="10298-105">This means putting information about the application into the registry, including how and where its objects are to be instantiated.</span></span> <span data-ttu-id="10298-106">Essas informações devem ser registradas para todos os CLSIDs.</span><span class="sxs-lookup"><span data-stu-id="10298-106">This information must be registered for all CLSIDs.</span></span> <span data-ttu-id="10298-107">Outras informações são opcionais.</span><span class="sxs-lookup"><span data-stu-id="10298-107">Other information is optional.</span></span> <span data-ttu-id="10298-108">Ferramentas como o regsvr32 simplificam a gravação de um programa de instalação que registra servidores na instalação.</span><span class="sxs-lookup"><span data-stu-id="10298-108">Tools such as Regsvr32 make it simple to write a setup program that registers servers at installation.</span></span>

<span data-ttu-id="10298-109">Se você não depender de padrões do sistema, haverá duas chaves importantes no registro: CLSID e AppID.</span><span class="sxs-lookup"><span data-stu-id="10298-109">If you are not relying on system defaults, there are two important keys in the registry: the CLSID and the AppID.</span></span> <span data-ttu-id="10298-110">Entre as informações importantes sob essas chaves está como o objeto deve ser instanciado.</span><span class="sxs-lookup"><span data-stu-id="10298-110">Among the important pieces of information under these keys is how the object is to be instantiated.</span></span> <span data-ttu-id="10298-111">Os objetos podem ser designados como em processo, fora do processo local ou remoto fora do processo.</span><span class="sxs-lookup"><span data-stu-id="10298-111">Objects can be designated as in-process, out-of-process local, or out-of-process remote.</span></span>

<span data-ttu-id="10298-112">Na chave AppID há vários valores que definem informações específicas para esse aplicativo.</span><span class="sxs-lookup"><span data-stu-id="10298-112">Under the AppID key are several values that define information specific to that application.</span></span> <span data-ttu-id="10298-113">Entre eles estão [RemoteServerName](remoteservername.md) e [ActivateAtStorage](activateatstorage.md), ambos podem ser usados para permitir que um cliente crie um objeto, com o cliente sem conhecimento interno do local do servidor.</span><span class="sxs-lookup"><span data-stu-id="10298-113">Among these are [RemoteServerName](remoteservername.md) and [ActivateAtStorage](activateatstorage.md), both of which can be used to permit a client to create an object, with the client having no built-in knowledge of the location of the server.</span></span> <span data-ttu-id="10298-114">(Para obter mais informações sobre a instanciação remota, consulte [localizando um objeto remoto](locating-a-remote-object.md) e [funções auxiliares de criação de instância](instance-creation-helper-functions.md).)</span><span class="sxs-lookup"><span data-stu-id="10298-114">(For more information about remote instantiation, see [Locating a Remote Object](locating-a-remote-object.md) and [Instance Creation Helper Functions](instance-creation-helper-functions.md).)</span></span>

<span data-ttu-id="10298-115">Um servidor também pode ser instalado como um serviço ou para ser executado em uma conta de usuário específica.</span><span class="sxs-lookup"><span data-stu-id="10298-115">A server can also be installed as a service, or to run under a specific user account.</span></span> <span data-ttu-id="10298-116">Para obter mais informações, consulte [instalando como um aplicativo de serviço](installing-as-a-service-application.md).</span><span class="sxs-lookup"><span data-stu-id="10298-116">For more information, see [Installing as a Service Application](installing-as-a-service-application.md).</span></span>

<span data-ttu-id="10298-117">Um servidor ou objeto corrompidos que não seja um serviço ou em execução em uma conta de usuário específica pode ser chamado de servidor "Activate as Activator".</span><span class="sxs-lookup"><span data-stu-id="10298-117">A server or ROT object that is not a service or running under a specific user account can be referred to as an "activate as activator" server.</span></span> <span data-ttu-id="10298-118">Para esses servidores, o contexto de segurança e a estação da janela/área de trabalho do cliente devem corresponder ao do servidor.</span><span class="sxs-lookup"><span data-stu-id="10298-118">For these servers, the security context and the window station/desktop of the client must match the server's.</span></span> <span data-ttu-id="10298-119">Um cliente que está tentando se conectar a um servidor remoto é considerado uma estação de janela/área de trabalho **nula** , portanto, somente o contexto de segurança do servidor é comparado nessa instância.</span><span class="sxs-lookup"><span data-stu-id="10298-119">A client attempting to connect to a remote server is considered to have a **NULL** window station/desktop, so only the server security context is compared in this instance.</span></span> <span data-ttu-id="10298-120">(Para obter mais informações sobre SIDs, consulte [segurança em com](security-in-com.md).) O COM armazena em cache a estação de janela/área de trabalho de um processo quando o processo se conecta pela primeira vez ao serviço COM distribuído.</span><span class="sxs-lookup"><span data-stu-id="10298-120">(For more information about SIDs, see [Security in COM](security-in-com.md).) COM caches the window station/desktop of a process when the process first connects to the distributed COM service.</span></span> <span data-ttu-id="10298-121">Portanto, os clientes e servidores COM não devem alterar sua estação de janela ou desktops de thread do processo depois de chamar o [**CoInitialize**](/windows/desktop/api/Objbase/nf-objbase-coinitialize) ou [**CoInitializeEx**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializeex).</span><span class="sxs-lookup"><span data-stu-id="10298-121">Therefore, COM clients and servers should not change their window station or thread desktops of the process after calling [**CoInitialize**](/windows/desktop/api/Objbase/nf-objbase-coinitialize) or [**CoInitializeEx**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializeex).</span></span>

<span data-ttu-id="10298-122">Quando uma classe é registrada como em processo, uma chamada para [**CoGetClassObject**](/windows/desktop/api/combaseapi/nf-combaseapi-cogetclassobject) para criar seu objeto de classe é passada automaticamente pelo com para a função [**DllGetClassObject**](/windows/desktop/api/combaseapi/nf-combaseapi-dllgetclassobject) , que a classe deve implementar para dar ao objeto de chamada um ponteiro para seu objeto de classe.</span><span class="sxs-lookup"><span data-stu-id="10298-122">When a class is registered as in-process, a call to [**CoGetClassObject**](/windows/desktop/api/combaseapi/nf-combaseapi-cogetclassobject) to create its class object is automatically passed by COM to the [**DllGetClassObject**](/windows/desktop/api/combaseapi/nf-combaseapi-dllgetclassobject) function, which the class must implement to give the calling object a pointer to its class object.</span></span>

<span data-ttu-id="10298-123">As classes implementadas em executáveis podem especificar que o COM deve executar seu processo e aguardar que o processo Registre a interface [**IClassFactory**](/windows/win32/api/unknwn/nn-unknwn-iclassfactory) do seu objeto de classe por meio de uma chamada para a função [**CoRegisterClassObject**](/windows/desktop/api/combaseapi/nf-combaseapi-coregisterclassobject) .</span><span class="sxs-lookup"><span data-stu-id="10298-123">Classes implemented in executables can specify that COM should execute their process and wait for the process to register their class object's [**IClassFactory**](/windows/win32/api/unknwn/nn-unknwn-iclassfactory) interface through a call to the [**CoRegisterClassObject**](/windows/desktop/api/combaseapi/nf-combaseapi-coregisterclassobject) function.</span></span>
