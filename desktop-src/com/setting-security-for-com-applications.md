# <a name="setting-security-for-com-applications"></a>Definindo a segurança para aplicativos COM

Há várias maneiras pelas quais você pode ajudar a controlar a segurança para seus aplicativos. Você pode alterar as configurações de segurança padrão de um computador, que são usadas por todos os aplicativos no computador que não fornecem seus próprios valores de segurança. Você pode definir a segurança para um processo específico, usando Dcomcnfg.exe ou ao chamar [**CoInitializeSecurity**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializesecurity). Você também pode controlar de forma programática as configurações de segurança no nível de proxy da interface.

Os tópicos a seguir fornecem procedimentos que explicam como definir a segurança:

-   [Modificando os padrões de segurança de um computador](modifying-the-security-defaults-for-a-computer.md)
-   [Definindo Process-Wide segurança](setting-processwide-security.md)
-   [Definindo a segurança no nível de proxy da interface](setting-security-at-the-interface-proxy-level.md)
