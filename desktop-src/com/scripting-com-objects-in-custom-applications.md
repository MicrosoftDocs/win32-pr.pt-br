# <a name="scripting-com-objects-in-custom-applications"></a>Script de objetos COM em aplicativos personalizados

A Microsoft fornece vários ambientes para script de objetos COM: Active Server Pages, Windows Host de Script e assim por diante. IsVs (fornecedores independentes de software) também podem licenciar mecanismos de script da Microsoft para uso em seus aplicativos. Por exemplo, um ISV que cria um processador de palavras pode querer adicionar uma linguagem de macro ao aplicativo para que os usuários possam automatizar tarefas simples. Ao licenciar um mecanismo de script, o ISV pode usar uma linguagem como VBScript ou JScript como a linguagem de macro do aplicativo.

Além de licenciar OVBScript e JScript de script, os ISVs podem escrever seus próprios mecanismos ActiveX script. Esses mecanismos de script podem ser conectados a qualquer ambiente de host que dá suporte ao padrão ActiveX de script. Por exemplo, um ISV pode escrever um mecanismo de script PerlScript e instalá-lo em um servidor ASP, permitindo que o servidor ASP processe scripts PerlScript.
