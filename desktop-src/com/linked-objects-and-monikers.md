# <a name="linked-objects-and-monikers"></a><span data-ttu-id="96ab7-103">Objetos vinculados e monikers</span><span class="sxs-lookup"><span data-stu-id="96ab7-103">Linked Objects and Monikers</span></span>

<span data-ttu-id="96ab7-104">Objetos vinculados, como objetos incorporados, dependem de um manipulador de objeto para se comunicar com aplicativos de servidor.</span><span class="sxs-lookup"><span data-stu-id="96ab7-104">Linked objects, like embedded objects, rely on an object handler to communicate with server applications.</span></span> <span data-ttu-id="96ab7-105">O próprio objeto vinculado, no entanto, gerencia a nomenclatura e o acompanhamento de fontes de link.</span><span class="sxs-lookup"><span data-stu-id="96ab7-105">The linked object itself, however, manages the naming and tracking of link sources.</span></span> <span data-ttu-id="96ab7-106">O objeto vinculado age como um servidor em processo.</span><span class="sxs-lookup"><span data-stu-id="96ab7-106">The linked object acts like an in-process server.</span></span> <span data-ttu-id="96ab7-107">Por exemplo, quando ativado, um objeto vinculado localiza e inicia o aplicativo de servidor OLE que é a origem do link.</span><span class="sxs-lookup"><span data-stu-id="96ab7-107">For example, when activated, a linked object locates and launches the OLE server application that is the link source.</span></span>

<span data-ttu-id="96ab7-108">O manipulador de um objeto vinculado é composto de dois componentes principais: o componente de manipulador e o componente de vinculação.</span><span class="sxs-lookup"><span data-stu-id="96ab7-108">A linked object's handler is made up of two main components: the handler component and the linking component.</span></span> <span data-ttu-id="96ab7-109">O componente de manipulador contém partes de controle e de comunicação remota e funções como um manipulador para um objeto incorporado.</span><span class="sxs-lookup"><span data-stu-id="96ab7-109">The handler component contains the controlling and remoting pieces and functions much like a handler for an embedded object.</span></span> <span data-ttu-id="96ab7-110">O componente de vinculação tem seu próprio controlador e cache e fornece acesso ao armazenamento estruturado do objeto.</span><span class="sxs-lookup"><span data-stu-id="96ab7-110">The linking component has its own controller and cache and provides access to the object's structured storage.</span></span> <span data-ttu-id="96ab7-111">O controlador de componentes de vinculação dá suporte à nomeação de origem por meio do uso de monikers e associação, o processo de localizar e executar a origem do link.</span><span class="sxs-lookup"><span data-stu-id="96ab7-111">The linking components controller supports source naming through the use of monikers, and binding, the process of locating and running the link source.</span></span> <span data-ttu-id="96ab7-112">(Para obter mais informações sobre monikers e associação, consulte [a Component Object Model](the-component-object-model.md).)</span><span class="sxs-lookup"><span data-stu-id="96ab7-112">(For more information on monikers and binding, see [The Component Object Model](the-component-object-model.md).)</span></span>

<span data-ttu-id="96ab7-113">Quando um usuário cria inicialmente um objeto vinculado ou carrega um existente do armazenamento, o contêiner carrega uma instância do componente de vinculação na memória, juntamente com o manipulador de objetos.</span><span class="sxs-lookup"><span data-stu-id="96ab7-113">When a user initially creates a linked object or loads an existing one from storage, the container loads an instance of the linking component into memory, along with the object handler.</span></span> <span data-ttu-id="96ab7-114">O componente de vinculação fornece interfaces – mais notavelmente [**IOleLink**](/windows/desktop/api/OleIdl/nn-oleidl-iolelink)– que identificam o objeto como um link e permitem que ele gerencie a nomenclatura, o controle e a atualização de sua origem de link.</span><span class="sxs-lookup"><span data-stu-id="96ab7-114">The linking component supplies interfaces — most notably [**IOleLink**](/windows/desktop/api/OleIdl/nn-oleidl-iolelink)— that identify the object as a link and enable it to manage the naming, tracking, and updating of its link source.</span></span>

<span data-ttu-id="96ab7-115">Ao implementar a interface [**IOleLink**](/windows/desktop/api/OleIdl/nn-oleidl-iolelink) , um objeto vinculado fornece seu contêiner com funções que dão suporte à vinculação.</span><span class="sxs-lookup"><span data-stu-id="96ab7-115">By implementing the [**IOleLink**](/windows/desktop/api/OleIdl/nn-oleidl-iolelink) interface, a linked object provides its container with functions that support linking.</span></span> <span data-ttu-id="96ab7-116">Somente objetos vinculados implementam **IOleLink** e, ao consultar essa interface, um contêiner pode determinar se um determinado objeto é inserido ou vinculado.</span><span class="sxs-lookup"><span data-stu-id="96ab7-116">Only linked objects implement **IOleLink**, and by querying for this interface a container can determine whether a given object is embedded or linked.</span></span> <span data-ttu-id="96ab7-117">A função mais importante fornecida pelo **IOleLink** permite que um contêiner seja vinculado à origem do objeto vinculado, ou seja, ative a conexão com o documento que armazena os dados nativos do objeto vinculado.</span><span class="sxs-lookup"><span data-stu-id="96ab7-117">The most important function provided by **IOleLink** enables a container to binding to the source of the linked object, that is, to activate the connection to the document that stores the linked object's native data.</span></span> <span data-ttu-id="96ab7-118">O **IOleLink** também define funções para gerenciar informações sobre o objeto vinculado, como dados de apresentação em cache e o local da origem do link.</span><span class="sxs-lookup"><span data-stu-id="96ab7-118">**IOleLink** also defines functions for managing information about the linked object, such as cached presentation data and the location of the link source.</span></span>

<span data-ttu-id="96ab7-119">Quando um documento composto que contém um objeto vinculado é salvo, os dados do link são salvos com a origem do link, não com o contêiner.</span><span class="sxs-lookup"><span data-stu-id="96ab7-119">When a compound document containing a linked object is saved, the link's data is saved with the link source, not with the container.</span></span> <span data-ttu-id="96ab7-120">Somente informações sobre seu nome e local são salvas junto com o documento composto.</span><span class="sxs-lookup"><span data-stu-id="96ab7-120">Only information about its name and location is saved along with the compound document.</span></span> <span data-ttu-id="96ab7-121">Esse comportamento está em contraste com o de um objeto incorporado, cujos dados são armazenados junto com o de seu contêiner.</span><span class="sxs-lookup"><span data-stu-id="96ab7-121">This behavior is in contrast to that of an embedded object, whose data is stored along with that of its container.</span></span>

<span data-ttu-id="96ab7-122">Os aplicativos de contêiner podem fornecer informações sobre seus objetos incorporados, de modo que os últimos, ou partes deles, possam atuar como fontes de link.</span><span class="sxs-lookup"><span data-stu-id="96ab7-122">Container applications can provide information about their embedded objects such that the latter, or portions thereof, can act as link sources.</span></span> <span data-ttu-id="96ab7-123">Ao implementar o suporte para vincular aos objetos incorporados do contêiner, você pode tornar as incorporações aninhadas possíveis, aliviando o usuário de ter que rastrear os originais de cada objeto de inserção ao qual um link é desejado.</span><span class="sxs-lookup"><span data-stu-id="96ab7-123">By implementing support for linking to your container's embedded objects, you make nested embeddings possible, relieving the user of having to track down the originals of every embedding object to which a link is desired.</span></span> <span data-ttu-id="96ab7-124">Por exemplo, se um usuário quiser inserir uma planilha do Microsoft Excel no Microsoft Word e a planilha contiver um bitmap criado no Paintbrush, o usuário poderá vincular a uma cópia do bitmap contido na planilha, e não ao original.</span><span class="sxs-lookup"><span data-stu-id="96ab7-124">For example, if a user wants to embed a Microsoft Excel worksheet in Microsoft Word, and the worksheet contains a bitmap created in Paintbrush, the user can link to a copy of the bitmap contained in the worksheet rather than the original.</span></span>
