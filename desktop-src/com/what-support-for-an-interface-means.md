# <a name="what-support-for-an-interface-means"></a>O que é o suporte para uma interface significa

além da interface [**IUnknown**](/windows/desktop/api/Unknwn/nn-unknwn-iunknown) , um controle de ActiveX ou objeto COM para esse assunto expressa qualquer funcionalidade opcional que ele dê suporte por meio de interfaces adicionais. Isso significa que nenhuma outra interface é necessária acima de **IUnknown**. para esse fim, a tabela a seguir lista as interfaces às quais um controle de ActiveX pode dar suporte e o que significa dar suporte a essa interface.

| [**IOleObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleobject)<br/>                                                                                                                                                                                                                                                                                                                                     | Se o controle exigir comunicação com seu site do cliente para qualquer coisa que não seja eventos (consulte [**IConnectionPointContainer**](/windows/desktop/api/OCIdl/nn-ocidl-iconnectionpointcontainer)), então [**IOleObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleobject) é uma necessidade. Ao implementar essa interface, o controle também deve oferecer suporte à semântica dos seguintes métodos: [**SetHostNames**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-sethostnames), [**fechar**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-close), [**EnumVerbs**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-enumverbs), [**Update**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-update), [**IsUpToDate**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-isuptodate), [**GetUserClassID**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-getuserclassid), [**GetUserType**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-getusertype), [**GetMiscStatus**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-getmiscstatus)e os métodos [**Advise**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-advise), [**Unadvise**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-unadvise)e [**EnumAdvise**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-enumadvise) que funcionam em conjunto com a implementação de [**IAdviseSink**](/windows/desktop/api/ObjIdl/nn-objidl-iadvisesink) de um contêiner. | [**IOleControl**](/windows/desktop/api/OCIdl/nn-ocidl-iolecontrol)<br/>                                                                                                                                                                                                                                                                                                                                   | Expressa a capacidade e o desejo do controle de lidar com (a) mnemônicos ([**GetControlInfo**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrol-getcontrolinfo), métodos [**onmnemônico**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrol-onmnemonic) ), (b) Propriedades de ambiente ([**OnAmbientPropertyChange**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrol-onambientpropertychange)) e/ou (c) eventos que o controle exige que o contêiner manipule ([**FreezeEvents**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrol-freezeevents)). Observe que os mnemônicos são diferentes dos aceleradores que são manipulados por meio de [**IOleInPlaceActiveObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleinplaceactiveobject): mnemônicos têm interface do usuário associada e estão ativos mesmo quando o controle não está ativo da interface do usuário. O suporte de um controle para mnemônicos significa que o controle também sabe como usar o método [**IOleControlSite:: OnControlInfoChanged**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrolsite-oncontrolinfochanged) do contêiner. Como isso requer que o controle saiba o site do contêiner, o suporte para mnemônicos também significa suporte para [**IOleObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleobject). Além disso, o conhecimento de mnemônicos requer suporte in-loco e, portanto, [**IOleInPlaceObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleinplaceobject).<br/> Se um controle usa qualquer propriedade de ambiente de contêiner, ele também deve implementar essa interface para receber notificações de alteração, conforme a semântica de alterações é necessária. Como as propriedades de ambiente só estão disponíveis por meio do **IDispatch** do site de contêiner, o suporte à propriedade de ambiente significa que o controle dá suporte a [**IOleObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleobject) (para obter o site), bem como a capacidade de gerar chamadas **IDispatch:: Invoke** .<br/> O método [**FreezeEvents**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrol-freezeevents) é necessário para controles que devem saber quando um contêiner não vai manipular um evento, essa é a única maneira de o controle conhecer essa condição. Se **FreezeEvents** só for necessário em isolamento, de modo que outros métodos [**IOleControl**](/windows/desktop/api/OCIdl/nn-ocidl-iolecontrol) não sejam implementados, **IOleControl** poderá ser autônomo sem [**IOleObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleobject) ou [**IOleInPlaceObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleinplaceobject).<br/> |
