# <a name="what-support-for-an-interface-means"></a>O que significa suporte para uma interface

Além da interface [**IUnknown,**](/windows/desktop/api/Unknwn/nn-unknwn-iunknown) um Objeto COM ou Controle ActiveX para esse assunto expressa qualquer funcionalidade opcional à qual ele dá suporte por meio de interfaces adicionais. Isso quer dizer que nenhuma outra interface é necessária acima **de IUnknown.** Para esse fim, a tabela a seguir lista as interfaces às quais um Controle ActiveX pode dar suporte e o que significa dar suporte a essa interface.

Como as propriedades de ambiente só estão disponíveis por meio do **IDispatch** do site de contêiner, o suporte à propriedade de ambiente significa que o controle dá suporte a [**IOleObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleobject) (para obter o site), bem como a capacidade de gerar chamadas **IDispatch:: Invoke** .<br/> O [**método FreezeEvents**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrol-freezeevents) é necessário para controles que devem saber quando um contêiner não manipulará um evento. Essa é a única maneira de o controle conhecer essa condição. Se **FreezeEvents** for necessário apenas isoladamente, de modo que outros [**métodos IOleControl**](/windows/desktop/api/OCIdl/nn-ocidl-iolecontrol) não sejam implementados, **IOleControl** poderá ficar sozinho sem [**IOleObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleobject) ou [**IOleInPlaceObject**](/windows/desktop/api/OleIdl/nn-oleidl-ioleinplaceobject).<br/> |
