# <a name="registering-objects-in-the-rot"></a><span data-ttu-id="bdb6f-103">Registrando objetos no corrompidos</span><span class="sxs-lookup"><span data-stu-id="bdb6f-103">Registering Objects in the ROT</span></span>

<span data-ttu-id="bdb6f-104">Normalmente, quando um cliente solicita que um servidor crie uma instância de objeto, o servidor normalmente cria um moniker para o objeto e o registra na tabela de objetos em execução (corrompidos) por meio de uma chamada para [**IRunningObjectTable:: Register**](/windows/desktop/api/ObjIdl/nf-objidl-irunningobjecttable-register).</span><span class="sxs-lookup"><span data-stu-id="bdb6f-104">Typically, when a client asks a server to create an object instance, the server typically creates a moniker for the object and registers it in the running object table (ROT) through a call to [**IRunningObjectTable::Register**](/windows/desktop/api/ObjIdl/nf-objidl-irunningobjecttable-register).</span></span>

<span data-ttu-id="bdb6f-105">Quando o servidor chama [**CreateFileMoniker**](/windows/desktop/api/Objbase/nf-objbase-createfilemoniker) para criar um moniker de arquivo a ser registrado no corrompidos, os servidores devem passar nomes de arquivos locais que são baseados em unidade, não no formato UNC.</span><span class="sxs-lookup"><span data-stu-id="bdb6f-105">When the server calls [**CreateFileMoniker**](/windows/desktop/api/Objbase/nf-objbase-createfilemoniker) to create a file moniker to be registered in the ROT, servers should pass local file names that are drive-based, not in UNC format.</span></span> <span data-ttu-id="bdb6f-106">Isso garante que os dados de comparação de moniker gerados pela chamada de registro corrompidos correspondam ao que é usado ao fazer uma pesquisa corrompidos na parte de um cliente remoto.</span><span class="sxs-lookup"><span data-stu-id="bdb6f-106">This ensures that the moniker comparison data that is generated by the ROT register call will match what is used while doing a ROT lookup on the part of a remote client.</span></span> <span data-ttu-id="bdb6f-107">Isso ocorre porque quando o serviço COM distribuído recebe uma solicitação de ativação de um arquivo local para o servidor de um cliente remoto, o arquivo é convertido em um caminho baseado em unidade local.</span><span class="sxs-lookup"><span data-stu-id="bdb6f-107">This is because when the distributed COM service receives an activation request for a file local to the server from a remote client, the file is converted to a local-drive-based path.</span></span>
