# <a name="security-in-com"></a><span data-ttu-id="c5bcd-103">Segurança em COM</span><span class="sxs-lookup"><span data-stu-id="c5bcd-103">Security in COM</span></span>

<span data-ttu-id="c5bcd-104">A segurança no COM é firmemente baseada na segurança fornecida pelo Windows e nos mecanismos de segurança RPC subjacentes.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-104">Security in COM is firmly based on the security provided by Windows and the underlying RPC security mechanisms.</span></span> <span data-ttu-id="c5bcd-105">A segurança COM conta COM a *autenticação* (o processo de verificar a identidade de um chamador) e a *autorização* (o processo de determinar se um chamador está autorizado a fazer o que está pedindo).</span><span class="sxs-lookup"><span data-stu-id="c5bcd-105">COM security relies on *authentication* (the process of verifying a caller's identity) and *authorization* (the process of determining whether a caller is authorized to do what it is asking to do).</span></span> <span data-ttu-id="c5bcd-106">Há dois tipos principais de segurança em COM: *segurança de ativação* e *chamada de segurança*.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-106">There are two main types of security in COM: *activation security* and *call security*.</span></span> <span data-ttu-id="c5bcd-107">A segurança de ativação determina se um cliente pode iniciar um servidor.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-107">Activation security determines whether a client can launch a server at all.</span></span> <span data-ttu-id="c5bcd-108">Depois que um servidor tiver sido iniciado, você poderá usar a segurança de chamada para controlar o acesso aos objetos de um servidor.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-108">After a server has been launched, you can use call security to control access to a server's objects.</span></span>

<span data-ttu-id="c5bcd-109">Nesse modelo de segurança, os servidores gerenciam e ajudam a proteger objetos, os clientes obtêm acesso a objetos por meio de servidores e os servidores podem tentar o acesso enquanto estão representando o cliente.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-109">In this security model, servers manage and help protect objects, clients get access to objects through servers, and servers can attempt access while impersonating the client.</span></span>

<span data-ttu-id="c5bcd-110">O sistema implementa o protocolo de autenticação Kerberos V5 e o pacote de segurança Schannel.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-110">The system implements the Kerberos v5 authentication protocol and the Schannel security package.</span></span> <span data-ttu-id="c5bcd-111">Ele também inclui recursos como representação de nível delegado, autenticação mútua, a capacidade de definir níveis de autenticação para um AppID no registro e o encobrimento.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-111">It also includes features such as delegate-level impersonation, mutual authentication, the ability to set authentication levels for an AppID in the registry, and cloaking.</span></span> <span data-ttu-id="c5bcd-112">Usando a segurança COM, você pode implementar objetos que podem executar operações privilegiadas sem comprometer a segurança.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-112">Using COM security, you can implement objects that can perform privileged operations without compromising security.</span></span>

<span data-ttu-id="c5bcd-113">Como há uma ampla variedade de recursos de segurança COM disponíveis, é útil determinar inicialmente o tipo de segurança de que seu aplicativo precisa.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-113">Because there is a wide range of COM security features available, it is helpful to initially determine what kind of security your application needs.</span></span> <span data-ttu-id="c5bcd-114">Para a maioria dos aplicativos, a definição de um nível aceitável de segurança pode ser um processo simples, mas você também pode usar a segurança COM para dar suporte a cenários de segurança muito complexos.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-114">For most applications, setting an acceptable level of security can be a painless process, but you can also use COM security to support very complex security scenarios.</span></span>

<span data-ttu-id="c5bcd-115">Você pode definir processwide de segurança usando Dcomcnfg.exe para definir o registro ou chamando [**CoInitializeSecurity**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializesecurity).</span><span class="sxs-lookup"><span data-stu-id="c5bcd-115">You can set security processwide, either by using Dcomcnfg.exe to set the registry or by calling [**CoInitializeSecurity**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializesecurity).</span></span> <span data-ttu-id="c5bcd-116">Duas interfaces primárias, [**IClientSecurity**](/windows/desktop/api/ObjIdl/nn-objidl-iclientsecurity) e [**IServerSecurity**](/windows/win32/api/objidlbase/nn-objidlbase-iserversecurity) (e funções auxiliares associadas) permitem que você defina a segurança em nível de chamada em seu programa.</span><span class="sxs-lookup"><span data-stu-id="c5bcd-116">Two primary interfaces, [**IClientSecurity**](/windows/desktop/api/ObjIdl/nn-objidl-iclientsecurity) and [**IServerSecurity**](/windows/win32/api/objidlbase/nn-objidlbase-iserversecurity) (and associated helper functions), allow you to set call-level security within your program.</span></span>
