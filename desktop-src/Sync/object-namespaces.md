<span data-ttu-id="c0033-113">Suponha que um processo solicite a criação de um namespace, NS1, que define um limite que contém dois elementos: o SID do administrador e o número da sessão atual.</span><span class="sxs-lookup"><span data-stu-id="c0033-113">Suppose that a process requests the creation of a namespace, NS1, that defines a boundary containing two elements: the administrator especificada.</span><span class="sxs-lookup"><span data-stu-id="c0033-114">The namespace is created if the process is running under the Administrator account in the specified session.</span></span> <span data-ttu-id="c0033-115">Outro processo pode acessar esse namespace usando a função [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) .</span><span class="sxs-lookup"><span data-stu-id="c0033-115">Another process can access this namespace using the [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) <span data-ttu-id="c0033-118">Os objetos criados nesse namespace têm nomes que são do *prefixo* de formulário \\ *objectname*.</span><span class="sxs-lookup"><span data-stu-id="c0033-118">The objects that are created in this namespace have names that are of the form *prefix*\\*objectname*.</span></span> <span data-ttu-id="c0033-119">O prefixo é o nome do namespace especificado pelo parâmetro *lpAliasPrefix* de [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea).</span><span class="sxs-lookup"><span data-stu-id="c0033-119">The prefix is the namespace name specified by the *lpAliasPrefix* parameter of [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea).</span></span> <span data-ttu-id="c0033-120">Por exemplo, para criar um objeto de evento chamado MyEvent no namespace NS1, chame a função [**CreateEvent**](/windows/win32/api/synchapi/nf-synchapi-createeventa) com o parâmetro *LPNAME* definido como ns1 \\ MyEvent.</span><span class="sxs-lookup"><span data-stu-id="c0033-120">For example, to create an event object named MyEvent in the NS1 namespace, call the [**CreateEvent**](/windows/win32/api/synchapi/nf-synchapi-createeventa) function with the *lpName* parameter set to NS1\\MyEvent.</span></span>

<span data-ttu-id="c0033-121">O processo que criou o namespace pode usar a função [**ClosePrivateNamespace**](/windows/win32/api/namespaceapi/nf-namespaceapi-closeprivatenamespace) para fechar o identificador para o namespace.</span><span class="sxs-lookup"><span data-stu-id="c0033-121">The process that created the namespace can use the [**ClosePrivateNamespace**](/windows/win32/api/namespaceapi/nf-namespaceapi-closeprivatenamespace) function to close the handle to the namespace.</span></span> <span data-ttu-id="c0033-122">O identificador também é fechado quando o processo que criou o namespace é encerrado.</span><span class="sxs-lookup"><span data-stu-id="c0033-122">The handle is also closed when the process that created the namespace terminates.</span></span> <span data-ttu-id="c0033-123">Depois que o identificador do namespace for fechado, as chamadas subsequentes para [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) falharão, mas todas as operações em objetos no namespace serão realizadas com êxito.</span><span class="sxs-lookup"><span data-stu-id="c0033-123">After the namespace handle is closed, subsequent calls to [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) fail, but all operations on objects in the namespace succeed.</span></span>
