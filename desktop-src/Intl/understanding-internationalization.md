# <a name="understanding-internationalization"></a><span data-ttu-id="406fb-103">Entendendo a internacionalização</span><span class="sxs-lookup"><span data-stu-id="406fb-103">Understanding Internationalization</span></span>

<span data-ttu-id="406fb-104">O desenvolvimento de produtos de software internacionalizados requer não apenas a produção do código correto, mas também um design que permite que o produto ofereça suporte a várias localidades.</span><span class="sxs-lookup"><span data-stu-id="406fb-104">Developing internationalized software products requires not only the production of correct code, but also a design that permits the product to support many locales.</span></span> <span data-ttu-id="406fb-105">Os desenvolvedores e seus gerentes devem estar cientes do nível de esforço e de atenção para os detalhes necessários para criar aplicativos preparados para o mundo, sejam eles entregues como binários únicos prontos para uso em vários mercados diferentes, ou como vários binários específicos de uma localidade.</span><span class="sxs-lookup"><span data-stu-id="406fb-105">Developers and their managers should be aware of the level of effort and attention to detail required to create world-ready applications, whether they deliver them as single binaries ready for use in many different markets, or as multiple binaries that are each specific to one locale.</span></span> <span data-ttu-id="406fb-106">Como desenvolvedor, certifique-se de que seu gerenciamento entenda que a internacionalização envolve mais do que apenas o desenvolvimento de código.</span><span class="sxs-lookup"><span data-stu-id="406fb-106">As a developer, make sure your management understands that internationalization involves more than just code development.</span></span> <span data-ttu-id="406fb-107">A criação de internacionalização no início do seu ciclo de produto economiza tempo, dinheiro e esforço.</span><span class="sxs-lookup"><span data-stu-id="406fb-107">Designing for internationalization at the beginning of your product cycle saves you time, money, and effort.</span></span>

<span data-ttu-id="406fb-108">A **internacionalização** é a criação de software que se adapta a localidades em todo o mundo.</span><span class="sxs-lookup"><span data-stu-id="406fb-108">**Internationalization** is the creation of software that adapts to locales around the world.</span></span> <span data-ttu-id="406fb-109">Uma **localidade** é uma coleção de preferências de usuário relacionadas a idiomas.</span><span class="sxs-lookup"><span data-stu-id="406fb-109">A **locale** is a collection of language-related user preferences.</span></span> <span data-ttu-id="406fb-110">Uma localidade é especificada pelo emparelhamento de um idioma e uma região geográfica.</span><span class="sxs-lookup"><span data-stu-id="406fb-110">A locale is specified by the pairing of a language and a geographical region.</span></span> <span data-ttu-id="406fb-111">Por exemplo, inglês (Estados Unidos) e inglês (Reino Unido) são duas localidades diferentes, como Inglês (Canadá) e francês (Canadá).</span><span class="sxs-lookup"><span data-stu-id="406fb-111">For example, English (United States) and English (United Kingdom) are two different locales, as are English (Canada) and French (Canada).</span></span>

<span data-ttu-id="406fb-112">O software internacionalizado tem estes atributos:</span><span class="sxs-lookup"><span data-stu-id="406fb-112">Internationalized software has these attributes:</span></span>

-   <span data-ttu-id="406fb-113">**A preparação para o mundo** abrange o design e a implementação que separam código independente de localidade de recursos dependentes de localidade e abrange duas áreas principais:</span><span class="sxs-lookup"><span data-stu-id="406fb-113">**World-readiness** covers design and implementation that separate locale-independent code from locale-dependent resources, and comprises two major areas:</span></span>
    -   <span data-ttu-id="406fb-114">A **globalização** é o processo de desenvolvimento de um núcleo de programa com recursos e design de código que são independentes de idioma ou localidade.</span><span class="sxs-lookup"><span data-stu-id="406fb-114">**Globalization** is the process of developing a program core with features and code design that are independent of language or locale.</span></span> <span data-ttu-id="406fb-115">Em vez disso, o design dá suporte à entrada, exibição e saída de scripts de idioma com suporte Unicode, bem como a dados relacionados a localidades específicas.</span><span class="sxs-lookup"><span data-stu-id="406fb-115">Instead, the design supports the input, display, and output of Unicode-supported language scripts, as well as data related to specific locales.</span></span>
    -   <span data-ttu-id="406fb-116">A **possibilidade de localização** é o design da base de código de software e dos recursos, de modo que um programa possa ser localizado, conforme definido abaixo, em diferentes edições de idioma, sem nenhuma alteração no código-fonte.</span><span class="sxs-lookup"><span data-stu-id="406fb-116">**Localizability** is the design of the software code base and resources such that a program can be localized, as defined below, into different language editions without any changes to the source code.</span></span> <span data-ttu-id="406fb-117">Por exemplo, buffers de cadeia de caracteres no código, e caixas de texto na interface do usuário, devem ser grandes o suficiente para acomodar cadeias de caracteres de texto mais longas em linguagens como alemão ou holandês.</span><span class="sxs-lookup"><span data-stu-id="406fb-117">For example, string buffers in the code, and text boxes in the UI, must be large enough to accommodate longer text strings in languages such as German or Dutch.</span></span>
-   <span data-ttu-id="406fb-118">**Localização** Isso envolve a tradução e a personalização de um produto para um mercado específico, e trata-se principalmente da criação de arquivos de recursos em vez de escrever código-fonte.</span><span class="sxs-lookup"><span data-stu-id="406fb-118">**Localization** This involves translating and customizing a product for a specific market, and it is mainly about creating resource files rather than writing source code.</span></span>

<span data-ttu-id="406fb-119">Por exemplo, o uso do NLS (suporte ao idioma nacional) fornecido pela API do Microsoft Win32 é um processo de desenvolvimento de software que dá suporte à preparação para o mundo, ao passo que modificar os elementos da interface do usuário, traduzir texto e padronização de terminologia são etapas de localização geralmente executadas por alguém que é especialista em idioma, como um tradutor.</span><span class="sxs-lookup"><span data-stu-id="406fb-119">For example, using the National Language Support (NLS) supplied by the Microsoft Win32 API is a software development process that supports world-readiness, whereas modifying the UI elements, translating text, and standardizing terminology are localization steps usually performed by someone who is a language specialist, such as a translator.</span></span> <span data-ttu-id="406fb-120">Mesmo quando o seu desenvolvimento de código enfatiza a preparação para o mundo, você precisa entender a localização porque suas decisões de design afetam o trabalho do tradutor.</span><span class="sxs-lookup"><span data-stu-id="406fb-120">Even when your code development emphasizes world-readiness, you need to understand localization because your design decisions affect the work of the translator.</span></span>
