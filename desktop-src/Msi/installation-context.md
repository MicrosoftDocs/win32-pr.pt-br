# <a name="installation-context"></a>Contexto de instalação

Windows Installer pode instalar um pacote em um computador em dois contextos de instalação: por máquina e por usuário. Uma instalação por computador do pacote é necessária para permitir que todos os usuários do computador acessem e usem o aplicativo. Como uma instalação por computador faz alterações no sistema que afetam todos os usuários, os usuários padrão com privilégios limitados podem ser impedidos de instalar um pacote no contexto por máquina sem primeiro obter permissão.

Você pode especificar o contexto de instalação criando o pacote para a instalação por usuário ou por computador e usando as propriedades [**AllUsers**](allusers.md) e [**MSIINSTALLPERUSER**](msiinstallperuser.md) . Com base nessas propriedades, Windows Installer redireciona automaticamente os valores de propriedades de pasta e registros para locais para o contexto por usuário ou por máquina.

> [!Note]  
> A propriedade [**MSIINSTALLPERUSER**](msiinstallperuser.md) , disponível a partir do Windows Installer 5,0 e do windows Server 2008 R2 e do Windows 7, pode facilitar o desenvolvimento de um único pacote capaz de ser instalado no contexto por máquina ou por usuário. Para obter informações sobre como desenvolver um pacote de uso duplo que dá ao usuário a capacidade de escolher o contexto de instalação no momento da instalação, consulte [criação de pacote único](single-package-authoring.md). Windows Installer ignora a propriedade **MSIINSTALLPERUSER** se o valor de [**AllUsers**](allusers.md) não for 2. Windows Installer sempre redefine o valor de **AllUsers** como 1 quando ele é instalado no contexto por máquina e redefine o valor de **AllUsers** para uma cadeia de caracteres vazia ("") quando ele é instalado no contexto por usuário.

A tabela a seguir compara os locais de atalhos para contextos de instalação por máquina e por usuário.



| Contexto de instalação do Per-Machine (ALLUSERS = 1)                                                             | Contexto de instalação do Per-User (ALLUSERS = "")                                                                                     |
|-----------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------|
| Os aplicativos aparecem em Adicionar/remover programas no painel de controle para todos os usuários do computador. <br/> | Os aplicativos aparecem somente em Adicionar/remover programas no painel de controle para usuários que instalaram os aplicativos. <br/> |
| Os atalhos são instalados no perfil todos os usuários.<br/>                                              | Os atalhos são instalados somente no perfil desse usuário.<br/>                                                                 |
| Os ícones e as transformações são armazenados em% WINDOWS% \\ Installer \\ {ProductCode}.<br/>                        | Os ícones e as transformações são armazenados em% USERPROFILE% \\ dados de aplicativo \\ Microsoft \\ Installer \\ {ProductCode GUID}<br/>         |

## <a name="folder-redirection"></a>Redirecionamento de pasta

Windows Installer define os valores das propriedades de pasta para o caminho completo da respectiva pasta para o contexto de instalação.

> [!Note]  
> As pastas são identificadas por suas constantes [**KNOWNFOLDERID**](../shell/knownfolderid.md) e [**CSIDL**](/windows/win32/api/shlobj_core/nf-shlobj_core-shgetfolderpatha) . A partir do Windows Vista, os aplicativos devem usar a função [**SHGetKnownFolderPath**](/windows/win32/api/shlobj_core/nf-shlobj_core-shgetknownfolderpath) e o **KNOWNFOLDERID** para determinar o caminho completo para as pastas especiais. Os aplicativos existentes que usam a função **SHGetFolderPath** e as IDs de item especial de constante ([**CSIDL**](/windows/win32/api/shlobj_core/nf-shlobj_core-shgetfolderpatha)) continuarão a funcionar.
