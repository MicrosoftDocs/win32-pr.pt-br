# <a name="radiobuttongroup-control"></a>Controle de botão de opção

O controle RadioButton é um grupo de botões de opção. Ele permite que o usuário selecione uma cadeia de caracteres ou um valor inteiro para uma propriedade de uma lista predeterminada de valores. Você pode associar esse controle a uma propriedade inserindo o nome da propriedade na coluna propriedade da tabela de [controle](control-table.md). Especifique os valores possíveis para a seleção na coluna valor da [tabela RadioButton](radiobutton-table.md). Observe que a cadeia de caracteres exibida não é necessariamente igual ao valor que o usuário está escolhendo.

Cada controle de botão de opção é associado a uma propriedade. O valor padrão para essa propriedade deve ser inicializado na [tabela de propriedades](property-table.md). Em cada um dos botões de opção especificados na [tabela RadioButton](radiobutton-table.md), pode haver um botão de opção com um valor no campo Value que corresponde ao valor padrão dessa propriedade. Este é o botão padrão para o controle de botão de opção. O botão **padrão** é mostrado inicialmente como selecionado no controle.

Observe que o foco em uma caixa de diálogo não pode ser movido para um controle de grupo de botões até que um dos botões no grupo tenha sido selecionado. Para fazer com que o foco se mova para esse grupo de botões, especifique um dos botões como um botão padrão para o grupo.

O botão de opção controla apenas valores de propriedade definidos e não pode ser usado para enviar um [ControlEvent,](control-events.md).

A implementação trata o grupo inteiro como um controle, portanto, não é possível ocultar ou Desabilitar botões individuais dentro do grupo. Da mesma forma, todos os botões precisam ser do mesmo estilo, ou seja, todos eles têm texto ou todos eles têm bitmaps (ou outros recursos semelhantes). A posição dos botões é criada em relação ao grupo. Dessa forma, todo o grupo pode ser movido alterando apenas as coordenadas do grupo sem alterar os botões individuais. Na criação, o controle verifica se os botões individuais não se estendem além dos limites do grupo.

## <a name="remarks"></a>Comentários

Devido à maneira como o Windows desenha o quadro, há uma lacuna entre a parte superior da janela do controle e o quadro visível, mesmo quando não há nenhuma legenda.

Esse controle pode ser criado a partir da classe BUTTON usando a função [**CreateWindowEx**](/windows/win32/api/winuser/nf-winuser-createwindowexa) . Se o bit [HasBorder](hasborder-control-attribute.md) for definido, ele terá o estilo de **\_ GroupBox da BS** , caso contrário, ele terá o estilo **BS \_ OWNERDRAW** .

O controle de botão de opção não deve sobrepor outros controles e outros controles não devem sobrepor um botão de opção. Sobrepor esse controle e outro pode fazer com que os controles funcionem ou sejam exibidos incorretamente. Para fornecer programas de leitor de tela com texto descritivo extra sobre um controle de botão de opção, siga o exemplo fornecido em [adicionando texto extra aos botões de opção](adding-extra-text-to-radio-buttons.md).
