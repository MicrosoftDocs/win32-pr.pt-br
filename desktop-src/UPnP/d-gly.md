# <a name="d-upnp-apis"></a>D (APIs UPnP)

Contém termos relacionados a UPnP que começam com a letra D.

[A](a-gly.md) B [C](c-gly.md) D [E](e-gly.md) F G [H](h-gly.md) i J K L M [N](n-gly.md) O [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.d_1_gly"></span><span id="UPNP.D_1_GLY"></span>**Vice**
</dt> <dd>

Um dispositivo é um contêiner de outros serviços ou dispositivos. Um dispositivo pode conter outros dispositivos lógicos.

</dd> <dt>

<span id="upnp.d_2_gly"></span><span id="UPNP.D_2_GLY"></span>**objeto de controle de dispositivo**
</dt> <dd>

Um objeto de controle de dispositivo é o ponto central de gerenciamento para os objetos de serviço de um dispositivo. Quando um dispositivo é registrado, seu objeto de controle de dispositivo é passado para o host do dispositivo.

</dd> <dt>

<span id="upnp.d_3_gly"></span><span id="UPNP.D_3_GLY"></span>**Descrição do dispositivo**
</dt> <dd>

Uma descrição do dispositivo é uma lista das propriedades genéricas de todos os dispositivos em um dispositivo físico, incluindo os serviços, a hierarquia do dispositivo e suas propriedades.

</dd> <dt>

<span id="upnp.d_4_gly"></span><span id="UPNP.D_4_GLY"></span>**host do dispositivo**
</dt> <dd>

O host do dispositivo manipula a descoberta, a descrição, o controle e as partes de eventos da funcionalidade de um dispositivo. Os dispositivos contêm o código para lidar com a funcionalidade específica do dispositivo, como a rebobinação de um videocassete.

</dd> <dt>

<span id="upnp.d_5_gly"></span><span id="UPNP.D_5_GLY"></span>**provedor de dispositivo**
</dt> <dd>

Um provedor de dispositivo é uma entidade que é registrada com o host do dispositivo. Este provedor de dispositivo não está publicado na rede. No entanto, o provedor do dispositivo registra outros dispositivos e os publica na rede.

</dd> <dt>

<span id="upnp.d_6_gly"></span><span id="UPNP.D_6_GLY"></span>**tipo de dispositivo**
</dt> <dd>

Um tipo de dispositivo padrão é indicado por urn: schemas-UPnP-org: Device: seguido por um nome exclusivo atribuído pelo Fórum UPnP.

Há uma relação um-para-um entre modelos de dispositivo baseado em UPnP e tipos de dispositivo. Os fornecedores UPnP podem especificar tipos de dispositivos adicionais; Eles são indicados por urn: nome-do-domínio: dispositivo: seguido de um nome exclusivo atribuído pelo fornecedor, em que Domain-Name é um nome de domínio registrado para o fornecedor.

</dd> </dl>
