# <a name="s-upnp-apis"></a>S (APIs UPnP)

Contém termos relacionados a UPnP que começam com a letra S.

[A](a-gly.md) B [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F G [H](h-gly.md) i J K L M [N](n-gly.md) O [P](p-gly.md) Q [R](r-gly.md) S T [U](u-gly.md) V W X Y Z

<dl> <dt>

<span id="upnp.s_1_gly"></span><span id="UPNP.S_1_GLY"></span>**serviço**
</dt> <dd>

Um serviço é uma unidade funcional lógica. Os serviços expõem ações e modelam alguns ou todos os Estados de um dispositivo físico usando variáveis de estado.

</dd> <dt>

<span id="upnp.s_2_gly"></span><span id="UPNP.S_2_GLY"></span>**Descrição do serviço**
</dt> <dd>

Uma descrição de serviço é uma lista das ações que um serviço fornece e as variáveis de estado associadas às ações.

</dd> <dt>

<span id="upnp.s_3_gly"></span><span id="UPNP.S_3_GLY"></span>**objeto de serviço**
</dt> <dd>

Um objeto de serviço é a representação da API do ponto de controle de um serviço. Um objeto de serviço implementa a interface [**IUPnPService**](/windows/desktop/api/Upnp/nn-upnp-iupnpservice) .

</dd> <dt>

<span id="upnp.s_4_gly"></span><span id="UPNP.S_4_GLY"></span>**tipo de serviço**
</dt> <dd>

Um tipo de serviço é um URN que identifica um serviço. Há uma relação um-para-um entre um modelo de serviço UPnP e um tipo de serviço. Vários tipos de serviço padrão são definidos pelos comitês de trabalho do fórum UPnP. Os tipos de serviço estão no formato urn: schemas-UPnP-org: Service: serviceType: Version. Por exemplo, um tipo de serviço do verificador poderia ser urn: schemas-UPnP-org: Service: scanner: 1.

Os fornecedores UPnP podem especificar serviços adicionais. Esses serviços são indicados por urn: nome-do-domínio: serviço: serviceType: versão em que Domain-Name é um nome de domínio registrado para o fornecedor.

</dd> <dt>

<span id="upnp.s_5_gly"></span><span id="UPNP.S_5_GLY"></span>**variável de estado**
</dt> <dd>

Uma variável de estado é um dado que descreve alguma parte do estado de um serviço.

</dd> </dl>
