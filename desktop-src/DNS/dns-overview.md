# <a name="dns-overview"></a><span data-ttu-id="e171e-103">Visão geral do DNS</span><span class="sxs-lookup"><span data-stu-id="e171e-103">DNS Overview</span></span>

<span data-ttu-id="e171e-104">O DNS é um serviço padrão do setor usado para localizar computadores em uma rede baseada em IP (Internet Protocol).</span><span class="sxs-lookup"><span data-stu-id="e171e-104">DNS is an industry-standard service used to locate computers on an Internet Protocol (IP)-based network.</span></span> <span data-ttu-id="e171e-105">Redes IP, como as redes Internet e Windows 2000, dependem de endereços baseados em número, como 207.46.131.137 para ferry informações em toda a rede.</span><span class="sxs-lookup"><span data-stu-id="e171e-105">IP networks, such as the Internet and Windows 2000 networks, rely on number-based addresses, such as 207.46.131.137 to ferry information throughout the network.</span></span> <span data-ttu-id="e171e-106">Os usuários de rede dependem de nomes baseados em caracteres, como `www.microsoft.com` .</span><span class="sxs-lookup"><span data-stu-id="e171e-106">Network users rely on character-based names, such as `www.microsoft.com`.</span></span> <span data-ttu-id="e171e-107">Portanto, é necessário converter caracteres ou endereços amigáveis ( `www.microsoft.com` ) em endereços com base em números (207.46.131.137) que a rede pode reconhecer.</span><span class="sxs-lookup"><span data-stu-id="e171e-107">Therefore, it is necessary to translate character or user-friendly addresses (`www.microsoft.com`) into the number-based addresses (207.46.131.137) that the network can recognize.</span></span> <span data-ttu-id="e171e-108">O DNS é o serviço de sua escolha no Windows 2000 para localizar recursos e convertê-los em seus endereços IP correspondentes.</span><span class="sxs-lookup"><span data-stu-id="e171e-108">DNS is the service of choice in Windows 2000 to locate resources and translate them into their corresponding IP addresses.</span></span>

<span data-ttu-id="e171e-109">O DNS usa um banco de dados especializado de registros de recursos, comumente conhecido como RRs, para responder a consultas de resolução de nomes de cliente.</span><span class="sxs-lookup"><span data-stu-id="e171e-109">DNS uses a specialized database of resource records, commonly referred to simply as RRs, to respond to client name-resolution queries.</span></span> <span data-ttu-id="e171e-110">Antes do DNS, a resolução de nomes na Internet foi obtida com o [*arquivo de hosts*](h-gly.md), que são manualmente criados para os nomes de host associados aos endereços IP.</span><span class="sxs-lookup"><span data-stu-id="e171e-110">Before DNS, name resolution on the Internet was achieved with [*hosts file*](h-gly.md), which are manually created files that associated host names with IP addresses.</span></span>

<span data-ttu-id="e171e-111">Quando um novo cliente foi adicionado à rede, um administrador precisava atualizar manualmente o arquivo de hosts e, em seguida, copiar (replicar) esse arquivo para todos os outros computadores na rede para que o novo host pudesse ser alcançado por todos.</span><span class="sxs-lookup"><span data-stu-id="e171e-111">When a new client was added to the network, an administrator had to manually update the hosts file and then copy (replicate) that file to all other computers on the network so that the new host could be reached by all.</span></span> <span data-ttu-id="e171e-112">À medida que a Internet cresceu, essa forma de resolução de nomes era claramente insuficiente; Ele era muito intensivo de gerenciamento e não foi [*dimensionado*](s-gly.md).</span><span class="sxs-lookup"><span data-stu-id="e171e-112">As the Internet grew, this form of name resolution was clearly insufficient; it was too management intensive, and it did not [*scale*](s-gly.md).</span></span> <span data-ttu-id="e171e-113">O arquivo de hosts acabou de ser maior, e como ele usou um [*espaço de nome simples*](f-gly.md) (consulte também o [espaço de nome](name-space.md)), ele não pôde ser particionado e teve que ser distribuído em sua totalidade.</span><span class="sxs-lookup"><span data-stu-id="e171e-113">The hosts file just got bigger, and because it used a [*flat name space*](f-gly.md) (see also [Name Space](name-space.md)), it could not be partitioned and had to be distributed in its entirety.</span></span> <span data-ttu-id="e171e-114">A solução era DNS.</span><span class="sxs-lookup"><span data-stu-id="e171e-114">The solution was DNS.</span></span>

-   <span data-ttu-id="e171e-115">O DNS substituiu o espaço de nome simples do arquivo de hosts por um [*espaço de nome hierárquico*](h-gly.md).</span><span class="sxs-lookup"><span data-stu-id="e171e-115">DNS replaced the hosts file's flat name space with a [*hierarchical name space*](h-gly.md).</span></span> <span data-ttu-id="e171e-116">Com um espaço de nome hierárquico, as informações sobre nomes de host e endereços IP podem ser particionadas e distribuídas; Portanto, a escalabilidade é alcançada.</span><span class="sxs-lookup"><span data-stu-id="e171e-116">With a hierarchical name space, information about host names and IP addresses can be partitioned and distributed; thus, scalability is achieved.</span></span> <span data-ttu-id="e171e-117">Por exemplo, no domínio fictício widgets.products.microsoft.com, a responsabilidade pela resolução de nomes pode ser particionada para que vários servidores possam lidar com a resolução de nomes para diferentes partes do espaço de nome:</span><span class="sxs-lookup"><span data-stu-id="e171e-117">For example, in the fictional widgets.products.microsoft.com domain, responsibility for name resolution can be partitioned so that various servers can handle name resolution for different parts of the name space:</span></span>
    -   <span data-ttu-id="e171e-118">Um servidor pode ser responsável por resolver a primeira parte (microsoft.com) e, em seguida, pode encaminhar a solicitação de resolução de nomes para o próximo servidor DNS na hierarquia.</span><span class="sxs-lookup"><span data-stu-id="e171e-118">One server can be responsible for resolving the first part (microsoft.com), and can then forward the name-resolution request to the next DNS Server in the hierarchy.</span></span>
    -   <span data-ttu-id="e171e-119">O próximo servidor DNS pode ser responsável por resolver a próxima parte do espaço de nome (produtos).</span><span class="sxs-lookup"><span data-stu-id="e171e-119">The next DNS Server can be responsible for resolving the next part of the name space (products).</span></span>
    -   <span data-ttu-id="e171e-120">Por fim, a solicitação pode ser encaminhada para um terceiro servidor DNS responsável por resolver a última parte do nome (widgets).</span><span class="sxs-lookup"><span data-stu-id="e171e-120">Finally, the request can be forwarded to a third DNS server that is responsible for resolving the last part of the name (widgets).</span></span>

<span data-ttu-id="e171e-121">Os servidores DNS em cada parte do espaço de nomes hierárquicos precisam manter um banco de dados de registros de recursos para hosts, mas apenas em sua parte da hierarquia.</span><span class="sxs-lookup"><span data-stu-id="e171e-121">DNS Servers in each part of the hierarchical name space need to maintain a database of resource records for hosts, but only in their part of the hierarchy.</span></span> <span data-ttu-id="e171e-122">Portanto, o servidor (ou servidores) na parte produtos da widgets.products.microsoft.com mantém RRs somente para os produtos que fazem parte do espaço de nome hierárquico — não para a parte microsoft.com ou os widgets que fazem parte do espaço de nome.</span><span class="sxs-lookup"><span data-stu-id="e171e-122">Thus, the server (or servers) in the products part of widgets.products.microsoft.com maintain RRs for only the products part of the hierarchical name space — not for the microsoft.com part or the widgets part of the name space.</span></span>
