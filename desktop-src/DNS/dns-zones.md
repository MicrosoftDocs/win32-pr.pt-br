<span data-ttu-id="eedcd-105">Uma zona DNS é um conjunto de arquivos ou registros (mais precisamente, um banco de dados de entradas de registro de recurso) que corresponde a parte do namespace hierárquico do DNS.</span><span class="sxs-lookup"><span data-stu-id="eedcd-105">A DNS zone is a set of files or records (more precisely, a database of resource record entries) that corresponds to part of the DNS hierarchical namespace.</span></span> <span data-ttu-id="eedcd-106">As zonas DNS são usadas para delinear quais servidores DNS são responsáveis (autoritativos) para resolver consultas de resolução de nomes para uma determinada seção da hierarquia de DNS.</span><span class="sxs-lookup"><span data-stu-id="eedcd-106">DNS zones are used to delineate which DNS Servers are responsible (authoritative) for resolving name-resolution queries for a given section of the DNS hierarchy.</span></span> <span data-ttu-id="eedcd-107">As zonas DNS diferem da estrutura de domínio do seguinte modo: as zonas podem ser compostas de um ou mais domínios DNS.</span><span class="sxs-lookup"><span data-stu-id="eedcd-107">DNS zones differ from the domain structure in the following fashion: zones can be composed of one or more DNS domains.</span></span> <span data-ttu-id="eedcd-108">Uma zona na árvore de domínio gadgets.widgets.microsoft.com pode ser autoritativa para os domínios de gadgets e widgets.</span><span class="sxs-lookup"><span data-stu-id="eedcd-108">One zone in the gadgets.widgets.microsoft.com domain tree might be authoritative for the gadgets and widgets domains.</span></span> <span data-ttu-id="eedcd-109">Em outras palavras, não há um requisito para que as zonas DNS tenham uma relação um-para-um com domínios DNS.</span><span class="sxs-lookup"><span data-stu-id="eedcd-109">In other words, there is not a requirement for DNS zones to have a one-to-one relationship with DNS domains.</span></span>
