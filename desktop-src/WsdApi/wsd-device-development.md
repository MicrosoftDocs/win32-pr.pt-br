# <a name="wsd-device-development"></a>Desenvolvimento de dispositivos WSD

Os fabricantes de dispositivos e outros implementadores de dispositivos podem criar dispositivos que interoperam com Windows e hosts WSDAPI baseados em Windows sem usar o WSDAPI.

Além de desenvolver a funcionalidade do dispositivo, os implementadores de dispositivo devem implementar o DPWS (Perfil de Dispositivos [para Serviços Web).](https://specs.xmlsoap.org/ws/2006/02/devprof/)

Sempre que possível, use o WSDAPI para criar um aplicativo de host de dispositivo ou cliente WSD baseado em Windows. O uso do WSDAPI reduz a complexidade do desenvolvimento e economiza tempo. Para obter mais informações, consulte [Desenvolvimento de aplicativos do WSD Windows](wsd-application-development-on-windows.md).
