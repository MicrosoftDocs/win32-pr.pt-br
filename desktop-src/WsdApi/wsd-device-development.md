# <a name="wsd-device-development"></a>Desenvolvimento de dispositivo WSD

Os fabricantes de dispositivos e outros implementadores de dispositivos podem criar dispositivos que interoperem com clientes e hosts WSDAPI baseados no Windows sem usar o WSDAPI.

Além de desenvolver a funcionalidade do dispositivo, os implementadores de dispositivos devem implementar o [perfil de dispositivos para serviços Web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS).

Sempre que possível, use o WSDAPI para criar um aplicativo de host de dispositivo ou cliente WSD baseado no Windows. O uso de WSDAPI reduz a complexidade do desenvolvimento e economiza tempo. Para obter mais informações, consulte [desenvolvimento de aplicativos WSD no Windows](wsd-application-development-on-windows.md).
