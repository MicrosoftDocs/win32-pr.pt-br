# <a name="wsdapi-xml-type-bytecodes"></a>Códigos de byte de tipo XML WSDAPI

Esses códigos de byte são usados pelo WSDAPI para analisar e gerar XML usando tabelas de código de byte que descrevem tipos complexos.

A tabela de código de byte para um tipo expressa as representações binárias (na memória) e XML (na transmissão) do tipo. Uma tabela de código de byte consiste em uma lista de operações encerradas por uma operação especial, ou seja, `OpEndOfTable` . Uma operação consiste em um código de operação de um byte seguido por zero ou mais bytes de dados de argumento.



| Bytecode                                     | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

| OpNone                                       | Não corresponde a nada.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| OpEndOfTable                                 | Indica o final de uma tabela de código de byte.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| OpBeginElement (nome)                        | Corresponde a um token inicial de elemento com o nome indicado. O nome aparece na tabela em formato codificado em 4 byte. OnBeginElement inicia uma cláusula que termina com um OpEndElement correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginElement e seu OpEndElement correspondente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| OpBeginAnyElement                            | Corresponde a um token inicial de elemento com qualquer nome. OnBeginAnyElement inicia uma cláusula que termina com um OpEndElement correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginAnyElement e seu OpEndElement correspondente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| OpEndElement                                 | Termina uma cláusula iniciada com OpBeginElement ou OpBeginAnyElement.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| OpElement (nome)                             | Corresponde a um elemento inteiro com o nome indicado. O nome aparece na tabela em formato codificado em 4 byte.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |

| OpAnyText                                    | Corresponde a um token de texto.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| OpAttribute (nome)                           | Corresponde ao token de rótulo de um atributo com o nome indicado. OpAttribute inicia uma cláusula que inclui a cláusula subsequente na tabela. A cláusula subsequente é usada para corresponder à parte do valor do atributo. As cláusulas OpAttribute sempre aparecem após as operações OpBeginElement ou OpBeginAnyElement ou após outra cláusula OpAttribute.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| OpBeginChoice                                | OpBeginChoice inicia uma cláusula que termina com um OpEndChoice correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginChoice e seu OpEndChoice correspondente. A cláusula externa corresponde aos tokens que corresponderam a qualquer uma das cláusulas internas. Todas as cláusulas internas devem começar com OpBeginElement, exceto que a última pode ser OpAnything. Esse constructo corresponde à partícula de escolha XSD.                                                                                                                                                                                                                                                                                                                                                                                                                             |
| OpEndChoice                                  | Termina uma cláusula iniciada com OpBeginChoice.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| OpBeginSequence                              | OpBeginSequence inicia uma cláusula que termina com um OpEndSequence correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginSequence e OpEndSequence. A cláusula externa corresponde aos tokens de acordo com todas as cláusulas internas em sequência. Esse constructo corresponde à partícula de sequência XSD.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| OpEndSequence                                | Termina uma cláusula iniciada com OpBeginSequence.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| OpBeginAll                                   | OpBeginAll inicia uma cláusula que termina com um OpEndAll correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginAll e OpEndAll. A cláusula externa corresponde aos tokens de acordo com as cláusulas internas em qualquer sequência. Operadores de ocorrência relacionados a cada cláusula interna indicam quantas vezes os tokens para cada cláusula podem ocorrer. O padrão é uma vez. Várias ocorrências de uma cláusula especificada podem ser intercaladas com ocorrências de outras cláusulas internas. Todas as cláusulas internas devem começar com OpBeginElement, exceto que a última pode ser OpAnything. Esse constructo corresponde à partícula toda XSD.                                                                                                                                                                                                        |
| OpEndAll                                     | Termina uma cláusula iniciada com OpBeginAll.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| OpAnyany                                   | Corresponde a qualquer número de elementos e tokens de texto.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| OpAnyNumber                                  | Indica que a cláusula subsequente pode ocorrer qualquer número de vezes. OpAnyNumber inicia uma cláusula que termina no final da cláusula subsequente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |

OpOneOrMore inicia uma cláusula que termina no final da cláusula subsequente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| OpOptional                                   | Indica que a cláusula subsequente pode ocorrer zero ou uma vez. OpOptional inicia uma cláusula que termina no final da cláusula subsequente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |

Um argumento de deslocamento de 4 byte indica o deslocamento da representação binária no contexto binário atual. A macro OpFormatInt16 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| OpFormatInt32 (tipo,campo)                   | Corresponde a um inteiro com sinal de 32 bits em um token de texto. Um argumento de deslocamento de 4 byte indica o deslocamento da representação binária no contexto binário atual. A macro OpFormatInt32 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| OpFormatInt64 (tipo,campo)                   | Corresponde a um inteiro com sinal de 64 bits em um token de texto. Um argumento de deslocamento de 4 byte indica o deslocamento da representação binária no contexto binário atual. A macro OpFormatInt64 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| OpFormatUInt8 (tipo,campo)                   | Corresponde a um inteiro sem sinal de 8 bits em um token de texto. Um argumento de deslocamento de 4 byte indica o deslocamento da representação binária no contexto binário atual. A macro OpFormatUInt8 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |

Um argumento de deslocamento de 4 byte indica o deslocamento da representação binária no contexto binário atual. A macro OpFormatUInt16 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

A macro OpFormatUInt32 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

Um argumento de deslocamento de 4 byte indica o deslocamento da representação binária no contexto binário atual. A macro OpFormatUInt64 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

Um argumento de deslocamento de 4 byte indica o deslocamento em que um ponteiro para a cadeia de caracteres aparece no contexto binário atual. A macro OpFormatUnicodeString calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento .                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
