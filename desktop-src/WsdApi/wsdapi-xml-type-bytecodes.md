# <a name="wsdapi-xml-type-bytecodes"></a>Códigos de bytes de tipo XML WSDAPI

Esses códigos de bytes são usados pelo WSDAPI para analisar e gerar XML usando tabelas de código de bytes que descrevem tipos complexos.

A tabela de código de bytes de um tipo expressa as representações binária (na memória) e XML (durante a transmissão) do tipo. Uma tabela de código de bytes consiste em uma lista de operações encerradas por uma operação especial, ou seja, `OpEndOfTable` Uma operação consiste em um código de operação de um byte seguido por zero ou mais bytes de dados de argumento.



| Verifica                                     | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

| OpNone                                       | Não corresponde a nada.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |

| OpBeginElement (nome)                        | Corresponde a um token de início de elemento com o nome indicado. O nome aparece na tabela no formato codificado de 4 bytes. Onbeginelement inicia uma cláusula que termina com um OpEndElement correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginElement e seu OpEndElement correspondente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |

| OpElement (nome)                             | Faz a correspondência de um elemento inteiro com o nome indicado. | OpAttribute (nome)                           | Faz a correspondência do token de rótulo de um atributo com o nome indicado. As cláusulas OpAttribute sempre aparecem após as operações OpBeginElement ou OpBeginAnyElement ou depois de outra cláusula OpAttribute.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| OpBeginChoice                                | OpBeginChoice inicia uma cláusula que termina com um OpEndChoice correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginChoice e seu OpEndChoice correspondente. A cláusula Outer corresponde aos tokens correspondidos por qualquer uma das cláusulas internas. Todas as cláusulas internas devem começar com OpBeginElement, exceto que a última pode ser OpAnything. Esse constructo corresponde à partícula Choice XSD.                                                                                                                                                                                                                                                                                                                                                                                                                             |
| OpEndChoice                                  | Termina uma cláusula iniciada com OpBeginChoice.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| OpBeginSequence                              | OpBeginSequence inicia uma cláusula que termina com um OpEndSequence correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginSequence e OpEndSequence. A cláusula Outer corresponde aos tokens correspondidos por todas as cláusulas internas em sequência. Esse constructo corresponde à partícula da sequência XSD.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| OpEndSequence                                | Termina uma cláusula iniciada com OpBeginSequence.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| OpBeginAll                                   | OpBeginAll inicia uma cláusula que termina com um OpEndAll correspondente. Zero ou mais cláusulas completas devem aparecer entre OpBeginAll e OpEndAll. A cláusula Outer corresponde aos tokens correspondidos pelas cláusulas internas em qualquer sequência. Os operadores de ocorrência relacionados a cada cláusula interna indicam Quantas vezes os tokens para cada cláusula podem ocorrer. O padrão é uma vez. Várias ocorrências de uma cláusula especificada podem ser combinadas com ocorrências de outras cláusulas internas. Todas as cláusulas internas devem começar com OpBeginElement, exceto que a última pode ser OpAnything. Esse constructo corresponde à partícula XSD ALL.                                                                                                                                                                                                        |

OpOptional inicia uma cláusula que termina no final da cláusula subsequente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |

A macro OpFormatInt32 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |

Um argumento de deslocamento de 4 bytes indica o deslocamento da representação binária no contexto binário atual. A macro OpFormatInt64 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |

A macro OpFormatUInt8 calcula o deslocamento do campo indicado no tipo indicado e fornece esse deslocamento como o argumento.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
