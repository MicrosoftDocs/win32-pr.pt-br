# <a name="web-services-on-devices"></a>Serviços Web em dispositivos

## <a name="purpose"></a>Finalidade

A API do Microsoft Web Services on Devices (WSDAPI) dá suporte à implementação de dispositivos e serviços controlados pelo cliente e aos hosts de dispositivo em conformidade com o [perfil de dispositivos para serviços Web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS). O WSDAPI usa o [WS-Discovery](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) para descoberta de dispositivo.

O WSDAPI pode ser usado para o desenvolvimento de implementações de cliente e serviço.

## <a name="where-applicable"></a>Quando aplicável

Os serviços Web em dispositivos permitem que um cliente descubra e acesse um dispositivo remoto e seus serviços associados em uma rede. Ele dá suporte à descoberta, descrição, controle e eventos do dispositivo. Os desenvolvedores podem criar proxies de cliente WSDAPI e stubs correspondentes para hosts de dispositivo.

## <a name="developer-audience"></a>Público de desenvolvedores

A documentação de serviços da Web em dispositivos é destinada a programadores C/C++ e fornecedores de dispositivos que criam produtos compatíveis com DPWS.

## <a name="run-time-requirements"></a>Requisitos de tempo de execução

Os aplicativos cliente que usam WSDAPI têm suporte a partir do Windows Vista e do Windows Server 2008.
