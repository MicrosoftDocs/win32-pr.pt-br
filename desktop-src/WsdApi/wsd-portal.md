# <a name="web-services-on-devices"></a>Serviços Web em dispositivos

## <a name="purpose"></a>Finalidade

A API WSDAPI (Serviços Web da Microsoft em Dispositivos) dá suporte à implementação de dispositivos e serviços controlados pelo cliente e hosts de dispositivo em conformidade com o DPWS (Perfil de Dispositivos para [Serviços Web).](https://specs.xmlsoap.org/ws/2006/02/devprof/) O WSDAPI usa [o WS-Discovery para](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) descoberta de dispositivos.

O WSDAPI pode ser usado para o desenvolvimento de implementações de cliente e serviço.

## <a name="where-applicable"></a>Quando aplicável

Os Serviços Web em Dispositivos permitem que um cliente descubra e acesse um dispositivo remoto e seus serviços associados em uma rede. Ele dá suporte à descoberta, à descrição, ao controle e ao evento do dispositivo. Os desenvolvedores podem criar proxies de cliente WSDAPI e stubs correspondentes para hosts de dispositivo.

## <a name="developer-audience"></a>Público de desenvolvedores

A documentação serviços Web em dispositivos destina-se a programadores C/C++ e fornecedores de dispositivos que criam produtos compatíveis com DPWS.

## <a name="run-time-requirements"></a>Requisitos de tempo de execução

Há suporte para aplicativos cliente que usam WSDAPI a partir do Windows Vista e do Windows Server 2008.
