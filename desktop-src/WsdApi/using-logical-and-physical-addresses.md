# <a name="using-logical-and-physical-addresses"></a><span data-ttu-id="a3f5d-103">Usando endereços lógicos e físicos</span><span class="sxs-lookup"><span data-stu-id="a3f5d-103">Using Logical and Physical Addresses</span></span>

<span data-ttu-id="a3f5d-104">O [WS-Discovery](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) define o endereçamento lógico usando URIs com base no `urn:uuid:` formato.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-104">[WS-Discovery](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf) defines logical addressing using URIs based on the `urn:uuid:` format.</span></span> <span data-ttu-id="a3f5d-105">A finalidade desse esquema de endereçamento é diferenciar a identidade do dispositivo de seu endereço IP atual.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-105">The purpose of this addressing scheme is to differentiate the identity of the device from its current IP address.</span></span> <span data-ttu-id="a3f5d-106">Basicamente, esse esquema fornece a funcionalidade de nomes DNS sem a necessidade de um servidor de nomes.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-106">This scheme essentially provides the functionality of DNS names without requiring a name server.</span></span> <span data-ttu-id="a3f5d-107">O [perfil de dispositivos para serviços Web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS) recomenda que os dispositivos usem esse esquema de endereçamento.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-107">[Devices Profile for Web Services](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS) recommends that devices use this addressing scheme.</span></span>

<span data-ttu-id="a3f5d-108">O DPWS também recomenda que os serviços usem endereços físicos (também chamados de transporte).</span><span class="sxs-lookup"><span data-stu-id="a3f5d-108">DPWS also recommends that services use physical (also called transport) addresses.</span></span> <span data-ttu-id="a3f5d-109">Isso permite que os clientes que não dão suporte nativo WS-Discovery mecanismos de endereçamento se comuniquem com os serviços do DPWS.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-109">This enables clients that do not natively support WS-Discovery addressing mechanisms to communicate with DPWS services.</span></span> <span data-ttu-id="a3f5d-110">Além disso, cada serviço pode definir seus endereços, o que permite o endereçamento de nível de transporte para implementações de dispositivo que gerenciam a expedição de serviço em uma camada inferior.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-110">Also, each service can define its addresses, which enables transport level addressing for device implementations that manage service dispatch at a lower layer.</span></span> <span data-ttu-id="a3f5d-111">Por fim, o uso de endereços físicos maximiza a interoperabilidade.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-111">Finally, using physical addresses maximizes interoperability.</span></span>

<span data-ttu-id="a3f5d-112">A desvantagem do endereçamento físico é que ele adiciona complexidade a implementações de dispositivo, pois o IP atual ou o endereço de transporte deve ser acompanhado e os metadados do dispositivo devem ser modificados de acordo.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-112">The disadvantage of physical addressing is that it adds complexity to device implementations, as the current IP or transport address must be tracked and device metadata must be modified accordingly.</span></span> <span data-ttu-id="a3f5d-113">Por esse motivo, o DPWS não exige que os serviços usem endereços de transporte.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-113">For this reason, DPWS does not require services to use transport addresses.</span></span>

<span data-ttu-id="a3f5d-114">Se forem usados endereços lógicos, haverá alguns cenários em que o comportamento de implementação é indefinido.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-114">If logical addresses are used, then there are some scenarios where the implementation behavior is undefined.</span></span> <span data-ttu-id="a3f5d-115">A especificação de WS-Discovery não descreve o que significa para um serviço residir em um endereço lógico.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-115">The WS-Discovery specification does not describe what it means for a service to reside at a logical address.</span></span> <span data-ttu-id="a3f5d-116">R1001 da especificação de WS-Discovery recomenda o uso de WS-Discovery em serviços hospedados por causa da informativa da rede associada.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-116">R1001 of the WS-Discovery specification recommends against using WS-Discovery on hosted services because of the associated network chatter.</span></span>

<span data-ttu-id="a3f5d-117">Não é recomendável que os serviços residam em endereços lógicos, pois isso reduz a interoperabilidade.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-117">It is not recommended that services reside at logical addresses as this reduces interoperability.</span></span> <span data-ttu-id="a3f5d-118">Se uma implementação absolutamente precisar residir em um endereço lógico, o serviço deverá usar o mesmo endereço lógico que o dispositivo.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-118">If an implementation absolutely must reside at a logical address, then the service should use the same logical address as the device.</span></span> <span data-ttu-id="a3f5d-119">Se isso adicionar muita complexidade ao modelo de expedição no dispositivo, a solução recomendada será usar parâmetros de referência para diferenciar os serviços.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-119">If this adds too much complexity to the dispatch model on the device, then the recommended solution is to use reference parameters to differentiate the services.</span></span> <span data-ttu-id="a3f5d-120">O WSDAPI enviará mensagens para os serviços corretamente se usarem o mesmo endereço de ponto de extremidade que o dispositivo.</span><span class="sxs-lookup"><span data-stu-id="a3f5d-120">WSDAPI will correctly send messages to services if they use the same endpoint address as the device.</span></span>
