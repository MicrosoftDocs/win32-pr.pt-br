# <a name="using-web-services-on-devices"></a>Usando serviços Web em dispositivos

A API do Microsoft Web Services on Devices (WSDAPI) dá suporte à implementação de dispositivos e serviços controlados pelo cliente e aos hosts de dispositivo em conformidade com o [perfil de dispositivos para serviços Web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS). O WSDAPI pode ser usado para o desenvolvimento de implementações de cliente e de servidor (dispositivo).

Sempre que possível, use o WSDAPI para criar um aplicativo de host de dispositivo ou cliente WSD baseado no Windows. O uso de WSDAPI reduz a complexidade do desenvolvimento e economiza tempo. Para obter mais informações, consulte [desenvolvimento de aplicativos WSD em ferramentas de desenvolvimento do Windows](wsd-application-development-on-windows.md) e [WSDAPI](wsdapi-development-tools.md).

Você também pode criar dispositivos WSD que não executam o Windows e são interoperáveis com clientes WSD baseados no Windows e hosts de dispositivo. Esses dispositivos devem atender a requisitos específicos para manter a interoperabilidade. Para obter mais informações, consulte o [desenvolvimento de dispositivo WSD](wsd-device-development.md).
