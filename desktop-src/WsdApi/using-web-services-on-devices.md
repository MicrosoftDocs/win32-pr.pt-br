# <a name="using-web-services-on-devices"></a>Usando serviços Web em dispositivos

A API WSDAPI (Serviços Web da Microsoft em Dispositivos) dá suporte à implementação de dispositivos e serviços controlados pelo cliente e hosts de dispositivo em conformidade com o DPWS (Perfil de Dispositivos para [Serviços Web).](https://specs.xmlsoap.org/ws/2006/02/devprof/) O WSDAPI pode ser usado para o desenvolvimento de implementações de cliente e servidor (dispositivo).

Sempre que possível, use o WSDAPI para criar um aplicativo de host de dispositivo ou cliente WSD baseado no Windows. O uso do WSDAPI reduz a complexidade do desenvolvimento e economiza tempo. Para obter mais informações, consulte [Desenvolvimento de aplicativos WSD no Windows](wsd-application-development-on-windows.md) e [WSDAPI Development Tools](wsdapi-development-tools.md).

Você também pode criar dispositivos WSD que não executem o Windows e sejam interoperáveis com clientes WSD baseados no Windows e hosts de dispositivo. Esses dispositivos devem atender a requisitos específicos para manter a interoperabilidade. Para obter mais informações, consulte [Desenvolvimento de dispositivos WSD.](wsd-device-development.md)
