# <a name="about-the-hyper-v-wmi-provider"></a>Sobre o provedor WMI do Hyper-V

O provedor WMI para Hyper-V permite que desenvolvedores e criadores de script criem rapidamente ferramentas personalizadas, utilitários e aprimoramentos para a plataforma de virtualização. As interfaces WMI podem gerenciar todos os aspectos dos serviços do Hyper-V.

## <a name="about-microsoft-windows-server-2008-r2-hyper-v"></a>Sobre o Microsoft Windows Server 2008 R2 Hyper-V

O Microsoft Windows Server 2008 R2 Hyper-V permite que os administradores do sistema consolidem servidores de hardware separados em um único servidor executando o Microsoft Windows Server 2008 R2 como o sistema operacional host.

Cada uma das máquinas virtuais hospedadas é executado em seu próprio ambiente virtual separado e isolado. Isso permite que o administrador gerencie, forneça e mantenha com facilidade um grande número de máquinas virtuais, reduzindo a necessidade de executar várias soluções de hardware para usar vários produtos e sistemas operacionais.

Outra vantagem do ambiente de máquina virtual está em teste e suporte. Novas configurações de servidor e sistema podem ser implantadas e testadas em paralelo com o sistema existente sem exigir tempo de inatividade caro durante a fase de implantação. Cada máquina virtual pode ser configurada para usar configurações variáveis durante a execução em uma plataforma padrão para produzir melhores resultados de teste. Com suporte para sistemas operacionais anteriores, é possível dar suporte e testar hardware, sistemas e aplicativos herdado. O suporte a instantâneos permite que você tire um instantâneo do estado de uma máquina virtual para que você possa retornar a um evento anterior, se necessário.

O Hyper-V expõe uma interface rica que permite que o usuário monitore e controle o ambiente da máquina virtual. A maioria do Hyper-V pode ser controlada usando o provedor WMI, que permite a personalização fácil, mas poderosa das máquinas virtuais.
