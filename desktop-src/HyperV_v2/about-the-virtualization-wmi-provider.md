# <a name="about-the-hyper-v-wmi-provider"></a>Sobre o provedor WMI do Hyper-V

O provedor WMI para Hyper-V permite que desenvolvedores e criadores de scripts criem rapidamente ferramentas, utilitários e aprimoramentos personalizados para a plataforma de virtualização. As interfaces WMI podem gerenciar todos os aspectos dos serviços do Hyper-V.

## <a name="about-microsoft-windows-server-2008-r2-hyper-v"></a>Sobre o Microsoft Windows Server 2008 R2 Hyper-V

O Microsoft Windows Server 2008 R2 Hyper-V permite que os administradores de sistema consolidem servidores de hardware separados em um único servidor que executa o Microsoft Windows Server 2008 R2 como o sistema operacional do host.

Cada uma das máquinas virtuais hospedadas é executada em seu próprio ambiente virtual separado e isolado. Isso permite que o administrador gerencie, forneça e mantenha um grande número de máquinas virtuais com facilidade, reduzindo a necessidade de executar várias soluções de hardware para usar vários produtos e sistemas operacionais.

Outra vantagem do ambiente de máquina virtual é o teste e o suporte. Novas configurações de servidor e sistema podem ser implantadas e testadas em paralelo com o sistema existente sem exigir um tempo de inatividade dispendioso durante a fase de distribuição. Cada máquina virtual pode ser configurada para usar diferentes configurações durante a execução em uma plataforma padrão para produzir melhores resultados de teste. Com suporte para sistemas operacionais anteriores, um pode dar suporte e testar hardware, sistemas e aplicativos herdados. O suporte a instantâneos permite tirar um instantâneo do estado de uma máquina virtual para que você possa retornar a um evento anterior, se necessário.

O Hyper-V expõe uma interface rica que permite ao usuário monitorar e controlar o ambiente de máquina virtual. A maior parte do Hyper-V pode ser controlada usando o provedor de WMI, que permite uma personalização fácil, mas eficiente, das máquinas virtuais.
