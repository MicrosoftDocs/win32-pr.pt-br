# <a name="data-coherency"></a><span data-ttu-id="aafa6-104">Coerência de dados</span><span class="sxs-lookup"><span data-stu-id="aafa6-104">Data Coherency</span></span>

<span data-ttu-id="aafa6-105">Os dados que são *coerentes* são dados que são os mesmos na rede.</span><span class="sxs-lookup"><span data-stu-id="aafa6-105">Data that is *coherent* is data that is the same across the network.</span></span> <span data-ttu-id="aafa6-106">Em outras palavras, se os dados forem coerentes, os dados no servidor e todos os clientes serão sincronizados.</span><span class="sxs-lookup"><span data-stu-id="aafa6-106">In other words, if data is coherent, data on the server and all the clients is synchronized.</span></span> <span data-ttu-id="aafa6-107">Um tipo de sistema de software que fornece a coerência de dados é um RCS (sistema de controle de revisão).</span><span class="sxs-lookup"><span data-stu-id="aafa6-107">One type of software system that provides data coherency is a revision control system (RCS).</span></span> <span data-ttu-id="aafa6-108">Esse sistema é geralmente bastante simples, com apenas um usuário com permissão para modificar um arquivo especificado de cada vez.</span><span class="sxs-lookup"><span data-stu-id="aafa6-108">Such a system is usually fairly simple, with only one user allowed to modify a specified file at a time.</span></span> <span data-ttu-id="aafa6-109">Outras pessoas podem ler o arquivo, mas não podem alterá-lo.</span><span class="sxs-lookup"><span data-stu-id="aafa6-109">Others can read the file but cannot change it.</span></span>

<span data-ttu-id="aafa6-110">O usuário que pode alterar um arquivo é dito para tê-lo verificado. <span data-ttu-id="aafa6-112">Um RCS requer a intervenção ativa dos usuários para operar de maneira útil.</span><span class="sxs-lookup"><span data-stu-id="aafa6-112">An RCS requires the active intervention of users to operate in a useful manner.</span></span> <span data-ttu-id="aafa6-113">Um sistema de arquivos que opera em uma rede deve tratar o problema automaticamente.</span><span class="sxs-lookup"><span data-stu-id="aafa6-113">A file system that operates across a network should handle the problem automatically.</span></span>

<span data-ttu-id="aafa6-114">Fornecer cache local de dados coerentes é bem simples quando você tem um thread em um cliente que acessa um arquivo pela rede por vez.</span><span class="sxs-lookup"><span data-stu-id="aafa6-114">Providing local caching of coherent data is fairly simple when you have one thread on one client accessing a file across the network at a time.</span></span> <span data-ttu-id="aafa6-115">No entanto, na maioria das situações, muitos threads diferentes em um ou mais computadores podem estar lendo o mesmo arquivo.</span><span class="sxs-lookup"><span data-stu-id="aafa6-115">However, in most situations many different threads on one or more computers may be reading the same file.</span></span> <span data-ttu-id="aafa6-116">Essa situação ainda é bem simples.</span><span class="sxs-lookup"><span data-stu-id="aafa6-116">This situation is still fairly straightforward.</span></span> <span data-ttu-id="aafa6-117">Como os dados no arquivo são estáticos, cada computador cliente pode ter sua própria cópia local sem implicações para a coerência dos dados.</span><span class="sxs-lookup"><span data-stu-id="aafa6-117">Because the data in the file is static, each client computer can have its own local copy with no implications for data coherency.</span></span>

<span data-ttu-id="aafa6-118">Uma situação mais comum é um thread que modifica o arquivo e muitos outros threads lendo.</span><span class="sxs-lookup"><span data-stu-id="aafa6-118">A more common situation is one thread modifying the file, and a lot of other threads reading it.</span></span> <span data-ttu-id="aafa6-119">No momento em que uma operação de gravação ocorre, todos os caches locais desse arquivo são obsoletos.</span><span class="sxs-lookup"><span data-stu-id="aafa6-119">The moment a write operation occurs, all of the local caches of that file are obsolete.</span></span> <span data-ttu-id="aafa6-120">O servidor deve notificar cada cliente para abandonar seu cache.</span><span class="sxs-lookup"><span data-stu-id="aafa6-120">The server must notify each client to abandon its cache.</span></span> <span data-ttu-id="aafa6-121">Todas as operações de leitura subsequentes para o arquivo devem ser executadas na rede.</span><span class="sxs-lookup"><span data-stu-id="aafa6-121">Any subsequent read operations for the file must be performed across the network.</span></span>

<span data-ttu-id="aafa6-122">Em outra situação comum, vários threads em um ou mais clientes de rede podem tentar gravar no mesmo arquivo.</span><span class="sxs-lookup"><span data-stu-id="aafa6-122">In another common situation, multiple threads on one or more network clients might try to write to the same file.</span></span> <span data-ttu-id="aafa6-123">Essa situação é semelhante a uma na qual vários usuários do RCS desejam fazer alterações no mesmo arquivo.</span><span class="sxs-lookup"><span data-stu-id="aafa6-123">This situation is similar to a one in which several RCS users all want to make changes to the same file.</span></span> <span data-ttu-id="aafa6-124">Cada usuário em sequência deve fazer check-out do arquivo, fazer alterações e, em seguida, verificar o arquivo novamente.</span><span class="sxs-lookup"><span data-stu-id="aafa6-124">Each user in sequence must check out the file, make changes, and then check the file back in.</span></span> <span data-ttu-id="aafa6-125">Da mesma forma, em um esquema de cache local, o servidor deve entregar o privilégio de gravar em um arquivo em um thread de cliente por vez.</span><span class="sxs-lookup"><span data-stu-id="aafa6-125">Similarly, in a local caching scheme the server must hand off the privilege of writing to a file to one client thread at a time.</span></span>
