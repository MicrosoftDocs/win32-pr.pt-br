# <a name="file-system-recognition"></a><span data-ttu-id="46fe4-103">Reconhecimento do sistema de arquivos</span><span class="sxs-lookup"><span data-stu-id="46fe4-103">File System Recognition</span></span>

<span data-ttu-id="46fe4-104">A meta do reconhecimento do sistema de arquivos é permitir que o sistema operacional Windows tenha uma opção adicional para um sistema de arquivos válido, mas não reconhecido, diferente de "RAW".</span><span class="sxs-lookup"><span data-stu-id="46fe4-104">The goal of file system recognition is to allow the Windows operating system to have an additional option for a valid but unrecognized file system other than "RAW".</span></span> <span data-ttu-id="46fe4-105">Para conseguir isso, começando com o Windows 7 e o Windows Server 2008 R2, o sistema define um tipo de estrutura de dados fixo que pode ser gravado na mídia na qual uma tecnologia habilitada que altera o formato do sistema de arquivos está ativa.</span><span class="sxs-lookup"><span data-stu-id="46fe4-105">To achieve this, beginning with Windows 7 and Windows Server 2008 R2, the system defines a fixed data structure type that can be written to the media on which an enabled technology that alters the file system format is active.</span></span> <span data-ttu-id="46fe4-106">Essa estrutura de dados, se presente no setor de disco lógico zero, seria reconhecida pelo sistema operacional e notificaria o usuário de que a mídia contém um sistema de arquivos válido, mas não reconhecido, e não será um volume bruto se os drivers do sistema de arquivos não estiverem instalados.</span><span class="sxs-lookup"><span data-stu-id="46fe4-106">This data structure, if present on logical disk sector zero, would then be recognized by the operating system and notify the user that the media contains a valid but unrecognized file system and is not a RAW volume if the drivers for the file system are not installed.</span></span>

-   <span data-ttu-id="46fe4-117">Código de validação da soma de verificação, armazenado na estrutura de dados da [**estrutura de reconhecimento do sistema de arquivos \_ \_ \_**](file-system-recognition-structure.md) .</span><span class="sxs-lookup"><span data-stu-id="46fe4-117">Checksum validation code, stored within the [**FILE\_SYSTEM\_RECOGNITION\_STRUCTURE**](file-system-recognition-structure.md) data structure.</span></span> <span data-ttu-id="46fe4-118">Para obter um exemplo de como computar essa soma de verificação, consulte [calculando uma soma de verificação de reconhecimento do sistema de arquivos](computing-a-file-system-recognition-checksum.md).</span><span class="sxs-lookup"><span data-stu-id="46fe4-118">For an example of how to compute this checksum, see [Computing a File System Recognition Checksum](computing-a-file-system-recognition-checksum.md).</span></span>

[**\_ \_ \_**](file-system-recognition-structure.md)</span><span class="sxs-lookup"><span data-stu-id="46fe4-119">The Windows Shell UI uses the previously listed features to provide more flexible and robust Autoplay and related support for unrecognized file systems, but it can work only if the [**FILE\_SYSTEM\_RECOGNITION\_STRUCTURE**](file-system-recognition-structure.md) data structure exists in logical disk sector zero.</span></span> <span data-ttu-id="46fe4-120">Os desenvolvedores que implementam novos sistemas de arquivos devem utilizar esse sistema para garantir que seu sistema de arquivos não seja considerado erroneamente como sendo do tipo "RAW".</span><span class="sxs-lookup"><span data-stu-id="46fe4-120">Developers implementing new file systems should utilize this system to ensure that their file system is not mistakenly assumed to be of type "RAW".</span></span>
