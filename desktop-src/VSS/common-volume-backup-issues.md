# <a name="common-volume-backup-issues"></a><span data-ttu-id="20a97-103">Problemas comuns de backup de volume</span><span class="sxs-lookup"><span data-stu-id="20a97-103">Common Volume Backup Issues</span></span>

<span data-ttu-id="20a97-104">Qualquer operação de backup que tenta copiar uma imagem completa e estável de um sistema deve lidar com as seguintes preocupações:</span><span class="sxs-lookup"><span data-stu-id="20a97-104">Any backup operation that attempts to copy a full and stable image of a system must deal with the following concerns:</span></span>

-   <span data-ttu-id="20a97-105">Arquivos inacessíveis durante um backup.</span><span class="sxs-lookup"><span data-stu-id="20a97-105">Inaccessible files during a backup.</span></span> <span data-ttu-id="20a97-106">A execução de aplicativos frequentemente precisa manter os arquivos abertos em modo exclusivo durante um backup, impedindo que os programas de backup os copiem.</span><span class="sxs-lookup"><span data-stu-id="20a97-106">Running applications frequently need to keep files open in exclusive mode during a backup, preventing backup programs from copying them.</span></span>
-   <span data-ttu-id="20a97-107">Estado de arquivo inconsistente.</span><span class="sxs-lookup"><span data-stu-id="20a97-107">Inconsistent file state.</span></span> <span data-ttu-id="20a97-108">Mesmo que um aplicativo não tenha seus arquivos abertos no modo exclusivo, é possível — devido ao tempo finito necessário para abrir, fazer backup e fechar um arquivo — os arquivos copiados para mídia de armazenamento podem não refletir o mesmo estado do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="20a97-108">Even if an application does not have its files open in exclusive mode, it is possible—because of the finite time needed to open, back up, and close a file—that files copied to storage media may not all reflect the same application state.</span></span>
-   <span data-ttu-id="20a97-109">É necessário minimizar as interrupções de serviço.</span><span class="sxs-lookup"><span data-stu-id="20a97-109">Need to minimize service interruptions.</span></span> <span data-ttu-id="20a97-110">Para garantir a acessibilidade do arquivo e a integridade dos dados cujo backup está sendo feito pode exigir a suspensão e/ou a rescisão de todos os programas em execução durante um backup de volume.</span><span class="sxs-lookup"><span data-stu-id="20a97-110">To ensure file accessibility and the integrity of the data being backed up can require the suspension and/or termination of all running programs during a volume backup.</span></span> <span data-ttu-id="20a97-111">Para sistemas de discos grandes, isso pode ser de horas de duração.</span><span class="sxs-lookup"><span data-stu-id="20a97-111">For large disk systems, this could be hours in duration.</span></span>

    <span data-ttu-id="20a97-112">Recentemente, alguns fornecedores de armazenamento tentaram solucionar esses problemas fornecendo um mecanismo de captura de volume – um meio de capturar uma imagem dos arquivos em disco em um determinado instante — usando um mecanismo de cópia de gravação ou de "espelho dividido".</span><span class="sxs-lookup"><span data-stu-id="20a97-112">Recently, some storage vendors have attempted to address these problems by providing a volume capture mechanism—a means to capture an image of the files on disk at a given instant in time—using either a copy-on-write or "split mirror" mechanism.</span></span> <span data-ttu-id="20a97-113">No entanto, essas soluções envolvem suas próprias dificuldades:</span><span class="sxs-lookup"><span data-stu-id="20a97-113">However, these solutions entail difficulties of their own:</span></span>

    -   <span data-ttu-id="20a97-114">Implementações de fornecedor incompatíveis da captura de volume.</span><span class="sxs-lookup"><span data-stu-id="20a97-114">Incompatible vendor implementations of volume capture.</span></span> <span data-ttu-id="20a97-115">Muitos provedores de dispositivos RAID fornecem mecanismos de captura de volume.</span><span class="sxs-lookup"><span data-stu-id="20a97-115">Many providers of RAID devices provide volume capture mechanisms.</span></span> <span data-ttu-id="20a97-116">No entanto, cada fornecedor tem sua própria interface e cada uma deve ter suporte dos fornecedores de backup para suas interfaces de captura de volume.</span><span class="sxs-lookup"><span data-stu-id="20a97-116">However, each vendor has its own interface and each must get support from the backup vendors for their volume capture interfaces.</span></span> <span data-ttu-id="20a97-117">Isso significa que os fornecedores de aplicativos de backup devem dar suporte a várias implementações de captura de volume, o que é indesejável.</span><span class="sxs-lookup"><span data-stu-id="20a97-117">This means that backup application vendors must support multiple volume capture implementations, which is undesirable.</span></span>
    -   <span data-ttu-id="20a97-118">Falta de coordenação de aplicativos.</span><span class="sxs-lookup"><span data-stu-id="20a97-118">Lack of application coordination.</span></span> <span data-ttu-id="20a97-119">Muitos dispositivos que dão suporte a uma captura de volume não dão suporte à coordenação de aplicativos em execução com o congelamento de dados em disco.</span><span class="sxs-lookup"><span data-stu-id="20a97-119">Many devices that support a volume capture do not support the coordination of running applications with the freeze of data on disk.</span></span> <span data-ttu-id="20a97-120">Para os dispositivos que fazem, assim como acontece com os aplicativos de backup, cada fornecedor tem uma interface diferente.</span><span class="sxs-lookup"><span data-stu-id="20a97-120">For those devices that do, as with the backup applications, each vendor has a different interface.</span></span>
    -   <span data-ttu-id="20a97-121">Suporte limitado para dispositivos não RAID.</span><span class="sxs-lookup"><span data-stu-id="20a97-121">Limited support for non-RAID devices.</span></span> <span data-ttu-id="20a97-122">Alguns se algum fornecedor de disco convencional fornecer suporte para qualquer tipo de captura de volume em seus drivers de dispositivo.</span><span class="sxs-lookup"><span data-stu-id="20a97-122">Few if any conventional disk vendors provide support for any sort of volume capture in their device drivers.</span></span> <span data-ttu-id="20a97-123">Isso significa que os mecanismos de captura são limitados a determinados sistemas de disco e normalmente não podem dar suporte ao backup de áreas do sistema.</span><span class="sxs-lookup"><span data-stu-id="20a97-123">This means that capture mechanisms are limited to certain disk systems, and cannot typically support the backup of system areas.</span></span>
    -   <span data-ttu-id="20a97-124">É necessário tratar as atualizações no disco durante a captura de volume.</span><span class="sxs-lookup"><span data-stu-id="20a97-124">Need to handle updates to disk during volume capture.</span></span> <span data-ttu-id="20a97-125">Embora os mecanismos de captura de volume fornecidos pelo fornecedor de armazenamento possam congelar o estado dos dados em disco, eles nem sempre interoperam com aplicativos em execução.</span><span class="sxs-lookup"><span data-stu-id="20a97-125">Although storage-vendor-provided volume capture mechanisms can freeze the state of data on disk, they do not always interoperate with running applications.</span></span> <span data-ttu-id="20a97-126">Isso geralmente significa que os dados enviados para o volume enquanto um dispositivo de armazenamento está passando por captura de volume podem ser perdidos.</span><span class="sxs-lookup"><span data-stu-id="20a97-126">This frequently means that data sent to the volume while a storage device is undergoing volume capture may be lost.</span></span>
    -   <span data-ttu-id="20a97-127">Backup de multivolume consistente.</span><span class="sxs-lookup"><span data-stu-id="20a97-127">Consistent multivolume backup.</span></span> <span data-ttu-id="20a97-128">O dispositivo de armazenamento executa essas capturas de volume, portanto, geralmente não há nenhum mecanismo para coordenar o tempo do congelamento de dados.</span><span class="sxs-lookup"><span data-stu-id="20a97-128">The storage device executes these volume captures, so there is generally no mechanism for coordinating the timing of the data freeze.</span></span> <span data-ttu-id="20a97-129">Isso será particularmente verdadeiro se os dispositivos vierem de fornecedores separados.</span><span class="sxs-lookup"><span data-stu-id="20a97-129">This is particularly true if the devices come from separate vendors.</span></span> <span data-ttu-id="20a97-130">Portanto, se vários volumes de armazenamento estiverem envolvidos em um backup com uma captura de volume, a imagem de tempo preservada para cada volume poderá não ser consistente.</span><span class="sxs-lookup"><span data-stu-id="20a97-130">Therefore, if several storage volumes are involved in a backup with a volume capture, the time image preserved for each volume may not be consistent.</span></span>
