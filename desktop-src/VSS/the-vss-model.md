# <a name="the-vss-model"></a><span data-ttu-id="6d590-103">O modelo VSS</span><span class="sxs-lookup"><span data-stu-id="6d590-103">The VSS Model</span></span>

<span data-ttu-id="6d590-105">O modelo VSS inclui o seguinte:</span><span class="sxs-lookup"><span data-stu-id="6d590-105">The VSS model includes the following:</span></span>

-   <span data-ttu-id="6d590-106">O mecanismo de cópia de sombra.</span><span class="sxs-lookup"><span data-stu-id="6d590-106">The shadow copy mechanism.</span></span> <span data-ttu-id="6d590-107">O VSS fornece captura rápida de volume do estado de um disco em um instante no tempo — uma [*cópia de sombra*](vssgloss-s.md) do volume.</span><span class="sxs-lookup"><span data-stu-id="6d590-107">VSS provides fast volume capture of the state of a disk at one instant in time—a [*shadow copy*](vssgloss-s.md) of the volume.</span></span> <span data-ttu-id="6d590-108">Essa cópia de volume existe lado a lado com o volume ativo e contém cópias de todos os arquivos em disco efetivamente salvas e disponíveis como um dispositivo separado.</span><span class="sxs-lookup"><span data-stu-id="6d590-108">This volume copy exists side by side with the live volume, and contains copies of all files on disk effectively saved and available as a separate device.</span></span>
-   <span data-ttu-id="6d590-109">Estado de arquivo consistente por meio da coordenação de aplicativos.</span><span class="sxs-lookup"><span data-stu-id="6d590-109">Consistent file state via application coordination.</span></span> <span data-ttu-id="6d590-110">O VSS fornece um mecanismo de comunicação interprocessamento baseado em eventos com base em um evento que os processos participantes podem usar para determinar o estado do sistema em relação a operações de backup, restauração e cópia de sombra (captura de volume).</span><span class="sxs-lookup"><span data-stu-id="6d590-110">VSS provides a COM-based, event-driven interprocess communication mechanism that participating processes can use to determine system state with respect to backup, restore, and shadow copy (volume capture) operations.</span></span> <span data-ttu-id="6d590-111">Esses eventos definem os estágios pelos quais os aplicativos que modificam os dados no disco ([*gravadores*](vssgloss-w.md)) podem colocar todos os seus arquivos em um estado consistente antes da criação da cópia de sombra.</span><span class="sxs-lookup"><span data-stu-id="6d590-111">These events define stages by which applications modifying data on disk ([*writers*](vssgloss-w.md)) can bring all their files into a consistent state prior to the creation of the shadow copy.</span></span>
-   <span data-ttu-id="6d590-112">Minimizando o tempo de inatividade do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="6d590-112">Minimizing application downtime.</span></span> <span data-ttu-id="6d590-113">A cópia de sombra do VSS existe em paralelo com uma cópia dinâmica do volume do qual será feito backup, portanto, exceto pelo breve período da preparação e da criação da cópia de sombra, um aplicativo pode continuar seu trabalho.</span><span class="sxs-lookup"><span data-stu-id="6d590-113">The VSS shadow copy exists in parallel with a live copy of the volume to be backed up, so except for the brief period of the shadow copy's preparation and creation, an application can continue its work.</span></span> <span data-ttu-id="6d590-114">O tempo necessário para realmente criar uma cópia de sombra, que ocorre entre [*congelar*](vssgloss-f.md) e [*descongelar*](vssgloss-t.md) eventos, normalmente leva cerca de um minuto.</span><span class="sxs-lookup"><span data-stu-id="6d590-114">The time needed to actually create a shadow copy, which occurs between [*Freeze*](vssgloss-f.md) and [*Thaw*](vssgloss-t.md) events, typically takes about one minute.</span></span>

    <span data-ttu-id="6d590-115">Enquanto a preparação de um gravador para uma cópia de sombra, incluindo a liberação de e/s e O estado de salvamento (consulte [visão geral das tarefas de pré-backup](overview-of-pre-backup-tasks.md)), pode não ser trivial, ela é significativamente menor do que o tempo necessário para realmente fazer backup de um volume — que para grandes volumes pode levar horas.</span><span class="sxs-lookup"><span data-stu-id="6d590-115">While a writer's preparation for a shadow copy, including flushing I/O and saving state (see [Overview of Pre-Backup Tasks](overview-of-pre-backup-tasks.md)), may be nontrivial, it is significantly shorter than the time required to actually back up a volume—which for large volumes may take hours.</span></span>

-   <span data-ttu-id="6d590-116">Interface unificada para VSS.</span><span class="sxs-lookup"><span data-stu-id="6d590-116">Unified interface to VSS.</span></span> <span data-ttu-id="6d590-117">O VSS abstrai os mecanismos de cópia de sombra em uma interface comum ao mesmo tempo em que permite que um fornecedor de hardware adicione e gerencie os recursos exclusivos de seus próprios provedores.</span><span class="sxs-lookup"><span data-stu-id="6d590-117">VSS abstracts the shadow copy mechanisms within a common interface while enabling a hardware vendor to add and manage the unique features of its own providers.</span></span> <span data-ttu-id="6d590-118">Qualquer aplicativo de backup ([*solicitante*](vssgloss-r.md)) e qualquer gravador deve ser capaz de ser executado em qualquer sistema de armazenamento em disco que dê suporte à interface VSS.</span><span class="sxs-lookup"><span data-stu-id="6d590-118">Any backup application ([*requester*](vssgloss-r.md)) and any writer should be able to run on any disk storage system that supports the VSS interface.</span></span>
-   <span data-ttu-id="6d590-119">Backup de multivolume.</span><span class="sxs-lookup"><span data-stu-id="6d590-119">Multivolume backup.</span></span> <span data-ttu-id="6d590-120">O VSS dá suporte a [*conjuntos de cópias de sombra*](vssgloss-s.md), que são coleções de cópias de sombra, em vários tipos de volumes de disco de vários fornecedores.</span><span class="sxs-lookup"><span data-stu-id="6d590-120">VSS supports [*shadow copy sets*](vssgloss-s.md), which are collections of shadow copies, across multiple types of disk volumes from multiple vendors.</span></span> <span data-ttu-id="6d590-121">Todas as cópias de sombra em um conjunto de cópias de sombra serão criadas com o mesmo carimbo de data/hora e apresentarão o mesmo estado de disco para um estado de disco de multivolume.</span><span class="sxs-lookup"><span data-stu-id="6d590-121">All shadow copies in a shadow copy set will be created with the same time stamp and will present the same disk state for a multivolume disk state.</span></span>
-   <span data-ttu-id="6d590-122">Suporte à cópia de sombra nativa.</span><span class="sxs-lookup"><span data-stu-id="6d590-122">Native shadow copy support.</span></span> <span data-ttu-id="6d590-123">A partir do Windows XP, o suporte à cópia de sombra está disponível por meio do VSS como parte nativa do sistema operacional Windows.</span><span class="sxs-lookup"><span data-stu-id="6d590-123">Beginning with Windows XP, shadow copy support is available through VSS as a native part of the Windows operating system.</span></span> <span data-ttu-id="6d590-124">Desde que pelo menos um disco NTFS esteja presente em um sistema, esses sistemas podem ser configurados para dar suporte a cópias de sombra de todos os sistemas de disco montados neles.</span><span class="sxs-lookup"><span data-stu-id="6d590-124">As long as at least one NTFS disk is present on a system, these systems can be configured to support shadow copies of all disk systems mounted on them.</span></span>
