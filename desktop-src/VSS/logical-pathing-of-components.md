# <a name="logical-pathing-of-components"></a><span data-ttu-id="4e404-103">Caminho lógico de componentes</span><span class="sxs-lookup"><span data-stu-id="4e404-103">Logical Pathing of Components</span></span>

<span data-ttu-id="4e404-104">O [*caminho lógico*](vssgloss-l.md) é usado para organizar os componentes gerenciados por um gravador em grupos bem definidos.</span><span class="sxs-lookup"><span data-stu-id="4e404-104">[*Logical pathing*](vssgloss-l.md) is used to organize components managed by a writer into well-defined groups.</span></span>

<span data-ttu-id="4e404-105">O caminho lógico é análogo na estrutura para o caminho de arquivo tradicional, usando a barra invertida " \\ " para separar elementos no caminho.</span><span class="sxs-lookup"><span data-stu-id="4e404-105">The logical path is analogous in structure to traditional file pathing, using the backslash "\\" to separate elements in the path.</span></span> <span data-ttu-id="4e404-106">Ao contrário dos caminhos de arquivo, a raiz de um caminho lógico é **nula**, em vez de " \\ ".</span><span class="sxs-lookup"><span data-stu-id="4e404-106">Unlike file paths, the root of a logical path is **NULL**, instead of "\\".</span></span>

<span data-ttu-id="4e404-107">O caminho lógico é expresso como uma cadeia de caracteres terminada em **nulo** e não há nenhuma outra restrição nos caracteres que o caminho pode conter.</span><span class="sxs-lookup"><span data-stu-id="4e404-107">The logical path is expressed as a **NULL**-terminated string, and there are no other restrictions on the characters the path can contain.</span></span>

<span data-ttu-id="4e404-108">O uso mais importante de caminhos lógicos é a definição de [*conjuntos de componentes*](vssgloss-c.md), em que a [*inclusão explícita de componentes*](vssgloss-e.md) em uma operação de backup ou restauração de um componente selecionável requer a inclusão de vários outros componentes ([*subcomponente*](vssgloss-s.md)).</span><span class="sxs-lookup"><span data-stu-id="4e404-108">The most important use of logical pathing is in defining [*component sets*](vssgloss-c.md), where the [*explicit component inclusion*](vssgloss-e.md) in a backup or restore operation of one selectable component requires the inclusion of a number of other components ([*subcomponent*](vssgloss-s.md)).</span></span> <span data-ttu-id="4e404-109">O caminho lógico do componente de definição de um conjunto de componentes é um pai para os caminhos lógicos de seus subcomponentes e:</span><span class="sxs-lookup"><span data-stu-id="4e404-109">The logical path of the defining component of a component set is a parent to the logical paths of its subcomponents and:</span></span>

-   <span data-ttu-id="4e404-110">Os subcomponentes devem compartilhar como um caminho raiz o caminho lógico do componente selecionável que define o conjunto de componentes.</span><span class="sxs-lookup"><span data-stu-id="4e404-110">Subcomponents must share as a root path the logical path of the selectable component that defines the component set.</span></span>
-   <span data-ttu-id="4e404-111">Um caminho raiz **nulo** é válido.</span><span class="sxs-lookup"><span data-stu-id="4e404-111">A root path of **NULL** is valid.</span></span>
-   <span data-ttu-id="4e404-112">O nome do componente selecionável de definição deve ser o primeiro elemento de caminho lógico após o caminho raiz para cada Subcomponente não selecionável do conjunto de componentes.</span><span class="sxs-lookup"><span data-stu-id="4e404-112">The name of the defining selectable component must be the first logical path element after the root path for every nonselectable subcomponent of the component set.</span></span>
-   <span data-ttu-id="4e404-113">Os conjuntos de componentes podem ser aninhados.</span><span class="sxs-lookup"><span data-stu-id="4e404-113">Component sets can be nested.</span></span>
