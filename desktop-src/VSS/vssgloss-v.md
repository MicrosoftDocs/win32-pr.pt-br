# <a name="v-volume-shadow-copy-service"></a>V (Serviço de Cópias de Sombra de Volume)

[A](vssgloss-a.md) [B](vssgloss-b.md) [C](vssgloss-c.md) [D](vssgloss-d.md) [E](vssgloss-e.md) [F](vssgloss-f.md) [G](vssgloss-g.md) [H](vssgloss-h.md) [i](vssgloss-i.md) J K [L](vssgloss-l.md) M [N](vssgloss-n.md) [O](vssgloss-o.md) [P](vssgloss-p.md) Q [R](vssgloss-r.md) [S](vssgloss-s.md) [T](vssgloss-t.md) U V [W](vssgloss-w.md) X Y Z

<dl> <dt>

<span id="base.vssgloss_volume"></span><span id="BASE.VSSGLOSS_VOLUME"></span>**volume**
</dt> <dd>

Uma unidade de armazenamento lógico. Um volume é o destino de operações de criação de cópia de sombra.

</dd> <dt>

<span id="base.vssgloss_volume_name"></span><span id="BASE.VSSGLOSS_VOLUME_NAME"></span>**nome do GUID do volume**
</dt> <dd>

Um nome exclusivo para um volume. Um nome GUID de volume é uma cadeia de caracteres do seguinte formato:

\\\\?\\ Volume {GUID}\\

(Observe o " \\ ") onde o GUID é um identificador global exclusivo para o volume. O sistema operacional atribui um nome de volume quando ele encontra um volume pela primeira vez, por exemplo, durante a formatação ou a instalação.

Observe que um volume pode ter mais de um nome de GUID de volume.

</dd> </dl>
