# <a name="working-with-new-targets-during-restore"></a><span data-ttu-id="1cbea-103">Trabalhando com novos destinos durante a restauração</span><span class="sxs-lookup"><span data-stu-id="1cbea-103">Working with New Targets during Restore</span></span>

<span data-ttu-id="1cbea-104">Um solicitante pode precisar restaurar arquivos para um local indicado por algo diferente do caminho padrão de um conjunto de arquivos ou seu [*mapeamento de local alternativo*](vssgloss-a.md).</span><span class="sxs-lookup"><span data-stu-id="1cbea-104">A requester may need to restore files to a location indicated by something other than a file set's default path or its [*alternate location mapping*](vssgloss-a.md).</span></span> <span data-ttu-id="1cbea-105">Há muitas razões pelas quais isso pode ocorrer — por exemplo, nenhum destino de restauração foi acessível ou um usuário solicitante solicita intencionalmente que os arquivos fossem restaurados para algum local desconhecido anteriormente.</span><span class="sxs-lookup"><span data-stu-id="1cbea-105">There are many reasons why this might happen—for example, neither restore destination was accessible, or a requester user intentionally requests that files be restored to some previously unknown location.</span></span> <span data-ttu-id="1cbea-106">Nesse caso, o solicitante usa o novo mecanismo de destino para indicar aos gravadores que ele restaurou um arquivo em uma área diferente no disco.</span><span class="sxs-lookup"><span data-stu-id="1cbea-106">In this case, the requester uses the new target mechanism to indicate to writers that it has restored a file to a different area on disk.</span></span>

<span data-ttu-id="1cbea-107">Nem todos os gravadores dão suporte a um solicitante que altera o destino de restauração de um arquivo.</span><span class="sxs-lookup"><span data-stu-id="1cbea-107">Not all writers support a requester changing the restore destination of a file.</span></span> <span data-ttu-id="1cbea-108">Um solicitante precisa verificar o suporte do gravador verificando a máscara de esquema de backup do gravador (retornada por [**IVssExamineWriterMetadata:: GetBackupSchema**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssexaminewritermetadata-getbackupschema)) e verificando se ele contém o gravador do VSS \_ BS que \_ \_ dá suporte ao \_ novo \_ sinalizador de destino.</span><span class="sxs-lookup"><span data-stu-id="1cbea-108">A requester needs to verify writer support by checking the writer's backup schema mask (returned by [**IVssExamineWriterMetadata::GetBackupSchema**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssexaminewritermetadata-getbackupschema)) and verifying that it contains the VSS\_BS\_WRITER\_SUPPORTS\_NEW\_TARGET flag.</span></span>

<span data-ttu-id="1cbea-109">O solicitante indica tal restauração por meio do método [**IVssBackupComponents:: AddNewTarget**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-addnewtarget) .</span><span class="sxs-lookup"><span data-stu-id="1cbea-109">The requester indicates such a restoration through the [**IVssBackupComponents::AddNewTarget**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-addnewtarget) method.</span></span> <span data-ttu-id="1cbea-110">Além de especificar uma especificação de arquivo e um destino de restauração original e um novo, o solicitante especifica informações de componente — um caminho lógico e um nome de componente.</span><span class="sxs-lookup"><span data-stu-id="1cbea-110">In addition to specifying a file specification and an original and a new restore destination, the requester specifies component information—a logical path and component name.</span></span>

<span data-ttu-id="1cbea-111">As informações do componente que são usadas dependem de se o componente que está gerenciando o arquivo que tem um novo destino adicionado foi [*explicitamente incluído*](vssgloss-e.md) ou [*incluído implicitamente*](vssgloss-i.md) no backup.</span><span class="sxs-lookup"><span data-stu-id="1cbea-111">Which component's information is used depends on whether or not the component managing the file having a new target added was [*explicitly included*](vssgloss-e.md) or [*implicitly included*](vssgloss-i.md) in the backup.</span></span>

<span data-ttu-id="1cbea-112">Se o componente de gerenciamento tiver sido explicitamente incluído, suas informações serão usadas.</span><span class="sxs-lookup"><span data-stu-id="1cbea-112">If the managing component was explicitly included, then its information is used.</span></span> <span data-ttu-id="1cbea-113">Se o componente de gerenciamento tiver sido incluído implicitamente, ele será um subcomponente em um conjunto de componentes.</span><span class="sxs-lookup"><span data-stu-id="1cbea-113">If the managing component was implicitly included, it is a subcomponent in a component set.</span></span> <span data-ttu-id="1cbea-114">Nesse caso, as informações do componente de definição do conjunto de componentes são usadas.</span><span class="sxs-lookup"><span data-stu-id="1cbea-114">In this case, the component set's defining component's information is used.</span></span>

<span data-ttu-id="1cbea-115">Ao manipular o evento de [**pós-restauração**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-postrestore) , os gravadores devem verificar se algum de seus arquivos foram restaurados para um novo local.</span><span class="sxs-lookup"><span data-stu-id="1cbea-115">While handling the [**PostRestore**](/windows/desktop/api/VsBackup/nf-vsbackup-ivssbackupcomponents-postrestore) event, writers should check to see if any of its files were restored to a new location.</span></span> <span data-ttu-id="1cbea-116">Isso pode ser feito usando os métodos [**IVssComponent:: GetNewTargetCount**](/windows/desktop/api/VsWriter/nf-vswriter-ivsscomponent-getnewtargetcount) e [**IVssComponent:: GetNewTarget**](/windows/desktop/api/VsWriter/nf-vswriter-ivsscomponent-getnewtarget) .</span><span class="sxs-lookup"><span data-stu-id="1cbea-116">This can be done by using the [**IVssComponent::GetNewTargetCount**](/windows/desktop/api/VsWriter/nf-vswriter-ivsscomponent-getnewtargetcount) and [**IVssComponent::GetNewTarget**](/windows/desktop/api/VsWriter/nf-vswriter-ivsscomponent-getnewtarget) methods.</span></span>

<span data-ttu-id="1cbea-117">A instância da interface [**IVssComponent**](/windows/desktop/api/VsWriter/nl-vswriter-ivsscomponent) que é usada depende de se o componente de gerenciamento do arquivo foi adicionado explicitamente ou implicitamente ao backup.</span><span class="sxs-lookup"><span data-stu-id="1cbea-117">The instance of the [**IVssComponent**](/windows/desktop/api/VsWriter/nl-vswriter-ivsscomponent) interface that is used depends on whether the file's managing component was explicitly or implicitly added to the backup.</span></span>
