# <a name="routing-and-remote-access-service"></a>Serviço de Roteamento e Acesso Remoto

## <a name="purpose"></a>Finalidade

O RAS (serviço de acesso remoto) pode ser usado para criar aplicativos cliente. Esses aplicativos exibem caixas de diálogo comuns de RAS, gerenciam dispositivos e conexões de acesso remoto e manipulam entradas de catálogo telefônico.

As APIs de roteamento possibilitam a criação de aplicativos para administrar os recursos de roteamento do sistema operacional.

Os desenvolvedores podem usar as APIs do protocolo de roteamento para implementar protocolos de roteamento.

## <a name="developer-audience"></a>Público de desenvolvedores

As APIs de serviço de roteamento e acesso remoto são projetadas para uso por programadores C/C++. Os programadores também devem estar familiarizados com os conceitos de rede.
