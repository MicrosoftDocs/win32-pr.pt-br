<dl> <dt>

<span id="_rras_aged_out_gly"></span><span id="_RRAS_AGED_OUT_GLY"></span>**antigo**
</dt> <dd>

Além disso, *antigo do cache*. No RRAS, uma rota é antiga quando é removida da tabela de roteamento após um período de tempo especificado.

</dd> <dt>

<span id="_rras_authentication_gly"></span><span id="_RRAS_AUTHENTICATION_GLY"></span>**Authentication**
</dt> <dd>

A autenticação é o processo de verificar se um usuário é quem ele alega ser. Normalmente, a autenticação é executada usando um nome de usuário e senha.

</dd> <dt>

<span id="_rras_autonomous_system_gly"></span><span id="_RRAS_AUTONOMOUS_SYSTEM_GLY"></span>**sistema autônomo (AS)**
</dt> <dd>

Um sistema autônomo é um grupo de roteadores que compartilham uma única política administrativa. Esses roteadores usam o mesmo protocolo de roteamento para se comunicar (chamado de [*protocolo de gateway interior*](i-gly.md)). Consulte também [*protocolo de gateway exterior*](e-gly.md).

</dd> <dt>

<span id="_rras_autostatic_route_gly"></span><span id="_RRAS_AUTOSTATIC_ROUTE_GLY"></span>**rota autoestática**
</dt> <dd>

Uma rota autoestática é uma rota obtida dinamicamente de um protocolo de roteamento. Depois que uma rota é obtida dessa maneira, ela se comporta como um [*STATICROUTE*](s-gly.md).

</dd> </dl>
