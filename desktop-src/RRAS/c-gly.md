<dl> <dt>

<span id="_rras_cyclic_redundancy_check_crc__gly"></span><span id="_RRAS_CYCLIC_REDUNDANCY_CHECK_CRC__GLY"></span>**CRC (verificação de redundância cíclica)**
</dt> <dd>

Um CRC é um valor numérico relacionado a um bloco de dados. Esse valor fornece informações que permitem que um aplicativo determine se os dados foram modificados. Um CRC é semelhante a uma soma de verificação, mas é considerado um mecanismo de verificação de erros mais forte.

</dd> <dt>

<span id="_rras_compression_control_protocol_ccp__gly"></span><span id="_RRAS_COMPRESSION_CONTROL_PROTOCOL_CCP__GLY"></span>**Protocolo de controle de compactação (CCP)**
</dt> <dd>

O protocolo de controle de compactação (CCP) permite que dois computadores que se comunicam por meio do PPP negociem algoritmos compatíveis para enviar e receber quadros PPP compactados. Os dois computadores não podem usar o CCP até a fase de protocolo de controle de rede da conexão PPP.

Para obter mais informações, consulte [RFC 1962](remote-access-request-for-comments.md). Consulte também [**\_ \_ informações do PPP CCP**](/windows/desktop/api/Mprapi/ns-mprapi-ppp_ccp_info) e [**RASPPPCCP**](/previous-versions/windows/desktop/legacy/aa377620(v=vs.85)).

</dd> </dl>
