<dl> <dt>

<span id="_rpc_partially_bound_handle_glos"></span><span id="_RPC_PARTIALLY_BOUND_HANDLE_GLOS"></span>**identificador parcialmente associado**
</dt> <dd>

[*Identificador de associação*](b-glos.md) que não inclui informações de [*ponto de extremidade*](e-glos.md) . Consulte também o [*identificador totalmente associado*](f-glos.md).

</dd> <dt>

<span id="_rpc_pickling_glos"></span><span id="_RPC_PICKLING_GLOS"></span>**seleção**
</dt> <dd>

Consulte [*serialização*](s-glos.md).

</dd> <dt>

<span id="_rpc_pipe_glos"></span><span id="_RPC_PIPE_GLOS"></span>**conexão**
</dt> <dd>

Construtor de tipo [*IDL*](i-glos.md)que dá suporte à transmissão de um fluxo de dados aberto e em aplicativos de cliente e servidor.

</dd> <dt>

<span id="_rpc_primary_enterprise_controller_glos"></span><span id="_RPC_PRIMARY_ENTERPRISE_CONTROLLER_GLOS"></span>**controlador corporativo primário**
</dt> <dd>

O banco de dados do serviço de informações de fila de mensagens (MQIS) do mestre para uma rede. Consulte também [*serviço de informações de fila de mensagens (MQIS)*](m-glos.md).

</dd> <dt>

<span id="_rpc_primary_site_controller_glos"></span><span id="_RPC_PRIMARY_SITE_CONTROLLER_GLOS"></span>**controlador de site primário**
</dt> <dd>

Banco de dados do serviço de informações de fila de mensagens (MQIS) para um site específico em uma empresa. Consulte também [*serviço de informações de fila de mensagens (MQIS)*](m-glos.md).

</dd> <dt>

<span id="_rpc_procedure_serialization_glos"></span><span id="_RPC_PROCEDURE_SERIALIZATION_GLOS"></span>**serialização de procedimento**
</dt> <dd>

[*Serialização*](s-glos.md) de dados que usa um stub de serialização gerado por MIDL para realizar a codificação e a decodificação de um ou mais tipos com uma única chamada de procedimento. A serialização de procedimento é realizada aplicando os atributos de \[ [codificação](/windows/desktop/Midl/encode) \] e \[ [decodificação](/windows/desktop/Midl/decode) \] a um protótipo de função no arquivo ACF. Consulte também [*serialização de tipo*](t-glos.md).

</dd> <dt>

<span id="_rpc_protocol_sequence_glos"></span><span id="_RPC_PROTOCOL_SEQUENCE_GLOS"></span>**sequência de protocolos**
</dt> <dd>

Cadeia de caracteres que representa uma combinação válida de um protocolo RPC, um protocolo de camada de rede e um protocolo de camada de transporte. Por exemplo, a sequência de protocolo NCACN \_ IP \_ TCP descreve uma conexão de [*NCA*](n-glos.md) por meio de um protocolo de Internet (IP) com um protocolo TCP como transporte.

</dd> <dt>

<span id="_rpc_proxy_stub_glos"></span><span id="_RPC_PROXY_STUB_GLOS"></span>**stub de proxy**
</dt> <dd>

Linguagem C ou C++ gerada por MIDL – código-fonte que contém todas as funções necessárias para uma interface OLE personalizada.

</dd> </dl>
