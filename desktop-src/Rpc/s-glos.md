# <a name="s-rpc"></a>S (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [E](e-glos.md) [F](f-glos.md) G H [i](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) S [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**escalabilidade**
</dt> <dd>

Capacidade de utilizar totalmente o poder de processamento adicional em um sistema multiprocessador (2, 4, 8, 32 ou mais processadores). A escalabilidade também é a capacidade de atender a um grande número de clientes.

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**Protocolo assequenciado de pacote (SPP)**
</dt> <dd>

Protocolo de comunicação de [*conexão orientada a conexões*](c-glos.md) Banyan Vines para roteamento de pacotes de informações em redes locais.

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**Intercâmbio de pacotes sequenciado (SPX)**
</dt> <dd>

Protocolo de comunicação [*orientado a conexões*](c-glos.md) Novell NetWare para roteamento de pacotes de informações em redes locais e redes de longa distância.

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**série**
</dt> <dd>

Processo de marshaling de dados para (codificação) e desempacotamento de dados (decodificação) de buffers que você controla. Isso é diferente do uso tradicional de RPC, em que os stubs e o tempo de execução RPC controlam os buffers de marshaling. Também chamado de seleção. Consulte também [*serialização de procedimento*](p-glos.md), [*serialização de tipo*](t-glos.md).

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**stub de servidor**
</dt> <dd>

Código-fonte do C-Language gerado pelo MIDL que contém todas as funções necessárias para o aplicativo de servidor manipular solicitações remotas usando chamadas de procedimento local. Consulte também [*stub de cliente*](c-glos.md).

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**sessão**
</dt> <dd>

Relação estabelecida entre um aplicativo cliente e um aplicativo de servidor. Consulte também [*Associação*](b-glos.md), [*identificador de associação*](b-glos.md).

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**função de retorno de chamada estática**
</dt> <dd>

Procedimento remoto que faz parte do lado do cliente de um aplicativo distribuído, que um servidor pode chamar para obter informações do cliente. O \[ atributo de [retorno de chamada](/windows/desktop/Midl/callback) \] designa uma função de retorno de chamada estática.

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**rastreamento de identidade estático**
</dt> <dd>

O controle de identidade estático especifica que o tempo de execução do RPC usa as credenciais de segurança no identificador de ligação do cliente para todas as chamadas RPC. Consulte também [*controle de identidade dinâmico*](d-glos.md).

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**Associação de cadeia de caracteres**
</dt> <dd>

Cadeia de caracteres que consiste nas opções [*UUID*](u-glos.md)do objeto, [*sequência de protocolos*](p-glos.md), [*endereço de rede*](n-glos.md), ponto de [*extremidade*](e-glos.md)e ponto de extremidade, que podem ser usadas para criar um [*identificador de associação*](b-glos.md) para o servidor especificado.

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**tipagem forte**
</dt> <dd>

Imposição de compilador de controle estrito sobre os tipos de dados. No MIDL e no RPC, a tipagem forte é usada para garantir que os dados sejam interpretados consistentemente por computadores diferentes em um ambiente distribuído.

</dd> </dl>
