# <a name="s-rpc"></a>S (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [E](e-glos.md) F [G](f-glos.md) H [I](i-glos.md) J K L [M](l-glos.md) [](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) S [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**Escalabilidade**
</dt> <dd>

Capacidade de utilizar totalmente a potência de processamento adicional em um sistema multiprocessador (2, 4, 8, 32 ou mais processadores). A escalabilidade também é a capacidade de serviço de um grande número de clientes.

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**Protocolo SPP (Sequenced Packet Protocol)**
</dt> <dd>

O protocolo de comunicação orientado a conexão Banyan [*Vines*](c-glos.md) para pacotes de informações de roteamento em redes locais.

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**SPX (sequenced Packet Exchange)**
</dt> <dd>

Protocolo de comunicação [*orientado*](c-glos.md) a conexão da Novell NetWare para pacotes de informações de roteamento em redes locais e redes de longa área.

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**Serialização**
</dt> <dd>

Processo de marshaling de dados para (codificação) e desmarque dados de buffers (decodificação) que você controla. Isso é diferente do uso de RPC tradicional, em que os stubs e o runtime RPC controlam os buffers de marshaling. Também chamado de selamento. Consulte também [*serialização de procedimento*](p-glos.md), [*serialização de tipo*](t-glos.md).

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**stub do servidor**
</dt> <dd>

Código-fonte da linguagem C gerado por MIDL que contém todas as funções necessárias para o aplicativo de servidor manipular solicitações remotas usando chamadas de procedimento local. Consulte também [*stub do cliente.*](c-glos.md)

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**Sessão**
</dt> <dd>

Estabeleceu uma relação entre um aplicativo cliente e um aplicativo de servidor. Consulte também [*bind*](b-glos.md), [*binding handle*](b-glos.md).

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**função de retorno de chamada estática**
</dt> <dd>

Procedimento remoto que faz parte do lado do cliente de um aplicativo distribuído, que um servidor pode chamar para obter informações do cliente. O \[ [atributo de retorno](/windows/desktop/Midl/callback) \] de chamada designa uma função de retorno de chamada estática.

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**acompanhamento de identidade estático**
</dt> <dd>

O rastreamento de identidade estática especifica que o tempo de executar RPC usa as credenciais de segurança no alçamento de associação do cliente para todas as chamadas RPC. Consulte também controle [*de identidade dinâmico.*](d-glos.md)

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**associação de cadeia de caracteres**
</dt> <dd>

Cadeia de caracteres que consiste nas opções [*UUID*](u-glos.md)do objeto [*,*](p-glos.md)sequência de protocolo [*,*](n-glos.md)endereço [](b-glos.md) de rede [*,*](e-glos.md)ponto de extremidade e ponto de extremidade, que podem ser usadas para criar um identificador de associação para o servidor especificado.

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**digitação forte**
</dt> <dd>

Imposição do compilador de controle estrito sobre tipos de dados. Em MIDL e RPC, a digitação forte é usada para garantir que os dados são interpretados consistentemente por computadores diferentes em um ambiente distribuído.

</dd> </dl>
