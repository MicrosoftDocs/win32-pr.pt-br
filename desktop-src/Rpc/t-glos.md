<dl> <dt>

<span id="_rpc_top_level_pointer_glos"></span><span id="_RPC_TOP_LEVEL_POINTER_GLOS"></span>**ponteiro de nível superior**
</dt> <dd>

Ponteiro especificado como o nome de um parâmetro em um protótipo de função. Consulte também [*ponteiro inserido*](e-glos.md).

</dd> <dt>

<span id="_rpc_tcp_glos"></span><span id="_RPC_TCP_GLOS"></span>**TCP (protocolo de controle de transmissão)**
</dt> <dd>

Transporte de rede [*orientado a conexão*](c-glos.md) na parte superior do protocolo de Internet ( [*IP)*](i-glos.md).

</dd> <dt>

<span id="_rpc_tunneling_glos"></span><span id="_RPC_TUNNELING_GLOS"></span>**túnel**
</dt> <dd>

Protocolo TCP/IP para transmissão de dados de um aplicativo Content-Server para um roteador de difusão.

</dd> <dt>

<span id="_rpc_type_serialization_glos"></span><span id="_RPC_TYPE_SERIALIZATION_GLOS"></span>**serialização de tipo**
</dt> <dd>

[*Serialização*](s-glos.md) de dados que usa rotinas geradas por MIDL para dimensionar, codificar e decodificar objetos de um tipo especificado. O aplicativo cliente chama essas rotinas para serializar os dados. A serialização de tipo é realizada aplicando os atributos de \[ [codificação](/windows/desktop/Midl/encode) \] e \[ [decodificação](/windows/desktop/Midl/decode) \] a um único tipo de dados, ou a uma interface, no arquivo ACF. Consulte também [*serialização de procedimento*](p-glos.md).

</dd> </dl>
