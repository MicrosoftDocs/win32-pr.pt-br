# <a name="using-asynchronous-rpc-with-dce-pipes"></a>Usando RPC assíncrono com pipes DCE

O Microsoft RPC dá suporte ao uso de pipes DCE. Embora compartilhem um nome semelhante, os pipes DCE não estão relacionados a [pipes nomeados.](asynchronous-rpc-over-the-named-pipe-protocol.md) Pipes nomeados são um protocolo de transporte. Os pipes DCE são um método independente de protocolo de comunicação de cliente/servidor.

Esta seção apresenta uma discussão sobre o RPC usando pipes DCE assíncronos. Ele é dividido nos seguintes tópicos:

-   [Pipes assíncronos](asynchronous-pipes.md)
-   [Declarando pipes assíncronos](declaring-asynchronous-pipes.md)
-   [Tratamento de pipe assíncrono do lado do cliente](client-side-asynchronous-pipe-handling.md)
-   [Tratamento de pipe assíncrono do lado do servidor](server-side-asynchronous-pipe-handling.md)

Para um exemplo de aplicativo RPC de pipe assíncrono, consulte o exemplo FileRep no SDK (Kit de Desenvolvimento de Software de Plataforma).
