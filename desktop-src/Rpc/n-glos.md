# <a name="n-rpc"></a>N (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [E](e-glos.md) [F](f-glos.md) G H [i](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) N [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_name_service_glos"></span><span id="_RPC_NAME_SERVICE_GLOS"></span>**serviço de nome**
</dt> <dd>

Serviço que mapeia nomes para objetos e armazena os pares de nome/objeto em um banco de dados. Por exemplo, o serviço de nome RPC mapeia um nome lógico para um [*identificador de associação*](b-glos.md) para que os aplicativos cliente possam se referir a esse nome lógico, em vez de uma sequência de protocolo e endereço de rede. Consulte também Name Service – interface daemon ([nsid)](/windows), serviço de diretório de cliente ([*CDs)*](c-glos.md), [*localizador*](l-glos.md).

</dd> <dt>

<span id="_rpc_nca_glos"></span><span id="_RPC_NCA_GLOS"></span>**NCA (arquitetura de computação de rede)**
</dt> <dd>

Coleção de diretrizes para computação distribuída. Os protocolos de comunicação RPC seguem estas diretrizes.

</dd> <dt>

<span id="_rpc_nsi_glos"></span><span id="_RPC_NSI_GLOS"></span>**Independent Service Name (NSI)**
</dt> <dd>

Padrão para funções de API que permite que um aplicativo distribuído acesse elementos de banco de dados de serviço de nome RPC por meio de vários provedores de serviço de nome, como o serviço de diretório de célula uso-DCE ou o Microsoft Locator. Consulte também [– interface daemon](/windows) (nsid).

</dd> <dt>

<span id="_rpc_nsid_glos"></span><span id="_RPC_NSID_GLOS"></span>**serviço de nome – daemon de interface (nsid)**
</dt> <dd>

Serviço que fornece uma interface entre o [*localizador*](l-glos.md) da Microsoft e os bancos de dados do serviço de diretório da célula uso-DCE para funções de nome de serviço RPC.

</dd> <dt>

<span id="_rpc_named_pipe_glos"></span><span id="_RPC_NAMED_PIPE_GLOS"></span>**pipe nomeado**
</dt> <dd>

Protocolo orientado a conexões, com base em SMBs (blocos de mensagens de servidor) e [NetBIOS](/windows), usado para comunicação entre um processo de servidor e um ou mais processos de cliente.

</dd> <dt>

<span id="_rpc_netbeui_glos"></span><span id="_RPC_NETBEUI_GLOS"></span>**Interface de usuário estendida NetBIOS (NetBEUI)**
</dt> <dd>

Protocolo de transporte nativo do LAN Manager e driver de dispositivo de rede. Consulte também [NetBIOS](/windows).

</dd> <dt>

<span id="_rpc_netbios_glos"></span><span id="_RPC_NETBIOS_GLOS"></span>**NetBIOS (sistema básico de entrada/saída) de rede**
</dt> <dd>

Interface de software entre o sistema operacional Microsoft MS-DOS, o barramento de e/s e uma rede de área local.

</dd> <dt>

<span id="_rpc_ndr_glos"></span><span id="_RPC_NDR_GLOS"></span>**NDR (representação de dados de rede)**
</dt> <dd>

Formato padrão usado durante a transmissão de rede que é independente do formato de tipo de dados em qualquer arquitetura de computador específica. Os dados transmitidos incluem informações que especificam seu formato de NDR.

</dd> <dt>

<span id="_rpc_network_address_glos"></span><span id="_RPC_NETWORK_ADDRESS_GLOS"></span>**Endereço de rede**
</dt> <dd>

Endereço que identifica um servidor em uma rede.

</dd> <dt>

<span id="_rpc_nonencapsulated_union_glos"></span><span id="_RPC_NONENCAPSULATED_UNION_GLOS"></span>**União não encapsulada**
</dt> <dd>

A [*união discriminada*](d-glos.md) que é menos restritiva do que uma União encapsulada em que o discriminante e a União não estão rigidamente ligados. Se Union for um parâmetro, o discriminante será outro parâmetro; se Union for um campo de estrutura, o discriminante será outro campo de estrutura. A opção de palavras-chave de IDL \[ [**\_ é**](/windows/desktop/Midl/switch-is) \] e o \[ [**\_ tipo de comutador**](/windows/desktop/Midl/switch-type) \] identificam o discriminante e seu tipo. Consulte também [*União encapsulada*](e-glos.md).

</dd> <dt>

<span id="_rpc_nonidempotent_glos"></span><span id="_RPC_NONIDEMPOTENT_GLOS"></span>**nonidempotent**
</dt> <dd>

Indicador de que uma chamada de procedimento remoto não pode ser executada mais de uma vez porque retornará um valor diferente ou alterará um estado.

</dd> </dl>
