# <a name="the-idl-and-acf-files"></a><span data-ttu-id="6ecf8-105">Os arquivos IDL e ACF</span><span class="sxs-lookup"><span data-stu-id="6ecf8-105">The IDL and ACF Files</span></span>

<span data-ttu-id="6ecf8-108">O design MIDL Especifica dois arquivos distintos: o arquivo IDL (Interface Definition Language) e o arquivo de configuração de aplicativo (ACF).</span><span class="sxs-lookup"><span data-stu-id="6ecf8-108">The MIDL design specifies two distinct files: the Interface Definition Language (IDL) file and the application configuration file (ACF).</span></span> <span data-ttu-id="6ecf8-109">Esses arquivos contêm atributos que direcionam a geração dos arquivos stub de linguagem C que gerenciam a chamada de procedimento remoto (RPC).</span><span class="sxs-lookup"><span data-stu-id="6ecf8-109">These files contain attributes that direct the generation of the C-language stub files that manage the remote procedure call (RPC).</span></span> <span data-ttu-id="6ecf8-110">O arquivo IDL contém uma descrição da interface entre o cliente e os programas do servidor.</span><span class="sxs-lookup"><span data-stu-id="6ecf8-110">The IDL file contains a description of the interface between the client and the server programs.</span></span> <span data-ttu-id="6ecf8-111">Os aplicativos RPC usam o arquivo ACF para descrever as características da interface que são específicas para o hardware e o sistema operacional que compõem um ambiente operacional específico.</span><span class="sxs-lookup"><span data-stu-id="6ecf8-111">RPC applications use the ACF file to describe the characteristics of the interface that are specific to the hardware and operating system that make up a particular operating environment.</span></span> <span data-ttu-id="6ecf8-112">A finalidade de dividir essas informações em dois arquivos é manter a interface do software separada das características que afetam apenas o ambiente operacional.</span><span class="sxs-lookup"><span data-stu-id="6ecf8-112">The purpose of dividing this information into two files is to keep the software interface separate from characteristics that affect only the operating environment.</span></span>

<span data-ttu-id="6ecf8-113">O arquivo IDL especifica um contrato de rede entre o cliente e o servidor, ou seja, o arquivo IDL especifica o que é transmitido entre o cliente e o servidor.</span><span class="sxs-lookup"><span data-stu-id="6ecf8-113">The IDL file specifies a network contract between the client and server—that is, the IDL file specifies what is transmitted between the client and the server.</span></span> <span data-ttu-id="6ecf8-114">Manter essas informações diferentes das informações sobre o ambiente operacional torna o arquivo IDL portátil para outros ambientes.</span><span class="sxs-lookup"><span data-stu-id="6ecf8-114">Keeping this information distinct from the information about the operating environment makes the IDL file portable to other environments.</span></span> <span data-ttu-id="6ecf8-115">O arquivo IDL consiste em duas partes: um [cabeçalho de interface](the-idl-interface-header.md) e um [corpo de interface](the-idl-interface-body.md).</span><span class="sxs-lookup"><span data-stu-id="6ecf8-115">The IDL file consists of two parts: an [interface header](the-idl-interface-header.md) and an [interface body](the-idl-interface-body.md).</span></span>
