# <a name="building-rpc-applications"></a>Criando aplicativos RPC

O procedimento para criar um aplicativo RPC (chamada de procedimento remoto) distribuído varia ligeiramente, dependendo de:

-   A plataforma do sistema operacional na qual você está desenvolvendo.
-   A plataforma de destino.
-   A versão do compilador MIDL e C ou C++ que você usa.
-   As bibliotecas de API que você usa.

Esta seção aborda o processo de criação de aplicativos cliente/servidor com a chamada de procedimento remoto da Microsoft. Ele apresenta as informações nos tópicos a seguir:

-   [Procedimento de Build geral](general-build-procedure.md)
-   [Tratamento de exceção](exception-handling.md)
