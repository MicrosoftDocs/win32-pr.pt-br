# <a name="d-rpc"></a>D (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) D [E](e-glos.md) [F G](f-glos.md) H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_datagram_glos"></span><span id="_RPC_DATAGRAM_GLOS"></span>**Datagrama**
</dt> <dd>

Protocolo de comunicação ou transporte no qual os pacotes de dados são roteados independentemente. Eles podem seguir rotas diferentes e chegar em uma ordem diferente da qual foram enviadas. [*UDP*](u-glos.md) e [*IPX são*](i-glos.md) exemplos de protocolos de datagrama de camada de transporte. Consulte também [*orientado a conexão.*](c-glos.md)

</dd> <dt>

<span id="_rpc_discriminant_glos"></span><span id="_RPC_DISCRIMINANT_GLOS"></span>**Discriminante**
</dt> <dd>

Variável que especifica os tipos de dados que podem ser armazenados em uma união.

</dd> <dt>

<span id="_rpc_discriminated_union_glos"></span><span id="_RPC_DISCRIMINATED_UNION_GLOS"></span>**união discriminada**
</dt> <dd>

(ou *registro variante*) União que inclui um discriminador como parte da estrutura de dados para que o tipo de dados válido no momento seja transmitido junto com a união. Consulte também [*união encapsulada*](e-glos.md), união não [*encapsulada*](n-glos.md).

</dd> <dt>

<span id="_rpc_dce_glos"></span><span id="_RPC_DCE_GLOS"></span>**DCE (Ambiente de Computação Distribuída)**
</dt> <dd>

Especificação do Open Software Foundation para um conjunto de serviços integrados, incluindo chamadas de procedimento remoto, sistemas de arquivos distribuídos e serviços de segurança. O padrão RPC osF-DCE é a base para o Microsoft RPC.

</dd> <dt>

<span id="_rpc_dynamic_endpoint_glos"></span><span id="_RPC_DYNAMIC_ENDPOINT_GLOS"></span>**ponto de extremidade dinâmico**
</dt> <dd>

Endereço de servidor específico da rede que é solicitado e atribuído em tempo de executar. Consulte também [*o ponto de extremidade conhecido.*](w-glos.md)

</dd> <dt>

<span id="_rpc_dynamic_identity_tracking_glos"></span><span id="_RPC_DYNAMIC_IDENTITY_TRACKING_GLOS"></span>**acompanhamento de identidade dinâmica**
</dt> <dd>

Especificação de que a biblioteca em tempo de executar RPC usará as credenciais do thread de chamada, em vez do alça de associação, para autenticação sempre que o cliente chamar um procedimento remoto. Consulte também controle [*de identidade estático.*](s-glos.md)

</dd> </dl>
