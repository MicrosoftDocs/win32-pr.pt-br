# <a name="asynchronous-dcom"></a>DCOM assíncrono

Para aproveitar o RPC assíncrono de forma transparente, os programadores COM podem usar as interfaces assíncronas \* geradas pelo compilador MIDL, conforme descrito no tópico atributo MIDL, [**\_ UUID Async**](../midl/async-uuid.md). Para obter mais informações, consulte [clientes e servidores com](../com/com-clients-and-servers.md),[fazendo e processando chamadas assíncronas](../com/making-and-processing-asynchronous-calls.md)e [chamadas de método assíncronos em clientes e servidores com](/archive/msdn-magazine/2000/april/windows-2000-asynchronous-method-calls-eliminate-the-wait-for-com-clients-and-servers).
