# <a name="configuring-the-name-service-provider"></a>Configurando o provedor de serviços de nome

Se o aplicativo distribuído registrar sua interface com um serviço de nome, o cliente e o servidor deverão estar usando o mesmo serviço de nome. O Microsoft RPC interopera com o Microsoft Locator e com qualquer NSP (provedor de serviços de nome) que adere à NSI (interface de serviço de nome) do Microsoft RPC – por exemplo, o Serviço de Diretório de Célula do DCE acessado por meio do daemon de serviço de nome da Digital Equipment Corporation (NSID). O Microsoft Locator, que foi projetado para uso Windows ambientes, é o provedor de serviços de nome padrão.

Esta seção apresenta uma discussão sobre como instalar e configurar provedores de serviços de nomes:

-   [Configurando o serviço de nome](configuring-the-name-service-for-windows-xp-windows-2000-or-windows-nt.md)
-   [Configurando o Serviço de Nome para Windows 95](configuring-the-name-service-for-windows-95.md)
-   [Configurando o Serviço de Nome para Windows 3.x ou MS-DOS](configuring-the-name-service-for-windows-3-x-or-ms-dos.md)
-   [Iniciando e parando o Localizador da Microsoft](starting-and-stopping-microsoft-locator.md)
