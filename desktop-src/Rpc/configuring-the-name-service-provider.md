# <a name="configuring-the-name-service-provider"></a>Configurando o provedor de serviços de nome

Se seu aplicativo distribuído registrar sua interface com um serviço de nome, o cliente e o servidor deverão estar usando o mesmo serviço de nome. O Microsoft RPC interopera com o Microsoft Locator e qualquer NSP (provedor de serviços de nome) que segue a interface de serviço de nome RPC (NSI) da Microsoft, por exemplo, o serviço de diretório de célula do DCE acessado por meio do NSID (daemon de serviço de nome) da Digital Equipment Corporation. O Microsoft Locator, que foi projetado para uso em ambientes Windows, é o provedor de serviço de nome padrão.

Esta seção apresenta uma discussão sobre como instalar e configurar provedores de serviço de nome:

-   [Configurando o serviço de nome](configuring-the-name-service-for-windows-xp-windows-2000-or-windows-nt.md)
-   [Configurando o serviço de nome para o Windows 95](configuring-the-name-service-for-windows-95.md)
-   [Configurando o serviço de nome para Windows 3. x ou MS-DOS](configuring-the-name-service-for-windows-3-x-or-ms-dos.md)
-   [Iniciando e parando o Microsoft Locator](starting-and-stopping-microsoft-locator.md)
