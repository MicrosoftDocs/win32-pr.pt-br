# <a name="security-rpc"></a>Segurança (RPC)

Com o maior uso de aplicativos distribuídos, a necessidade de comunicação segura entre as partes do cliente e do servidor de aplicativos é fundamental. A biblioteca de tempo de execução RPC (chamada de procedimento remoto) fornece uma interface padronizada para serviços de autenticação para clientes e servidores. Os serviços de autenticação no sistema host do servidor fornecem autenticação RPC. Os aplicativos usam chamadas de procedimento remoto autenticadas para garantir que todas as chamadas sejam provenientes de clientes autorizados. Eles também podem ajudar a garantir que todas as respostas do servidor venham de servidores autenticados.

Esta seção apresenta uma visão geral do RPC autenticado nas seguintes seções:

-   [Noções básicas de segurança RPC](rpc-security-essentials.md)
-   [Métodos de segurança](security-methods.md)
