# <a name="a-programmers-model-of-replication-in-active-directory-domain-services"></a><span data-ttu-id="9c52e-105">Um modelo de replicação do programador no Active Directory Domain Services</span><span class="sxs-lookup"><span data-stu-id="9c52e-105">A Programmer's Model of Replication in Active Directory Domain Services</span></span>

<span data-ttu-id="9c52e-106">Veja a seguir uma descrição do modelo de replicação para Active Directory Domain Services.</span><span class="sxs-lookup"><span data-stu-id="9c52e-106">The following is a description of the replication model for Active Directory Domain Services.</span></span>

<span data-ttu-id="9c52e-107">Todas as atualizações para o controlador de Domínio do Active Directory (DC) são executadas usando solicitações LDAP que criam, modificam ou excluem um objeto para cada solicitação.</span><span class="sxs-lookup"><span data-stu-id="9c52e-107">All updates to the Active Directory Domain Controller (DC) are performed using LDAP requests that create, modify, or delete one object for each request.</span></span> <span data-ttu-id="9c52e-108">Uma única solicitação pode definir ou modificar vários atributos em um objeto.</span><span class="sxs-lookup"><span data-stu-id="9c52e-108">A single request can set or modify multiple attributes on an object.</span></span>

<span data-ttu-id="9c52e-109">Uma solicitação de atualização é processada como uma transação atômica em algum DC.</span><span class="sxs-lookup"><span data-stu-id="9c52e-109">An update request is processed as an atomic transaction at some DC.</span></span> <span data-ttu-id="9c52e-110">Toda a atualização ocorre ou nenhuma delas.</span><span class="sxs-lookup"><span data-stu-id="9c52e-110">Either the entire update happens or none of it does.</span></span> <span data-ttu-id="9c52e-111">Se o solicitante receber uma resposta bem-sucedida a uma solicitação de atualização, essa solicitação inteira terá êxito (confirmada).</span><span class="sxs-lookup"><span data-stu-id="9c52e-111">If the requester receives a successful response to an update request, that entire request has succeeded (committed).</span></span> <span data-ttu-id="9c52e-112">Isso é chamado de "gravação de origem".</span><span class="sxs-lookup"><span data-stu-id="9c52e-112">This is called an "originating write."</span></span> <span data-ttu-id="9c52e-113">Várias solicitações LDAP não podem ser agrupadas em uma única transação maior.</span><span class="sxs-lookup"><span data-stu-id="9c52e-113">Multiple LDAP requests cannot be grouped into a single larger transaction.</span></span>

<span data-ttu-id="9c52e-114">Em uma gravação de origem, um DC computa um "carimbo" para cada valor de atributo novo ou modificado e anexa esse carimbo ao valor, de modo que, quando o valor for replicado, o carimbo também seja replicado.</span><span class="sxs-lookup"><span data-stu-id="9c52e-114">In an originating write, a DC computes a "stamp" for each new or modified attribute value, and attaches this stamp to the value so when the value is replicated, the stamp is replicated too.</span></span> <span data-ttu-id="9c52e-115">O novo carimbo é exclusivo e, no caso de uma atualização, o novo carimbo é maior do que o carimbo no valor antigo nesse DC.</span><span class="sxs-lookup"><span data-stu-id="9c52e-115">The new stamp is unique, and in case of an update, the new stamp is larger than the stamp on the old value at that DC.</span></span>

<span data-ttu-id="9c52e-116">Ocasionalmente, um DC seleciona o conjunto de objetos que foram alterados desde a última vez em que o DC realizou a replicação.</span><span class="sxs-lookup"><span data-stu-id="9c52e-116">Occasionally, a DC selects the set of objects that have changed since the last time the DC performed replication.</span></span> <span data-ttu-id="9c52e-117">Em seguida, para cada objeto, ele envia uma única mensagem para todos os outros DCs que contêm todos os valores atuais de atributos alterados desde a última vez em que o objeto foi replicado.</span><span class="sxs-lookup"><span data-stu-id="9c52e-117">Then, for each object, it sends a single message to all other DCs that contain all the current values of attributes changed since the last time the object was replicated.</span></span> <span data-ttu-id="9c52e-118">As mensagens de replicação são confiáveis e são entregues em ordem, mas podem exigir mais tempo para serem entregues.</span><span class="sxs-lookup"><span data-stu-id="9c52e-118">Replication messages are reliable and are delivered in order, but may require more time to be delivered.</span></span>

<span data-ttu-id="9c52e-119">Quando um DC recebe uma mensagem de replicação de outro DC, ele a processa da seguinte maneira: para cada atributo modificado, se o carimbo no valor na mensagem de replicação for maior do que o carimbo no valor atual, o controlador de domínio aplicará a atualização; caso contrário, o DC descartará a atualização.</span><span class="sxs-lookup"><span data-stu-id="9c52e-119">When one DC receives a replication message from another DC, it processes it as follows: For each modified attribute, if the stamp on the value in the replication message is larger than the stamp on the current value, the DC applies the update; otherwise the DC discards the update.</span></span> <span data-ttu-id="9c52e-120">Cada mensagem de replicação é aplicada como uma transação atômica, assim como uma gravação de origem.</span><span class="sxs-lookup"><span data-stu-id="9c52e-120">Each replication message is applied as an atomic transaction, just like an originating write.</span></span>

<span data-ttu-id="9c52e-121">Esse é o modelo de replicação de Active Directory Domain Services.</span><span class="sxs-lookup"><span data-stu-id="9c52e-121">That is the Active Directory Domain Services replication model.</span></span> <span data-ttu-id="9c52e-122">As principais propriedades desse modelo incluem:</span><span class="sxs-lookup"><span data-stu-id="9c52e-122">Key properties of this model include:</span></span>

-   <span data-ttu-id="9c52e-123">Uma gravação de origem em um único objeto é atômica.</span><span class="sxs-lookup"><span data-stu-id="9c52e-123">An originating write to a single object is atomic.</span></span>
-   <span data-ttu-id="9c52e-124">Ao replicar as alterações, todas as alterações feitas por uma gravação de origem são enviadas ou nenhuma delas é.</span><span class="sxs-lookup"><span data-stu-id="9c52e-124">When replicating changes, either all the changes made by an originating write are sent or none of them are.</span></span>
-   <span data-ttu-id="9c52e-125">Uma gravação replicada em um único objeto é atômica, mas os conflitos são resolvidos atributo-por-atributo.</span><span class="sxs-lookup"><span data-stu-id="9c52e-125">A replicated write to a single object is atomic, but conflicts are resolved attribute-by-attribute.</span></span>

<span data-ttu-id="9c52e-126">O modelo não garante a ordenação de replicação de alterações feitas em objetos diferentes.</span><span class="sxs-lookup"><span data-stu-id="9c52e-126">The model does not guarantee the replication ordering of changes made to different objects.</span></span> <span data-ttu-id="9c52e-127">Não grave aplicativos que supõem que as alterações sejam replicadas na ordem de gravação de origem.</span><span class="sxs-lookup"><span data-stu-id="9c52e-127">Do not write applications that assume that changes are replicated in originating-write order.</span></span> <span data-ttu-id="9c52e-128">O modelo não garante que, se um atributo de um objeto for alterado duas vezes, ambos os valores serão replicados; A replicação envia apenas o valor atual no momento da replicação.</span><span class="sxs-lookup"><span data-stu-id="9c52e-128">The model does not guarantee that if an attribute of an object is changed twice, both values will be replicated; Replication sends only the current value at the time of replication.</span></span>

<span data-ttu-id="9c52e-129">O modelo difere da realidade de várias maneiras que afetam apenas o desempenho.</span><span class="sxs-lookup"><span data-stu-id="9c52e-129">The model differs from reality in several ways that only affect performance.</span></span> <span data-ttu-id="9c52e-130">Por exemplo, o servidor de Active Directory envia mensagens de replicação que contêm as alterações a vários objetos, mas processa o conteúdo de uma mensagem de vários objetos como se fosse uma série de mensagens de objeto único.</span><span class="sxs-lookup"><span data-stu-id="9c52e-130">For example, the Active Directory Server sends replication messages containing the changes to multiple objects, but processes the contents of such a multi-object message as if it were a series of single-object messages.</span></span> <span data-ttu-id="9c52e-131">O servidor de Active Directory não executa a replicação ponto a ponto conforme descrito no modelo, mas realiza uma replicação transitiva mais complexa e eficiente que é funcionalmente equivalente ao modelo.</span><span class="sxs-lookup"><span data-stu-id="9c52e-131">The Active Directory Server does not perform point-to-point replication as described in the model, but instead performs a more complex and more efficient transitive replication that is functionally equivalent to the model.</span></span>
