# <a name="determining-shim-status"></a>Determinando o status do Shim

## <a name="platforms"></a>Plataformas

<dl> Clientes-Windows 8  
Servidores-Windows Server 2012  
</dl>

## <a name="description"></a>Descrição

O Windows 8 registra eventos quando os aplicativos estão sendo corrigido por vários motivos. A maneira mais fácil de determinar se seu aplicativo está sendo corrigido é verificar o Visualizador de eventos. Acesse logs de aplicativos e serviços \\ Microsoft Windows Application-Experience \\ telemetria de programa.

## <a name="mitigation"></a>Atenuação

A melhor maneira de se ficar fora do shims é renomear o executável ou aumentar a versão principal em 1 (recompilar o executável).
