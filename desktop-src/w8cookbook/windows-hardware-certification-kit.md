# <a name="windows-hardware-certification-kit"></a>Kit de certificação de hardware do Windows

## <a name="platforms"></a>Plataformas

 **Clientes** -Windows 7 Windows \| 8  
**Servidores** -windows Server 2008 R2 \| Windows Server 2012  
**Servidor/controlador** -Windows Server 2008 R2  


## <a name="description"></a>Descrição

O kit de certificação de hardware do Windows permite que desenvolvedores, ISVs, IHVs e OEMs certificem seus dispositivos de hardware, sistemas e drivers de filtro para os sistemas operacionais Windows mais recentes. Ele contém todas as ferramentas e a documentação de que você precisa para certificar seus drivers de hardware e filtro para esses sistemas operacionais:

-   Windows 8
-   Windows Server 2012
-   Windows 7
-   Windows Server 2008 R2

O kit de certificação de hardware do Windows substitui o kit de logotipo do Windows e faz parte do programa de certificação do Windows.

## <a name="usage-or-best-practices"></a>Uso ou práticas recomendadas

Antes de poder testar qualquer driver de hardware ou de filtro, você deve configurar o ambiente de teste adequado com base no driver de filtro ou hardware que você está certificando. Isso inclui o controlador, o cliente e o hardware potencialmente adicional (por exemplo, dispositivos com várias funções) ou software. Depois que o ambiente estiver configurado, você poderá testar seu hardware usando a nova ferramenta do HCK Studio. Estas etapas descrevem o processo de teste de certificação.

1.  Configurar ambiente de teste.
2.  Criar um projeto.
3.  Crie um ou mais pools de computadores.
4.  Selecione os recursos a serem validados.
5.  Selecione e execute testes nesses recursos.
6.  Exibir resultados de teste.
7.  Envie seu pacote.
