# <a name="key-changes-since-windows-7-to-ensure-compatibility"></a>Principais alterações desde o Windows 7 para garantir a compatibilidade

Entendemos que compatibilidade é importante para os desenvolvedores. Os ISVs e desenvolvedores querem verificar se os aplicativos deles serão executados como esperado em todas as versões compatíveis do sistema operacional Windows. Os consumidores e as empresas têm um importante investimento aqui — eles querem a certeza de que os aplicativos pelos quais eles pagaram continuarão funcionando. Nós sabemos que compatibilidade é o principal critério para decisões de compra. Aplicativos bem escritos com base em práticas recomendadas levarão a muito menos variação de código quando uma nova versão do Windows for lançada e reduzirão a fragmentação— esses aplicativos têm um investimento de engenharia reduzido a ser mantido e um tempo de comercialização mais rápido.

No cronograma do Windows 7, compatibilidade era muito mais uma abordagem reativa. No Windows 8, começamos a observar isso de forma diferente, trabalhando dentro do Windows para garantir que a compatibilidade foi projetada no lugar de ser considerada algo secundário. Windows 10 é a versão mais compatível por design do sistema operacional até a data. Veja algumas importantes maneiras por meio das quais conseguimos isso:

-   Telemetria do aplicativo: isso nos ajuda a entender a popularidade do aplicativo no ecossistema Windows para informar os testes de compatibilidade.
-   Parcerias ISV: trabalhe diretamente com parceiros externos para fornecer dados a eles e ajudar a corrigir problemas que nossos usuários experimentam.
-   Revisões de design, detecção upstream: parceiro com equipes de recursos para reduzir o número de alterações significativas Windows. A análise de compatibilidade é uma entrada pela qual as nossas equipes de recurso devem passar.
-   Controle mais rígido sobre alterações de API & comunicação aprimorada
-   Loop de voo e comentários: a população do Windows Insider recebe builds voos que ajudam a melhorar nossa capacidade de encontrar problemas de compatibilidade antes que um build final seja liberado para os clientes. Esse processo de comentários não só expõe os bugs, mas verifica se entregamos os recursos que nossos usuários querem.
