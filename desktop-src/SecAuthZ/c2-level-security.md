# <a name="c2-level-security"></a>Segurança em nível C2

A lista a seguir inclui alguns dos requisitos mais importantes da segurança em nível C2, conforme definido pelo departamento de defesa dos EUA:

-   Deve ser possível controlar o acesso a um recurso concedendo ou negando acesso a usuários individuais ou a grupos de usuários nomeados.
-   A memória deve ser protegida para que seu conteúdo não possa ser lido depois que um [*processo*](/windows/desktop/SecGloss/p-gly) o liberar. Da mesma forma, um sistema de arquivos seguro, como NTFS, deve proteger arquivos excluídos de serem lidos.
-   Os usuários devem se identificar de uma maneira exclusiva, como por senha, quando fazem logon. Todas as ações auditáveis devem identificar o usuário que executa a ação.
-   Os administradores do sistema devem ser capazes de auditar eventos relacionados à segurança. No entanto, o acesso aos dados de auditoria de eventos relacionados à segurança deve ser limitado a administradores autorizados.
-   O sistema deve ser protegido contra interferência externa ou violação, como a modificação do sistema em execução ou de arquivos do sistema armazenados no disco.
