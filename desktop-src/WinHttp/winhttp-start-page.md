# <a name="windows-http-services"></a>Windows Serviços HTTP

## <a name="purpose"></a>Finalidade

O Microsoft Windows Http Services (WinHTTP) fornece aos desenvolvedores uma API (interface de programação de aplicativo) cliente HTTP para enviar solicitações por meio do protocolo HTTP para outros servidores HTTP.

## <a name="where-applicable"></a>Quando aplicável

O WinHTTP dá suporte a aplicativos cliente da área de trabalho, Windows serviços e Windows aplicativos baseados em servidor.

Para obter mais informações sobre como usar o WinHTTP para aplicativos com base no Microsoft .NET Framework, consulte a [API WinHttpHandler](/dotnet/api/system.net.http?view=netframework-4.8)

## <a name="developer-audience"></a>Público de desenvolvedores

O WinHTTP oferece uma API (interface de programação de aplicativos) C/C++ e um componente de automação com Component Object Model (COM) adequado para uso em aplicativos baseados em PÁGINAS Active Server (ASP).

Uma compreensão básica do protocolo HTTP é importante para usar qualquer interface.

## <a name="run-time-requirements"></a>Requisitos de tempo de execução

O WinHTTP 5.1 oferece melhorias em relação à versão 5.0. Ele está incluído no sistema operacional. Para obter mais informações sobre novos recursos, consulte Novidades no [WinHTTP 5.1](what-s-new-in-winhttp-5-1.md) e Novidades no [Windows Server 2008](what-s-new-in-windows-longhorn.md)e Windows Vista .

> [!IMPORTANT]
> O download do WinHTTP 5.0 não está mais disponível. A partir de 1º de outubro de 2004, a Microsoft removeu o download do SDK do WinHTTP 5.0 do MSDN e terminou o suporte ao produto para a versão 5.0.
