# <a name="using-winhttp-tools"></a>Usando ferramentas WinHTTP

Para os comandos que você pode usar no Windows Vista e posterior para definir as configurações de proxy e rastreamento para o HTTP do Windows , consulte Comandos Netsh para [Windows WINHTTP (Hypertext Transfer Protocol).](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc731131(v=ws.10))

**Windows Server 2003 e Anterior:** Há várias ferramentas disponíveis para ajudá-lo a escrever e depurar seus aplicativos. Eles são explicados nos tópicos a seguir:

-   [WinHttpCfg.exe,](winhttpcertcfg-exe--a-certificate-configuration-tool.md) uma Ferramenta de Configuração de Certificados permite que [](glossary.md) os administradores instalem e configurem certificados de cliente em qualquer armazenamento de certificados que possa ser acessado pela conta do IWAM (Gerenciador de Aplicativos Web) do Servidor internet. A ferramenta também elimina a necessidade de fazer qualquer coisa especial para contas como a conta IWAM para obter acesso a certificados ao usar Active Server Pages (ASP).
-   [Netsh.exe ou ProxyCfg.exe proxy Ferramentas de Configuração](proxycfg-exe--a-proxy-configuration-tool.md) pode ser usado para configurar servidores proxy com o WinHTTP (Serviços HTTP do Microsoft Windows).
-   [WinHttpTraceCfg, uma Ferramenta](winhttptracecfg-exe--a-trace-configuration-tool.md) de Configuração de Rastreamento fornece a capacidade de monitorar funções WinHTTP e o tráfego de rede correspondente. Os aplicativos de depuração que utilizam protocolos de transmissão criptografados, como protocolo SSL (SSL), impedem a interceptação de pacotes na camada de transporte de rede e exigem a capacidade de interceptar o tráfego entre o cliente e o servidor depois que ele for descriptografado. O recurso de rastreamento WinHTTP tem uma variedade de recursos para interceptar o fluxo de tráfego entre o cliente e o servidor.
