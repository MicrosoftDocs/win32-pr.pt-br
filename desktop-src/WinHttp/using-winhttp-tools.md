# <a name="using-winhttp-tools"></a>Usando as ferramentas do WinHTTP

Para os comandos que você pode usar no Windows Vista e versões posteriores para definir configurações de proxy e rastreamento para HTTP do Windows, consulte [comandos netsh para o protocolo de transferência de hipertexto do Windows (WinHTTP)](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc731131(v=ws.10)).

**Windows Server 2003 e anterior:** Há várias ferramentas que estão disponíveis para ajudá-lo a escrever e depurar seus aplicativos. Eles são explicados nos seguintes tópicos:

-   [WinHttpCfg.exe, uma ferramenta de configuração de certificado](winhttpcertcfg-exe--a-certificate-configuration-tool.md) permite que os administradores instalem e configurem certificados de cliente em qualquer [*repositório de certificados*](glossary.md) que possa ser acessado pela conta do Gerenciador de aplicativos da Web do servidor da Internet (IWAM). A ferramenta também elimina a necessidade de fazer qualquer coisa especial para contas como a conta IWAM para obter acesso aos certificados ao usar páginas de Active Server (ASP).
-   [ Ferramentas de configuração de proxy deNetsh.exe ou ProxyCfg.exe](proxycfg-exe--a-proxy-configuration-tool.md) pode ser usado para configurar servidores proxy com o Microsoft Windows http Services (WinHTTP).
-   [WinHttpTraceCfg, uma ferramenta de configuração de rastreamento](winhttptracecfg-exe--a-trace-configuration-tool.md) fornece a capacidade de monitorar as funções WinHTTP e o tráfego de rede correspondente. Depurar aplicativos que utilizam protocolos de fio criptografados, como protocolo SSL (SSL), impedir a detecção de pacotes na camada de transporte de rede e exigir a capacidade de interceptar o tráfego entre o cliente e o servidor depois que ele tiver sido descriptografado. O recurso de rastreamento do WinHTTP tem uma variedade de recursos para interceptar o fluxo de tráfego entre o cliente e o servidor.
