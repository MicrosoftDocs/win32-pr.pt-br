# <a name="programming-with-the-system-registry-provider"></a>Programando com o provedor de registro do sistema

Os tópicos a seguir descrevem como os aplicativos podem usar o provedor de registro do sistema para fornecer dados para as classes que você criar.



| Tópico                                                                                                                      | Descrição                                                                                                                                                                                                                    |
|----------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [Definindo classes para o provedor de registro do sistema](defining-classes-for-the-system-registry-provider.md)                 | Para acessar o registro do sistema, você deve criar classes do Windows que armazenam informações de registro no repositório Instrumentação de Gerenciamento do Windows (WMI).                                                                     |
| [Usando o provedor de registro do sistema como um provedor de propriedades](using-the-system-registry-provider-as-a-property-provider.md) | O provedor de registro do sistema requer que você use vários qualificadores para exibir as propriedades do registro.                                                                                                                             |
| [Descrevendo um recurso para o registro](describing-a-resource-for-the-registry.md)                                       | Você pode acessar dados no registro que descrevem um recurso do sistema. Para modificar as informações, o provedor de registro do sistema requer uma sintaxe específica.                                                                      |
| [Acessando um valor de registro sem nome](accessing-an-unnamed-registry-value.md)                                             | Para acessar a maioria dos valores do registro, crie uma classe para receber os valores no WMI, ou você pode examinar propriedades específicas do registro. No entanto, o provedor de registro do sistema requer uma sintaxe especial para exibir valores de registro sem nome. |
