# <a name="i-wmi"></a>I (WMI)

<dl> <dt>

<span id="wmi.gloss_impersonation"></span><span id="WMI.GLOSS_IMPERSONATION"></span>**Representação**
</dt> <dd>

A capacidade de um thread em um processo assumir a identidade de segurança e os privilégios de outro processo.

</dd> <dt>

<span id="wmi.gloss_instance_provider"></span><span id="WMI.GLOSS_INSTANCE_PROVIDER"></span>**provedor de instância**
</dt> <dd>

Um servidor COM que fornece instâncias de classes e implementa métodos da interface [**IWbemServices**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) para dar suporte à recuperação de dados, modificação, exclusão, enumeração ou processamento de consulta.

</dd> <dt>

<span id="wmi.gloss_intrinsic_event"></span><span id="WMI.GLOSS_INTRINSIC_EVENT"></span>**evento intrínseco**
</dt> <dd>

Um evento que o WMI fornece, que é uma notificação de uma alteração no modelo de dados WMI padrão. Por exemplo, a criação de uma nova instância do [**Processo Win32 \_**](/windows/desktop/CIMWin32Prov/win32-process) é um [**\_ \_ tipo InstanceCreationEvent**](--instancecreationevent.md) de evento intrínseco. Consulte também [*evento extrífico*](gloss-e.md).

</dd> <dt>

<span id="wmi.gloss_instrumentation"></span><span id="WMI.GLOSS_INSTRUMENTATION"></span>**Instrumentação**
</dt> <dd>

Um mecanismo usado para relatar o status de hardware e software do sistema. O WMI implementa a instrumentação modelando um sistema usando classes baseadas no [*CIM (modelo CIM)*](gloss-c.md). O WMI no .NET Framework cria classes instrumentadas adicionando atributos.

</dd> </dl>
