Descreve os erros do provedor SNMP WMI 281 a 290.

[Erro fatal 282](#fatal-error-282)

[Erro fatal 283](#fatal-error-283)

## <a name="fatal-error-282"></a>Erro fatal 282

<dl> <dt>

<span id="_282__Fatal_____fileName___line____Syntax_error._Last_token_read_is__token__"></span><span id="_282__fatal_____filename___line____syntax_error._last_token_read_is__token__"></span><span id="_282__FATAL_____FILENAME___LINE____SYNTAX_ERROR._LAST_TOKEN_READ_IS__TOKEN__"></span>**<282,> fatal: " <fileName> : <linha \#>: erro de sintaxe. O último token lido é <token> "**
</dt> <dd>

Esse erro ocorre quando o compilador sabe que ocorreu um erro de sintaxe, mas não pode fornecer informações mais específicas sobre o erro de sintaxe.

</dd> </dl>

## <a name="fatal-error-283"></a>Erro fatal 283

<dl> <dt>

<span id="_283__Fatal_____fileName___line_____Unrecognized_character__character__in_input_"></span><span id="_283__fatal_____filename___line_____unrecognized_character__character__in_input_"></span><span id="_283__FATAL_____FILENAME___LINE_____UNRECOGNIZED_CHARACTER__CHARACTER__IN_INPUT_"></span>**<283,> fatal: " <fileName> : linha de <\# \#>: caractere não reconhecido <character> na entrada"**
</dt> <dd>

Há um caractere não reconhecido na entrada. Normalmente, os caracteres não reconhecidos são ignorados e essa mensagem de erro relata o primeiro caractere não reconhecido.

</dd> </dl>
