Consulte *modelo CIM (CIM)*.

</dd> <dt>

<span id="wmi.gloss_cim_datetime"></span><span id="WMI.GLOSS_CIM_DATETIME"></span>**Datetime cim**
</dt> <dd>

O formato de data e hora combinado definido pela DMTF (Distributed Management Task Force).

</dd> <dt>

<span id="wmi.gloss_cim_object_manager"></span><span id="WMI.GLOSS_CIM_OBJECT_MANAGER"></span>**Gerenciador de Objetos CIM**
</dt> <dd>

Um componente na infraestrutura WMI que lida com a interação entre aplicativos de gerenciamento e [*provedores*](gloss-p.md).

</dd> <dt>

<span id="wmi.gloss_cim_repository"></span><span id="WMI.GLOSS_CIM_REPOSITORY"></span>**Repositório CIM**
</dt> <dd>

Consulte [*Repositório WMI*](gloss-w.md).

</dd> <dt>

<span id="wmi.gloss_cim_schema"></span><span id="WMI.GLOSS_CIM_SCHEMA"></span>**Esquema CIM**
</dt> <dd>

Uma coleção de definições de classe usada para representar [*objetos gerenciados*](gloss-m.md) que ocorrem em todos os ambientes gerenciados. Consulte também *modelo comum* e esquema [*de extensão*](gloss-e.md).

</dd> <dt>

<span id="wmi.gloss_common_information_model"></span><span id="WMI.GLOSS_COMMON_INFORMATION_MODEL"></span>**modelo CIM (CIM)**
</dt> <dd>

Um modelo usado para descrever como representar objetos gerenciados do mundo real.

</dd> <dt>

<span id="wmi.gloss_common_model"></span><span id="WMI.GLOSS_COMMON_MODEL"></span>**modelo comum**
</dt> <dd>

A segunda camada do *esquema CIM,* que inclui uma série de classes específicas do domínio, mas independentes de plataforma. Os domínios são sistemas, redes, aplicativos e outros dados relacionados ao gerenciamento. O modelo comum é derivado do modelo principal. Consulte também [*o esquema de extensão*](gloss-e.md).

</dd> <dt>

<span id="wmi.gloss_consumer_filter_binding"></span><span id="WMI.GLOSS_CONSUMER_FILTER_BINDING"></span>**associação de filtro de consumidor**
</dt> <dd>

Uma instância da classe de sistema [**\_ \_ FilterToConsumerBinding**](--filtertoconsumerbinding.md) que descreve uma relação entre um consumidor [*lógico*](gloss-l.md) e um [*filtro de evento*](gloss-e.md). A associação de filtro de consumidor é um componente da arquitetura permanente do consumidor.

</dd> <dt>

<span id="wmi.gloss_consumer_provider"></span><span id="WMI.GLOSS_CONSUMER_PROVIDER"></span>**provedor de consumidor**
</dt> <dd>

Consulte [*provedor de consumidor de eventos*](gloss-e.md).

</dd> </dl>
