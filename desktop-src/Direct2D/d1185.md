# <a name="d1185-fillgeometry-opacity-mask-brush-restriction"></a>D1185: restrição de pincel de máscara de opacidade FillGeometry

Se FillGeometry for usado com uma máscara de opacidade, o pincel primário deverá ser um pincel de bitmap clamped.






 

## <a name="possible-causes"></a>Possíveis causas

Ao usar ID2D1RenderTarget:: FillGeometry, o pincel primário era um tipo diferente de ID2D1BitmapBrush. Ou, o pincel primário tinha um modo de extensão x ou y diferente do modo D2D1 \_ Extend \_ \_ fixe.

## <a name="possible-fixes"></a>Possíveis correções

Se o aplicativo usar FillGeometry com uma máscara de opacidade, verifique se o pincel primário (não deve ser confundido com o pincel de opacidade) é um ID2D1BitmapBrush com modos de extensão clamped. Se o comportamento de máscara de opacidade não for importante, deixe o argumento de pincel de opacidade como nulo para remover essas restrições no pincel primário.
