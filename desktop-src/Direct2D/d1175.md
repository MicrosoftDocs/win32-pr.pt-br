# <a name="d1175-dxgi-target-must-not-use-software-flag"></a>D1175: o destino DXGI não deve usar um sinalizador de software

Esse tipo de renderTarget deve ser sinalizado como padrão ou HARDWARE.






 

## <a name="possible-causes"></a>Possíveis causas

Um aplicativo chamado ID2D1Factory:: CreateDxgiSurfaceRenderTarget com um \_ objeto de propriedades de destino de RENDERIZAÇÃO d2d1 \_ \_ cujo campo de tipo foi definido como d2d1 \_ software de tipo de destino de renderização \_ \_ \_ . Direct2D considera que essa combinação de especificadores está incorreta, pois o sinalizador de SOFTWARE é reservado para destinos criados usando CreateWicBitmapRenderTarget e CreateHwndRenderTarget.

## <a name="possible-fixes"></a>Possíveis correções

Ao usar CreateDxgiSurfaceRenderTarget, os aplicativos devem especificar o tipo de destino de D2D1 de \_ processamento \_ \_ \_ .
