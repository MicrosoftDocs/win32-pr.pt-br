# <a name="d1234-null-resource-brush"></a>D1234: pincel de recurso nulo

Um pincel de bitmap ou um pincel de imagem não pode ser usado para desenhar quando seu bitmap/imagem associado está definido como nulo.






 

## <a name="possible-causes"></a>Possíveis causas

O aplicativo tentou renderizar um primitivo usando um ID2D1BitmapBrush ou ID2D1ImageBrush cujo bitmap ou imagem era nulo.

## <a name="possible-fixes"></a>Possíveis correções

Verifique se os pincéis de bitmap ou imagem têm um bitmap ou imagem válidos atribuídos a eles antes de usá-los. O aplicativo pode fazer isso no momento da criação ou depois, usando ID2D1BitmapBrush:: SetBitmap ou ID2D1ImageBrush:: SetImage. O Direct2D permite recursos de pincel nulo para que os aplicativos possam ter a liberdade de liberar recursos de pincel depois de usá-los para renderizar. Os aplicativos podem optar por fazer isso se o recurso de bitmap ou imagem for particularmente dispendioso e o cenário exigir que ele seja liberado imediatamente. Definir um recurso de pincel como nulo impede que o Direct2D mantenha uma referência pendente para ele. No entanto, se um aplicativo atribuir um recurso de pincel a NULL, ele não deverá tentar usar o pincel até que um novo recurso válido tenha sido atribuído a ele.
