# <a name="d1251-d3d-state-issue"></a>D1251: problema de estado do D3D

O estado do dispositivo Direct3D foi alterado quando o Direct2D ainda o estava usando. Isso pode indicar uma violação de Threading porque apenas um thread pode usar o dispositivo de cada vez.






 

## <a name="possible-causes"></a>Possíveis causas

O aplicativo pode estar usando uma única fábrica de D2D threaded em dois threads diferentes ao mesmo tempo.

## <a name="possible-fixes"></a>Possíveis correções

Verifique se o aplicativo está respeitando adequadamente as políticas de threading de Direct2D's.
