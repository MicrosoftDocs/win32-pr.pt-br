# <a name="d1123-dxgi-format-unknown-not-allowed"></a>D1123: formato DXGI desconhecido não permitido

\_Formato dxgi \_ desconhecido não é permitido como parâmetro para esta API.






 

## <a name="possible-causes"></a>Possíveis causas

Para a funcionalidade Direct2D que aceita um formato de superfície como parâmetro, o formato da superfície deve ser totalmente especificado. Em alguns casos, pode ser deixado desconhecido; por exemplo, ao chamar ID2D1RenderTarget:: CreateHwndRenderTarget ou ID2D1RenderTarget:: CreateDxgiSurfaceRenderTarget, Direct2D pode inferir um formato de superfície do formato da cadeia de permuta ou do formato de superfície do Direct3D de backup, como o caso pode ser.

## <a name="possible-fixes"></a>Possíveis correções

Nos casos em que não foi possível inferir o formato da superfície, o Direct2D não criará o recurso e produzirá essa mensagem. Para resolver isso, modifique o aplicativo para especificar um formato de superfície diferente de desconhecido para a operação que causa essa mensagem. Um formato de superfície padrão razoável a ser usado, em caso de dúvida, é o \_ formato dxgi \_ B8G8R8A8 \_ UNORM. Se o aplicativo precisar criar um recurso Direct2D para interoperar com um recurso de Direct3D ou WIC, escolha um formato que corresponda ao recurso interoperado. A semântica para recursos de WIC difere ligeiramente dos recursos de DXGI; consulte [formatos de pixel com suporte e modos alfa](supported-pixel-formats-and-alpha-modes.md) para obter diretrizes adicionais.
