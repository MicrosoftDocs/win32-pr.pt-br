# <a name="d1180-begindraw-incorrect-state"></a>D1180: BeginDraw estado incorreto

BeginDraw foi chamado várias vezes, sem uma chamada correspondente para EndDraw.






 

## <a name="possible-causes"></a>Possíveis causas

O aplicativo fez várias chamadas para BeginDraw sem chamar EndDraw no mesmo destino. Ao usar Direct2D, os aplicativos devem corresponder a cada chamada para BeginDraw com um EndDraw subsequente. Como alternativa, essa mensagem pode ocorrer se o destino estiver em um estado incorreto e a chamada BeginDraw não permitisse que o destino fosse renderizado.

## <a name="possible-fixes"></a>Possíveis correções

Certifique-se de que cada chamada para BeginDraw seja correspondida com um EndDraw subsequente. Ao enviar comandos para um destino usando flush ou EndDraw, verifique se o resultado retornado é S \_ OK antes de tentar usar o destino novamente.
