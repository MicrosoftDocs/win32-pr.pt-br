# <a name="d1127-alpha-mode-mismatch"></a>D1127: Incompatibilidade de modo alfa

O modo alfa passado para essa API não é compatível com o formato de pixel da \[ *superfície* \] .

## <a name="placeholders"></a>Espaços reservados

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*Superfície*
</dt> <dd>

A superfície direct3D que não dá suporte ao modo alfa.

</dd> </dl> 



 

## <a name="possible-causes"></a>Possíveis causas

Operações como ID2D1RenderTarget::CreateBitmapFromWicBitmap ou ID2D1RenderTarget::CreateDxgiSurfaceRendertarget exigem uma superfície e um modo alfa de superfície especificado. Direct2D verifica se o modo alfa especificado é compatível com a superfície. Se não for compatível, Direct2D a saída dessa mensagem.

## <a name="possible-fixes"></a>Possíveis correções

Modifique o aplicativo para usar um MODO ALFA que corresponde ou é compatível com o \_ modo alfa da superfície subjacente. Consulte [Formatos de pixel e modos](supported-pixel-formats-and-alpha-modes.md) alfa com suporte para obter diretrizes adicionais sobre os modos alfa expostos por Direct2D.
