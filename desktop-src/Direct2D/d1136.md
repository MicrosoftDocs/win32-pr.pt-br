# <a name="d1136-text-rendering-mode-conflict"></a>D1136: conflito de modo de renderização de texto

O \_ modo de RENDERIZAÇÃO DWRITE \_ e o \_ \_ modo AntiAlias de texto d2d1 \_ definido nesse destino de renderização não são compatíveis entre si.






 

## <a name="possible-causes"></a>Possíveis causas

para operações baseadas em texto, como ID2D1RenderTarget::D rawglyphrun e ID2D1DeviceContext:: GetGlyphRunWorldBounds, Direct2D considera a opção de suavização de texto atual do contexto do dispositivo, bem como a opção de anti-aliasing DirectWrite. a semântica de anti-aliasing difere ligeiramente entre Direct2D e DirectWrite. Essa mensagem ocorre quando as duas opções de suavização especificadas não são aceitas.

## <a name="possible-fixes"></a>Possíveis correções

para garantir que o modo de anti-aliasing de texto Direct2D concorde com o modo de renderização de DirectWrite, verifique se eles atendem aos seguintes requisitos: • para \_ \_ modo antialias de texto D2D1 \_ \_ CLEARTYPE, o modo de renderização DirectWrite não pode ser contorno ou com alias. • para o modo de suavização de texto D2D1 em escala de cinza, o modo de renderização \_ \_ \_ \_ de DirectWrite pode não ter um alias. • para o \_ modo antialias de texto D2D1 com \_ \_ \_ alias, o modo de renderização DirectWrite pode não ser \[nova linha \] para alterar o modo de anti-aliasing de texto Direct2D atual, use ID2D1RenderTarget:: SetTextAntialiasMode.
