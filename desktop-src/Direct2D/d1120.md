# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120: Tentativa inválida de desenhar bitmap

Uma operação falhou porque o bitmap tem a opção D2D1 \_ BITMAP \_ OPTIONS CANNOT \_ \_ DRAW.





 

## <a name="possible-causes"></a>Possíveis causas

Essa mensagem ocorre quando um aplicativo tenta desenhar um bitmap sinalizado com OPÇÕES D2D1 \_ BITMAP \_ NÃO É \_ POSSÍVEL \_ DESENHAR. Isso inclui chamadas diretas para DrawBitmap, bem como o desenho de primitivos usando um pincel de bitmap.

## <a name="possible-fixes"></a>Possíveis correções

O sinalizador CANNOT DRAW é usado para anotação correta de recursos que podem ser usados apenas como destinos, como backbuffers de cadeia \_ de permuta. No entanto, também é permitido omitir o sinalizador CANNOT DRAW para alguns bitmaps de destino se eles também puderem \_ ser usados como uma origem. Se esse for o caso, basta omitir o sinalizador CANNOT \_ DRAW. Se o bitmap precisar ser anotado com NÃO É POSSÍVEL DESENHAR devido ao seu uso exclusivo como um destino ou para permitir a capacidade de bloqueio da CPU, considere usar uma textura de preparação com o mesmo tamanho que a fonte desejada, mas sem a designação \_ CANNOT \_ DRAW. Copie para a textura de preparação usando ID3D11DeviceContext::CopyResource e use a textura de preparação como uma origem para a operação de desenho. O uso de uma textura de preparação gera implicações de desempenho; eles poderão ser aceitáveis se o bitmap de origem não mudar com frequência.
