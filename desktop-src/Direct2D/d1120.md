# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120: tentativa inválida de desenhar bitmap

Uma operação falhou porque o bitmap tem as \_ Opções de bitmap d2d1 \_ \_ não podem \_ desenhar a opção.



|             |             |
|-------------|-------------|
| Nível de erro | Informações do |



 

## <a name="possible-causes"></a>Possíveis causas

Essa mensagem ocorre quando um aplicativo tenta desenhar um bitmap sinalizado com \_ Opções de bitmap d2d1 \_ \_ não pode \_ desenhar. Isso inclui chamadas diretas para DrawBitmap, bem como o desenho de primitivos usando um pincel de bitmap.

## <a name="possible-fixes"></a>Possíveis correções

O sinalizador não pode \_ desenhar é usado para anotar corretamente os recursos que são utilizáveis somente como destinos, como backbuffers da cadeia de permuta. No entanto, também é permitido omitir o sinalizador não é possível \_ desenhar para alguns bitmaps de destino se eles também podem ser usados como uma origem. Se esse for o caso, simplesmente omita o sinalizador não é possível \_ desenhar. Se o bitmap precisar ser anotado porque não pode ser \_ desenhado devido a seu uso exclusivo como um destino ou para permitir o bloqueio de CPU, considere usar uma textura de preparo com o mesmo tamanho que a origem desejada, mas sem a \_ designação não pode desenhar. Copie para a textura de preparo usando ID3D11DeviceContext:: CopyResource e use a textura de preparo como uma fonte para a operação de desenho. O uso de uma textura de preparo oferece aumento nas implicações de desempenho; Eles poderão ser aceitáveis se o bitmap de origem não for alterado com frequência.
