# <a name="d1194-flush-outstanding-dc"></a>D1194: liberar o controlador de domínio pendente

É inválido chamar Flush ou EndDraw entre um par de GetDC/ReleaseDC.






 

## <a name="possible-causes"></a>Possíveis causas

Ao usar um renderTarget interoperado GDI, um aplicativo pode controlar a propriedade do DC de destino usando IGdiInteropRenderTarget:: GetDC ou IGdiInteropRenderTarget:: ReleaseDC. No entanto, não é válido chamar ID2D1RenderTarget:: flush ou ID2D1RenderTarget:: EndDraw enquanto o aplicativo tem controle do controlador de domínio. Flush e EndDraw podem fazer com que o processamento ocorra no recurso de destino e Direct2D não deve tentar modificar o recurso ao mesmo tempo que o aplicativo.

## <a name="possible-fixes"></a>Possíveis correções

Para resolver essa mensagem, verifique se o aplicativo não chama ID2D1RenderTarget:: flush ou ID2D1RenderTarget:: EndDraw enquanto tem Propriedade do DC de um IGdiInteropRenderTarget. Se possível, organize o aplicativo para separar seu comportamento de renderização Direct2D de seu comportamento GDI para tornar a distinção de propriedade de DC mais clara.
