# <a name="d1141-print-requires-srgb-color-space"></a>D1141: Print requer espaço de cores sRGB

Direct2D Imprimir: não há suporte para o espaço de cores personalizado. Usando o sRGB em vez disso.






 

## <a name="possible-causes"></a>Possíveis causas

ao criar um Direct2D controle de impressão, um aplicativo usa \_ propriedades de controle de impressão D2D1 \_ \_ com um atributo de espaço de cor. Direct2D dá suporte ao [**\_ espaço de cores D2D1 \_ \_ personalizado**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_color_space) para algumas operações, como a criação de um contexto de cor usando dados ICC. no entanto, ao criar um controle de impressão, Direct2D dá suporte apenas a espaços de cores internos como sRGB ou scRGB.

## <a name="possible-fixes"></a>Possíveis correções

Para endereçar a mensagem, especifique um espaço de cores não personalizado, como sRGB ou scRGB, ao criar um controle de impressão. a mensagem não indica um erro, mas indica que Direct2D ignorará o especificador personalizado e criará o controle de impressão usando um espaço de cores sRGB.
