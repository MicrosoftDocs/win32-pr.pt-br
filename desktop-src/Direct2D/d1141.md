# <a name="d1141-print-requires-srgb-color-space"></a>D1141: Print requer espaço de cores sRGB

Direct2D imprimir: não há suporte para o espaço de cores personalizado. Usando o sRGB em vez disso.






 

## <a name="possible-causes"></a>Possíveis causas

Ao criar um controle de impressão Direct2D, um aplicativo usa \_ Propriedades de controle de impressão d2d1 \_ \_ com um atributo de espaço de cores. O Direct2D dá suporte ao [**\_ espaço de cores d2d1 \_ \_ personalizado**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_color_space) para algumas operações, como a criação de um contexto de cor usando dados ICC. No entanto, ao criar um controle de impressão, o Direct2D dá suporte apenas a espaços de cores internos como sRGB ou scRGB.

## <a name="possible-fixes"></a>Possíveis correções

Para endereçar a mensagem, especifique um espaço de cores não personalizado, como sRGB ou scRGB, ao criar um controle de impressão. A mensagem não indica um erro, mas indica que o Direct2D irá ignorar o especificador personalizado e criar o controle de impressão usando um espaço de cores sRGB.
