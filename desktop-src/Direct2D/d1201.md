# <a name="d1201-pushaxisalignedclip-with-antialiased-multisampled-target"></a>D1201: PushAxisAlignedClip com destino multiamostrado com suavização de alias

PushAxisAlignedClip foi usado com um destino de uma amostra. A chamada ainda será emitida, mas poderão ocorrer artefatos se os limites do retângulo do clipe não estiverem alinhados em pixel. Além disso, o resultado renderizado poderá não representar o padrão de multiamostração esperado.






 

## <a name="possible-causes"></a>Possíveis causas

O aplicativo interoperau uma textura multiamostrada do Direct3D como um destino Direct2D e chamou PushAxisAlignedClip no destino.

## <a name="possible-fixes"></a>Possíveis correções

Evite a interoperação de texturas com multiamostrar se a funcionalidade do clipe for importante. Usar camadas ou clipes com esses tipos de recursos pode incorrer em uma penalidade de desempenho ou resultados de renderização inesperados. Isso será particularmente verdadeiro se os limites do clipe não estiverem alinhados em pixel. Se possível, use um meio alternativo de mascarar o conteúdo em vez de um clipe, como uma tesoura de Direct3D. Ou então, use um intermediário em vez de um recurso de uma amostra. Se o recurso de multiamostrar for destinada ao conteúdo 2D de suavização de alias, considere se a suavização interna por primitivo Direct2D's é suficiente.
