# <a name="d1125-dxgi-format-mismatch"></a>D1125: formato de DXGI incompatível

O formato de pixel passado para essa API não é compatível com o formato de pixel da \[ superfície \] .

## <a name="placeholders"></a>Espaços reservados

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*tona*
</dt> <dd>

O destino de renderização ou o bitmap especificado.

</dd> </dl>

## <a name="possible-causes"></a>Possíveis causas

Operações como ID2D1RenderTarget:: CreateBitmapFromWicBitmap ou ID2D1RenderTarget:: CreateDxgiSurfaceRendertarget exigem uma superfície e um formato de superfície especificado. Direct2D verifica se o tipo de superfície especificado é compatível com a superfície. Se não for compatível, o Direct2D produzirá essa mensagem.

## <a name="possible-fixes"></a>Possíveis correções

Verifique se o formato DXGI especificado corresponde a, \_ ou é compatível com, o tipo da superfície subjacente. Para recursos de superfície do Direct3D criados pelo aplicativo, use o mesmo \_ formato dxgi usado para criar o recurso. Consulte [formatos de pixel com suporte e modos alfa](supported-pixel-formats-and-alpha-modes.md) para obter diretrizes adicionais.
