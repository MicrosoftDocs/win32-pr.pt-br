# <a name="d1125-dxgi-format-mismatch"></a>D1125: Incompatibilidade de formato DXGI

O formato de pixel passado para essa API não é compatível com o formato de pixel da \[ superfície \] .

## <a name="placeholders"></a>Espaços reservados

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*Superfície*
</dt> <dd>

O destino de renderização ou bitmap especificado.

</dd> </dl>

## <a name="possible-causes"></a>Possíveis causas

Operações como ID2D1RenderTarget::CreateBitmapFromWicBitmap ou ID2D1RenderTarget::CreateDxgiSurfaceRendertarget exigem uma superfície e um formato de superfície especificado. Direct2D verifica se o tipo de superfície especificado é compatível com a superfície. Se não for compatível, Direct2D a saída dessa mensagem.

## <a name="possible-fixes"></a>Possíveis correções

Verifique se o FORMATO DXGI especificado corresponde ou é compatível com \_ o tipo da superfície subjacente. Para recursos de superfície do Direct3D criados pelo aplicativo, use o mesmo FORMATO DXGI \_ usado para criar o recurso. Confira [Formatos de pixel com suporte e modos alfa para](supported-pixel-formats-and-alpha-modes.md) obter diretrizes adicionais.
