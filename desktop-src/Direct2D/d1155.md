# <a name="d1155-fillopacitymask-requires-aliased-rendering"></a>D1155: FillOpacityMask requer renderização com alias

FillOpacityMask requer que o modo AntiAlias seja definido como \_ alias do modo ANTIALIAS d2d1 \_ \_ .



|             |             |
|-------------|-------------|
| Nível de erro | Informações do |



 

## <a name="possible-causes"></a>Possíveis causas

Quando um aplicativo chama ID2D1RenderTarget:: FillOpacityMask, o modo AntiAlias do renderTarget envolvido deve ser o \_ modo de ANTIALIAS d2d1 com \_ \_ alias. O comportamento padrão de um destino de renderização é usar a suavização, portanto, é necessário chamar ID2D1RenderTarget:: SetAntialiasMode antes de preencher uma máscara de opacidade.

## <a name="possible-fixes"></a>Possíveis correções

Use ID2D1RenderTarget:: SetAntialiasMode para garantir que um renderTarget use a renderização com alias ao preencher uma máscara de opacidade. Se as operações subsequentes exigirem a suavização, defina o modo de antialias do destino de volta para o \_ modo de ANTIALIAS d2d1 \_ \_ por \_ primitiva. Verifique se as chamadas para ID2D1RenderTarget:: SetAntialiasMode e ID2D1RenderTarget:: FillOpacityMask são emitidas no mesmo destino.
