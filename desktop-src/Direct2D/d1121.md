# <a name="d1121-wrong-resource-domain"></a>D1121: domínio de recurso incorreto

Uma operação falhou porque um recurso dependente de dispositivo está associado ao ID2D1Device errado (domínio de recurso).






 

## <a name="possible-causes"></a>Possíveis causas

Essa mensagem ocorre quando um aplicativo tenta uma operação de desenho que combina recursos de domínios de recurso diferentes. Contextos, pincéis e bitmaps de dispositivo, por exemplo, são específicos do dispositivo. Ao executar uma operação como ID2D1DeviceContext::D rawBitmap, o bitmap de argumento deve pertencer ao mesmo domínio de recurso que o contexto do dispositivo. Neste exemplo, uma maneira de garantir isso é criar o bitmap usando a função de membro CreateBitmap do contexto do dispositivo, em vez de qualquer outro contexto de dispositivo.

## <a name="possible-fixes"></a>Possíveis correções

Quando grupos de recursos devem ser usados juntos, verifique se eles são criados a partir do mesmo ID2D1Device. Recursos não-alvo criados diretamente de um ID2D1Factory, como blocos de estado e geometrias, e são considerados independentes de dispositivo e não têm essa restrição.
