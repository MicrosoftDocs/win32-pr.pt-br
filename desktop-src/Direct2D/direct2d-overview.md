## <a name="why-direct2d"></a><span data-ttu-id="d4c47-122">Por que Direct2D?</span><span class="sxs-lookup"><span data-stu-id="d4c47-122">Why Direct2D?</span></span>

<span data-ttu-id="d4c47-135">Por fim, a necessidade de uma API gráfica que possa ser usada em um serviço tornou-se um requisito emergente para os desenvolvedores que trabalham em cenários empresariais e de desenvolvimento na Web.</span><span class="sxs-lookup"><span data-stu-id="d4c47-135">Finally, the need for a graphics API that can be used in a service has become an emerging requirement for developers working in enterprise and Web-development scenarios.</span></span> <span data-ttu-id="d4c47-136">As APIs de renderização existentes se concentram na renderização do lado do cliente em uma única sessão de usuário.</span><span class="sxs-lookup"><span data-stu-id="d4c47-136">Existing rendering APIs focus on client-side rendering in a single user session.</span></span> <span data-ttu-id="d4c47-137">Assim, eles podem cair em áreas de robustez e escalabilidade quando usados em um contexto de serviço.</span><span class="sxs-lookup"><span data-stu-id="d4c47-137">As such, they can fall short in areas of robustness and scalability when used in a service context.</span></span> <span data-ttu-id="d4c47-138">Uma nova API é necessária para resolver isso.</span><span class="sxs-lookup"><span data-stu-id="d4c47-138">A new API is required to address this.</span></span>
