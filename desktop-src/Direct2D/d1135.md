# <a name="d1135-bitmap-option-must-be-subset"></a>D1135: a opção de bitmap deve ser subconjunto

O valor das opções de bitmap \[  \] deve ser um subconjunto dos sinalizadores associados à superfície DXGI.






 

## <a name="placeholders"></a>Espaços reservados

<dl> <dt>

<span id="value"></span><span id="VALUE"></span>*valor*
</dt> <dd>

O valor UINT da enumeração opções de bitmap.

</dd> </dl>

## <a name="possible-causes"></a>Possíveis causas

ao interoperar texturas de Direct3D com Direct2D como ID2D1Bitmaps, as opções especificadas no \_ campo de opções de BITMAP D2D1 \_ devem corresponder aos atributos do Direct3D da textura. Especialmente:

-   A opção de [**leitura de CPU de \_ Opções de bitmap \_ \_ \_ d2d1**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) requer a opção de leitura de acesso de CPU Direct3D \_ \_ .
-   A [**opção \_ \_ \_ destino de opções de bitmap d2d1**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) requer a opção de destino de processamento de associação \_ \_ .
-   A [**opção \_ Opções de bitmap d2d1 \_ \_ None**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) não é compatível com a opção de preparo de uso do Direct3D \_ com acesso de CPU \_ \_ somente gravação.

## <a name="possible-fixes"></a>Possíveis correções

Para resolver esse problema, verifique se a criação de [**ID2D1Bitmap**](/windows/win32/api/d2d1/nn-d2d1-id2d1bitmap) usa especificadores de [**\_ \_ Opções de bitmap d2d1**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) que concordam com as propriedades da textura do Direct3D subjacente. Para usar **\_ \_ Opções de bitmap d2d1**, isso pode envolver o uso de [**ID2D1DeviceContext:: CreateBitmap**](id2d1devicecontext-createbitmap-overload.md) em vez de [**ID2D1RenderTarget:: CreateBitmap**](id2d1rendertarget-createbitmap.md), porque ele é membro de [**d2d1 \_ bitmap \_ PROPERTIES1**](/windows/desktop/api/D2D1_1/ns-d2d1_1-d2d1_bitmap_properties1).
