# <a name="the-evolution-of-file-systems"></a><span data-ttu-id="7b79a-103">A evolução dos sistemas de arquivos</span><span class="sxs-lookup"><span data-stu-id="7b79a-103">The Evolution of File Systems</span></span>

<span data-ttu-id="7b79a-104">Antes que os computadores tenham sido desenvolvidos para funcionar em sistemas operacionais de disco, cada computador foi criado para executar um único aplicativo proprietário, que tinha controle completo e exclusivo de todo o computador.</span><span class="sxs-lookup"><span data-stu-id="7b79a-104">Before computers were developed to function on disk operating systems, each computer was built to run a single, proprietary application, which had complete and exclusive control of the entire machine.</span></span> <span data-ttu-id="7b79a-105">O aplicativo gravaria seus dados persistentes diretamente em um disco, ou tambor, enviando comandos diretamente para o controlador de disco.</span><span class="sxs-lookup"><span data-stu-id="7b79a-105">The application would write its persistent data directly to a disk, or drum, by sending commands directly to the disk controller.</span></span> <span data-ttu-id="7b79a-106">O aplicativo foi responsável por gerenciar os locais absolutos dos dados no disco, garantindo que ele não estava substituindo os dados já existentes.</span><span class="sxs-lookup"><span data-stu-id="7b79a-106">The application was responsible for managing the absolute locations of data on the disk, making sure that it was not overwriting already-existing data.</span></span> <span data-ttu-id="7b79a-107">Como apenas um aplicativo estava em execução no computador a qualquer momento, essa tarefa não era muito difícil.</span><span class="sxs-lookup"><span data-stu-id="7b79a-107">Since only one application was running on the computer at any time, this task was not too difficult.</span></span>

<span data-ttu-id="7b79a-108">O advento dos sistemas de computador que podiam executar mais de um aplicativo exigia um mecanismo para garantir que os aplicativos não gravassem os dados uns dos outros.</span><span class="sxs-lookup"><span data-stu-id="7b79a-108">The advent of computer systems that could run more than one application required a mechanism to ensure that applications did not write over each other's data.</span></span> <span data-ttu-id="7b79a-109">Os desenvolvedores de aplicativos resolveram esse problema adotando um único padrão para diferenciar os setores de disco em uso daqueles que estavam livres, marcando-os adequadamente.</span><span class="sxs-lookup"><span data-stu-id="7b79a-109">Application developers addressed this problem by adopting a single standard for distinguishing disk sectors in use from those that were free by marking them accordingly.</span></span> <span data-ttu-id="7b79a-110">No tempo, esses padrões são Unidos para se tornar um sistema operacional de disco, que forneceu vários serviços para os aplicativos, incluindo um sistema de arquivos para gerenciar o armazenamento persistente.</span><span class="sxs-lookup"><span data-stu-id="7b79a-110">In time, these standards coalesced to become a disk operating system, which provided various services to the applications, including a file system for managing persistent storage.</span></span> <span data-ttu-id="7b79a-111">Com o advento de um sistema de arquivos, os aplicativos não tinham mais que lidar diretamente com o meio de armazenamento físico.</span><span class="sxs-lookup"><span data-stu-id="7b79a-111">With the advent of a file system, applications no longer had to deal directly with the physical storage medium.</span></span> <span data-ttu-id="7b79a-112">Em vez disso, eles simplesmente disseram ao sistema de arquivos para gravar blocos de dados no disco e deixar que o sistema de arquivos se preocupe com a maneira de fazer isso.</span><span class="sxs-lookup"><span data-stu-id="7b79a-112">Instead, they simply told the file system to write blocks of data to the disk and let the file system worry about how to do it.</span></span> <span data-ttu-id="7b79a-113">Além disso, o sistema de arquivos permitia que os aplicativos criassem hierarquias de dados por meio de uma abstração conhecida como um diretório.</span><span class="sxs-lookup"><span data-stu-id="7b79a-113">In addition, the file system allowed applications to create data hierarchies through an abstraction known as a directory.</span></span> <span data-ttu-id="7b79a-114">Um diretório pode conter não apenas arquivos, mas outros diretórios que, por sua vez, podem conter seus próprios arquivos e diretórios, e assim por diante.</span><span class="sxs-lookup"><span data-stu-id="7b79a-114">A directory could contain not only files but other directories, which in turn could contain their own files and directories, and so on.</span></span>

<span data-ttu-id="7b79a-115">O sistema de arquivos forneceu um único nível de indireção entre aplicativos e o disco, e o resultado era que cada aplicativo viu um arquivo como um único fluxo contíguo de bytes no disco, embora o sistema de arquivos estivesse realmente armazenando o arquivo em setores descontínuos.</span><span class="sxs-lookup"><span data-stu-id="7b79a-115">The file system provided a single level of indirection between applications and the disk, and the result was that every application saw a file as a single contiguous stream of bytes on the disk even though the file system was actually storing the file in discontiguous sectors.</span></span> <span data-ttu-id="7b79a-116">O indireção liberou os aplicativos de ter que controlar a posição absoluta dos dados em um dispositivo de armazenamento.</span><span class="sxs-lookup"><span data-stu-id="7b79a-116">The indirection freed the applications from having to track the absolute position of data on a storage device.</span></span>

<span data-ttu-id="7b79a-117">Atualmente, praticamente todas as APIs de sistema para entrada e saída de arquivo fornecem aplicativos para gravar informações em um arquivo simples.</span><span class="sxs-lookup"><span data-stu-id="7b79a-117">Today, virtually all system APIs for file input and output provide applications for writing information into a flat file.</span></span> <span data-ttu-id="7b79a-118">Os aplicativos veem esse arquivo como um único fluxo de bytes que pode crescer tão grande quanto necessário até que o disco esteja cheio.</span><span class="sxs-lookup"><span data-stu-id="7b79a-118">Applications see this file as a single stream of bytes that can grow as large as necessary until the disk is full.</span></span> <span data-ttu-id="7b79a-119">Por muito tempo, essas APIs têm sido suficientes para que os aplicativos armazenem suas informações persistentes.</span><span class="sxs-lookup"><span data-stu-id="7b79a-119">For a long time these APIs have been sufficient for applications to store their persistent information.</span></span> <span data-ttu-id="7b79a-120">Os aplicativos fizeram inovações significativas em como lidam com um único fluxo de informações para fornecer recursos como gravações incrementais "rápidas".</span><span class="sxs-lookup"><span data-stu-id="7b79a-120">Applications have made significant innovations in how they deal with a single stream of information to provide features like incremental "fast" saves.</span></span>

<span data-ttu-id="7b79a-121">No entanto, em um mundo de objetos de componente, o armazenamento de dados em um único arquivo simples não é mais eficiente.</span><span class="sxs-lookup"><span data-stu-id="7b79a-121">However, in a world of component objects, storing data in a single flat file is no longer efficient.</span></span> <span data-ttu-id="7b79a-122">Assim como os sistemas de arquivos surgiu da necessidade de vários aplicativos compartilharem o mesmo meio de armazenamento, por isso, agora, os objetos de componente exigem um sistema que permite que eles compartilhem armazenamento dentro da estrutura conceitual de um único arquivo.</span><span class="sxs-lookup"><span data-stu-id="7b79a-122">Just as file systems arose out of the need for multiple applications to share the same storage medium, so, now, do component objects require a system that allows them to share storage within the conceptual framework of a single file.</span></span> <span data-ttu-id="7b79a-123">Embora seja possível armazenar os objetos separados usando o armazenamento de arquivo simples convencional, se um dos objetos aumentar de tamanho, ou se você simplesmente adicionar outro objeto, será necessário carregar o arquivo inteiro na memória, inserir o novo objeto e, em seguida, salvar o arquivo inteiro.</span><span class="sxs-lookup"><span data-stu-id="7b79a-123">Even though it is possible to store the separate objects using conventional flat-file storage, if one of the objects increases in size, or if you simply add another object, it becomes necessary to load the entire file into memory, insert the new object, and then save the whole file.</span></span> <span data-ttu-id="7b79a-124">Esse processo pode ser extremamente demorado.</span><span class="sxs-lookup"><span data-stu-id="7b79a-124">This process can be extremely time-consuming.</span></span>

<span data-ttu-id="7b79a-125">A solução fornecida pelo COM é implementar um segundo nível de indireção: um sistema de arquivos em um arquivo.</span><span class="sxs-lookup"><span data-stu-id="7b79a-125">The solution provided by COM is to implement a second level of indirection: a file system within a file.</span></span> <span data-ttu-id="7b79a-126">O armazenamento de arquivos simples requer que uma sequência contígua grande de bytes no disco seja manipulada por meio de um único identificador de arquivo com um único ponteiro de busca.</span><span class="sxs-lookup"><span data-stu-id="7b79a-126">Flat-file storage requires that a large contiguous sequence of bytes on the disk be manipulated through a single file handle with a single seek pointer.</span></span> <span data-ttu-id="7b79a-127">Por outro lado, o armazenamento estruturado COM define como tratar uma única entidade do sistema de arquivos como uma coleção estruturada de dois tipos de objetos — armazenamentos e fluxos — que atuam como diretórios e arquivos.</span><span class="sxs-lookup"><span data-stu-id="7b79a-127">By contrast, COM structured storage defines how to treat a single file system entity as a structured collection of two types of objects — storages and streams — that act like directories and files.</span></span>
