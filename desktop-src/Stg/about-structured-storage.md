# <a name="about-structured-storage"></a>Sobre o armazenamento estruturado

Os sistemas de arquivos tradicionais encontram desafios quando tentam armazenar com eficiência vários tipos de objetos em um documento. O COM fornece uma solução: um sistema de arquivos em um arquivo. O armazenamento estruturado COM define como tratar uma única entidade de arquivo como uma coleção estruturada de dois tipos de objetos — armazenamentos e fluxos — que executam diretórios e arquivos. Esse esquema é chamado de *armazenamento estruturado*. A finalidade do armazenamento estruturado é reduzir as penalidades de desempenho e a sobrecarga associada ao armazenamento de objetos separados em um arquivo simples.

Esta seção contém uma visão geral dos benefícios e conceitos básicos do armazenamento estruturado, conjuntos de propriedades e armazenamento estruturado assíncrono, bem como uma visão histórica dos sistemas de arquivos:

-   Implementação de arquivo composto
-   Implementação do sistema de arquivos NTFS
-   Implementação autônoma

Esta seção inclui links para [a evolução de sistemas de arquivos](the-evolution-of-file-systems.md), [armazenamentos e fluxos](storages-and-streams.md), [arquivos compostos](compound-files.md), [conceitos básicos de armazenamento estruturado](structured-storage-fundamentals.md)e [exemplos de armazenamento estruturado](using-structured-storage.md).
