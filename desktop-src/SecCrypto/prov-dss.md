# <a name="prov_dss"></a>PROV \_ DSS

O tipo de provedor \_ PROV DSS, como PROV RSA SIG, dá suporte apenas \_ a \_ [*hashes*](../secgloss/h-gly.md) e [*assinaturas digitais*](../secgloss/d-gly.md). O algoritmo de assinatura especificado pelo tipo de provedor \_ PROV DSS é o DSA (Algoritmo de Assinatura [*Digital).*](../secgloss/d-gly.md)

## <a name="algorithms-supported"></a>Algoritmos com suporte

Para ver descrições de cada um desses algoritmos, confira o glossário.



| Finalidade      | Algoritmos compatíveis                                                                                                              |
|--------------|-----------------------------------------------------------------------------------------------------------------------------------|
| Chave Exchange | Nenhum                                                                                                                              |
| Assinatura    | [*Dss*](../secgloss/d-gly.md)                                                                       |
| Criptografia   | Nenhum                                                                                                                              |
| Hash      | [*SHA MD5*](../secgloss/m-gly.md)[](../secgloss/s-gly.md)<br/> |



 

## <a name="related-documentation"></a>Documentação relacionada

A DSA foi proposta pelo NIST (National [*Institute of Standards and Technology).*](../secgloss/n-gly.md) Uma descrição do algoritmo pode ser encontrada na seguinte referência governamental: "Proposta padrão FIPS DSS (Digital Signature Standard)," Federal Register, v. 57, não. 21 de janeiro de 1992, pp. 3747-3749.
