# <a name="prov_dss"></a>PROV \_ DSS

O tipo de provedor do PROV \_ DSS, como o Prov \_ RSA \_ SIG, dá suporte apenas a [*hashes*](../secgloss/h-gly.md) e [*assinaturas digitais*](../secgloss/d-gly.md). O algoritmo de assinatura especificado pelo tipo de provedor do PROV \_ DSS é o DSA ( [*algoritmo de assinatura digital*](../secgloss/d-gly.md) ).

## <a name="algorithms-supported"></a>Algoritmos com suporte

Para obter descrições de cada um desses algoritmos, consulte o Glossário.



| Finalidade      | Algoritmos compatíveis                                                                                                              |
|--------------|-----------------------------------------------------------------------------------------------------------------------------------|
| Troca de chaves | Nenhum                                                                                                                              |
| Assinatura    | [*DSS*](../secgloss/d-gly.md)                                                                       |
| Criptografia   | Nenhum                                                                                                                              |
| Hash      | [](../secgloss/m-gly.md)[*Sha* MD5](../secgloss/s-gly.md)<br/> |



 

## <a name="related-documentation"></a>Documentação relacionada

O DSA foi proposto pelo [*National Institute of Standards and Technology*](../secgloss/n-gly.md) (NIST). Uma descrição do algoritmo pode ser encontrada na seguinte referência do governo: "proposta padrão FIPS para DSS (padrão de assinatura digital)," registro Federal, v. 57, não. 21, 31 de janeiro de 1992, pp. 3747-3749.
