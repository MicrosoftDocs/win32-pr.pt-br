# <a name="standards"></a>Padrões

Windows Internet Explorer 8 contém um novo mecanismo de renderização e layout que implementa totalmente o CSS 2.1 e inclui conformidade adicional do HTML 4.0 (ao contrário das versões anteriores do Windows Internet Explorer). O suporte mais amplo dos padrões da Web no Internet Explorer 8 pode fazer com que alguns aplicativos Web operem icorretamente porque eles não são escritos para o Microsoft Internet Explorer 6 ou de maneira não compatível com padrões. Os aplicativos podem falhar de maneiras diferentes, incluindo imagens ou texto exibidos em diferentes locais ou aplicativos inteiros que são renderizáveis de uma maneira que torna o aplicativo inutilizável.

Internet Explorer 8 também inclui um mecanismo JavaScript aprimorado. Esse novo mecanismo de script melhora o desempenho, dá suporte a novos métodos na verdade, fornece maior segurança e altera algumas funções para operar de maneira mais compatível com padrões. O desempenho aprimorado não introduz problemas de compatibilidade do aplicativo, mas outras alterações podem fazer com que alguns aplicativos Web param de funcionar conforme o esperado. Para obter mais informações sobre problemas de compatibilidade relacionados a alterações javaScript, consulte Compatibilidade de [site e Internet Explorer 8](/archive/blogs/ie/site-compatibility-and-ie8).
