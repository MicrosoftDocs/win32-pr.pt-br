# <a name="standards"></a>Padrões

O Windows Internet Explorer 8 contém um novo mecanismo de renderização e layout que implementa totalmente o CSS 2,1 e inclui a conformidade HTML 4,0 adicional (ao contrário das versões anteriores do Windows Internet Explorer). O suporte mais amplo dos padrões da Web no Internet Explorer 8 pode fazer com que alguns aplicativos Web operem incorretamente porque não são escritos para o Microsoft Internet Explorer 6 ou em uma maneira não compatível com os padrões. Os aplicativos podem falhar de maneiras diferentes, incluindo imagens ou texto que são exibidos em diferentes locais ou aplicativos inteiros que são renderizados de forma a tornar o aplicativo inutilizável.

O Internet Explorer 8 também inclui um mecanismo de JavaScript aprimorado. Esse novo mecanismo de script melhora o desempenho, dá suporte a novos métodos nativamente, fornece maior segurança e altera algumas funções para operar de forma compatível com mais padrões. O desempenho aprimorado não introduz problemas de compatibilidade de aplicativos, mas outras alterações podem fazer com que alguns aplicativos da Web parem de funcionar conforme o esperado. Para obter mais informações sobre problemas de compatibilidade relacionados a alterações de JavaScript, consulte [compatibilidade de site e Internet Explorer 8](/archive/blogs/ie/site-compatibility-and-ie8).
