# <a name="windows-7-client-software-logo-program"></a>Programa de logotipo de software cliente do Windows 7

## <a name="affected-platforms"></a>Plataformas afetadas

 **Clientes** -Windows 7  


## <a name="description-client-application-certification"></a>Descrição: certificação de aplicativo cliente

No período de tempo do Windows 7, queremos continuar criando o sucesso do Windows e expandindo o programa de logotipo para que ele seja mais relevante para os clientes. Com base nos comentários de parceiros e clientes, o programa de logotipo do Windows 7 se concentrará em garantir que dispositivos, sistemas e aplicativos sejam compatíveis, confiáveis e possam ser executados para atender aos padrões do Windows. O programa proposto para o Windows 7 terá apenas um logotipo para sistemas, dispositivos e aplicativos. Este tópico se concentra especificamente na compatibilidade do software cliente.

Melhoramos o processo para qualificar aplicativos de software para o logotipo "compatível com o Windows 7". A Microsoft não precisará mais que os desenvolvedores tenham seus aplicativos testados para certificação por autoridades de teste autorizadas. Em vez disso, a Microsoft está desenvolvendo um kit de ferramentas de logotipo que você pode usar para garantir que seu aplicativo seja compatível com o Windows 7 e atenda aos requisitos do programa de logotipo de software cliente do Windows 7. Reduzimos significativamente o número de requisitos. Semelhante ao programa de hardware, criamos algumas políticas; Embora não esteja vinculado a casos de teste específicos, iremos impor essas políticas. Os parceiros devem atender a todos os requisitos e observar todas as políticas para que seus aplicativos de software se qualifiquem para o logotipo "compatível com o Windows 7".

## <a name="process"></a>Processar

Quando os aplicativos de software estiverem prontos para serem testados, vá para a página do programa de logotipo de software do Microsoft Connect Windows 7 e siga as instruções:

1.  Comece a examinar antecipadamente os requisitos e identifique as alterações de código necessárias para atendê-los.
2.  Use a ferramenta de autoteste Alpha para verificação do download e instale a versão mais recente do kit de ferramentas. Observe a arquitetura de 64 bits e os requisitos do sistema operacional no arquivo readme.txt. Execute o aplicativo por meio do kit de ferramentas, conforme explicado na visão geral para verificar a conformidade do aplicativo com os requisitos.
3.  Faça quaisquer alterações de código (se necessário) e teste novamente se qualquer alteração no aplicativo for necessária para atender aos requisitos, faça as alterações necessárias e teste novamente com o kit de ferramentas para verificar a conformidade.
4.  Aceitar notificações por email a Microsoft começará a aceitar envios de logotipo com o lançamento do kit de ferramentas beta, devido ao fim de maio. Recomendamos que você opte por receber notificações por email quando o kit de ferramentas for lançado e quando o programa for aberto para envios.

    As etapas a seguir podem ser executadas somente após o lançamento do kit de ferramentas beta:

5.  Baixe o kit de ferramentas beta, teste e envie o logotipo assim que o kit de ferramentas beta estiver disponível, baixe, instale e teste seu aplicativo. Envie os resultados de aprovação à Microsoft.
6.  Assinar o contrato de licença de logotipo de software após a aprovação dos resultados, assine o [contrato de licença](/windows-hardware/drivers/dashboard/winqual-submission-tool--winqualexe-). Em seguida, você pode baixar e começar a usar a arte do logotipo e estará qualificado para receber todos os benefícios do logotipo disponíveis.
