# <a name="windows-7-and-windows-server-2008-r2-application-quality-cookbook"></a>Guia de qualidade de aplicativos para Windows 7 e Windows Server 2008 R2

O guia de *qualidade de aplicativos do Windows 7 e do Windows Server 2008 R2* familiariza os desenvolvedores de aplicativos com a forma de verificar a compatibilidade de seus aplicativos com o novo sistema operacional e fornece uma visão geral dos problemas conhecidos de compatibilidade de aplicativos no Windows 7 e no windows Server 2008 R2. Ele também destaca as diferenças de desempenho, confiabilidade e usabilidade e fornece links para White papers detalhados e outras diretrizes para desenvolvedores.

## <a name="overview"></a>Visão geral

O Windows 7 e o Windows Server 2008 R2 introduzem a mais recente tecnologia de sistema operacional e a plataforma de desenvolvimento de software para uso por desenvolvedores de aplicativos e empresas em todo o mundo. Como parte do aprimoramento da segurança, da confiabilidade, do desempenho e da experiência do usuário do Windows, muitos novos recursos foram introduzidos, os recursos existentes foram aprimorados e alguns recursos foram removidos.

Embora o Windows 7 e o Windows Server 2008 R2 sejam altamente compatíveis com a maioria dos seus respectivos aplicativos escritos para o Windows XP, o Windows Server 2003, o Windows Vista, o Windows Server 2008, o Windows Server 2008 R2 e seus Service Packs, algumas quebras de compatibilidade são inevitáveis devido a inovações, fortalecimento da segurança e maior confiabilidade. Em geral, a compatibilidade do Windows 7 e do Windows Server 2008 R2 com aplicativos existentes é alta.

Este documento baseia-se nos conceitos incorporados no [Windows Vista e no Windows Server 2008 Application Compatibility Cookbook](/previous-versions/bb757005(v=msdn.10)). (Esse recurso pode não estar disponível em alguns idiomas e países/regiões.) Assim, este documento fornece os meios para se familiarizar com a forma de verificar a compatibilidade de seus aplicativos com o novo sistema operacional e fornece uma visão geral dos problemas de incompatibilidade de aplicativos conhecidos no Windows 7 e no Windows Server 2008 R2. Mas, mais do que isso, ele também destaca as diferenças de desempenho, confiabilidade e usabilidade e fornece links para White papers detalhados e outras diretrizes para desenvolvedores.

Além disso, a Microsoft está investindo em vários recursos e ferramentas novos e aprimorados para permitir que você crie aplicativos de maior qualidade e solucione problemas quando os aplicativos não funcionarem corretamente no Windows 7 e no Windows Server 2008 R2. Entre essas ferramentas estão as instruções sobre como qualificar seus aplicativos de cliente e servidor para participar do programa de logotipo do Windows.

Este *manual* contém mais de três dúzias de tópicos divididos em três seções principais:

-   [Compatibilidade](compatibility.md) (geral e específica do servidor)
-   [Novos recursos e melhorias](new-features-and-enhancements.md)
-   [Ferramentas, práticas recomendadas e diretrizes](tools--resources--and-guidance.md)

Convidamos você a conferir esses tópicos para aprender a otimizar seu aplicativo para aproveitar o que o Windows 7 tem a oferecer.

Levamos suas sugestões para aprimoramentos neste documento.
