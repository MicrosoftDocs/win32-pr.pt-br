# <a name="windows-7-and-windows-server-2008-r2-application-quality-cookbook"></a>Windows 7 e Windows Server 2008 R2 Application Quality Cookbook

O Guia de Qualidade de Aplicativo do Windows 7 e *Windows Server 2008 R2* familiariza os desenvolvedores de aplicativos com como verificar a compatibilidade de seus aplicativos com o novo sistema operacional e fornece uma visão geral dos poucos problemas conhecidos de compatibilidade de aplicativos no Windows 7 e no Windows Server 2008 R2. Ele também aponta diferenças de desempenho, confiabilidade e usabilidade e fornece links para white papers detalhados e outras diretrizes para desenvolvedores.

## <a name="overview"></a>Visão geral

Windows 7 e Windows Server 2008 R2 apresentam a plataforma de desenvolvimento de software e tecnologia de sistema operacional mais recente para uso por desenvolvedores de aplicativos e empresas em todo o mundo. Como parte do aprimoramento da segurança, confiabilidade, desempenho e experiência do usuário do Windows, muitos novos recursos foram introduzidos, recursos existentes foram aprimorados e alguns recursos foram removidos.

Embora o Windows 7 e o Windows Server 2008 R2 sejam altamente compatíveis com a maioria de seus respectivos aplicativos escritos para o Windows XP, o Windows Server 2003, o Windows Vista, o Windows Server 2008, o Windows Server 2008 R2 e seus service packs, algumas quebras de compatibilidade são inevitáveis devido a inovações, à segurança e ao aumento da confiabilidade. Em geral, a compatibilidade do Windows 7 e Windows Server 2008 R2 com aplicativos existentes é alta.

Este documento se baseia nos conceitos embodiados no guia de compatibilidade de aplicativos Windows Vista e [Windows Server 2008.](/previous-versions/bb757005(v=msdn.10)) (Esse recurso pode não estar disponível em alguns idiomas e países/regiões.) Como ele, este documento fornece os meios para se familiarizar com como verificar a compatibilidade de seus aplicativos com o novo sistema operacional e fornece uma visão geral dos poucos problemas de incompatibilidade de aplicativo conhecidos no Windows 7 e no Windows Server 2008 R2. Porém, mais do que isso, ele também aponta diferenças no desempenho, na confiabilidade e na usabilidade e fornece links para white papers detalhados e outras diretrizes para desenvolvedores.

Além disso, a Microsoft está investindo em vários recursos e ferramentas novos e aprimorados para permitir que você crie aplicativos de alta qualidade e solucione problemas quando os aplicativos não funcionam corretamente no Windows 7 e no Windows Server 2008 R2. Entre essas ferramentas estão instruções sobre como qualificar seus aplicativos cliente e servidor para participar do programa Windows Logo.

Este *Guia contém* mais de três dezenas de tópicos divididos em três seções principais:

-   [Compatibilidade](compatibility.md) (geral e específica do servidor)
-   [Novos recursos e melhorias](new-features-and-enhancements.md)
-   [Ferramentas, práticas recomendadas e diretrizes](tools--resources--and-guidance.md)

Convidamos você a conferir esses tópicos para saber como otimizar seu aplicativo para aproveitar o que o Windows 7 tem a oferecer.

Vamos levar suas sugestões para melhorias neste documento a sério.
