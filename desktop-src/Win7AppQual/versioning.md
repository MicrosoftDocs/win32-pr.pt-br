# <a name="versioning-windows-7-and-windows-server-2008-r2-application-quality-cookbook"></a>Controle de versão (Windows 7 e Windows Server 2008 R2 Application Quality Cookbook)

Um dos problemas mais comuns de compatibilidade de aplicativos para o Windows Internet Explorer são as cadeias de caracteres do agente do usuário (UA) e os vetores de versão. O Windows Internet Explorer 8 apresenta uma nova cadeia de caracteres de UA para ajudar os aplicativos a detectar qual versão os usuários estão executando. Além de simplesmente aumentar o valor de MSIE associado ao Internet Explorer na cadeia de caracteres UA, o Internet Explorer 8 adiciona um valor Trident/4.0 para ajudá-lo a detectar se o Internet Explorer 8 está sendo executado no modo de exibição de compatibilidade. Essas alterações, além de verificações de versão codificadas, podem apresentar problemas de compatibilidade entre navegadores.
