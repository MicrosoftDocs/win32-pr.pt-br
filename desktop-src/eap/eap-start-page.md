# <a name="extensible-authentication-protocol"></a>Protocolo EAP (Extensible Authentication Protocol)

## <a name="purpose"></a>Finalidade

O EAP (protocolo de autenticação extensível) é um padrão compatível com vários componentes do sistema. O EAP é crucial para proteger a segurança de LANs sem fio (802.1 X) e com fio, dial-up e redes virtuais privadas (VPNs).

## <a name="where-applicable"></a>Quando aplicável

O EAP melhora os protocolos de autenticação anteriores, como PAP (protocolo de autenticação de senha) e CHAP (Challenge Handshake Authentication Protocol).

Para o novo desenvolvimento de método EAP, consulte [host de protocolo de autenticação extensível](../eaphost/portal.md).

## <a name="developer-audience"></a>Público de desenvolvedores

A API do EAP foi projetada para ser usada por programadores C/C++. Os programadores devem estar familiarizados com os conceitos de rede.

## <a name="run-time-requirements"></a>Requisitos de tempo de execução

o EAP tem suporte em computadores cliente e servidor em execução no Windows 2000 e posterior. o EAP também tem suporte em computadores que executam o Windows Server 2000 e posterior se estiverem executando o IAS (serviço de autenticação da Internet). Para obter mais informações sobre sistemas operacionais com suporte, consulte a seção requisitos na documentação do.
