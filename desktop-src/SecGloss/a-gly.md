<dl> <dt>

<span id="_security_absolute_security_descriptor_gly"></span><span id="_SECURITY_ABSOLUTE_SECURITY_DESCRIPTOR_GLY"></span>**descritor de segurança absoluto**
</dt> <dd>

Uma estrutura de descritor de segurança que contém ponteiros para as informações de segurança associadas a um objeto.

Consulte também [*descritor de segurança*](s-gly.md) e [*descritor de segurança auto-relativo*](s-gly.md).

</dd> <dt>

<span id="_security_abstract_syntax_notation_one_gly"></span><span id="_SECURITY_ABSTRACT_SYNTAX_NOTATION_ONE_GLY"></span>**Notação de sintaxe abstrata One**
</dt> <dd>

(ASN. 1) um método usado para especificar objetos abstratos destinados à transmissão serial.

</dd> <dt>

<span id="_security_access_block_gly"></span><span id="_SECURITY_ACCESS_BLOCK_GLY"></span>**bloco de acesso**
</dt> <dd>

Um BLOB de chave que contém a chave da cifra simétrica usada para criptografar um arquivo ou uma mensagem. O bloco de acesso só pode ser aberto com uma chave privada.

</dd> <dt>

<span id="_security_access_control_entry_gly"></span><span id="_SECURITY_ACCESS_CONTROL_ENTRY_GLY"></span>**entrada de controle de acesso**
</dt> <dd>

PERFEITA Uma entrada em uma ACL (lista de controle de acesso). Uma ACE contém um conjunto de direitos de acesso e um SID (identificador de segurança) que identifica um grupo de confiança para o qual os direitos são permitidos, negados ou auditados.

Consulte também *lista de controle de acesso*, [*identificador de segurança*](s-gly.md)e [*confiança*](t-gly.md).

</dd> <dt>

<span id="_security_access_control_list_gly"></span><span id="_SECURITY_ACCESS_CONTROL_LIST_GLY"></span>**lista de controle de acesso**
</dt> <dd>

LCA Uma lista de proteções de segurança que se aplica a um objeto. (Um objeto pode ser um arquivo, processo, evento ou qualquer outra coisa que tenha um descritor de segurança.) Uma entrada em uma ACL (lista de controle de acesso) é uma ACE (entrada de controle de acesso). Há dois tipos de lista de controle de acesso, discricionárias e de sistema.

Consulte também *entrada de controle de acesso*, [*lista de controle de acesso discricional*](d-gly.md), [*descritor de segurança*](s-gly.md)e lista de controle de acesso do [*sistema*](s-gly.md).

</dd> <dt>

<span id="_security_access_mask_gly"></span><span id="_SECURITY_ACCESS_MASK_GLY"></span>**máscara de acesso**
</dt> <dd>

Um valor de 32 bits que especifica os direitos permitidos ou negados em uma ACE (entrada de controle de acesso). Uma máscara de acesso também é usada para solicitar direitos de acesso quando um objeto é aberto.

Consulte também *entrada de controle de acesso*.

</dd> <dt>

<span id="_security_access_token_gly"></span><span id="_SECURITY_ACCESS_TOKEN_GLY"></span>**token de acesso**
</dt> <dd>

Um token de acesso contém as informações de segurança para uma sessão de logon. O sistema cria um token de acesso quando um usuário faz logon e cada processo executado em nome do usuário tem uma cópia do token. O token identifica o usuário, os grupos do usuário e os privilégios do usuário. O sistema usa o token para controlar o acesso a objetos protegíveis e controlar a capacidade do usuário de executar várias operações relacionadas ao sistema no computador local. Há dois tipos de token de acesso, primário e representação.

Consulte também [*token de representação*](i-gly.md), [*token primário*](p-gly.md), [*privilégio*](p-gly.md), [*processo*](p-gly.md)e [*identificador de segurança*](s-gly.md).

</dd> <dt>

<span id="_security_ace_gly"></span><span id="_SECURITY_ACE_GLY"></span>**PERFEITA**
</dt> <dd>

Consulte *entrada de controle de acesso*.

</dd> <dt>

<span id="_security_acl_gly"></span><span id="_SECURITY_ACL_GLY"></span>**LCA**

Consulte *lista de controle de acesso*.

</dd> <dt>

<span id="_security_aes_gly"></span><span id="_SECURITY_AES_GLY"></span>**criptografia AES**
</dt> <dd>

AES Um algoritmo criptográfico especificado pelo National Institute of Standards and Technology (NIST) para proteger informações confidenciais.

</dd> <dt>

<span id="_security_alg_class_data_encrypt_gly"></span><span id="_SECURITY_ALG_CLASS_DATA_ENCRYPT_GLY"></span>**\_criptografia de \_ dados da classe alg \_**

</dd> <dt>

<span id="_security_alg_class_hash_gly"></span><span id="_SECURITY_ALG_CLASS_HASH_GLY"></span>**\_hash de classe alg \_**

A classe de algoritmo CryptoAPI para algoritmos de hash. Os algoritmos de hash típicos incluem MD2, MD5, SHA-1 e MAC.

</dd> <dt>

<span id="_security_alg_class_key_exchange_gly"></span><span id="_SECURITY_ALG_CLASS_KEY_EXCHANGE_GLY"></span>**\_troca de \_ chave de classe alg \_**

A classe de algoritmo CryptoAPI para algoritmos de troca de chave. Um algoritmo típico de troca de chaves é o RSA \_ KEYX.

</dd> <dt>

<span id="_security_alg_class_signature_gly"></span><span id="_SECURITY_ALG_CLASS_SIGNATURE_GLY"></span>**\_assinatura da classe alg \_**

A classe de algoritmo CryptoAPI para algoritmos de assinatura. Um algoritmo de assinatura digital típico é o RSA \_ Sign.

</dd> <dt>

<span id="_security_apdu_gly"></span><span id="_SECURITY_APDU_GLY"></span>**APDU**
</dt> <dd>

<span id="_security_application_protocol_data_unit_gly"></span><span id="_SECURITY_APPLICATION_PROTOCOL_DATA_UNIT_GLY"></span>**unidade de dados do protocolo de aplicativo**

APDU Uma sequência de comandos (uma unidade de dados de protocolo de aplicativo) que pode ser enviada pelo cartão inteligente ou retornada pelo aplicativo.

Consulte também [*responder APDU*](r-gly.md).

</dd> <dt>

<span id="_security_application_protocol_gly"></span><span id="_SECURITY_APPLICATION_PROTOCOL_GLY"></span>**Protocolo de aplicativo**

<span id="_security_asn.1_gly"></span><span id="_SECURITY_ASN.1_GLY"></span>**ASN. 1**

</dd> <dt>

<span id="_security_ascii_gly"></span><span id="_SECURITY_ASCII_GLY"></span>**LOCALIZADOS**
</dt> <dd>

ASCII. Um esquema de codificação que atribui valores numéricos a letras, números, marcas de Pontuação e outros caracteres.

</dd> <dt>

<span id="_security_asymmetric_algorithm_gly"></span><span id="_SECURITY_ASYMMETRIC_ALGORITHM_GLY"></span>**algoritmo assimétrico**
</dt> <dd>

Consulte [*algoritmo de chave pública*](p-gly.md).

</dd> <dt>

<span id="_security_asymmetric_key_gly"></span><span id="_SECURITY_ASYMMETRIC_KEY_GLY"></span>**chave assimétrica**
</dt> <dd>

Um de um par de chaves usado com um algoritmo de criptografia assimétrica. Esse algoritmo usa duas chaves de criptografia: uma "chave pública" para criptografia e uma "chave privada" para descriptografia. Na assinatura e na verificação, as funções são revertidas: a chave pública é usada para verificação e a chave privada é usada para a geração de assinatura. O recurso mais importante desses algoritmos é que sua segurança não depende de manter o segredo da chave pública (embora possa exigir alguma garantia de autenticidade das chaves públicas, por exemplo, que elas sejam obtidas de uma fonte confiável). O sigilo da chave privada é necessário. Exemplos de algoritmos de chave pública são [*DSA (algoritmo de assinatura digital)*](d-gly.md), ECDSA (algoritmo de assinatura digital de curva elíptica) e a família de algoritmos do Rivest-Shamir-adlemat (RSA).

</dd> <dt>

<span id="_security_atr_string_gly"></span><span id="_SECURITY_ATR_STRING_GLY"></span>**Cadeia de caracteres ATR**
</dt> <dd>

Uma sequência de bytes retornada de um cartão inteligente quando ele é ativado. Esses bytes são usados para identificar o cartão para o sistema.

</dd> <dt>

<span id="_security_attribute_gly"></span><span id="_SECURITY_ATTRIBUTE_GLY"></span>**Attribute**
</dt> <dd>

Um elemento de um RDN (nome distinto relativo). Alguns atributos típicos incluem nome comum, sobrenome, endereço de email, endereço postal e nome de país/região.

</dd> <dt>

<span id="_security_attribute_blob_gly"></span><span id="_SECURITY_ATTRIBUTE_BLOB_GLY"></span>**BLOB de atributo**
</dt> <dd>

Uma representação codificada das informações de atributo armazenadas em uma solicitação de certificado.

</dd> <dt>

<span id="_security_audit_security_object_gly"></span><span id="_SECURITY_AUDIT_SECURITY_OBJECT_GLY"></span>**Auditar objeto de segurança**
</dt> <dd>

Um descritor de segurança que controla o acesso ao subsistema de diretiva de auditoria.

</dd> <dt>

<span id="_security_authentication_gly"></span><span id="_SECURITY_AUTHENTICATION_GLY"></span>**Authentication**
</dt> <dd>

O processo para verificar se um usuário, computador, serviço ou processo é quem ou o que ele alega ser.

</dd> <dt>

<span id="_security_authentication_package_gly"></span><span id="_SECURITY_AUTHENTICATION_PACKAGE_GLY"></span>**pacote de autenticação**
</dt> <dd>

Uma DLL que encapsula a lógica de autenticação usada para determinar se deve permitir que um usuário faça logon. O LSA autentica um logon de usuário enviando a solicitação a um pacote de autenticação. Em seguida, o pacote de autenticação examina as informações de logon e autentica ou rejeita a tentativa de logon do usuário.

</dd> <dt>

<span id="_security_authenticode_gly"></span><span id="_SECURITY_AUTHENTICODE_GLY"></span>**Authenticode**
</dt> <dd>

Um recurso de segurança do Internet Explorer. O Authenticode permite que os fornecedores de códigos executáveis baixáveis (plug-ins ou controles ActiveX, por exemplo) anexem certificados digitais a seus produtos para garantir aos usuários finais que o código é do desenvolvedor original e que não foram alterados. O Authenticode permite que os usuários finais decidam se aceitam ou rejeitam componentes de software postados na Internet antes do início do download.

</dd> </dl>
