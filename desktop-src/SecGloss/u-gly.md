# <a name="u-security-glossary"></a>U (Glossário de segurança)

[A](a-gly.md) [B](b-gly.md) [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F [G](g-gly.md) [H](h-gly.md) [i](i-gly.md) J [K](k-gly.md) [L](l-gly.md) [M](m-gly.md) [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) U [V](v-gly.md) [W](w-gly.md) [X](x-gly.md) Y Z

<dl> <dt>

<span id="_security_ucs_gly"></span><span id="_SECURITY_UCS_GLY"></span>**UCS**
</dt> <dd>

Consulte *conjunto de caracteres universais*.

</dd> <dt>

<span id="_security_unicode_gly"></span><span id="_SECURITY_UNICODE_GLY"></span>**Unicode**
</dt> <dd>

Um padrão de codificação de caractere mundial que permite que mais informações sejam contidas em cada cadeia de caracteres definindo cadeias de caracteres de 16 bits em vez de cadeias de caracteres de 8 bits padrão. O Unicode permite a troca de dados universal e melhora o processamento de texto multilíngüe. Cadeias de caracteres Unicode também são chamadas de *cadeias de caracteres grandes*.

</dd> <dt>

<span id="_security_universal_character_set_gly"></span><span id="_SECURITY_UNIVERSAL_CHARACTER_SET_GLY"></span>**Conjunto de caracteres universal**
</dt> <dd>

UCS Um conjunto de caracteres de 16 bits.

</dd> <dt>

<span id="_security_universal_well_known_security_identifier_gly"></span><span id="_SECURITY_UNIVERSAL_WELL_KNOWN_SECURITY_IDENTIFIER_GLY"></span>**identificador de segurança bem conhecido Universal**
</dt> <dd>

Um SID ( [*identificador de segurança*](s-gly.md) ) que é conhecido em todos os sistemas que aderem ao modelo de segurança baseado em usuários autenticados e controle de acesso discricionário.

</dd> <dt>

<span id="_security_upn_gly"></span><span id="_SECURITY_UPN_GLY"></span>**SUFIXO**
</dt> <dd>

Consulte *nome principal do usuário*.

</dd> <dt>

<span id="_security_user_gly"></span><span id="_SECURITY_USER_GLY"></span>**usuário**
</dt> <dd>

Um usuário de um sistema é identificado por um procedimento de logon, que estabelece um processo pelo qual os aplicativos podem executar e acessar outros objetos relevantes de segurança. Um exemplo de um procedimento de logon é usar um cartão inteligente, possivelmente em conjunto com uma senha ou um PIN (número de identificação pessoal).

</dd> <dt>

<span id="_security_user_interface_smart_card_gly"></span><span id="_SECURITY_USER_INTERFACE_SMART_CARD_GLY"></span>**interface do usuário (cartão inteligente)**
</dt> <dd>

Uma caixa de diálogo comum que permite ao usuário se conectar a um cartão inteligente e usá-lo em um aplicativo. Usando a caixa de diálogo, o usuário pode especificar um cartão específico ou pesquisar o cartão inteligente a ser aberto.

</dd> <dt>

<span id="_security_user_principal_name_gly"></span><span id="_SECURITY_USER_PRINCIPAL_NAME_GLY"></span>**nome principal do usuário**
</dt> <dd>

SUFIXO Um nome de conta de usuário (às vezes chamado de nome de logon do usuário) e um nome de domínio que identifica o domínio no qual a conta de usuário está localizada. Esse é o uso padrão para fazer logon em um domínio do Windows. O formato é: someone@example.com (como para um endereço de email).

</dd> </dl>
