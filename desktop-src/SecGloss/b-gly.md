# <a name="b-security-glossary"></a>B (Glossário de segurança)

[A](a-gly.md) B [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F [G](g-gly.md) [H](h-gly.md) [i](i-gly.md) J [K](k-gly.md) [L](l-gly.md) [M](m-gly.md) [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) [U](u-gly.md) [V](v-gly.md) [W](w-gly.md) [X](x-gly.md) Y Z

<dl> <dt>

<span id="_security_backup_authority_gly"></span><span id="_SECURITY_BACKUP_AUTHORITY_GLY"></span>**autoridade de backup**
</dt> <dd>

Um aplicativo confiável em execução em um computador seguro que fornece armazenamento secundário para as chaves de sessão de seus clientes. A autoridade de backup armazena chaves de sessão como BLOBs de chave que são criptografados com a chave pública da autoridade de backup.

</dd> <dt>

<span id="_security_base_content_type_gly"></span><span id="_SECURITY_BASE_CONTENT_TYPE_GLY"></span>**tipo de conteúdo base**
</dt> <dd>

Um tipo de dados contidos em uma \# mensagem PKCS 7. Os tipos de conteúdo base contêm apenas dados, sem aprimoramentos de criptografia, como hashes ou assinaturas. Atualmente, o único tipo de conteúdo base é o tipo de conteúdo de dados.

</dd> <dt>

<span id="_security_base_cryptographic_functions_gly"></span><span id="_SECURITY_BASE_CRYPTOGRAPHIC_FUNCTIONS_GLY"></span>**funções de criptografia base**
</dt> <dd>

O menor nível de funções na arquitetura de CryptoAPI. Eles são usados por aplicativos e outras funções de CryptoAPI de alto nível para fornecer acesso aos algoritmos criptográficos fornecidos pelo CSP, à geração de chave segura e ao armazenamento seguro de segredos.

Consulte também [*provedores de serviço de criptografia*](c-gly.md).

</dd> <dt>

<span id="_security_basic_encoding_rules_gly"></span><span id="_SECURITY_BASIC_ENCODING_RULES_GLY"></span>**Regras básicas de codificação**
</dt> <dd>

BRO O conjunto de regras usadas para codificar os dados definidos pelo ASN. 1 em um fluxo de bits (zeros ou uns) para armazenamento ou transmissão externa. Um único objeto ASN. 1 pode ter vários codificadores do BER equivalentes. O BER é definido na recomendação de CCITT X. 209. Esse é um dos dois métodos de codificação usados atualmente pelo CryptoAPI.

</dd> <dt>

<span id="_security_ber_gly"></span><span id="_SECURITY_BER_GLY"></span>**BRO**
</dt> <dd>

Consulte *regras básicas de codificação*.

</dd> <dt>

<span id="_security_big_endian_gly"></span><span id="_SECURITY_BIG_ENDIAN_GLY"></span>**big endian**
</dt> <dd>

Um formato de memória ou de dados no qual o byte mais significativo é armazenado no endereço inferior ou chega primeiro.

Veja também [*little-endian*](l-gly.md).

</dd> <dt>

<span id="_security_blob_gly"></span><span id="_SECURITY_BLOB_GLY"></span>**BLOB**
</dt> <dd>

Uma sequência genérica de bits que contém uma ou mais estruturas de cabeçalho de comprimento fixo, além de dados específicos de contexto.

Consulte também [*blobs de chave*](k-gly.md), [*blobs de certificado*](c-gly.md), [*blobs de nome de certificado*](c-gly.md)e [*blobs de atributo*](a-gly.md).

</dd> <dt>

<span id="_security_block_cipher_gly"></span><span id="_SECURITY_BLOCK_CIPHER_GLY"></span>**codificação de bloco**
</dt> <dd>

Um algoritmo de codificação que criptografa dados em unidades discretas (chamadas de blocos), em vez de como um fluxo contínuo de bits. O tamanho de bloco mais comum é de 64 bits. Por exemplo, o DES é uma codificação de bloco.

Consulte também [*codificação de fluxo*](s-gly.md).

</dd> <dt>

<span id="_security_bulk_encryption_key_gly"></span><span id="_SECURITY_BULK_ENCRYPTION_KEY_GLY"></span>**chave de criptografia em massa**
</dt> <dd>

Uma chave de sessão derivada de uma chave mestra. As chaves de criptografia em massa são usadas na criptografia [*Schannel*](s-gly.md) .

</dd> </dl>
