# <a name="t-security-glossary"></a>T (Glossário de segurança)

<dl> <dt>

<span id="_security_t_0_protocol_gly"></span><span id="_SECURITY_T_0_PROTOCOL_GLY"></span>**T = 0 protocolo**
</dt> <dd>

Um protocolo assíncrono de transmissão de Half-duplex orientado por caracteres.

</dd> <dt>

<span id="_security_t_1_protocol_gly"></span><span id="_SECURITY_T_1_PROTOCOL_GLY"></span>**T = 1 protocolo**
</dt> <dd>

Um protocolo assíncrono de transmissão de Half-duplex orientado por blocos.

</dd> <dt>

<span id="_security_terminal_gly"></span><span id="_SECURITY_TERMINAL_GLY"></span>**componentes**
</dt> <dd>

Uma combinação de monitor, teclado, mouse e periféricos colocalizados, como leitores de cartão inteligente. Vários processos podem ser associados a um único terminal, mas apenas um processo controla o terminal em um determinado momento.

</dd> <dt>

<span id="_security_three_phase_key_exchange_protocol_gly"></span><span id="_SECURITY_THREE_PHASE_KEY_EXCHANGE_PROTOCOL_GLY"></span>**Protocolo de troca de chaves de três fases**
</dt> <dd>

Um protocolo usado para gerar uma conexão autenticada e criptografada entre dois usuários em uma rede não segura. Os usuários trocam um conjunto de mensagens para negociar um par de chaves de criptografia. Uma chave é usada pelo remetente para criptografar mensagens enviadas ao receptor e a outra é usada pelo destinatário para criptografar as mensagens enviadas ao remetente. Esse protocolo garante que ambos os usuários estejam ativos e enviem mensagens diretamente entre si.

</dd> <dt>

<span id="_security_thumbprint_gly"></span><span id="_SECURITY_THUMBPRINT_GLY"></span>**digitais**
</dt> <dd>

O hash [*SHA-1*](s-gly.md) de um certificado.

</dd> <dt>

<span id="_security_tls_gly"></span><span id="_SECURITY_TLS_GLY"></span>**TLS**
</dt> <dd>

Consulte *protocolo de segurança da camada de transporte*.

</dd> <dt>

<span id="_security_token_gly"></span><span id="_SECURITY_TOKEN_GLY"></span>**token**
</dt> <dd>

Consulte [*token de acesso*](a-gly.md).

</dd> <dt>

<span id="_security_transaction_gly"></span><span id="_SECURITY_TRANSACTION_GLY"></span>**aciona**
</dt> <dd>

Uma operação que permite executar várias interações como uma única operação. As etapas intermediárias não são realmente realizadas até que todas as interações sejam concluídas com êxito. Se alguma interação falhar, todas as etapas serão retornadas à configuração original.

</dd> <dt>

<span id="_security_transport_layer_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_GLY"></span>**camada de transporte**
</dt> <dd>

A camada de rede que é responsável por qualidade de serviço e entrega precisa de informações. Entre as tarefas executadas nesta camada estão a detecção de erros e a correção.

</dd> <dt>

<span id="_security_transport_layer_security_protocol_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_SECURITY_PROTOCOL_GLY"></span>**Protocolo de segurança da camada de transporte**
</dt> <dd>

TLS Um protocolo que fornece privacidade e segurança de comunicações entre dois aplicativos se comunicando por meio de uma rede.

</dd> <dt>

<span id="_security_triple_des_gly"></span><span id="_SECURITY_TRIPLE_DES_GLY"></span>**DES triplo**
</dt> <dd>

Uma variação do algoritmo de codificação de bloco DES que criptografa o texto sem formatação com uma chave, criptografa o cifrado resultante com uma segunda chave e, por fim, criptografa o resultado da segunda criptografia com uma terceira chave. O DES triplo é um algoritmo simétrico que usa o mesmo algoritmo e chaves para criptografia e descriptografia.

</dd> <dt>

<span id="_security_trustee_gly"></span><span id="_SECURITY_TRUSTEE_GLY"></span>**confiança**
</dt> <dd>

A conta de usuário, a conta de grupo ou a sessão de logon à qual se aplica uma ACE (entrada de controle de acesso). Cada ACE em uma ACL (lista de controle de acesso) se aplica a um confiável.

Consulte também [*entrada de controle de acesso*](a-gly.md) e [*lista de controle de acesso*](a-gly.md).

</dd> <dt>

<span id="_security_trust_list_gly"></span><span id="_SECURITY_TRUST_LIST_GLY"></span>**lista de confiança**
</dt> <dd>

Consulte [*lista de certificados confiáveis*](c-gly.md).

</dd> <dt>

<span id="_security_trust_provider_gly"></span><span id="_SECURITY_TRUST_PROVIDER_GLY"></span>**provedor de confiança**
</dt> <dd>

O software que decide se um determinado arquivo é confiável. Essa decisão se baseia no certificado associado ao arquivo.

</dd> </dl>
