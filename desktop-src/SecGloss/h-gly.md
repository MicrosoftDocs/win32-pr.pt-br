# <a name="h-security-glossary"></a>H (Glossário de segurança)

<dl> <dt>

<span id="_security_handle_gly"></span><span id="_SECURITY_HANDLE_GLY"></span>**processamento**
</dt> <dd>

Um token usado para identificar ou acessar um objeto, como o identificador para um provedor criptográfico, repositório de certificados, mensagem ou par de chaves.

</dd> <dt>

<span id="_security_hash_gly"></span><span id="_SECURITY_HASH_GLY"></span>**tralha**
</dt> <dd>

Um resultado de tamanho fixo obtido pela aplicação de uma função matemática (o *algoritmo de hash*) a uma quantidade arbitrária de dados. (Também conhecido como "síntese de mensagem".)

Consulte também *funções de hash*.

</dd> <dt>

<span id="_security_hash_object_gly"></span><span id="_SECURITY_HASH_OBJECT_GLY"></span>**objeto hash**
</dt> <dd>

Um objeto usado para mensagens de hash ou chaves de sessão. O objeto hash é criado por uma chamada para **CryptCreateHash**. A definição do objeto é definida pelo CSP especificado na chamada.

</dd> <dt>

<span id="_security_hashing_algorithm_gly"></span><span id="_SECURITY_HASHING_ALGORITHM_GLY"></span>**algoritmo de hash**
</dt> <dd>

Um algoritmo usado para produzir um valor de hash de algum dado, como uma mensagem ou chave de sessão. Os algoritmos de hash típicos incluem MD2, MD4, MD5 e SHA-1.

</dd> <dt>

<span id="_security_hashing_functions_gly"></span><span id="_SECURITY_HASHING_FUNCTIONS_GLY"></span>**funções de hash**
</dt> <dd>

Um conjunto de funções usadas para criar e destruir objetos de hash, obter ou definir os parâmetros de um objeto de hash e as chaves de sessão e de dados de hash.

</dd> <dt>

<span id="_security_hash_based_message_authentication_code_gly"></span><span id="_SECURITY_HASH_BASED_MESSAGE_AUTHENTICATION_CODE_GLY"></span>**Message Authentication Code com base em hash**
</dt> <dd>

HMAC Um algoritmo de hash com chave simétrica implementado pelos provedores de serviços de criptografia da Microsoft. Um HMAC é usado para verificar a integridade dos dados para ajudar a garantir que ele não tenha sido modificado enquanto estiver em armazenamento ou em trânsito. Ele pode ser usado com qualquer algoritmo de hash criptográfico iterado, como MD5 ou SHA-1. O CryptoAPI faz referência a esse algoritmo por seu identificador de algoritmo (CALG \_ HMAC) e classe ( \_ hash de classe alg \_ ).

Consulte também [*Message Authentication Code*](m-gly.md).

</dd> <dt>

<span id="_security_hcsbc_gly"></span><span id="_SECURITY_HCSBC_GLY"></span>**HCSBC**
</dt> <dd>

Tipo de dados que serve como um identificador para um contexto de backup dos serviços de certificados. Sua função é manter o estado de contexto entre o servidor e as APIs de backup quando um backup está sendo executado.

</dd> <dt>

<span id="_security_hmac_gly"></span><span id="_SECURITY_HMAC_GLY"></span>**HMAC**
</dt> <dd>

Consulte *Message Authentication Code com base em hash*.

</dd> </dl>
