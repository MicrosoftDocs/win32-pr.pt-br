# <a name="h-security-glossary"></a>H (Glossário de Segurança)

[A](a-gly.md) [B](b-gly.md) [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F [G](g-gly.md) H [I](i-gly.md) J [K](k-gly.md) L [M](l-gly.md) [](m-gly.md) [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) [U](u-gly.md) [V](v-gly.md) [W](w-gly.md) [X](x-gly.md) Y Z

<dl> <dt>

<span id="_security_handle_gly"></span><span id="_SECURITY_HANDLE_GLY"></span>**Lidar com**
</dt> <dd>

Um token usado para identificar ou acessar um objeto, como o identificador para um provedor de criptografia, um armazenamento de certificados, uma mensagem ou um par de chaves.

</dd> <dt>

<span id="_security_hash_gly"></span><span id="_SECURITY_HASH_GLY"></span>**Hash**
</dt> <dd>

Um resultado de tamanho fixo obtido pela aplicação de uma função matemática (o *algoritmo de hash*) a uma quantidade arbitrária de dados. (Também conhecido como "resumo da mensagem").

Consulte também *funções de hash*.

</dd> <dt>

<span id="_security_hash_object_gly"></span><span id="_SECURITY_HASH_OBJECT_GLY"></span>**objeto hash**
</dt> <dd>

Um objeto usado para hash de mensagens ou chaves de sessão. O objeto hash é criado por uma chamada para **CryptCreateHash**. A definição do objeto é definida pelo CSP especificado na chamada.

</dd> <dt>

<span id="_security_hashing_algorithm_gly"></span><span id="_SECURITY_HASHING_ALGORITHM_GLY"></span>**algoritmo de hash**
</dt> <dd>

Um algoritmo usado para produzir um valor de hash de alguns dados, como uma mensagem ou uma chave de sessão. Os algoritmos de hash típicos incluem MD2, MD4, MD5 e SHA-1.

</dd> <dt>

<span id="_security_hashing_functions_gly"></span><span id="_SECURITY_HASHING_FUNCTIONS_GLY"></span>**funções de hash**
</dt> <dd>

Um conjunto de funções usadas para criar e destruir objetos hash, obter ou definir os parâmetros de um objeto de hash e dados de hash e chaves de sessão.

</dd> <dt>

<span id="_security_hash_based_message_authentication_code_gly"></span><span id="_SECURITY_HASH_BASED_MESSAGE_AUTHENTICATION_CODE_GLY"></span>**Dados baseados em hash Message Authentication Code**
</dt> <dd>

(HMAC) Um algoritmo de hash com chave simétrica implementado pelos provedores de serviços criptográficos da Microsoft. Um HMAC é usado para verificar a integridade dos dados para ajudar a garantir que eles não foram modificados durante o armazenamento ou o trânsito. Ele pode ser usado com qualquer algoritmo de hash criptográfico iterado, como MD5 ou SHA-1. CryptoAPI faz referência a esse algoritmo por seu identificador de algoritmo (CALG \_ HMAC) e classe \_ \_ (ALG CLASS HASH).

Consulte também [*Message Authentication Code*](m-gly.md).

</dd> <dt>

<span id="_security_hcsbc_gly"></span><span id="_SECURITY_HCSBC_GLY"></span>**HCSBC**
</dt> <dd>

Tipo de dados que serve como um identificador para um contexto de backup dos Serviços de Certificados. Sua função é manter o estado de contexto entre o servidor e as APIs de backup quando um backup está sendo executado.

</dd> <dt>

<span id="_security_hmac_gly"></span><span id="_SECURITY_HMAC_GLY"></span>**Hmac**
</dt> <dd>

Consulte *Hash-based Message Authentication Code*.

</dd> </dl>
