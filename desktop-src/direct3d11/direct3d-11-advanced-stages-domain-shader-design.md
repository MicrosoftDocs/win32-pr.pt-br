# <a name="how-to-design-a-domain-shader"></a><span data-ttu-id="60cf5-103">Como criar um sombreador de domínio</span><span class="sxs-lookup"><span data-stu-id="60cf5-103">How To: Design a Domain Shader</span></span>

<span data-ttu-id="60cf5-104">Um sombreador de domínio é o terceiro de três estágios que trabalham em conjunto para implementar o [mosaico](direct3d-11-advanced-stages-tessellation.md).</span><span class="sxs-lookup"><span data-stu-id="60cf5-104">A domain shader is the third of three stages that work together to implement [tessellation](direct3d-11-advanced-stages-tessellation.md).</span></span> <span data-ttu-id="60cf5-105">O sombreador de domínio gera a geometria da superfície dos pontos de controle transformados de um sombreador envoltória e das coordenadas UV.</span><span class="sxs-lookup"><span data-stu-id="60cf5-105">The domain shader generates the surface geometry from the transformed control points from a hull shader and the UV coordinates.</span></span> <span data-ttu-id="60cf5-106">Este tópico mostra como criar um sombreador de domínio.</span><span class="sxs-lookup"><span data-stu-id="60cf5-106">This topics shows how to design a domain shader.</span></span>

<span data-ttu-id="60cf5-107">Um sombreador de domínio é invocado uma vez para cada ponto gerado pela função fixa Tessellator.</span><span class="sxs-lookup"><span data-stu-id="60cf5-107">A domain shader is invoked once for each point generated by the fixed function tessellator.</span></span> <span data-ttu-id="60cf5-108">As entradas são as \[ coordenadas UV W \] do ponto no patch, bem como todos os dados de saída do sombreador envoltória, incluindo pontos de controle e constantes de patch.</span><span class="sxs-lookup"><span data-stu-id="60cf5-108">The inputs are the UV\[W\] coordinates of the point on the patch, as well as all of the output data from the hull shader including control points and patch constants.</span></span> <span data-ttu-id="60cf5-109">A saída é um vértice definido de qualquer maneira desejada.</span><span class="sxs-lookup"><span data-stu-id="60cf5-109">The output is a vertex defined in whatever way is desired.</span></span> <span data-ttu-id="60cf5-110">Se a saída estiver sendo enviada para o sombreador de pixel, a saída deverá incluir uma posição (denotada com uma semântica de posição de VA \_ ).</span><span class="sxs-lookup"><span data-stu-id="60cf5-110">If the output is being sent to the pixel shader, the output must include a position (denoted with a SV\_Position semantic).</span></span>

<span data-ttu-id="60cf5-111">**Para criar um sombreador de domínio**</span><span class="sxs-lookup"><span data-stu-id="60cf5-111">**To design a domain shader**</span></span>

1.  <span data-ttu-id="60cf5-112">Defina o atributo de domínio.</span><span class="sxs-lookup"><span data-stu-id="60cf5-112">Define the domain attribute.</span></span>

    ```
    [domain("quad")]
    ```

    

    <span data-ttu-id="60cf5-113">O domínio é definido para um patch quádruplo.</span><span class="sxs-lookup"><span data-stu-id="60cf5-113">The domain is defined for a quad patch.</span></span>

2.  <span data-ttu-id="60cf5-114">Declare o local no envoltória com o valor do sistema de [local de domínio](/windows/desktop/direct3dhlsl/sv-domainlocation) .</span><span class="sxs-lookup"><span data-stu-id="60cf5-114">Declare the location on the hull with the [domain location](/windows/desktop/direct3dhlsl/sv-domainlocation) system value.</span></span>

    -   <span data-ttu-id="60cf5-115">Para um patch quádruplo, use um float2.</span><span class="sxs-lookup"><span data-stu-id="60cf5-115">For a quad patch, use a float2.</span></span>
    -   <span data-ttu-id="60cf5-116">Para um patch Tri, use um float3 (para coordenadas barycentric)</span><span class="sxs-lookup"><span data-stu-id="60cf5-116">For a tri patch, use a float3 (for barycentric coordinates)</span></span>
    -   <span data-ttu-id="60cf5-117">Para um Isoline, use um float2.</span><span class="sxs-lookup"><span data-stu-id="60cf5-117">For an isoline, use a float2.</span></span>

    <span data-ttu-id="60cf5-118">Portanto, o local do domínio para um patch quádruplo tem esta aparência:</span><span class="sxs-lookup"><span data-stu-id="60cf5-118">Therefore, the domain location for a quad patch looks like this:</span></span>

    ```
    float2 UV : SV_DomainLocation
    ```

    

3.  <span data-ttu-id="60cf5-119">Defina as outras entradas.</span><span class="sxs-lookup"><span data-stu-id="60cf5-119">Define the other inputs.</span></span>

    <span data-ttu-id="60cf5-120">As outras entradas são provenientes do sombreador envoltória e são definidas pelo usuário.</span><span class="sxs-lookup"><span data-stu-id="60cf5-120">The other inputs come from the hull shader and are user defined.</span></span> <span data-ttu-id="60cf5-121">Isso inclui os pontos de controle de entrada para patch, dos quais pode haver entre 1 e 32 pontos e dados constantes de patch de entrada.</span><span class="sxs-lookup"><span data-stu-id="60cf5-121">This includes the input control points for patch, of which there can be between 1 and 32 points, and input patch constant data.</span></span>

    <span data-ttu-id="60cf5-122">Os pontos de controle são definidos pelo usuário, geralmente com uma estrutura como esta (definida em [How to: design a Shader envoltória](direct3d-11-advanced-stages-hull-shader-design.md)):</span><span class="sxs-lookup"><span data-stu-id="60cf5-122">The control points are user defined, usually with a structure such as this one (defined in [How To: Design a Hull Shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span></span>

    ```
    const OutputPatch<BEZIER_CONTROL_POINT, 16> bezpatch
    ```

    

    <span data-ttu-id="60cf5-123">Os dados constantes de patch também são definidos pelo usuário e podem ser semelhantes a este (definido em [como criar um sombreador envoltória](direct3d-11-advanced-stages-hull-shader-design.md)):</span><span class="sxs-lookup"><span data-stu-id="60cf5-123">The patch constant data is also user defined, and might look like this one (defined in [How To: Design a Hull Shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span></span>

    ```
    HS_CONSTANT_DATA_OUTPUT input
    ```

    

4.  <span data-ttu-id="60cf5-124">Adicionar código definido pelo usuário para computar as saídas; Isso constitui o corpo do sombreador de domínio.</span><span class="sxs-lookup"><span data-stu-id="60cf5-124">Add user-defined code to compute the outputs; this makes up the body of the domain shader.</span></span>

    <span data-ttu-id="60cf5-125">Essa estrutura contém saídas de sombreador de domínio definidas pelo usuário.</span><span class="sxs-lookup"><span data-stu-id="60cf5-125">This structure contains user-defined domain shader outputs.</span></span>

    ```
    struct DS_OUTPUT
    {
        float3 vNormal    : NORMAL;
        float2 vUV        : TEXCOORD;
        float3 vTangent   : TANGENT;
        float3 vBiTangent : BITANGENT;
        
        float4 vPosition  : SV_POSITION;
    };
    ```

    

    <span data-ttu-id="60cf5-126">A função usa cada UV de entrada (do Tessellator) e avalia o patch Bezier nessa posição.</span><span class="sxs-lookup"><span data-stu-id="60cf5-126">The function takes each input UV (from the tessellator) and evaluates the Bezier patch at this position.</span></span>

    ```
    [domain("quad")]
    DS_OUTPUT BezierEvalDS( HS_CONSTANT_DATA_OUTPUT input, 
                            float2 UV : SV_DomainLocation,
                            const OutputPatch<BEZIER_CONTROL_POINT, 16> bezpatch )
    {
        DS_OUTPUT Output;

        // Insert code to compute the output here.

        return Output;    
    }
    ```

    

    <span data-ttu-id="60cf5-127">A função é invocada uma vez para cada ponto gerado pela função fixa Tessellator.</span><span class="sxs-lookup"><span data-stu-id="60cf5-127">The function is invoked once for each point generated by the fixed function tessellator.</span></span> <span data-ttu-id="60cf5-128">Como este exemplo usa um patch quádruplo, o local do domínio de entrada ([VA \_ DomainLocation](/windows/desktop/direct3dhlsl/sv-domainlocation)) é um float2 (UV); um patch Tri teria um local de entrada float3 (coordenadas barycentric UVW) e um Isoline teria um local de domínio de entrada float2.</span><span class="sxs-lookup"><span data-stu-id="60cf5-128">Since this example uses a quad patch, the input domain location ([SV\_DomainLocation](/windows/desktop/direct3dhlsl/sv-domainlocation)) is a float2 (UV); a tri patch would have a float3 input location (UVW barycentric coordinates), and an isoline would have a float2 input domain location.</span></span>

    <span data-ttu-id="60cf5-129">As outras entradas para a função vêm do sombreador envoltória diretamente.</span><span class="sxs-lookup"><span data-stu-id="60cf5-129">The other inputs for the function come from the hull shader directly.</span></span> <span data-ttu-id="60cf5-130">Neste exemplo, é 16 pontos de controle, cada um sendo **um \_ \_ ponto de controle de Bézier**, bem como patches de dados constantes (**\_ \_ \_ saída de dados constante HS**).</span><span class="sxs-lookup"><span data-stu-id="60cf5-130">In this example it is 16 control points each one being a **BEZIER\_CONTROL\_POINT**, as well as patch constant data (**HS\_CONSTANT\_DATA\_OUTPUT**).</span></span> <span data-ttu-id="60cf5-131">A saída é um vértice que contém qualquer **\_ saída** de dados desejada-DS neste exemplo.</span><span class="sxs-lookup"><span data-stu-id="60cf5-131">The output is a vertex containing any data desired - **DS\_OUTPUT** in this example.</span></span>
