# <a name="whats-new-in-the-nov-2008-direct3d-11-tech-preview"></a>Novidades na Versão Prévia Técnica do Direct3D 11 de novembro de 2008

Esta versão do Direct3D 11 contém novos recursos, ferramentas e documentação.




| Item | Descrição | 
|------|-------------|
| <span id="Tessellation"></span><span id="tessellation"></span><span id="TESSELLATION"></span>Mosaico<br /> | O Direct3D 11 fornece estágios de pipeline adicionais para dar suporte ao <a href="direct3d-11-advanced-stages-tessellation.md">mosaico</a> em tempo real de primitivos de ordem alta. Com recursos amplamente programáveis, esse recurso permite muitos métodos diferentes para avaliar superfícies de alta ordem, incluindo superfícies de subdivisão usando técnicas de aproximação, patches de Bezier, mosaico adaptável e mapeamento de deslocamento. Esse recurso só estará disponível no hardware de classe Direct3D 11, portanto, para avaliar esse recurso, você precisará usar o Rasterizador de Referência. Para ver uma demonstração do mosaico em ação, confira o <a href="https://msdn.microsoft.com/library/Ee416576(v=VS.85).aspx">exemplo de SubD11</a> disponível por meio do Sample Browser.<br /> | 
| <span id="Compute_Shader"></span><span id="compute_shader"></span><span id="COMPUTE_SHADER"></span>Sombreador de computação<br /> | O <a href="direct3d-11-advanced-stages-compute-shader.md">Sombreador de Computação</a> é um estágio adicional independente do pipeline do Direct3D 11 que permite a computação de uso geral na GPU. Além de todos os recursos de sombreador fornecidos pelo núcleo de sombreador unificado, o Sombreador de Computação também dá suporte a leituras e gravações dispersas em recursos por meio de Exibições de Acesso Não Organizado, um pool de memória compartilhada dentro de um grupo de threads em execução, primitivos de sincronização, operadores atômicos e muitos outros recursos avançados paralelos de dados. Uma variante do Sombreador de Computação direct3D 11 foi habilitada nesta versão que pode operar no hardware da classe Direct3D 10. Portanto, é possível desenvolver sombreadores de computação em hardware real, mas um driver atualizado é necessário. A funcionalidade completa do Sombreador de Computação direct3D 11 destina-se ao suporte do hardware de classe Direct3D 11, portanto, para avaliar a funcionalidade completa, você precisará usar o Rasterizador de Referência até que esse hardware seja disponibilizado. Para ver uma demonstração do Sombreador de Computação em ação, confira o exemplo <a href="https://msdn.microsoft.com/library/Ee416569(v=VS.85).aspx">HDRToneMappingCS11</a> disponível por meio do Navegador de Exemplo.<br /> | 
| <span id="Multithreaded_Rendering"></span><span id="multithreaded_rendering"></span><span id="MULTITHREADED_RENDERING"></span>Renderização multithread<br /> | A principal diferença de API do Direct3D 10 no Direct3D 11 é a adição de <a href="overviews-direct3d-11-render-multi-thread.md">contextos adiados,</a>que permite a execução escalonável de comandos Direct3D distribuídos em vários núcleos. Um Contexto Adiado captura e monta ações como alterações de estado e envios de desenho que podem ser executados no dispositivo real posteriormente. Utilizando Contextos Adiados em vários threads, um aplicativo pode distribuir a sobrecarga de CPU necessária no runtime direct3D11 e o driver para vários núcleos, permitindo um melhor uso da configuração do computador do usuário final. Esse recurso está disponível para uso no hardware atual do Direct3D 10, bem como no rasterizador de referência. Para ver uma demonstração do uso da API, confira o exemplo <a href="https://msdn.microsoft.com/library/Ee416570(v=VS.85).aspx">MultithreadedRendering11</a> disponível por meio do Sample Browser.<br /> | 
| <span id="Dynamic_Shader_Linkage_"></span><span id="dynamic_shader_linkage_"></span><span id="DYNAMIC_SHADER_LINKAGE_"></span>Vinculação dinâmica do sombreador <br /> | Para resolver o problema de explosão combinatória visto na especialização de sombreadores para desempenho, o Direct3D 11 apresenta uma forma limitada de vinculação de sombreador de <a href="/windows/desktop/direct3dhlsl/overviews-direct3d-11-hlsl-dynamic-linking">runtime</a> que permite especialização de sombreador quase ideal durante a execução de um aplicativo. Isso é feito especificando as implementações de funções específicas no código do sombreador quando o sombreador é atribuído ao pipeline, permitindo que o driver emlinee instruções de sombreador nativo rapidamente em vez de forçar o driver a recompilar o idioma intermediário em instruções nativas com a nova configuração. O desenvolvimento de sombreador é exposto por meio da introdução de classes e interfaces ao HLSL. Para uma demonstração, confira o exemplo <a href="https://msdn.microsoft.com/library/Ee416565(v=VS.85).aspx">de Vinculação dinâmica do Sombreador 11</a> disponível por meio do Navegador de Exemplo.<br /> | 
| <span id="Windows_Advanced_Rasterizer__WARP_"></span><span id="windows_advanced_rasterizer__warp_"></span><span id="WINDOWS_ADVANCED_RASTERIZER__WARP_"></span>Windows Advanced Rasterizer (WARP)<br /> | Disponível nesse SDK por meio do Direct3D 11 e, eventualmente, também por meio do Direct3D 10.1, o WARP é um rasterizador de dimensionamento rápido e de vários núcleos que está totalmente em conformidade com o Direct3D 10.1. Utilizar essa tecnologia é tão simples quanto passar o sinalizador D3D_DRIVER_TYPE_WARP na criação do dispositivo.<br /> | 
| <span id="Direct3D_10_and_Direct3D_11_on_Direct3D_9_Hardware__D3D10_Level_9_"></span><span id="direct3d_10_and_direct3d_11_on_direct3d_9_hardware__d3d10_level_9_"></span><span id="DIRECT3D_10_AND_DIRECT3D_11_ON_DIRECT3D_9_HARDWARE__D3D10_LEVEL_9_"></span>Direct3D 10 e Direct3D 11 no hardware direct3D 9 (D3D10 nível 9)<br /> | Disponível nesse SDK por meio do Direct3D 11 e, eventualmente, também por meio do Direct3D 10.1, a API do Direct3D pode direcionar a maioria dos hardwares direct3D 9, bem como hardware Direct3D 10, Direct3D 10.1 e Direct3D 11. Isso é feito fornecendo o mecanismo de Nível de Recurso, que grupos de hardware em seis categorias, dependendo da funcionalidade: 9_1, 9_2, 9_3, 10_0, 10_1 e 11_0. Um cartão só atenderá a um nível de recurso se ele estiver totalmente em conformidade com esse nível e cada nível for um super-conjunto estrito daqueles abaixo dele. A funcionalidade é minimamente emulada para garantir que nenhum desempenho inesperado seja encontrado. Portanto, recursos como Sombreadores de Geometria não estão disponíveis para destinos de nível 9 do Direct3D. <br /> | 
| <span id="Runtime_Binaries"></span><span id="runtime_binaries"></span><span id="RUNTIME_BINARIES"></span>Binários de runtime<br /> | Todos os binários de runtime fornecidos na visualização técnica do Direct3D 11 que estarão disponíveis no Windows 7 e no Windows Vista SP1 são instalados com o SDK e rotulados como componentes "Beta" (ou seja, D3D11_beta.DLL). Todos os componentes rotulados como beta são com tempo. Para criar projetos para avaliar esses novos componentes, você deve vincular às bibliotecas de importação com rótulo beta equivalentes (ou seja, D3D11_beta.lib). Se você tiver uma cópia PDC do Windows 7, os headers, libs e pdbs fornecidos no SDK do Windows com o build serão apropriados para desenvolvimento usando os componentes direct3D 11 fornecidos no Windows 7. Reserve o uso dos headers, libs e pdbs neste SDK para os componentes beta fornecidos aqui.<br /> | 
| <span id="D3DX11"></span><span id="d3dx11"></span>D3DX11<br /> | O D3DX11 atualmente dá suporte ao carregamento de textura, compilação de sombreador, carregamento de dados e funções de thread de trabalho para recursos do Direct3D 11. No futuro, esse componente fornecerá mais das tecnologias disponíveis no D3DX10. A funcionalidade de compilação do sombreador também é fornecida diretamente por meio do componente compilador Direct3D, descrito a seguir.<br /> | 
| <span id="HLSL_and_Direct3D_Compiler"></span><span id="hlsl_and_direct3d_compiler"></span><span id="HLSL_AND_DIRECT3D_COMPILER"></span>Compilador HLSL e Direct3D<br /> | O compilador HLSL tem vários novos recursos para dar suporte às novas tecnologias disponíveis no Direct3D 11. Isso inclui programação orientada a objeto por meio de interfaces e classes, uma sintaxe de indexação direta para cargas de recursos e a palavra-chave "precisa" para garantir que todas as operações executadas com uma variável específica aderam às regras de ponto flutuante estritas. Quase todos os novos recursos linguísticos têm funcionalidade válida em destinos de sombreador existentes. Além de dar suporte a todos os sombreadores Direct3D 9, Direct3D 10, Direct3D 10.1 e Direct3D 11, o compilador HLSL também dá suporte aos destinos especiais necessários para gravar sombreadores para destinos do Direct3D 10 Nível 9. O Compilador D3D agora está diretamente acessível fora do D3DX10 e D3DX11 por meio de D3DCompiler.H e D3DCompiler.lib. Com esses novos arquivos, não é necessário vincular um aplicativo ao D3DX para executar a compilação em runtime, e um aplicativo não é necessário para incluir o compilador se apenas a funcionalidade D3DX for necessária.<br /> | 
| <span id="D3D11_Reference_Rasterizer"></span><span id="d3d11_reference_rasterizer"></span><span id="D3D11_REFERENCE_RASTERIZER"></span>Rasterizador de referência D3D11<br /> | O Rasterizador de Referência fornece uma implementação de rasterização gold-standard para avaliação de recursos do Direct3D 11 ainda não disponíveis no hardware. O Rasterizador de Referência também é fornecido como uma maneira de verificar a precisão de uma implementação de hardware específica para o padrão de rasterização. O rasterizador de referência foi projetado para a correção, não para o desempenho. Para criar um dispositivo de referência, basta passar o sinalizador D3D_DRIVER_TYPE_REFERENCE na criação do dispositivo. <br /> | 
| <span id="D3D11_SDK_Layers"></span><span id="d3d11_sdk_layers"></span><span id="D3D11_SDK_LAYERS"></span>Camadas do SDK D3D11<br /> | As camadas do SDK do Direct3D11 fornecem um mecanismo para acompanhar a operação do runtime do Direct3D 11 durante o desenvolvimento. Atualmente, isso é usado para fornecer informações úteis de depuração, que não só incluem erros de uso inadequado, mas também avisos que recomendam o uso de melhores práticas do runtime e geralmente fornece informações detalhadas e úteis para depuração. É altamente recomendável que a saída de depuração das Camadas do SDK D3D11 seja sempre ativas durante o desenvolvimento e um aplicativo não gere nenhuma saída de depuração durante a execução antes que ela seja lançada ou usada com a OPÇÃO WINDOWS para criação de perfil. A habilitação da camada de depuração é tão simples quanto passar o sinalizador D3D11_CREATE_DEVICE_DEBUG no momento da criação do dispositivo. Os desenvolvedores são incentivados a usar camadas em builds de depuração. As camadas não são recomendadas para uso em builds de perfil ou versão.<br /> | 
| <span id="New_Samples"></span><span id="new_samples"></span><span id="NEW_SAMPLES"></span>Novos exemplos<br /> | Esta versão tem quatro novos exemplos.<br /><ul><li>O exemplo de Vinculação dinâmica do Sombreador <a href="https://msdn.microsoft.com/library/Ee416565(v=VS.85).aspx">11</a> demonstra o uso de interfaces de sombreador do Modelo de Sombreador 5 e suporte ao Direct3D 11 para vincular métodos de interface de sombreador em runtime.</li><li>O exemplo <a href="https://msdn.microsoft.com/library/Ee416569(v=VS.85).aspx">HDRToneMappingCS11</a> demonstra como configurar e executar o Sombreador de Computação (CS para breve posteriormente), que é um dos novos recursos mais interessantes do Direct3D 11. Embora o exemplo utilize apenas o CS para implementar o mapeamento de tom do HDR, o conceito deve se estender facilmente para outros algoritmos de pós-processamento, bem como cálculos mais gerais.</li><li>O <a href="https://msdn.microsoft.com/library/Ee416570(v=VS.85).aspx">exemplo MultithreadedRendering11</a> demonstra como dividir a renderização entre vários threads, com uma sobrecarga muito baixa.</li><li>O novo exemplo <a href="https://msdn.microsoft.com/library/Ee416576(v=VS.85).aspx">de SubD11</a> é muito semelhante ao exemplo subD10 no SDK do DirectX, exceto pelo fato de que ele foi aprimorado para aproveitar três novos estágios de pipeline do Direct3D 11: o sombreador de enxuto, o mosaico e o sombreador de domínio.</li></ul> |
