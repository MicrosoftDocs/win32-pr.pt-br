# <a name="kernel-objects"></a><span data-ttu-id="a5299-103">Objetos kernel</span><span class="sxs-lookup"><span data-stu-id="a5299-103">Kernel Objects</span></span>

<span data-ttu-id="a5299-104">Identificadores de objeto kernel são específicos do processo.</span><span class="sxs-lookup"><span data-stu-id="a5299-104">Kernel object handles are process specific.</span></span> <span data-ttu-id="a5299-105">Ou seja, um processo deve criar o objeto ou abrir um objeto existente para obter um identificador de objeto de kernel.</span><span class="sxs-lookup"><span data-stu-id="a5299-105">That is, a process must either create the object or open an existing object to obtain a kernel object handle.</span></span> <span data-ttu-id="a5299-106">O limite por processo em identificadores de kernel é 2 ^ 24.</span><span class="sxs-lookup"><span data-stu-id="a5299-106">The per-process limit on kernel handles is 2^24.</span></span> <span data-ttu-id="a5299-107">No entanto, os identificadores são armazenados no pool paginável, portanto, o número real de identificadores que você pode criar baseia-se na memória disponível.</span><span class="sxs-lookup"><span data-stu-id="a5299-107">However, handles are stored in the paged pool, so the actual number of handles you can create is based on available memory.</span></span> <span data-ttu-id="a5299-108">O número de identificadores que você pode criar em Windows de 32 bits é significativamente inferior a 2 ^ 24.</span><span class="sxs-lookup"><span data-stu-id="a5299-108">The number of handles that you can create on 32-bit Windows is significantly lower than 2^24.</span></span>

<span data-ttu-id="a5299-109">Qualquer processo pode criar um novo identificador para um objeto de kernel existente (mesmo um criado por outro processo), desde que o processo saiba o nome do objeto e tenha acesso de segurança ao objeto.</span><span class="sxs-lookup"><span data-stu-id="a5299-109">Any process can create a new handle to an existing kernel object (even one created by another indicate the actions that can be granted or denied to a process.</span></span> <span data-ttu-id="a5299-111">Um aplicativo especifica direitos de acesso quando cria um objeto ou Obtém um identificador de objeto existente.</span><span class="sxs-lookup"><span data-stu-id="a5299-111">An application specifies access rights when it creates an object or obtains an existing object handle.</span></span> <span data-ttu-id="a5299-112">Cada tipo de objeto de kernel dá suporte a seu próprio conjunto de direitos de acesso.</span><span class="sxs-lookup"><span data-stu-id="a5299-112">Each type of kernel object supports its own set of access rights.</span></span> <span data-ttu-id="a5299-113">Por exemplo, os identificadores de eventos podem ter acesso de Set ou Wait (ou ambos), os identificadores de Objects](/windows/desktop/SecAuthZ/securable-objects).</span></span>

<span data-ttu-id="a5299-125">Se outro processo souber o nome do evento e tiver acesso de segurança ao objeto, ele poderá criar seu próprio identificador de objeto de evento usando [**OpenEvent**](/windows/desktop/api/synchapi/nf-synchapi-openeventa).</span><span class="sxs-lookup"><span data-stu-id="a5299-125">If another process knows the event name and has security access to the object, it can create its own event object handle by using [**OpenEvent**](/windows/desktop/api/synchapi/nf-synchapi-openeventa).</span></span> <span data-ttu-id="a5299-126">O aplicativo de criação também poderia duplicar um de seus identificadores no mesmo processo ou em outro processo usando a função [**DuplicateHandle**](/windows/win32/api/handleapi/nf-handleapi-duplicatehandle) .</span><span class="sxs-lookup"><span data-stu-id="a5299-126">The creating application could also duplicate one of its handles into the same process or into another process by using the [**DuplicateHandle**](/windows/win32/api/handleapi/nf-handleapi-duplicatehandle) function.</span></span>

<span data-ttu-id="a5299-127">Um objeto permanece na memória contanto que haja pelo menos um identificador de objeto.</span><span class="sxs-lookup"><span data-stu-id="a5299-127">An object remains in memory as long as at least one object handle exists.</span></span> <span data-ttu-id="a5299-128">Na ilustração a seguir, os aplicativos usam a função [**CloseHandle**](/windows/win32/api/handleapi/nf-handleapi-closehandle) para fechar seus identificadores de objeto de evento.</span><span class="sxs-lookup"><span data-stu-id="a5299-128">In the following illustration, the applications use the [**CloseHandle**](/windows/win32/api/handleapi/nf-handleapi-closehandle) function to close their event object handles.</span></span> <span data-ttu-id="a5299-129">Quando não há identificadores de eventos, o sistema remove o objeto da memória, conforme mostrado na ilustração a seguir.</span><span class="sxs-lookup"><span data-stu-id="a5299-129">When there are no event handles, the system removes the object from memory, as shown in the following illustration.</span></span>
