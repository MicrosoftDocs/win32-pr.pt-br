# <a name="predefined-keys"></a>Chaves predefinidas

Um aplicativo deve abrir uma chave para poder adicionar dados ao registro. Para abrir uma chave, um aplicativo deve fornecer um identificador para outra chave no registro que já está aberto. O sistema define as chaves predefinidas que estão sempre abertas. Chaves predefinidas ajudam um aplicativo a navegar no registro e possibilitar o desenvolvimento de ferramentas que permitem a um administrador de sistema manipular categorias de dados. Os aplicativos que adicionam dados ao registro devem sempre funcionar dentro da estrutura de chaves predefinidas, portanto, as ferramentas administrativas podem localizar e usar os novos dados.

Um aplicativo pode usar identificadores para essas chaves como pontos de entrada para o registro. Esses identificadores são válidos para todas as implementações do registro, embora o uso dos identificadores possa variar de plataforma para plataforma. Além disso, outros identificadores predefinidos foram definidos para plataformas específicas. Veja a seguir os identificadores para as chaves predefinidas.

A função [**RegOverridePredefKey**](/windows/desktop/api/Winreg/nf-winreg-regoverridepredefkey) permite mapear uma chave de registro predefinida para uma chave especificada no registro. Por exemplo, um programa de instalação de software poderia remapear uma chave predefinida antes de instalar um componente DLL. Isso permite que o programa de instalação examine facilmente as informações que o procedimento de instalação da DLL grava na chave predefinida.

As funções [**RegDisablePredefinedCache**](/windows/desktop/api/Winreg/nf-winreg-regdisablepredefinedcache) e [**RegDisablePredefinedCacheEx**](/windows/desktop/api/Winreg/nf-winreg-regdisablepredefinedcacheex) desabilitam o cache de identificadores para identificadores de registro predefinidos. Os serviços que usam a representação devem chamar **RegDisablePredefinedCacheEx** antes de usar identificadores de registro predefinidos.

Os identificadores predefinidos não são thread-safe. Fechar um identificador predefinido em um thread afeta quaisquer outros threads que estejam usando o identificador.
