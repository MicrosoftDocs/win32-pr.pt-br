# <a name="terms-used-in-the-print-schema"></a>Termos usados no esquema de impressão

Este tópico não é atual. Para obter as informações mais atuais, consulte a [Especificação de Esquema de Impressão](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

A lista a seguir contém termos específicos para o esquema de impressão e tecnologias relacionadas.



| Termo                                                                                                                                                                 | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>nome do atributo<br/>                                                                          | Um nome de atributo é o token no lado esquerdo do sinal de igual em um atributo XML padrão. A representação padrão de um atributo XML é *o nome do atributo*=" valor do *atributo*". Embora semelhante, o termo "nome do atributo" não deve ser confundido com o atributo name definido na Estrutura de Esquema de Impressão. O atributo name definido no Print Schema Framework é uma instância específica de um nome de atributo. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>Candidato<br/>                                                                                         | Opção. Qualquer Opção que pertence à lista que está sendo avaliada durante o processo de pontuação. Consulte os termos, a opção de pontuação ou mapeamento e a opção de referência.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Configuração<br/>                                             | Descreve um dos estados em que um dispositivo pode assumir ou ser colocado. O dispositivo executa sua tarefa de forma diferente em cada estado. Há uma diferença distinta e identificável entre o comportamento do dispositivo em cada um desses estados. Por exemplo, em um estado, a mídia é retirada de uma bandeja e termina em outra bandeja. Uma alteração em uma ou em ambas as bandejas determina um estado diferente.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Restrição<br/>                                                         | Informações necessárias para determinar a validade de uma configuração. Uma restrição geralmente está no formato: FeatureA definido como Option1 não é compatível com FeatureB definido como Option2.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>elementos correspondentes<br/>                                                  | Dois elementos que têm o mesmo local em relação a um par especificado de elementos de origem. Os elementos de origem são frequentemente dois elementos Option que estão sendo comparados.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>Padrão<br/>                                                                                               | Opção. A Opção que é designada pelo esquema de impressão ou pelo provedor PrintCapabilities como a Opção atribuída a um Recurso quando o PrintTicket não contém uma configuração explícita para esse Recurso. Cada recurso tem uma opção padrão designada.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>atributo de configuração do dispositivo<br/>                          | Um atributo de dispositivo que contribui para a definição da configuração do dispositivo. Um atributo de configuração de dispositivo é representado na Estrutura de Esquema de Impressão por constructos de recurso/opção ou parâmetro. Observe que o atributo usado nesse contexto não tem relação com atributos XML ou com elementos de Atributo definidos pela Estrutura de Esquema de Impressão.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>tipo de elemento<br/>                                                                                | O tipo de cada elemento XML definido na Estrutura de Esquema de Impressão é identificado por sua marca de elemento XML.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>elementos em comum<br/>                                                              | Consulte elementos correspondentes.<br/> Recurso. Um constructo definido pelo Print Schema Framework que representa um atributo de configuração de dispositivo específico. Cada instância de recurso representa um atributo de dispositivo. Uma instância de recurso é definida definindo explicitamente cada um de seus estados.<br/> Opção de identidade. A Opção que não tem nenhum efeito observável quando selecionada; na verdade, um "no-op". O resultado é sempre o mesmo, independentemente do número de vezes que essa Opção está selecionada.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>nome da instância<br/>                                                                             | Um atributo XML que identifica a instância específica de um tipo de elemento. Consulte atributo de nome.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>cliente de interface<br/>                                                                    | A entidade que consome documentos PrintCapabilities e que constrói e consome PrintTickets. Exemplos comuns de clientes de interface incluem módulos de interface do usuário e módulos de renderização.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>provedor de interface<br/>                                                              | A entidade que implementa e expõe a funcionalidade por meio de uma interface . No Esquema de Impressão, isso equivale à criação de documentos PrintCapabilities e à validação de PrintTickets. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>local de um elemento<br/>                                                  | A posição de uma instância específica de um elemento dentro da hierarquia de elementos. O local é definido pelo elemento específico e por todos os ancestrais desse elemento. Em outras palavras, location é o equivalente a um caminho totalmente qualificado em um sistema de arquivos. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>atributo name<br/>                                                                          | A designação dada ao atributo XML com um nome de atributo de "name". Esse atributo aparece na maioria dos tipos de elemento definidos na Estrutura de Esquema de Impressão. Ele identifica e define instâncias de cada tipo de elemento. Os dois termos a seguir referem-se a atributos de nome: a Regra de Exclusividade De Irmão e a Convenção de Namespace.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Convenção de namespace<br/>                 | Declara que os atributos de nome definidos pelo esquema, bem como qualquer objeto definido pelo esquema, existem em um namespace dedicado, no namespace público e que os atributos de nome definidos de forma privada devem existir em um namespace diferente, um que pode ser identificado como pertencente à parte que o define.<br/> Opção. Um constructo da Estrutura de Esquema de Impressão usado para representar estados em que cada Recurso pode assumir ou ser colocado. Por exemplo, se o Recurso representar o tamanho da mídia, os valores típicos de Opção poderão ser Letra, Legal, A4, entre outros.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Parâmetro<br/>                                                             | O Esquema de Impressão dá suporte à capacidade de descrever dados parametrizados para fins de entrada do usuário. Há três tipos de elemento print schema Framework que reproduzem funções na entrada paramétrica: ParameterDef, ParameterInit e ParameterRef. Um elemento ParameterDef caracteriza um parâmetro (seu intervalo de valores, o tipo de seus dados e outras características). Um elemento ParameterInit é usado para inicializar um parâmetro. Um elemento ParameterRef é usado para referenciar uma instância específica de um elemento ParameterInit.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>Parametrizada<br/>                                                                             | Opção. Uma instância option que contém pelo menos um elemento ScoredProperty que, por sua vez, contém um elemento ParameterRef em vez de um elemento Value.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>perfeitamente correspondente<br/>                                                              | Opções. Duas instâncias option são consideradas correspondentes se, para cada elemento ScoredProperty em uma instância option, há uma instância ScoredProperty correspondente na outra instância option e ambos os elementos Value das instâncias ScoredProperty correspondentes são idênticos.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>Esquema PrintCapabilities<br/> | O esquema que define a sintaxe e a semântica usadas por um documento PrintCapabilities. O esquema PrintCapabilities e o esquema PrintTicket são subconjunto do esquema de impressão. Consulte também Imprimir Estrutura de Esquema.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>Esquema PrintTicket<br/>                         | O esquema que define a sintaxe e a semântica usadas pelo PrintTicket. O esquema PrintTicket e o esquema PrintCapabilities são subconjunto do esquema de impressão. Consulte também Imprimir Estrutura de Esquema.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Estrutura de esquema de impressão<br/>         | O esquema que define a sintaxe e a semântica usadas pelas tecnologias de Esquema de Impressão. Por exemplo, a Estrutura de Esquema de Impressão define uma coleção de tipos de elemento e seus usos, mas não enumera todas as instâncias de elemento.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Palavras-chave de esquema de impressão<br/>             | O esquema que define instâncias individuais dos elementos Property, Feature e Option que são comumente usados para descrever dispositivos e formatação de trabalho. Esse esquema define uma linguagem comum que torna os documentos PrintCapabilities e PrintTickets portáteis. Também conhecido como esquema público.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>definidos de forma privada<br/>                                                                 | Todas as instâncias de elementos que aparecem em um documento PrintCapabilities ou PrintTicket que não são definidas explicitamente nas Palavras-chave de esquema de impressão. Essas instâncias de elemento devem ser qualificadas explicitamente com um namespace privado por fornecedor.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>Referência<br/>                                                                                         | Opção. A Opção à qual cada opção candidata é comparada. Uma opção de referência normalmente é originada do PrintTicket. Consulte os termos, a opção de pontuação ou mapeamento e a opção candidate.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>nível raiz<br/>                                                                                      | O nível raiz é a base da estrutura hierárquica da Estrutura de Esquema de Impressão. Um elemento de nível raiz é definido como um filho de um elemento PrintCapabilities ou um elemento PrintTicket.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>prefixo de scoping<br/>                                                                          | Um rótulo textual pré-anexado a uma palavra-chave de esquema para fornecer escopo contextual. Os prefixos de definição de esquema de impressão permitidos são Trabalho, Documento e Página. Por exemplo, o prefixo de scoping para JobInputBin é 'Job'.<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>Scoredproperty<br/>                                         | Um constructo do Print Schema Framework usado para representar as qualidades que distinguem ou diferenciam uma instância option de outra. Um exemplo é a diferença entre letras e papel de tamanho legal.<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>pontuação ou mapeamento<br/>                                                              | Opção. O processo de seleção em uma lista da instância de Opção (candidato) que melhor corresponde a uma determinada instância de Opção (referência). Os termos intimamente relacionados são Opção candidata e opção de referência. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Regra de exclusividade de irmão<br/>     | Declara que todos os irmãos do mesmo tipo de elemento devem ter atributos de nome exclusivos. Alguns tipos de elemento são isentos dessa regra. Essas exceções são definidas na Estrutura de Esquema de Impressão.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>valor único versus multivalor<br/>                    | Propriedade. Uma Propriedade cujo Valor é invariável (não depende da configuração) é considerada com valor único. Essa propriedade tem apenas um único Valor em qualquer configuração possível. Uma Propriedade que depende da configuração é considerada com vários valores.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>sufixo/sufixos<br/>                                                                       | Uma palavra-chave de esquema, que representa o nome do recurso ou do parâmetro. Por exemplo, o sufixo para JobInputBin é 'InputBin'.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Validação<br/>                                                         | O processo de verificar se um PrintTicket está em conformidade com o esquema e representa uma configuração que pode ser executada por um dispositivo específico.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Indefinido\_<br/>                                                   | Os dados específicos representados aqui são um espaço reservado para o driver, dispositivo ou aplicativo específico a ser produzido. Ele não transmite que os dados devem ser uma cadeia de caracteres.<br/>                                                                                                                                                                                                                                                                                                                                                            |
