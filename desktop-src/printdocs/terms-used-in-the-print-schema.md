# <a name="terms-used-in-the-print-schema"></a>Termos usados no esquema de impressão

Este tópico não é atual. Para obter as informações mais atuais, consulte a [especificação do esquema de impressão](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

A lista a seguir contém termos que são específicos para o esquema de impressão e tecnologias relacionadas.



| Termo                                                                                                                                                                 | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>nome do atributo<br/>                                                                          | Um nome de atributo é o token no lado esquerdo do sinal de igual em um atributo XML padrão. A representação padrão de um atributo XML é *nome do atributo*= "*valor do atributo*". Embora seja semelhante, o termo "nome do atributo" não deve ser confundido com o atributo Name definido na estrutura do esquema de impressão. O atributo de nome definido na estrutura de esquema de impressão é uma instância específica de um nome de atributo. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>Candidate<br/>                                                                                         | Option. Qualquer opção que pertença à lista que está sendo avaliada durante o processo de pontuação. Consulte a opção termos, pontuação ou mapeamento e a opção referência.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Configuração<br/>                                             | Descreve um dos Estados que um dispositivo pode assumir ou ser colocado. O dispositivo executa sua tarefa de forma diferente em cada Estado. Há uma diferença distinta e identificável entre o comportamento do dispositivo em cada um desses Estados. Por exemplo, em um estado, a mídia é retirada de uma bandeja e termina em outra bandeja. Uma alteração em uma ou ambas as bandejas determina um estado diferente.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Constraint<br/>                                                         | Informações necessárias para determinar a validade de uma configuração. Uma restrição é geralmente do formato: o recurso de definido como opção 1 não é compatível com FeatureB definido como opção 2.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>elementos correspondentes<br/>                                                  | Dois elementos que têm o mesmo local em relação a um par especificado de elementos de origem. Os elementos de origem são frequentemente dois elementos de opção que estão sendo comparados.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>os<br/>                                                                                               | Option. A opção que é designada pelo esquema de impressão ou pelo provedor de PrintCapabilities como a opção que é atribuída a um recurso quando o PrintTicket não contém uma configuração explícita para esse recurso. Cada recurso tem uma opção padrão designada.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>atributo de configuração do dispositivo<br/>                          | Um atributo de dispositivo que contribui para a definição da configuração do dispositivo. Um atributo de configuração de dispositivo é representado na estrutura de esquema de impressão por recurso/opção ou construções de parâmetro. Observe que o atributo usado neste contexto não tem relação com os atributos XML ou para imprimir elementos de atributo definidos pelo Framework Schema.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>tipo de elemento<br/>                                                                                | O tipo de cada elemento XML definido na estrutura de esquema de impressão é identificado por sua marca de elemento XML.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>elementos em comum<br/>                                                              | Consulte os elementos correspondentes.<br/> Recurso. Uma construção definida pela estrutura de esquema de impressão que representa um atributo de configuração de dispositivo específico. Cada instância de recurso representa um atributo de dispositivo. Uma instância de recurso é definida por meio da definição explícita de cada um dos seus Estados.<br/> Opção de identidade. A opção que não tem nenhum efeito observável quando selecionada; em vigor, um "não op". O resultado é sempre o mesmo, independentemente do número de vezes que essa opção é selecionada.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>nome da instância<br/>                                                                             | Um atributo XML que identifica a instância específica de um tipo de elemento. Consulte atributo de nome.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>cliente de interface<br/>                                                                    | A entidade que consome documentos de PrintCapabilities e que constrói e consome PrintTickets. Exemplos comuns de clientes de interface incluem módulos de interface do usuário e módulos de renderização.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>provedor de interface<br/>                                                              | A entidade que implementa e expõe a funcionalidade por meio de uma interface. No esquema de impressão, esse valor é para a criação de documentos de PrintCapabilities e a validação de PrintTickets. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>local de um elemento<br/>                                                  | A posição de uma instância específica de um elemento dentro da hierarquia do elemento. O local é definido pelo elemento específico e por todos os ancestrais desse elemento. Em outras palavras, Location é o equivalente a um caminho totalmente qualificado em um sistema de arquivos. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>atributo de nome<br/>                                                                          | A designação fornecida ao atributo XML com um nome de atributo de "Name". Esse atributo aparece na maioria dos tipos de elementos definidos na estrutura de esquema de impressão. Ele identifica e define instâncias de cada tipo de elemento. Os dois termos a seguir se referem a atributos de nome: a regra de exclusividade irmã e a Convenção de namespace.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Convenção de namespace<br/>                 | Declara que os atributos de nome definidos pelo esquema, bem como qualquer objeto definido pelo esquema, existem em um namespace dedicado, no namespace público e que os atributos de nome definidos de forma privada devem existir em um namespace diferente, um que possa ser identificado como pertencente à entidade de definição.<br/> Option. Uma construção de estrutura de esquema de impressão usada para representar Estados que cada recurso pode assumir ou ser colocado. Por exemplo, se o recurso representa o tamanho da mídia, os valores de opção típicos podem ser carta, ofício, A4, entre outros.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Meter<br/>                                                             | O esquema de impressão dá suporte à capacidade de descrever dados com parâmetros para fins de entrada do usuário. Há três tipos de elementos de estrutura de esquema de impressão que desempenham funções na entrada paramétrica: ParameterDef, ParameterInit e ParameterRef. Um elemento ParameterDef caracteriza um parâmetro (seu intervalo de valores, o tipo de seus dados e outras características). Um elemento ParameterInit é usado para inicializar um parâmetro. Um elemento ParameterRef é usado para fazer referência a uma instância específica de um elemento ParameterInit.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>com parâmetros<br/>                                                                             | Option. Uma instância de opção que contém pelo menos um elemento ScoredProperty que, por sua vez, contém um elemento ParameterRef em vez de um elemento Value.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>correspondência perfeita<br/>                                                              | Opções. Duas instâncias de opção são consideradas perfeitamente se para cada elemento ScoredProperty em uma instância de opção, há uma instância ScoredProperty correspondente na outra instância de opção e ambos os elementos de valor das instâncias ScoredProperty correspondentes são idênticos.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>Esquema de PrintCapabilities<br/> | O esquema que define a sintaxe e a semântica usada por um documento de PrintCapabilities. O esquema de PrintCapabilities e o esquema PrintTicket são subconjuntos do esquema de impressão. Consulte também estrutura de esquema de impressão.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>Esquema PrintTicket<br/>                         | O esquema que define a sintaxe e a semântica usada pelo PrintTicket. O esquema PrintTicket e o esquema PrintCapabilities são subconjuntos do esquema de impressão. Consulte também estrutura de esquema de impressão.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Estrutura de esquema de impressão<br/>         | O esquema que define a sintaxe e a semântica usada pelas tecnologias de esquema de impressão. Por exemplo, a estrutura de esquema de impressão define uma coleção de tipos de elementos e seus usos, mas não enumera todas as instâncias de elemento.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Imprimir palavras-chave de esquema<br/>             | O esquema que define instâncias individuais de propriedade, recurso e elementos de opção que são comumente usados para descrever dispositivos e formatação de trabalho. Esse esquema define uma linguagem comum que torna os documentos de PrintCapabilities e os PrintTickets em formato portátil. Também conhecido como esquema público.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>definido de modo privado<br/>                                                                 | Todas as instâncias de elementos que aparecem em um documento de PrintCapabilities ou PrintTicket que não estão explicitamente definidos nas palavras-chave do esquema de impressão. Essas instâncias de elemento devem ser qualificadas explicitamente com um namespace privado por fornecedor.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>referência<br/>                                                                                         | Option. A opção em que cada opção de candidato é comparada. Normalmente, uma opção de referência origina-se do PrintTicket. Consulte a opção termos, pontuação ou mapeamento e a opção candidato.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>nível raiz<br/>                                                                                      | O nível raiz é a base da estrutura hierárquica da estrutura de esquema de impressão. Um elemento de nível raiz é definido como um filho de um elemento PrintCapabilities ou um elemento PrintTicket.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>prefixo de escopo<br/>                                                                          | Um rótulo textual previamente acrescentado a uma palavra-chave de esquema para fornecer escopo contextual. Os prefixos de escopo permitidos pelo esquema de impressão são trabalho, Doc e página. Por exemplo, o prefixo de escopo para JobInputBin é ' Job '.<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>ScoredProperty<br/>                                         | Uma construção de estrutura de esquema de impressão usada para representar as qualidades que distinguem ou diferenciam uma instância de opção de outra. Um exemplo é a diferença entre letras e papéis de tamanho ofício.<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>Pontuação ou mapeamento<br/>                                                              | Option. O processo de selecionar de uma lista a instância de opção (candidato) que melhor corresponde a uma determinada instância de opção (referência). Os termos bem relacionados são opções de candidato e opção de referência. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Regra de exclusividade de irmãos<br/>     | Declara que todos os irmãos do mesmo tipo de elemento devem ter atributos de nome exclusivos. Alguns tipos de elementos são isentos dessa regra. Essas exceções são definidas na estrutura de esquema de impressão.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>com valor único versus com múltiplos valores<br/>                    | Propriedade. Uma propriedade cujo valor é invariável (não depende da configuração) é considerada de valor único. Essa propriedade tem apenas um único valor em qualquer configuração possível. Uma propriedade que depende da configuração é considerada com valores variados.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>sufixo/sufixos<br/>                                                                       | Uma palavra-chave de esquema, que representa o nome do recurso ou do parâmetro. Por exemplo, o sufixo para JobInputBin é ' InputBin'.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Confirmação<br/>                                                         | O processo de verificar se um PrintTicket está em conformidade com o esquema e representa uma configuração que pode ser executada por um dispositivo específico.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Indefinido\_<br/>                                                   | Os dados específicos que são representados aqui são um espaço reservado para o driver específico, o dispositivo ou o aplicativo a ser produzido. Ele não transmite que os dados devem ser uma cadeia de caracteres.<br/>                                                                                                                                                                                                                                                                                                                                                            |
