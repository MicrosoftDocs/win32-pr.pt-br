# <a name="purpose-of-the-printticket"></a>Finalidade do PrintTicket

Este tópico não é atual. Para obter as informações mais atuais, consulte a [especificação do esquema de impressão](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

A principal finalidade do PrintTicket é expressar as configurações de dispositivo. Ao contrário do documento PrintCapabilities, que define todas as configurações possíveis que o dispositivo pode assumir (o espaço de configuração do dispositivo), a finalidade do PrintTicket é definir uma única configuração específica que é aplicada ao objeto de trabalho (neste caso, a página) que está sendo processada. Para promover a portabilidade e a consistência das configurações, as palavras-chave específicas e a hierarquia apresentadas na seção de palavras-chave do esquema de impressão são usadas pelos PrintCapabilities e pelas tecnologias PrintTicket. Além disso, a [estrutura de esquema de impressão](print-schema-framework.md) define as construções básicas de ambas as tecnologias, para promover a comunicação não ambígua, aberta e extensível de formatação de trabalho e PrintCapabilities.
