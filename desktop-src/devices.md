# <a name="devices"></a>Dispositivos

## <a name="in-this-section"></a>Nesta seção

| Tópico | Descrição |
|---|---|
| [Acesso ao dispositivo](deviceaccess/device-access-broker-api-portal.md)<br/> | As APIs de acesso do dispositivo permitem que os desenvolvedores gravem aplicativos da Windows Store para dispositivos personalizados. A API aplica uma política para conceder a um aplicativo da Windows Store o nível apropriado de acesso a um dispositivo e, se o acesso for concedido, fornece métodos para enviar códigos de controle para operar o dispositivo.<br/> |
| [API AllJoyn](/previous-versions/windows/desktop/alljoyn/alljoyn-api-portal)<br/> | O AllJoyn é uma estrutura do tipo DCOM e um protocolo para fazer chamadas de método remotas e/ou enviar sinais unidirecionais entre aplicativos em um barramento distribuído. Ele se destina a ser usado principalmente para cenários de tipo de Internet das Coisas. <br/> |
| [API de ponto de serviço](/uwp/api/Windows.Devices.PointOfService)<br/> | A API de ponto de serviço permite que os desenvolvedores de aplicativos acessem dispositivos periféricos de POS (ponto de serviço). O namespace fornece uma interface neutra de fornecedor para acessar dispositivos de POS de um aplicativo UWP.<br/> |
| [Recursos de comunicação](/windows/desktop/DevIO/communications-resources)<br/>           | Um recurso de comunicação é um dispositivo físico ou lógico que fornece um único fluxo de dados bidirecional e assíncrono. Portas seriais, portas paralelas, máquinas de fax e modems são exemplos de recursos de comunicação. Para cada recurso de comunicação, há um provedor de serviços, que consiste em uma biblioteca ou um driver, que permite que os aplicativos acessem o recurso. <br/> |
| [Gerenciamento de dispositivo](/windows/desktop/DevIO/device-management)<br/>                         | O gerenciamento de dispositivos fornece uma maneira de notificar de forma uniforme todos os aplicativos e componentes do sistema de alterações que podem afetar a operação ou o acesso aos recursos. Os aplicativos e o sistema usam e processam eventos de dispositivo para aproveitar os novos recursos quando eles se tornam disponíveis e para evitar a perda de dados quando os recursos existentes se tornam indisponíveis. <br/>                 |
| [Armazenamento Avançado](/previous-versions/windows/desktop/enstor/enhanced-storage-portal)<br/>                  | A API de armazenamento aprimorada permite a autenticação de ponta a ponta consistente para dispositivos de armazenamento pessoal que diferem no fator forma. <br/>                                                                                                                                                                                                                                                 |
| [Descoberta de função](/previous-versions/windows/desktop/fundisc/fd-portal)<br/>                                 | A descoberta de função fornece uma interface programática uniforme para a enumeração de recursos do sistema, como dispositivos de hardware, sejam eles locais ou conectados por meio de uma rede.<br/>                                                                                                                                                                                                 |
| [PnP-X](/previous-versions/windows/desktop/fundisc/pnp-x)<br/>                                                  | Extensões de Plug and Play (PnP-X) permitem que um sistema de computador descubra dispositivos em rede e instale-os no sistema local usando Plug and Play (PnP). <br/>                                                                                                                                                                                                                     |
| [API de localização](/windows/desktop/LocationAPI/windows-location-api-portal)<br/>             | A API de localização ajuda a simplificar a programação com reconhecimento de local, fornecendo uma maneira padrão de recuperar dados sobre o local do usuário e padronizar formatos para relatórios de dados de localização. A API de localização manipula automaticamente as transições entre os provedores de dados de localização e sempre escolhe o provedor mais preciso para a situação atual.<br/>                                 |
| [API de sensor](/windows/desktop/SensorsAPI/portal)<br/>                                     | A API do sensor permite que os aplicativos obtenham e usem dados de sensores de forma padronizada.<br/>                                                                                                                                                                                                                                                                                     |
| [APIs de UPnP](/windows/desktop/UPnP/universal-plug-and-play-start-page)<br/>                | A estrutura UPnP permite a rede dinâmica de dispositivos inteligentes, computadores sem fio e PCs. <br/>                                                                                                                                                                                                                                                                            |
| [Serviços Web em dispositivos](/windows/desktop/WsdApi/wsd-portal)<br/>                           | A API do Microsoft Web Services on Devices (WSDAPI) dá suporte à implementação de dispositivos e serviços controlados pelo cliente e aos hosts de dispositivo em conformidade com o perfil de dispositivos para serviços Web (DPWS). <br/>                                                                                                                                                                          |
| [SDK do Windows Media Gerenciador de Dispositivos 11](./wmdm/windows-media-device-manager-11-sdk.md)<br/> | Aplicativos ou componentes criados no Windows Media Gerenciador de Dispositivos têm uma API consistente para se comunicar com uma ampla gama de dispositivos, incluindo o MTP (Media Transfer Protocol), o MSC (classe de armazenamento em massa), o RAPI e outros dispositivos criados nas versões mais recentes e anteriores da tecnologia Windows Media.<br/>                                                                    |
| [Windows Mixed Reality](https://msdn.microsoft.com/library/Mt844799(v=WIN.10).aspx)<br/>          | O Windows Mixed Reality permite a criação de uma experiência de realidade misturada para o Microsoft HoloLens e outros headsets de imersão. Para obter mais informações, consulte o [centro de desenvolvimento da realidade misturada](https://developer.microsoft.com/windows/mixed-reality).<br/>                                                                                                                                    |
| [Dispositivos portáteis do Windows](./windows-portable-devices.md)<br/>            | Os dispositivos portáteis do Windows (WPD) permitem que os computadores se comuniquem com dispositivos de mídia e armazenamento anexados. O WPD fornece uma maneira flexível e robusta para que os computadores se comuniquem com players de música, dispositivos de armazenamento, celulares, câmeras e muitos outros tipos de dispositivos conectados.<br/>                                                                                               |
