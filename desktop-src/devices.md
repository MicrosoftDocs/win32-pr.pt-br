# <a name="devices"></a>Dispositivos

## <a name="in-this-section"></a>Nesta seção

| Tópico | Descrição |
|---|---|
| [Acesso ao dispositivo](deviceaccess/device-access-broker-api-portal.md)<br/> | As APIs de Acesso ao Dispositivo permitem que os desenvolvedores escrevam Windows da Store para dispositivos personalizados. A API aplica uma política para conceder a um aplicativo Windows Store o nível apropriado de acesso a um dispositivo e, se o acesso for concedido, fornece métodos para enviar códigos de controle para operar o dispositivo.<br/> |
| [AllJoyn API](/previous-versions/windows/desktop/alljoyn/alljoyn-api-portal)<br/> | AllJoyn é uma estrutura e protocolo do tipo DCOM para fazer chamadas de método remotamente e/ou enviar sinais avuário entre aplicativos em um barramento distribuído. Ele destina-se a ser usado principalmente para cenários Internet das Coisas tipo. <br/> |
| [API de Ponto de Serviço](/uwp/api/Windows.Devices.PointOfService)<br/> | A API de Ponto de Serviço permite que os desenvolvedores de aplicativos acessem dispositivos periféricos de PONTO de Serviço (POS). O namespace fornece uma interface neutra do fornecedor para acessar dispositivos POS de um aplicativo UWP.<br/> |
| [Recursos de comunicação](/windows/desktop/DevIO/communications-resources)<br/>           | Um recurso de comunicação é um dispositivo físico ou lógico que fornece um único fluxo de dados bidirecional e assíncrono. Portas seriais, portas paralelas, máquinas de fax e modems são exemplos de recursos de comunicação. Para cada recurso de comunicação, há um provedor de serviços, que consiste em uma biblioteca ou driver, que permite que os aplicativos acessem o recurso. <br/> |
| [Gerenciamento de Dispositivos](/windows/desktop/DevIO/device-management)<br/>                         | O gerenciamento de dispositivos fornece uma maneira de notificar uniformemente todos os aplicativos e componentes do sistema de alterações que podem afetar sua operação ou o acesso aos recursos. Os aplicativos e o sistema usam e processam eventos de dispositivo para aproveitar novos recursos quando eles ficam disponíveis e para evitar a perda de dados quando os recursos existentes ficam indisponíveis. <br/>                 |
| [Armazenamento Avançado](/previous-versions/windows/desktop/enstor/enhanced-storage-portal)<br/>                  | A API de Armazenamento aprimorada permite autenticação consistente de ponta a ponta para dispositivos de armazenamento pessoal que diferem no fator forma. <br/>                                                                                                                                                                                                                                                 |
| [Descoberta de funções](/previous-versions/windows/desktop/fundisc/fd-portal)<br/>                                 | A Descoberta de Funções fornece uma interface programática uniforme para enumerar recursos do sistema, como dispositivos de hardware, sejam eles locais ou conectados por meio de uma rede.<br/>                                                                                                                                                                                                 |
| [PnP-X](/previous-versions/windows/desktop/fundisc/pnp-x)<br/>                                                  | Plug and Play Extensões (PnP-X) permitem que um sistema de computador descubra dispositivos em rede e instale-os no sistema local usando Plug and Play (PnP). <br/>                                                                                                                                                                                                                     |
| [API de localização](/windows/desktop/LocationAPI/windows-location-api-portal)<br/>             | A API de Localização ajuda a simplificar a programação com conhecimento de localização, fornecendo uma maneira padrão de recuperar dados sobre a localização do usuário e padronizar formatos para relatórios de dados de localização. A API de Localização lida automaticamente com transições entre provedores de dados de localização e sempre escolhe o provedor mais preciso para a situação atual.<br/>                                 |
| [API de sensor](/windows/desktop/SensorsAPI/portal)<br/>                                     | A API do Sensor permite que os aplicativos recebam e usem dados de sensores de maneira padronizada.<br/>                                                                                                                                                                                                                                                                                     |
| [APIs de UPnP](/windows/desktop/UPnP/universal-plug-and-play-start-page)<br/>                | A estrutura UPnP permite a rede dinâmica de dispositivos inteligentes, dispositivos sem fio e computadores. <br/>                                                                                                                                                                                                                                                                            |
| [Serviços Web em dispositivos](/windows/desktop/WsdApi/wsd-portal)<br/>                           | A API WSDAPI (Serviços Web da Microsoft em Dispositivos) dá suporte à implementação de dispositivos e serviços controlados pelo cliente e hosts de dispositivo em conformidade com o DPWS (Perfil de Dispositivos para Serviços Web). <br/>                                                                                                                                                                          |
| [Windows SDK Gerenciador de Dispositivos 11](./wmdm/windows-media-device-manager-11-sdk.md)<br/> | Os aplicativos ou componentes do Windows Media Gerenciador de Dispositivos têm uma API consistente para se comunicar com uma ampla variedade de dispositivos, incluindo o Protocolo de Transferência de Mídia (MTP), a MSC (Classe Armazenamento em Massa), oWLI e outros dispositivos com base nas versões mais recentes e anteriores da tecnologia Windows Media.<br/>                                                                    |
| [Windows Mixed Reality](https://msdn.microsoft.com/library/Mt844799(v=WIN.10).aspx)<br/>          | Windows Mixed Reality permite a criação da experiência de realidade misturada para Microsoft HoloLens e outros headsets imersivos. Para obter mais informações, consulte o guia [realidade misturada Centro de Desenvolvimento](https://developer.microsoft.com/windows/mixed-reality).<br/>                                                                                                                                    |
| [Windows Dispositivos portáteis](./windows-portable-devices.md)<br/>            | Windows O WPD (Dispositivos Portáteis) permite que os computadores se comuniquem com dispositivos de mídia e armazenamento anexados. O WPD fornece uma maneira flexível e robusta para computadores se comunicarem com players de música, dispositivos de armazenamento, celulares, câmeras e muitos outros tipos de dispositivos conectados.<br/>                                                                                               |
