# <a name="s-network-monitor"></a>S (Monitor de Rede)

<dl> <dt>

<span id="_netmon_sap_gly"></span><span id="_NETMON_SAP_GLY"></span>**SAP**
</dt> <dd>

Consulte ponto de acesso de serviço.

</dd> <dt>

<span id="_netmon_server_message_block_gly"></span><span id="_NETMON_SERVER_MESSAGE_BLOCK_GLY"></span>**protocolo SMB (Server Message Block)**
</dt> <dd>

Um protocolo de nível de arquivo que contém primitivos e é usado para operações como **abrir**, **fechar**, **ler** e **gravar**.

</dd> <dt>

<span id="_netmon_service_access_point_gly"></span><span id="_NETMON_SERVICE_ACCESS_POINT_GLY"></span>**ponto de acesso a serviços (SAP)**
</dt> <dd>

Um identificador de quadro.

</dd> <dt>

<span id="_netmon_session_gly"></span><span id="_NETMON_SESSION_GLY"></span>**sessão**
</dt> <dd>

Um conjunto de pacotes de dados que uma estação envia para outra estação durante uma captura. Uma sessão é um intercâmbio unidirecional de pacotes no qual cada sessão tem um proprietário (a estação que envia os pacotes de dados) e um parceiro (a estação que recebe os pacotes de dados). Consulte também informações da sessão \_ .

</dd> <dt>

<span id="_netmon_session_information_gly"></span><span id="_NETMON_SESSION_INFORMATION_GLY"></span>**informações da sessão**
</dt> <dd>

Informações que descrevem uma sessão. Monitor de Rede salva informações de sessão durante a captura de dados. As informações de sessão são conhecidas como estatísticas de conversa e podem ser recuperadas por uma chamada para o método **GetConversationStatistics** . Consulte também [*\_ estatísticas*](c.md)de sessão, [*captura*](c.md)e conversa.

</dd> <dt>

<span id="_netmon_smb_gly"></span><span id="_NETMON_SMB_GLY"></span>**SMBs**
</dt> <dd>

Confira o bloco de mensagens do servidor.

</dd> <dt>

<span id="_netmon_source_address_gly"></span><span id="_NETMON_SOURCE_ADDRESS_GLY"></span>**Endereço de origem**
</dt> <dd>

Um número hexadecimal exclusivo que identifica o computador do qual os dados são enviados.

</dd> <dt>

<span id="_netmon_special_blobs_gly"></span><span id="_NETMON_SPECIAL_BLOBS_GLY"></span>**BLOB especial**
</dt> <dd>

Um NPP (provedor de pacotes de rede) ou um BLOB (objeto binário grande) de filtro que contém informações complementares que outros BLOBs não fornecem. Consulte também [*provedor de pacotes de rede*](n.md) e [*blob*](b.md).

</dd> <dt>

<span id="_netmon_station_gly"></span><span id="_NETMON_STATION_GLY"></span>**Estação**
</dt> <dd>

Uma NIC específica instalada em um computador que está capturando dados. Consulte também informações de estação.

</dd> <dt>

<span id="_netmon_station_information_gly"></span><span id="_NETMON_STATION_INFORMATION_GLY"></span>**informações da estação**
</dt> <dd>

Informações que descrevem uma estação. Monitor de Rede salva as informações da estação durante uma captura de dados. As informações da estação também são conhecidas como estatísticas de conversa que você pode recuperar por uma chamada para o método **GetConversationStatistics** . Confira também estatísticas de estação e de [*conversa*](c.md).

</dd> <dt>

<span id="_netmon_summary_property_gly"></span><span id="_NETMON_SUMMARY_PROPERTY_GLY"></span>**Propriedade Summary**
</dt> <dd>

Uma propriedade definida pelo analisador usada para agrupar as propriedades de um protocolo para exibição na interface do usuário do Monitor de Rede. Uma estrutura **PROPERTYINFO** define uma propriedade Summary da mesma maneira que outras propriedades definidas pelo analisador são definidas. A propriedade Summary normalmente é a primeira definição de propriedade anexada a um quadro quando Monitor de Rede chama a função **attachproperties** .

</dd> <dt>

<span id="_netmon_summary_pane_gly"></span><span id="_NETMON_SUMMARY_PANE_GLY"></span>**painel de resumo**
</dt> <dd>

O painel em uma janela do Visualizador de quadro Monitor de Rede que lista os quadros contidos em um arquivo de captura. Para cada quadro, o painel Resumo lista o número do quadro, a hora em que o quadro é capturado, o endereço de origem e de destino do quadro, os protocolos usados para enviar o quadro e uma breve descrição do quadro. Se um endereço bruto tiver um endereço IP ou IPX/XNS associado, esse endereço será exibido em **outro** endereço de origem e em outro endereço de **horário de verão**. O tipo de endereço (IP ou IPX/XNS) aparece em **outro endereço de tipo**.

</dd> </dl>
