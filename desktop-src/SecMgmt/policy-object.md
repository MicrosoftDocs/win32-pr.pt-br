# <a name="policy-object"></a><span data-ttu-id="5528f-103">Objeto de política</span><span class="sxs-lookup"><span data-stu-id="5528f-103">Policy Object</span></span>

<span data-ttu-id="5528f-104">O objeto de **política** é usado para controlar o acesso ao banco de dados da [*autoridade de segurança local*](/windows/desktop/SecGloss/l-gly) (LSA) e contém informações que se aplicam ao sistema inteiro ou estabelecem padrões para o sistema.</span><span class="sxs-lookup"><span data-stu-id="5528f-104">The **Policy** object is used to control access to the [*Local Security Authority*](/windows/desktop/SecGloss/l-gly) (LSA) database and contains information that applies to the entire system or establishes defaults for the system.</span></span> <span data-ttu-id="5528f-105">Cada sistema tem apenas um objeto de **política** .</span><span class="sxs-lookup"><span data-stu-id="5528f-105">Each system has only one **Policy** object.</span></span> <span data-ttu-id="5528f-106">Esse objeto de **política** é criado pela LSA quando o sistema é iniciado e os aplicativos não podem criá-lo ou destruí-lo.</span><span class="sxs-lookup"><span data-stu-id="5528f-106">This **Policy** object is created by the LSA when the system starts up, and applications cannot create or destroy it.</span></span>

<span data-ttu-id="5528f-107">As informações armazenadas em um objeto de **política** incluem:</span><span class="sxs-lookup"><span data-stu-id="5528f-107">The information stored in a **Policy** object includes:</span></span>

-   <span data-ttu-id="5528f-108">Cota de memória padrão do sistema.</span><span class="sxs-lookup"><span data-stu-id="5528f-108">System default memory quota.</span></span> <span data-ttu-id="5528f-109">A menos que especificado de outra forma, cada usuário que fizer logon no sistema receberá essa cota de memória.</span><span class="sxs-lookup"><span data-stu-id="5528f-109">Unless otherwise specified, each user logging on to the system will be assigned this memory quota.</span></span> <span data-ttu-id="5528f-110">Cotas de memória especiais podem ser atribuídas a indivíduos ou membros de grupos ou grupos locais por meio de um objeto de [**conta**](account-object.md) .</span><span class="sxs-lookup"><span data-stu-id="5528f-110">Special memory quotas can be assigned to individuals or members of groups or local groups through an [**Account**](account-object.md) object.</span></span>
-   <span data-ttu-id="5528f-111">Requisitos de auditoria de segurança em todo o sistema.</span><span class="sxs-lookup"><span data-stu-id="5528f-111">System-wide security auditing requirements.</span></span>
-   <span data-ttu-id="5528f-112">O nome e o SID do domínio da conta deste sistema.</span><span class="sxs-lookup"><span data-stu-id="5528f-112">The name and SID of the account domain of this system.</span></span>
-   <span data-ttu-id="5528f-113">Informações sobre o domínio primário deste sistema.</span><span class="sxs-lookup"><span data-stu-id="5528f-113">Information about the primary domain of this system.</span></span> <span data-ttu-id="5528f-114">Essas informações incluem o nome e o SID do domínio primário, o nome da conta no domínio primário que deve ser usado para solicitações de autenticação, traduções de nome e SID e a obtenção dos nomes dos controladores de domínio no domínio.</span><span class="sxs-lookup"><span data-stu-id="5528f-114">This information includes the name and SID of the primary domain, the name of the account within the primary domain that is to be used for authentication requests, name and SID translations, and obtaining the names of domain controllers within the domain.</span></span> <span data-ttu-id="5528f-115">Esses nomes podem estar desatualizados e devem ser levados apenas em uma dica.</span><span class="sxs-lookup"><span data-stu-id="5528f-115">These names may be out of date and should be taken only as a hint.</span></span> <span data-ttu-id="5528f-116">A ordem dessa lista é considerada significativa e será mantida.</span><span class="sxs-lookup"><span data-stu-id="5528f-116">The order of this list is assumed to be significant and will be maintained.</span></span> <span data-ttu-id="5528f-117">Isso permite, por exemplo, o primeiro nome na lista para representar o último controlador de domínio primário conhecido.</span><span class="sxs-lookup"><span data-stu-id="5528f-117">This allows, for example, the first name in the list to represent the last-known primary domain controller.</span></span>
-   <span data-ttu-id="5528f-118">Informações sobre se a LSA contém a cópia mestra das informações de política ou uma réplica.</span><span class="sxs-lookup"><span data-stu-id="5528f-118">Information about whether the LSA holds the master copy of the policy information or a replica.</span></span> <span data-ttu-id="5528f-119">Somente parte das informações de política é replicada; o restante é estabelecido em uma base por sistema.</span><span class="sxs-lookup"><span data-stu-id="5528f-119">Only part of the policy information is replicated; the remainder is established on a per-system basis.</span></span>

<span data-ttu-id="5528f-120">Os campos **AccountDomain** e **PrimaryDomain** do objeto de **política** são usados para finalidades diferentes, dependendo do tipo de relações de confiança e do sistema:</span><span class="sxs-lookup"><span data-stu-id="5528f-120">The **AccountDomain** and **PrimaryDomain** fields of the **Policy** object are used for different purposes depending upon the type of system and trust relationships:</span></span>

-   <span data-ttu-id="5528f-121">Em um sistema que não tem um domínio primário, o campo **AccountDomain** contém o nome e o SID do domínio de conta local do sistema, que é o mesmo que o nome do computador.</span><span class="sxs-lookup"><span data-stu-id="5528f-121">On a system that does not have a primary domain, the **AccountDomain** field contains the name and SID of the system's local account domain, which is the same as the computer name.</span></span> <span data-ttu-id="5528f-122">O campo **PrimaryDomain** contém o nome do grupo de trabalho do qual este computador é membro.</span><span class="sxs-lookup"><span data-stu-id="5528f-122">The **PrimaryDomain** field contains the name of the workgroup this machine is a member of.</span></span> <span data-ttu-id="5528f-123">Os objetos [**TrustedDomain**](trusteddomain-object.md) são ignorados com uma exceção – não pode haver um objeto **TrustedDomain** com o mesmo nome que o grupo de trabalho, pois ele será exibido como se fosse o domínio primário do computador.</span><span class="sxs-lookup"><span data-stu-id="5528f-123">[**TrustedDomain**](trusteddomain-object.md) objects are ignored with one exception—there cannot be a **TrustedDomain** object with the same name as the workgroup because it will appear as though it is the machine's primary domain.</span></span>
-   <span data-ttu-id="5528f-124">Em um sistema que tem um domínio primário, o campo **AccountDomain** identifica o nome e o SID do domínio da conta local, como antes.</span><span class="sxs-lookup"><span data-stu-id="5528f-124">On a system that has a primary domain, the **AccountDomain** field identifies the name and SID of the local account domain, as before.</span></span> <span data-ttu-id="5528f-125">No entanto, o campo **PrimaryDomain** contém o nome e o SID do domínio primário para o sistema.</span><span class="sxs-lookup"><span data-stu-id="5528f-125">However, the **PrimaryDomain** field contains the name and SID of the primary domain for the system.</span></span> <span data-ttu-id="5528f-126">Além disso, deve haver um objeto [**TrustedDomain**](trusteddomain-object.md) com o nome e o Sid identificados no campo **PrimaryDomain** .</span><span class="sxs-lookup"><span data-stu-id="5528f-126">Additionally, there should be a [**TrustedDomain**](trusteddomain-object.md) object with the name and SID identified in the **PrimaryDomain** field.</span></span> <span data-ttu-id="5528f-127">Esse objeto **TrustedDomain** contém as informações de conta e servidor necessárias para estabelecer um canal seguro para um controlador de domínio no domínio primário.</span><span class="sxs-lookup"><span data-stu-id="5528f-127">This **TrustedDomain** object contains the account and server information necessary to establish a secure channel to a domain controller in the primary domain.</span></span> <span data-ttu-id="5528f-128">Todos os outros objetos **TrustedDomain** são ignorados.</span><span class="sxs-lookup"><span data-stu-id="5528f-128">Any other **TrustedDomain** objects are ignored.</span></span>
-   <span data-ttu-id="5528f-129">Em controladores de domínio, o campo **AccountDomain** identifica o domínio de conta local para o sistema; no entanto, o nome da conta é atribuído pelo usuário em vez de ser um nome conhecido.</span><span class="sxs-lookup"><span data-stu-id="5528f-129">On domain controllers, the **AccountDomain** field identifies the local account domain for the system; however, the account name is user assigned rather than being a well-known name.</span></span> <span data-ttu-id="5528f-130">Como o domínio primário é o mesmo que o domínio da conta, o campo **PrimaryDomain** deve conter o mesmo valor que o campo **AccountDomain** .</span><span class="sxs-lookup"><span data-stu-id="5528f-130">Since the primary domain is the same as the account domain, the **PrimaryDomain** field must contain the same value as the **AccountDomain** field.</span></span> <span data-ttu-id="5528f-131">Além disso, espera-se que todos os objetos [**TrustedDomain**](trusteddomain-object.md) sejam válidos e representem relações de confiança com outros domínios.</span><span class="sxs-lookup"><span data-stu-id="5528f-131">Furthermore, all [**TrustedDomain**](trusteddomain-object.md) objects are expected to be valid and represent trust relationships with other domains.</span></span> <span data-ttu-id="5528f-132">Se o sistema não confiar em nenhum outro domínio, não deverá haver nenhum objeto **TrustedDomain** .</span><span class="sxs-lookup"><span data-stu-id="5528f-132">If the system does not trust any other domains, there should not be any **TrustedDomain** objects.</span></span>
