# <a name="other-resources"></a>Outros recursos

Esta seção contém uma lista de livros e recursos recomendados relacionados ao design de interface do usuário. (Esses livros e recursos podem não estar disponíveis em alguns idiomas e países.)

-   [Lista de Leitura](#reading-list)
-   [Organizações](#organizations)

## <a name="reading-list"></a>Lista de Leitura

<dl> <dt>

<span id="_Windows_Interface_Guidelines_for_Software_Design"></span><span id="_windows_interface_guidelines_for_software_design"></span><span id="_WINDOWS_INTERFACE_GUIDELINES_FOR_SOFTWARE_DESIGN"></span>*Diretrizes de interface do Windows para o design de software*
</dt> <dd>

(Microsoft Press, 1995; ISBN 1-55615-679-0)

Este livro explica todas as principais regras da interface do usuário a serem seguidas, com vários exemplos de como dispor caixas de diálogo, assistentes, barras de ferramentas e menus de contexto.

</dd> <dt>

<span id="Usability_Engineering_by_Jakob_Nielsen_"></span><span id="usability_engineering_by_jakob_nielsen_"></span><span id="USABILITY_ENGINEERING_BY_JAKOB_NIELSEN_"></span>*Engenharia de usabilidade* por Jakob Nielsen 
</dt> <dd>

(AP Professional, 1994; ISBN 0125184069)

Este livro se concentra nas técnicas para medir efetivamente a usabilidade de uma interface. Ele explica muitas técnicas de teste diferentes para comparar diferentes ideias de design, incluindo sugestões para manter os custos de teste de usabilidade inativos. Este livro aborda técnicas de análise que podem ser aplicadas a sites, páginas da Web ou qualquer tipo de software. É ótimo para as fases iniciais de aprendizado sobre os usuários e as fases intermediárias da iteração e dos testes de usabilidade.

</dd> <dt>

<span id="Design_of_Everyday_Things_by_Don_Norman"></span><span id="design_of_everyday_things_by_don_norman"></span><span id="DESIGN_OF_EVERYDAY_THINGS_BY_DON_NORMAN"></span>*Design de coisas diárias* por Don Norman
</dt> <dd>

(Moeda/Doubleday, 1988; ISBN: 0385267746)

Este livro aborda os princípios de psicológica em funcionamento que tornam as interfaces tão difíceis de usar. Recomendado para qualquer pessoa interessada em interfaces do usuário e que queira uma ampla compreensão dos problemas envolvidos. Os exemplos incluem dispositivos de cozinha, portas de escritório e máquinas de venda.

</dd> <dt>

<span id="About_Face_by_Alan_Cooper_"></span><span id="about_face_by_alan_cooper_"></span><span id="ABOUT_FACE_BY_ALAN_COOPER_"></span>*Sobre a face* de Alan Cooper 
</dt> <dd>

(Livros de IDG em todo o mundo, 1995; ISBN 1568843224)

Ao pensar sobre as soluções de interface do usuário, uma rápida inversão por esse livro deve ajudar a tornar as ideias fluindo.

</dd> <dt>

<span id="The_Elements_of_Friendly_Software_Design_by_Paul_Heckel_"></span><span id="the_elements_of_friendly_software_design_by_paul_heckel_"></span><span id="THE_ELEMENTS_OF_FRIENDLY_SOFTWARE_DESIGN_BY_PAUL_HECKEL_"></span>*Os elementos de design de software amigável* por Paul Heckel 
</dt> <dd>

(SYBEX, 1991; ISBN 0-89588-768-1)

Este livro continua sendo uma soma de pontos de referência de várias abordagens ao design de interface.

</dd> <dt>

<span id="Microsoft_Windows_User_Experience"></span><span id="microsoft_windows_user_experience"></span><span id="MICROSOFT_WINDOWS_USER_EXPERIENCE"></span>*Experiência do usuário do Microsoft Windows*
</dt> <dd>

(Microsoft Press, 1999; ISBN 0735605661)

As diretrizes oficiais da Microsoft para a criação de interfaces de usuário bem projetadas, visualmente e funcionalmente consistentes para aplicativos.

</dd> <dt>

<span id="The_Case_Against_User_Interface_Consistency_by__Jonathan_Grudin"></span><span id="the_case_against_user_interface_consistency_by__jonathan_grudin"></span><span id="THE_CASE_AGAINST_USER_INTERFACE_CONSISTENCY_BY__JONATHAN_GRUDIN"></span>*O caso em relação à consistência da interface do usuário* por Jonathan Grudin
</dt> <dd>

(Comunicações do ACM, outubro de 1989)

</dd> <dt>

<span id="Contextual_Design__Defining_Customer-Centered_Systems_by_Hugh_Beyer_and_Karen_Holtzblatt"></span><span id="contextual_design__defining_customer-centered_systems_by_hugh_beyer_and_karen_holtzblatt"></span><span id="CONTEXTUAL_DESIGN__DEFINING_CUSTOMER-CENTERED_SYSTEMS_BY_HUGH_BEYER_AND_KAREN_HOLTZBLATT"></span>*Design contextual: definindo sistemas de Customer-Centered* de Hugh Beyer e Karen Holtzblatt
</dt> <dd>

Dumas e Janice C. redish
</dt> <dd>
