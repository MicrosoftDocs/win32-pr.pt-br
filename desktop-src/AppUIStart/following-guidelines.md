# <a name="following-user-interface-guidelines"></a>Diretrizes de interface do usuário a seguir

Este tópico define quais diretrizes de interface do usuário são e descreve por que as diretrizes a seguir podem não ser a melhor estratégia.

-   [O que são diretrizes de interface do usuário?](#what-are-ui-guidelines)
-   [Não confie exclusivamente nas diretrizes](#do-not-rely-on-guidelines-exclusively)

## <a name="what-are-ui-guidelines"></a>O que são diretrizes de interface do usuário?

As diretrizes de interface do usuário são coleções de recomendações que os designers e desenvolvedores seguem ao criar a interface do usuário para aplicativos. As diretrizes podem incluir:

-   Princípios de design geral derivados por meio de pesquisa.

    Esses princípios podem incluir a expressão de uma filosofia de design fundamental, suposições sobre comportamento humano, metodologia de design e conceitos incorporados na interface.

-   Padrões.

    A maioria das plataformas de software principais, se não todas, publicou as diretrizes para o design da interface do usuário. Um exemplo é a experiência do usuário do Microsoft Windows, que é subtitulada as diretrizes oficiais para desenvolvedores e designers de interface do usuário.

-   Regras locais ou guia de estilo.

    Muitas empresas e organizações individuais têm seu próprio conjunto de regras de interface do usuário documentadas ou estilos de design de interface que os desenvolvedores da empresa usam. Isso é comum em grandes empresas, especialmente, em que um pacote de aplicativos é criado internamente.

## <a name="do-not-rely-on-guidelines-exclusively"></a>Não confie exclusivamente nas diretrizes

Como um designer de aplicativos, as diretrizes de interface do usuário a seguir ajudam a garantir que o produto permita que os usuários apliquem habilidades que já aprenderam a tarefas comuns e aprendam novas tarefas com mais facilidade. No entanto, você não pode confiar exclusivamente nas diretrizes para garantir a usabilidade do seu produto. As diretrizes da interface do usuário geralmente são muito gerais. Ainda assim, é tão geral que dificulta a aplicação. Quando você está tentando tomar decisões específicas dentro do contexto do seu produto, um conjunto geral de diretrizes pode não fornecer a você informações suficientes para tomar uma decisão. Um exemplo é quando você está tentando decidir entre o método A e o método B de apresentar informações em uma caixa de diálogo.

Por outro lado, as diretrizes podem ser muito específicas. Por exemplo, uma diretriz pode especificar não ter mais de sete itens em um menu. No entanto, a adição de menus adicionais pode ser mais confusa para os usuários do que ter mais de sete opções em um menu. Além disso, as diretrizes podem entrar em conflito umas com as outras. Por exemplo, uma diretriz pode especificar não ter mais de sete itens em um menu e outra pode especificar itens semelhantes agrupados em menus. Qual diretriz tem precedência?

No entanto, ao criar um aplicativo, a consistência Visual incentivada pelas diretrizes da interface do usuário pode ser útil. Considere a consistência que você encontra no Microsoft Word e no Microsoft Excel. As interfaces de usuário nesses produtos de software são muito semelhantes nos elementos básicos, como menus, barras de ferramentas e posicionamento de botões em caixas de diálogo (ou seja, a interface de nível de superfície). Além disso, eles são consistentes em como eles manipulam muitas tarefas comuns: formatação de texto, salvamento de arquivos e assim por diante. A consistência nesses e em outros elementos pode facilitar para os usuários a transferência de habilidades ao aprender diferentes aplicativos. Diretrizes de interface do usuário específicas podem ajudar a manter a consistência entre diferentes produtos.

Tudo se resume ao contexto: uma interface do usuário precisa ser projetada para usuários, metas e tarefas específicas. As diretrizes podem ser um ponto de partida razoável, mas são apenas um ponto de partida. O valor na consistência da interface do usuário está em aprendizado efetivo, facilitando a transferência de conhecimento de outro produto sem interferir na facilidade de uso.
