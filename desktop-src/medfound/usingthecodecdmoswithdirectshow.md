## <a name="two-pass-encoding"></a><span data-ttu-id="cf1f2-126">Codificação de duas passagens</span><span class="sxs-lookup"><span data-stu-id="cf1f2-126">Two-pass Encoding</span></span>

<span data-ttu-id="cf1f2-127">Os modos de codificação de duas passagens exigem uma abordagem mais complexa para criação e streaming de grafo para impedir que o DMO libere seu conteúdo da primeira passagem antes de iniciar a segunda passagem.</span><span class="sxs-lookup"><span data-stu-id="cf1f2-127">The two-pass encoding modes require a more complex approach to graph building and streaming in order to prevent the DMO from flushing its contents from the first pass before beginning the second pass.</span></span> <span data-ttu-id="cf1f2-128">Em codificação de dois passos, é necessário executar o grafo uma vez para que o DMO possa executar sua análise de pré-processamento dos dados do arquivo e, em seguida, retroceder o grafo e executá-lo novamente para que o DMO possa fazer a codificação real.</span><span class="sxs-lookup"><span data-stu-id="cf1f2-128">In two-pass encoding, it is necessary to run the graph once so the DMO can perform its preprocessing analysis of the file data, and then rewind the graph and run it again so that the DMO can do the actual encoding.</span></span>
