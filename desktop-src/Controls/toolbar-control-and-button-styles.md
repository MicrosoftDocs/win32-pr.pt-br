# <a name="toolbar-control-and-button-styles"></a>Controle da barra de ferramentas e estilos de botão

Os estilos de janela a seguir são específicos para barras de ferramentas. Eles são combinados com outros estilos de janela quando a barra de ferramentas é criada.

**Observação** Para os controles comuns [versão 6, 0](common-control-versions.md), se um [estilo visual](themes-overview.md) estiver sendo usado com a barra de ferramentas, os botões sempre serão transparentes, independentemente da configuração de estilo. Caso contrário, o comportamento de transparência é normal, conforme indicado pelo uso do \_ estilo transparente TBSTYLE ou TBSTYLE \_ .

> [!Note]  
> O Comctl32.dll versão 6 não é redistribuível, mas está incluído no Windows. Para usar Comctl32.dll versão 6, especifique-o em um manifesto. Para obter mais informações sobre manifestos, consulte [habilitando estilos visuais](cookbook-overview.md).

**Observação:** Um botão da barra de ferramentas pode ter uma combinação dos estilos a seguir. Para especificar um estilo de botão, defina os sinalizadores apropriados no membro **fsStyle** da estrutura [**TBBUTTON**](/windows/desktop/api/Commctrl/ns-commctrl-tbbutton) do botão. Nem todos os estilos podem ser combinados.

Para o Shell versão 4,72 e anterior, os sinalizadores de estilo de janela e de botão tinham o formato TBSTYLE \_ xxx. Se você estiver Compilando um aplicativo com a [versão 4,72](common-control-versions.md) ou anterior do commctrl. h, deverá usar os \_ sinalizadores de estilo do botão TBSTYLE xxx. Para a versão 5,80 e posterior, todos os estilos de botão têm o formato BTNS \_ xxx. Todos os valores de TBSTYLE \_ xxx têm valores equivalentes de BTNS \_ xxx, com o mesmo significado e valor numérico. Para sua conveniência, ambos os formulários são fornecidos na tabela a seguir.
