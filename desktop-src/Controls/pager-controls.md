# <a name="about-pager-controls"></a>Sobre os controles de pager

Um *controle de pager* é um contêiner de janela que é usado com uma janela que não tem área de exibição suficiente para mostrar todo o seu conteúdo. O controle de pager permite que o usuário Role até a área da janela que não está em exibição no momento.

-   [Sobre os controles de pager](#about-pager-controls)

## <a name="about-pager-controls"></a>Sobre os controles de pager

O Microsoft Internet Explorer versão 4,0 (commctrl.dll versão 4,71) apresenta o controle de pager. Esse controle é útil em situações em que uma janela não tem área suficiente para exibir uma janela filho. Por exemplo, se seu aplicativo tiver uma barra de ferramentas que não seja grande o suficiente para mostrar todos os seus itens, você poderá atribuir a barra de ferramentas a um controle de pager e os usuários poderão rolar para a esquerda ou para a direita para acessar todos os itens. Você também pode criar controles de pager que rolam verticalmente.

Uma janela atribuída ao controle de pager é conhecida como a *janela contida*.

A captura de tela a seguir mostra uma barra de ferramentas contida dentro de um controle de pager. O controle de pager é exibido em vermelho para mostrar quais áreas do controle estão visíveis.

![captura de tela de uma janela estreita com uma barra de ferramentas de exemplo dentro de um controle de pager](images/pager.jpg)

> [!Note]  
> O controle de pager é implementado na versão 4,71 e posterior do Comctl32.dll.
