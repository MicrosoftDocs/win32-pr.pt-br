# <a name="parts-and-states"></a>Partes e Estados

Este tópico descreve as partes e os Estados que você usa para alterar a aparência de controles quando os estilos visuais são habilitados. Windows desenha determinadas partes para controles e janelas e essas partes são divididas em partes pai e filho. A maioria das partes tem um estado, por exemplo, desabilitado, ativo ou selecionado. Ao criar um controle, você decide a quais partes do controle aplicar estilos visuais. Você divide o controle em uma parte principal e zero ou mais partes filhas nomeadas. Exemplos de partes são áreas de texto, formas desenhadas, imagens e linhas. Para cada parte, você pode definir uma ou mais imagens de plano de fundo em um único arquivo de bitmap. Os diferentes planos de fundo geralmente são associados a diferentes Estados do controle, mas não precisam ser. Por exemplo, se o controle estiver desabilitado, um plano de fundo sombreado poderá ser usado. Em tempo de execução, a imagem apropriada pode ser selecionada usando um valor *istateid* como o índice de 1 base para a imagem correta.

> [!Note]  
> Grande parte da API de tema permite que você use um valor de (0, 0) para a parte e a ID de estado na lista de parâmetros, mais comumente ao desenhar o plano de fundo usando [**DrawThemeBackground**](/windows/desktop/api/Uxtheme/nf-uxtheme-drawthemebackground). Isso indica que os dados de um elemento de nível superior serão usados. A parte 0 pode ser interpretada como "dados que são os mesmos para todas as partes deste elemento" e estado 0 como "dados que são os mesmos para todos os Estados desta parte". Use *stateId*= 0 somente quando não houver Estados definidos.
