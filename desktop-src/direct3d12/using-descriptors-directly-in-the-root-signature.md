# <a name="using-descriptors-directly-in-the-root-signature"></a><span data-ttu-id="8383c-103">Usando descritores diretamente na assinatura raiz</span><span class="sxs-lookup"><span data-stu-id="8383c-103">Using descriptors directly in the root signature</span></span>

<span data-ttu-id="8383c-104">Para evitar a necessidade de passar por um heap de descritor, você pode colocar um descritor diretamente na assinatura raiz.</span><span class="sxs-lookup"><span data-stu-id="8383c-104">To avoid the need to go through a descriptor heap, you can put a descriptor directly into the root signature.</span></span> <span data-ttu-id="8383c-105">Esses descritores ocupam muito espaço na assinatura raiz (consulte [limites de assinatura raiz](/windows/win32/direct3d12/root-signature-limits)), portanto, recomendamos que você os use com moderação.</span><span class="sxs-lookup"><span data-stu-id="8383c-105">These descriptors take up a lot of space in the root signature (see [Root signature limits](/windows/win32/direct3d12/root-signature-limits)), so we recommend that you use them sparingly.</span></span>

<span data-ttu-id="8383c-106">Um exemplo de uso seria colocar no layout raiz uma CBV (exibição de buffer constante) que está sendo alterada por empate.</span><span class="sxs-lookup"><span data-stu-id="8383c-106">An example usage would be to place in the root layout a constant buffer view (CBV) that is changing per draw.</span></span> <span data-ttu-id="8383c-107">Isso é para que o espaço de heap do descritor não precise ser alocado pelo aplicativo por empate (e salva o apontador de uma tabela de descritores no novo local no heap do descritor).</span><span class="sxs-lookup"><span data-stu-id="8383c-107">That's so that descriptor heap space doesn't have to be allocated by the application per draw (and saves pointing a descriptor table at the new location in the descriptor heap).</span></span> <span data-ttu-id="8383c-108">Ao colocar algo na assinatura de raiz, o aplicativo está apenas entregando a responsabilidade de controle de versão para o driver; Mas essa é a infraestrutura que os drivers já têm.</span><span class="sxs-lookup"><span data-stu-id="8383c-108">By putting something in the root signature, the application is merely handing the versioning responsibility to the driver; but that's infrastructure that drivers already have.</span></span>

<span data-ttu-id="8383c-109">Para a renderização que usa extremamente poucos recursos, o uso de tabela/heap de descritor pode não ser necessário se todos os descritores necessários puderem ser colocados diretamente na assinatura raiz.</span><span class="sxs-lookup"><span data-stu-id="8383c-109">For rendering that uses extremely few resources, descriptor table/heap use may not be needed at all if all of the needed descriptors can be placed directly in the root signature.</span></span>

<span data-ttu-id="8383c-110">Esses são os únicos tipos de descritores com suporte na assinatura raiz.</span><span class="sxs-lookup"><span data-stu-id="8383c-110">These are the only types of descriptors supported in the root signature.</span></span>

- <span data-ttu-id="8383c-111">CBVs (exibição de buffer constante).</span><span class="sxs-lookup"><span data-stu-id="8383c-111">Constant buffer view (CBVs).</span></span>
- <span data-ttu-id="8383c-112">Modos de exibição de recurso do sombreador (SRVs)/exibições de acesso não ordenado (UAVs) de recursos de buffer em que a conversão de formato não é necessária (buffers não tipados).</span><span class="sxs-lookup"><span data-stu-id="8383c-112">Shader resource views (SRVs) / unordered access views (UAVs) of buffer resources where format conversion is not required (untyped buffers).</span></span> <span data-ttu-id="8383c-113">Alguns exemplos de buffers não tipados que podem ser associados a descritores raiz incluem `StructuredBuffer<type>` , `RWStructuredBuffer<type>` `ByteAddressBuffer` e `RWByteAddressBuffer` .</span><span class="sxs-lookup"><span data-stu-id="8383c-113">Some examples of untyped buffers that can be bound with root descriptors include `StructuredBuffer<type>`, `RWStructuredBuffer<type>`, `ByteAddressBuffer` and `RWByteAddressBuffer`.</span></span> <span data-ttu-id="8383c-114">Buffers tipados como `Buffer<uint>` e `Buffer<float2>` não podem.</span><span class="sxs-lookup"><span data-stu-id="8383c-114">Typed buffers such as `Buffer<uint>` and `Buffer<float2>` can't.</span></span>

<span data-ttu-id="8383c-116">Um UAV na raiz não pode ter contadores associados a ele.</span><span class="sxs-lookup"><span data-stu-id="8383c-116">A UAV in the root cannot have counters associated with it.</span></span> <span data-ttu-id="8383c-117">Os descritores na assinatura raiz aparecem cada um dos descritores individuais separados &mdash; que não podem ser indexados dinamicamente.</span><span class="sxs-lookup"><span data-stu-id="8383c-117">Descriptors in the root signature appear each as individual separate descriptors&mdash;they cannot be dynamically indexed.</span></span>

``` syntax

   uint foo;
   float bar[2];
   int moo;
};

```

<span data-ttu-id="8383c-118">No exemplo acima, `mySceneData` não pode ser declarado como uma matriz, como em `cbuffer mySceneData[2]` se ele for mapeado para um descritor na assinatura raiz.</span><span class="sxs-lookup"><span data-stu-id="8383c-118">In the above example, `mySceneData` can't be declared as an array, as in `cbuffer mySceneData[2]` if it is going to be mapped onto a descriptor in the root signature.</span></span> <span data-ttu-id="8383c-119">Isso ocorre porque a indexação entre descritores não tem suporte na assinatura raiz.</span><span class="sxs-lookup"><span data-stu-id="8383c-119">That's because indexing across descriptors isn't supported in the root signature.</span></span> <span data-ttu-id="8383c-120">Se desejar, você pode definir buffers de constantes individuais separados e defini-los como uma entrada separada na assinatura raiz.</span><span class="sxs-lookup"><span data-stu-id="8383c-120">If desired, you can define separate individual constant buffers, and define them each as a separate entry in the root signature.</span></span> <span data-ttu-id="8383c-121">Observe que, dentro de `mySceneData` acima, há uma matriz `bar[2]` .</span><span class="sxs-lookup"><span data-stu-id="8383c-121">Note that within `mySceneData` above, there is an array `bar[2]`.</span></span> <span data-ttu-id="8383c-122">A indexação dinâmica dentro do buffer de constantes é válida &mdash;
