# <a name="vml-introduction"></a>Introdução ao VML

Este tópico descreve o VML, um recurso que foi preterido a partir Windows Internet Explorer 9. As páginas da Web e os aplicativos que dependem do VML devem ser migrados para o SVG ou outros padrões amplamente suportados.

> [!Note]  
> A partir de dezembro de 2011, este tópico foi arquivado. Como resultado, ele não é mais mantido ativamente. Para obter mais informações, consulte [Conteúdo arquivado.](/previous-versions/windows/internet-explorer/ie-developer/) Para obter informações, recomendações e diretrizes sobre a versão atual do Windows Internet Explorer, consulte [Internet Explorer Developer Center.](https://msdn.microsoft.com/ie/)

 

Este documento fornece uma referência detalhada completa à implementação do linguagem VML (VML) que é fornecido com Microsoft Office 2000. Outras implementações podem variar. Consulte a [especificação de VML](https://www.w3.org/TR/NOTE-datetime.html) para obter mais informações sobre o VML como um padrão.

O VML é definido como um conjunto de elementos XML e os atributos correspondentes para cada elemento. Como o **elemento Shape** é o bloco de construção básico do VML, clique [aqui](shape-element--vml.md) para iniciar a referência.

Observe que essa referência contém vários exemplos e demonstrações. Você deve ter o Microsoft Internet Explorer 5.0 ou superior instalado para exibir o VML ao vivo.

Além das informações sobre como usar elementos e atributos como marcas XML que estendem HTML, essa referência contém sintaxe e exemplos que mostram como usar o script e os recursos Modelo de Objeto do Documento do Internet Explorer 5 ou superior. O uso de script com VML permite que você crie elementos "em tempo real" e manipule atributos em tempo real.

Este exemplo nesta referência usa VBScript e Javascript. Se você usar uma linguagem de script diferente da mostrada em um exemplo, deverá corresponder ao caso de todos os nomes de elemento e atributo. A referência fornece sintaxe de script que está correta para linguagens que não são sensíveis a minúsculas, como JavaScript. Em caso de dúvida sobre o caso de caractere correto, use um navegador de objeto (como OLEView) para explorar o VGX.DLL para determinar o caso exato de um elemento ou atributo. Observe que quando o VML é usado como marcas XML, a sensibilidade a casos depende do tipo de documento do documento subjacente. Se você estiver usando um modo estrito ! DOCTYPE, elementos e atributos serão sensíveis a minúsculas.
