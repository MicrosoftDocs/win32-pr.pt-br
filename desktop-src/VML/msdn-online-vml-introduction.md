# <a name="vml-introduction"></a>Introdução à VML

Este tópico descreve a VML, um recurso que foi preterido a partir do Windows Internet Explorer 9. As páginas da Web e os aplicativos que dependem de VML devem ser migrados para o SVG ou outros padrões amplamente suportados.

> [!Note]  
> A partir de dezembro de 2011, este tópico foi arquivado. Como resultado, ele não é mais mantido ativamente. Para obter mais informações, consulte [conteúdo arquivado](/previous-versions/windows/internet-explorer/ie-developer/). Para obter informações, recomendações e orientações sobre a versão atual do Windows Internet Explorer, consulte [Internet Explorer Developer Center](https://msdn.microsoft.com/ie/).

 

Este documento fornece uma referência detalhada completa para a implementação do linguagem VML (VML) que é enviado com o Microsoft Office 2000. Outras implementações podem variar. Consulte a [especificação da VML](https://www.w3.org/TR/NOTE-datetime.html) para obter mais informações sobre a VML como padrão.

A VML é definida como um conjunto de elementos XML e os atributos correspondentes para cada elemento. Como o elemento **Shape** é o bloco de construção básico de VML, clique [aqui](shape-element--vml.md) para iniciar a referência.

Observe que essa referência contém vários exemplos e demonstrações. Você deve ter o Microsoft Internet Explorer 5,0 ou superior instalado para exibir a VML ao vivo.

Além das informações sobre como usar elementos e atributos como marcas XML que estendem HTML, essa referência contém sintaxe e exemplos que mostram como usar os recursos de script e Modelo de Objeto do Documento do Internet Explorer 5 ou superior. O uso de script com VML permite que você crie elementos "imediatamente" e manipule atributos em tempo real.

Esses exemplos nesta referência usam VBScript e JavaScript. Se usar uma linguagem de script diferente daquela mostrada em um exemplo, você deverá corresponder ao caso de todos os nomes de elementos e atributos. A referência fornece a sintaxe de script correta para linguagens que diferenciam maiúsculas de minúsculas, como JavaScript. Se estiver em dúvida sobre o caso de caractere correto, use um pesquisador de objetos (como OLEView) para explorar o VGX.DLL para determinar o caso exato de um elemento ou atributo. Observe que quando a VML é usada como marcações XML, a diferenciação de maiúsculas e minúsculas depende do tipo de documento do documento subjacente. Se você estiver usando um modo estrito! DOCTYPE, elementos e atributos serão sensíveis a maiúsculas e minúsculas.
