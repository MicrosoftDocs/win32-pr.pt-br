# <a name="xml-layer-overview"></a>Visão geral da camada XML

A API XML no WWSAPI é baseada nos objetos [leitor de XML](xml-reader.md) e [gravador XML](xml-writer.md) , que permitem a leitura ou gravação de documentos XML em uma forma somente para frente. A camada XML concede ao aplicativo acesso completo e controle sobre o conteúdo das mensagens.

## <a name="encoding"></a>Codificação

A API XML dá suporte a documentos codificados como:

-   Text (UTF-8, UTF-16LE, UTF-16BE)
-   Binário
-   MTOM

## <a name="storage"></a>Armazenamento

A API XML dá suporte ao processamento de documentos armazenados como:

-   Um buffer na memória de bytes codificados
-   Um fluxo
-   Um [buffer XML](xml-buffer.md)

Um [buffer XML](xml-buffer.md) é uma representação estruturada na memória de um documento XML. Essa é uma representação mais eficiente do que um documento codificado como bytes. Um documento XML armazenado em um buffer XML pode ser navegado, lido ou gravado.

## <a name="io"></a>E/S

A API XML nunca executará a e/s, a menos que solicitado especificamente. Além disso, qualquer e/s pode ser iniciada de maneira assíncrona. Consulte [**WsFillReader**](/windows/desktop/api/WebServices/nf-webservices-wsfillreader) e [**WsFlushWriter**](/windows/desktop/api/WebServices/nf-webservices-wsflushwriter) para obter detalhes sobre o processamento assíncrono com a API XML.

## <a name="processing"></a>Processing

A API XML tem três níveis distintos nos quais o documento pode ser processado.

Um documento pode ser processado em um [**nó**](/windows/desktop/api/WebServices/ns-webservices-ws_xml_node) por vez. Isso oferece a manipulação mais refinada do conteúdo XML e fornece fidelidade completa dos dados do documento. Nesse nível, as funções [**WsReadNode**](/windows/desktop/api/WebServices/nf-webservices-wsreadnode) e [**WsWriteNode**](/windows/desktop/api/WebServices/nf-webservices-wswritenode) e [**WsCopyNode**](/windows/desktop/api/WebServices/nf-webservices-wscopynode) seriam usadas.

O próximo nível de controle são APIs como [**WsReadStartElement**](/windows/desktop/api/WebServices/nf-webservices-wsreadstartelement), [**WsReadValue**](/windows/desktop/api/WebServices/nf-webservices-wsreadvalue) e [**WsReadEndElement**](/windows/desktop/api/WebServices/nf-webservices-wsreadendelement). Essas APIs fornecem vários tipos de validação, ignoram espaços em branco e comentários e normalizam o texto e CDATA para apresentar ao consumidor uma visão mais simples do XML.

O nível mais alto de controle é usar a API de serialização. Essas APIs são acionadas por um mapeamento entre tipos de dados C e XML e podem ler ou gravar uma estrutura complexa na memória em XML e voltar com uma única função como [**WsWriteElement**](/windows/desktop/api/WebServices/nf-webservices-wswriteelement) e [**WsReadElement**](/windows/desktop/api/WebServices/nf-webservices-wsreadelement).

As APIs de canonização XML podem ser usadas para gerar uma forma canônica de XML que, por sua vez, pode ser usada para gerar assinaturas criptográficas em conteúdo XML.

## <a name="creating-a-writer"></a>Criando um gravador

Para criar e usar um gravador para gravar em um buffer na memória:

``` syntax
WsCreateWriter              // Create an instance of a WS_XML_WRITER
// Initialize a WS_XML_WRITER_BUFFER_OUTPUT
WsSetOutput                 // Set the encoding and output of the writer along with any other writer properties
// Write Elements
WsGetWriterProperty(..., WS_XML_WRITER_PROPERTY_BYTES, ...)  // Get the generated bytes as a single byte array
// Use the generated bytes
WsFreeWriter                // Free the writer
```

Para criar e usar um gravador para gravar em um fluxo:

``` syntax
WsCreateWriter              // Create an instance of a WS_XML_WRITER
// Initialize a WS_XML_WRITER_STREAM_OUTPUT
WsSetOutput                 // Set the encoding and output of the writer along with any other writer properties
// Write Elements
WsFlushWriter               // Force any buffered data to be written
WsFreeWriter                // Free the writer
```

Para criar e usar um gravador para gravar em um [ \_ \_ buffer XML do WS](ws-xml-buffer.md):

``` syntax
WsCreateXmlBuffer           // Create the buffer to write to
WsCreateWriter              // Create an instance of a WS_XML_WRITER
WsSetOutputToBuffer         // Set the output buffer along with any other writer properties
// Write Elements
WsFreeWriter                // Free the writer
// The buffer has the generated document
```

Em todos os casos, o [**recuo da propriedade WS \_ XML \_ Writer \_ \_**](/windows/desktop/api/WebServices/ne-webservices-ws_xml_writer_property_id) da propriedade pode ser incluído para formatar o XML.

## <a name="writing-elements"></a>Elementos de gravação

Para gravar um elemento em um gravador:

``` syntax
WsWriteStartElement          // Write a start element
for each attribute
{
// Write an attribute using either
WsWriteStartAttribute    // Write a start attribute
// Write Content
WsWriteEndAttribute      // Write an end attribute
// Or one of the following
WsWriteXmlnsAttribute    // Write an explicit xmlns attribute
}
// Write Elements or Content
WsWriteEndElement
```

O seguinte também pode ser usado:

``` syntax
WsWriteArray                 // Write an array of primitive values as a series of repeated elements
```

## <a name="writing-content"></a>Gravando conteúdo

Para gravar o conteúdo em um elemento ou atributo, o seguinte pode ser usado:

``` syntax
WsWriteChars                 // Write unicode characters from memory
WsWriteCharsUtf8             // Write UTF-8 encoded characters from memory
WsWriteBytes                 // Write binary data encoded as base64
WsPushBytes                  // Direct the writer to request that bytes be written
WsPullBytes                  // Direct the writer to read the bytes to be written
WsWriteValue                 // Write primitive values such as ints and BOOLs
WsWriteText                  // Write an WS_XML_TEXT
WsWriteQualifiedName         // Write a qualified name
```

O seguinte pode ser usado para gravar em um documento, mas não pode ser usado quando estiver dentro de um atributo.

``` syntax
WsWriteNode                  // Write a single WS_XML_NODE
WsCopyNode                   // Copy a single node, or an entire WS_XML_ELEMENT_NODE and children from an WS_XML_READER
```

O seguinte pode ser usado para escrever uma seção CDATA em um documento de texto:

``` syntax
WsWriteStartCData            // Start a CDATA section in a text encoding
// Write Content
WsWriteEndCData              // End a CDATA section in text encoding
```

WsGetPrefixFromNamespace     // Find a prefix bound to a namespace

## <a name="creating-a-reader"></a>Criando um leitor

Para criar e usar um leitor para ler de um buffer na memória:

``` syntax
WsCreateReader              // Create an instance of a WS_XML_READER
// Initialize a WS_XML_READER_BUFFER_INPUT
WsSetInput                  // Set the encoding and input of the reader along with any other reader properties
// Read Elements
WsFreeReader                // Free the reader
```

Para criar e usar um leitor para leitor de um fluxo:

``` syntax
WsCreateReader              // Create an instance of a WS_XML_READER
// Initialize a WS_XML_READER_STREAM_INPUT
WsSetInput                  // Set the encoding and input of the reader along with any other reader properties
WsFillReader                // Populate the reader with data from the underlying stream
// Read Elements
WsFreeReader                // Free the reader
```

Para criar e usar um leitor para ler de um [ \_ \_ buffer XML WS](ws-xml-buffer.md):

``` syntax
WsCreateXmlBuffer           // Create the buffer to write to
WsCreateReader              // Create an instance of a WS_XML_READER
WsSetInputToBuffer          // Set the input buffer along with any other reader properties
// Read Elements
WsFreeReader                // Free the reader
```

## <a name="reading-elements"></a>Lendo elementos

Para ler um elemento de um leitor:

``` syntax
WsReadToStartElement         // Skip whitespace and comments to position the reader on a specific element
for each attribute of interest
{
WsFindAttribute          // Try Locate the attribute
if (found)
{
WsReadStartAttribute // Set the reader to read the attribute
// Read Content
WsReadEndAttribute   // Return the reader to the element
}
}
WsReadStartElement           // Advance the reader past the current element
// Read Elements or Content
WsWriteEndElement            // Advance the reader past the corresponding end element
```

O seguinte também pode ser usado:

``` syntax
WsReadArray                  // Read an array of primitive values as a series of repeated elements
```

## <a name="reading-content"></a>Lendo o conteúdo

Para ler o conteúdo de um elemento ou atributo, o seguinte pode ser usado:

``` syntax
WsReadChars                 // Read characters to memory as unicode
WsReadCharsUtf8             // Read characters to memory encoded as UTF-8
WsReadBytes                 // Read binary data encoded as base64
WsReadValue                 // Read primitive values such as ints and BOOLs
WsReadQualifiedName         // Read a qualified name
```

O seguinte pode ser usado para inspecionar o nó atual no qual o leitor está posicionado:

``` syntax
WsGetReaderNode             // Get the current node
```

## <a name="using-a-buffer"></a>Usando um buffer

Ao gravar em um [ \_ \_ buffer XML do WS](ws-xml-buffer.md) , o seguinte pode ser usado:

``` syntax
WsGetWriterPosition          // Get the current position of the writer in the document
WsSetWriterPosition          // Set the current position of the writer in the document
WsMoveWriter                 // Move relative to the current position in the document
WsRemoveNode                 // Delete an element or text from a document
```

Ao ler de um [ \_ \_ buffer XML do WS](ws-xml-buffer.md) , o seguinte pode ser usado:

``` syntax
WsGetReaderPosition          // Get the current position of the reader in the document
WsSetReaderPosition          // Set the current position of the reader in the document
WsMoveReader                 // Move relative to the current position in the document
```

O seguinte pode ser usado para modificar um [ \_ \_ buffer XML do WS](ws-xml-buffer.md):

``` syntax

WsRemoveNode                 // Delete an element or text from a document
```
