## <a name="remarks"></a>Comentários

Você pode se interessar apenas se o método teve êxito ou falhou. A melhor maneira de testar se um valor **HRESULT** indica êxito ou falha é passar o valor para uma das macros a seguir, definida em Winerror. h:

-   A macro [**Succeeded**](/windows/win32/api/winerror/nf-winerror-succeeded) retorna **true** para um código de êxito e **false** para um código de falha.
-   A macro [**com falha**](/windows/win32/api/winerror/nf-winerror-failed) retorna **true** para um código de falha e **false** para um código de êxito.

O valor **HRESULT** para cada valor de **\_ erro dxgi** é determinado por essa macro que é definida em DXGItype. h:


```
#define _FACDXGI    0x87a
#define MAKE_DXGI_HRESULT(code) MAKE_HRESULT(1, _FACDXGI, code)
```



Por exemplo, **a \_ \_ \_ chamada inválida de erro dxgi** é definida como **0x887A0001**:


```
#define DXGI_ERROR_INVALID_CALL                 MAKE_DXGI_HRESULT(1)
```
