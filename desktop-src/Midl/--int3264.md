# <a name="__int3264-attribute"></a>\_\_atributo int3264

A palavra-chave **\_ \_ int3264** especifica um tipo integral com as seguintes propriedades:

-   É 32 bits em plataformas de 32 bits
-   É 64 bits em plataformas de 64 bits
-   É de 32 bits na conexão para compatibilidade com versões anteriores. Ele é truncado no lado de envio e estendido adequadamente (assinado ou não assinado) no lado de recebimento.

``` syntax
[ signed | unsigned ] __int3264 [ int ] declarator-list;
```

## <a name="parameters"></a>Parâmetros

<dl> <dt>

*Declarador-lista* 
</dt> <dd>

Especifica um ou mais declaradores C padrão, como identificadores, declaradores de ponteiro e declaradores de matriz. (Os declaradores de função e as declarações de campo de bits não são permitidas em estruturas que são transmitidas em chamadas de procedimento remoto. Esses declaradores são permitidos em estruturas que não são transmitidas.) Separe vários declaradores com vírgulas.

</dd> </dl>

## <a name="remarks"></a>Comentários

Esse tipo foi introduzido especificamente para facilitar a portabilidade de interfaces de 32 bits para 64 bits. Em geral, ele é útil apenas em casos em que as interfaces existentes enviam ponteiros como inteiros ou usam técnicas de solução alternativa semelhantes. A Microsoft não recomenda usar esse tipo (ou outros tipos públicos derivados dele, como **int \_ PTR**, **DWORD \_ PTR**), onde pode ser evitado. Em particular, esses tipos não podem ser usados em interfaces de automação OLE e bibliotecas de tipos. O uso deste tipo, ou tipos derivados dele, em novas interfaces não é recomendado. Consulte o linguagem IDL da Microsoft (MIDL): guia de portabilidade de 64 bits para obter mais informações sobre esses problemas, bem como comentários e conselhos sobre outros problemas de 64 bits.

A \_ \_ palavra-chave **int3264** pode ser precedida por uma palavra-chave assinada ou por uma palavra-chave não assinada. A palavra-chave int é opcional e pode ser omitida. Para o compilador MIDL \_ \_ , o inteiro **int3264** é assinado por padrão e é sinônimo de \_ \_ int3264 int assinado.

O tipo de inteiro **\_ \_ int3264** é um dos tipos base da linguagem IDL. O tipo de inteiro **\_ \_ int3264** pode aparecer como um especificador de tipo em declarações const, declarações de typedef, declarações gerais e declaradores de função (como um especificador de tipo de retorno de função e um especificador de tipo de parâmetro). Para o contexto no qual os especificadores de tipo aparecem, consulte IDL.
