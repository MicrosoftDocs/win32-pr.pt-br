<span data-ttu-id="5338c-111">Os pacotes de autenticação são responsáveis pelas seguintes tarefas:</span><span class="sxs-lookup"><span data-stu-id="5338c-111">Authentication packages are responsible for the following tasks:</span></span>

-   <span data-ttu-id="5338c-113">Estabelecendo uma nova [*sessão de logon*](/windows/desktop/SecGloss/l-gly) e criando um identificador de [*logon*](/windows/desktop/SecGloss/l-gly) exclusivo para a entidade de segurança autenticada com êxito.</span><span class="sxs-lookup"><span data-stu-id="5338c-113">Establishing a new [*logon session*](/windows/desktop/SecGloss/l-gly) and creating a unique [*logon identifier*](/windows/desktop/SecGloss/l-gly) for the successfully authenticated principal.</span></span>

<span data-ttu-id="5338c-120">Depois de autenticar com êxito as credenciais de uma [*entidade de segurança*](/windows/desktop/SecGloss/s-gly) , um pacote de autenticação é responsável por criar uma nova sessão de logon LSA para a entidade e alocar o [*identificador de logon*](/windows/desktop/SecGloss/l-gly) que identifica exclusivamente a sessão de logon.</span><span class="sxs-lookup"><span data-stu-id="5338c-120">After successfully authenticating a [*security principal's*](/windows/desktop/SecGloss/s-gly) credentials, an authentication package is responsible for creating a new LSA logon session for the principal and allocating the [*logon identifier*](/windows/desktop/SecGloss/l-gly) that uniquely identifies the logon session.</span></span> <span data-ttu-id="5338c-121">O pacote de autenticação pode associar informações de credenciais com a sessão de logon para solicitações de autenticação subsequentes.</span><span class="sxs-lookup"><span data-stu-id="5338c-121">The authentication package may associate credential information with the logon session for subsequent authentication requests.</span></span> <span data-ttu-id="5338c-122">Por exemplo, o \_ pacote de autenticação MSV1 0 (fornecido pela Microsoft) associa o nome da conta de usuário e um hash da senha do usuário a cada sessão de logon.</span><span class="sxs-lookup"><span data-stu-id="5338c-122">For example, the MSV1\_0 authentication package (provided by Microsoft) associates the user account name and a hash of the user's password with each logon session.</span></span>

<span data-ttu-id="5338c-123">O pacote de autenticação também fornece um conjunto de SIDs ( [*identificadores de segurança*](/windows/desktop/SecGloss/s-gly) ) e outras informações apropriadas para inclusão no token de segurança criado pela LSA.</span><span class="sxs-lookup"><span data-stu-id="5338c-123">The authentication package also provides a set of [*security identifiers*](/windows/desktop/SecGloss/s-gly) (SIDs) and other information appropriate for inclusion in the security token created by the LSA.</span></span> <span data-ttu-id="5338c-124">Esse token representará o [*contexto*](/windows/desktop/SecGloss/c-gly) de segurança da entidade para acesso às operações do Windows.</span><span class="sxs-lookup"><span data-stu-id="5338c-124">This token will represent the principal's security [*context*](/windows/desktop/SecGloss/c-gly) for access to Windows operations.</span></span>
