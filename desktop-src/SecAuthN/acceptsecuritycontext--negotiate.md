## <a name="remarks"></a><span data-ttu-id="0c032-209">Comentários</span><span class="sxs-lookup"><span data-stu-id="0c032-209">Remarks</span></span>

<span data-ttu-id="0c032-210">A função **AcceptSecurityContext (Negotiate)** é a contraparte do servidor para a função [**InitializeSecurityContext (Negotiate)**](initializesecuritycontext--negotiate.md) .</span><span class="sxs-lookup"><span data-stu-id="0c032-210">The **AcceptSecurityContext (Negotiate)** function is the server counterpart to the [**InitializeSecurityContext (Negotiate)**](initializesecuritycontext--negotiate.md) function.</span></span>

<span data-ttu-id="0c032-211">Quando o servidor recebe uma solicitação de um cliente, o servidor usa o parâmetro *fContextReq* para especificar o que é necessário para a sessão.</span><span class="sxs-lookup"><span data-stu-id="0c032-211">When the server receives a request from a client, the server uses the *fContextReq* parameter to specify what it requires of the session.</span></span> <span data-ttu-id="0c032-212">Dessa maneira, um servidor pode especificar que os clientes devem ser capazes de usar uma sessão confidencial ou com verificação de [*integridade*](../secgloss/i-gly.md), e pode rejeitar clientes que não podem atender a essa demanda.</span><span class="sxs-lookup"><span data-stu-id="0c032-212">In this fashion, a server can specify that clients must be capable of using a confidential or [*integrity*](../secgloss/i-gly.md)-checked session, and it can reject clients that cannot meet that demand.</span></span> <span data-ttu-id="0c032-213">Como alternativa, um servidor pode não exigir nada e tudo o que o cliente pode fornecer ou exigir é retornado no parâmetro *pfContextAttr* .</span><span class="sxs-lookup"><span data-stu-id="0c032-213">Alternatively, a server can require nothing, and whatever the client can provide or requires is returned in the *pfContextAttr* parameter.</span></span>
