# <a name="authenticator-messages"></a><span data-ttu-id="5b5b3-103">Mensagens do autenticador</span><span class="sxs-lookup"><span data-stu-id="5b5b3-103">Authenticator Messages</span></span>

<span data-ttu-id="5b5b3-104">Em um protocolo simples usando a autenticação de chave secreta, um cliente apresenta uma mensagem de autenticador na forma de uma informação criptografada na [*chave de sessão*](/windows/desktop/SecGloss/s-gly).</span><span class="sxs-lookup"><span data-stu-id="5b5b3-104">In a simple protocol using secret key authentication, a client presents an authenticator message in the form of a piece of information encrypted in the [*session key*](/windows/desktop/SecGloss/s-gly).</span></span> <span data-ttu-id="5b5b3-105">As informações na mensagem do autenticador devem ser diferentes cada vez que o protocolo de autenticação é executado, ou uma mensagem de autenticador criptografado pode ser reutilizada por uma entidade não autorizada.</span><span class="sxs-lookup"><span data-stu-id="5b5b3-105">The information in the authenticator message must be different each time the authentication protocol is executed, or an encrypted authenticator message could be reused by an unauthorized entity.</span></span>

<span data-ttu-id="5b5b3-106">Ao receber a mensagem do autenticador, o servidor a descriptografa e pode contar com o conteúdo da mensagem descriptografada, independentemente de a descriptografia ter sido bem-sucedida.</span><span class="sxs-lookup"><span data-stu-id="5b5b3-106">On receiving the authenticator message, the server decrypts it and can tell from the contents of the decrypted message whether decryption was successful.</span></span> <span data-ttu-id="5b5b3-107">Se a mensagem descriptografada não for ininteligível, o servidor saberá que o cliente que está apresentando a mensagem do autenticador usou a chave correta para criptografar a mensagem.</span><span class="sxs-lookup"><span data-stu-id="5b5b3-107">If the decrypted message is not gibberish, the server knows that the client presenting the authenticator message used the correct key to encrypt the message.</span></span> <span data-ttu-id="5b5b3-108">Somente duas entidades têm acesso à [*chave de sessão*](/windows/desktop/SecGloss/s-gly) e, se o servidor for um deles, o cliente que criptografou a mensagem de autenticador deverá ser o outro.</span><span class="sxs-lookup"><span data-stu-id="5b5b3-108">Only two entities have access to the [*session key*](/windows/desktop/SecGloss/s-gly) and if the server is one of those, the client who encrypted the authenticator message must be the other.</span></span>

<span data-ttu-id="5b5b3-109">Para a autenticação mútua, um protocolo semelhante é executado.</span><span class="sxs-lookup"><span data-stu-id="5b5b3-109">For mutual authentication, a similar protocol is executed.</span></span> <span data-ttu-id="5b5b3-110">O servidor extrai parte das informações da mensagem de autenticador descriptografada e original, criptografa-a com a chave de sessão compartilhada e envia a mensagem criptografada para o cliente.</span><span class="sxs-lookup"><span data-stu-id="5b5b3-110">The server extracts part of the information from the decrypted, original authenticator message, encrypts it with the shared session key, and sends the encrypted message to the client.</span></span> <span data-ttu-id="5b5b3-111">O cliente descriptografa a mensagem e compara o resultado com o original.</span><span class="sxs-lookup"><span data-stu-id="5b5b3-111">The client decrypts the message and compares the result with the original.</span></span> <span data-ttu-id="5b5b3-112">Se a mensagem descriptografada corresponder à parte correta da mensagem original, o cliente saberá que o servidor foi capaz de descriptografar a mensagem original com a chave de sessão de segredo compartilhado e foi capaz de criptografar novamente uma parte dessa mensagem com a mesma [*chave de sessão*](/windows/desktop/SecGloss/s-gly)de segredo compartilhado.</span><span class="sxs-lookup"><span data-stu-id="5b5b3-112">If the decrypted message matches the correct part of the original message, the client knows that the server was able to decrypt the original message with the shared secret session key and was able to re-encrypt a portion of that message with that same shared secret [*session key*](/windows/desktop/SecGloss/s-gly).</span></span>
