<span data-ttu-id="01271-104">A taxonomia descrita nesta seção primeiro distingue o plano de controle que se preocupa com a maneira como uma sessão do MultiPoint é estabelecida, do plano de dados que lida com a transferência de dados entre os participantes da sessão.</span><span class="sxs-lookup"><span data-stu-id="01271-104">The taxonomy described in this section first distinguishes the control plane that concerns itself with the way a multipoint session is established, from the data plane that deals with the transfer of data among session participants.</span></span>

## <a name="session-establishment-in-the-control-plane"></a><span data-ttu-id="01271-105">Estabelecimento de sessão no plano de controle</span><span class="sxs-lookup"><span data-stu-id="01271-105">Session Establishment in the Control Plane</span></span>

<span data-ttu-id="01271-106">No plano de controle, há dois tipos distintos de estabelecimento de sessão:</span><span class="sxs-lookup"><span data-stu-id="01271-106">In the control plane there are two distinct types of session establishment:</span></span>

-   <span data-ttu-id="01271-107">com raiz</span><span class="sxs-lookup"><span data-stu-id="01271-107">rooted</span></span>
-   <span data-ttu-id="01271-108">não raiz</span><span class="sxs-lookup"><span data-stu-id="01271-108">nonrooted</span></span>

<span data-ttu-id="01271-109">No caso do controle com raiz, um participante especial, a \_ raiz c, que é diferente do restante dos membros dessa sessão do MultiPoint, cada um deles é chamado de \_ folha c.</span><span class="sxs-lookup"><span data-stu-id="01271-109">In the case of rooted control, a special participant, c\_root, exists that is different from the rest of the members of this multipoint session, each of which are called a c\_leaf.</span></span> <span data-ttu-id="01271-110">A \_ raiz c deve permanecer presente por toda a duração da sessão do MultiPoint, pois a sessão é dividida na ausência da \_ raiz c.</span><span class="sxs-lookup"><span data-stu-id="01271-110">The c\_root must remain present for the whole duration of the multipoint session, as the session is broken up in the absence of the c\_root.</span></span> <span data-ttu-id="01271-111">A \_ raiz c geralmente inicia a sessão do MultiPoint Configurando a conexão com uma \_ folha c ou um número de folhas c \_ .</span><span class="sxs-lookup"><span data-stu-id="01271-111">The c\_root usually initiates the multipoint session by setting up the connection to a c\_leaf, or a number of c\_leafs.</span></span> <span data-ttu-id="01271-112">A \_ raiz c pode adicionar mais \_ folhas c ou (em alguns casos) uma folha c \_ pode unir a raiz c \_ em um momento posterior.</span><span class="sxs-lookup"><span data-stu-id="01271-112">The c\_root may add more c\_leafs, or (in some cases) a c\_leaf can join the c\_root at a later time.</span></span> <span data-ttu-id="01271-113">Exemplos do plano de controle com raiz podem ser encontrados em ATM e ST-II.</span><span class="sxs-lookup"><span data-stu-id="01271-113">Examples of the rooted control plane can be found in ATM and ST-II.</span></span>

<span data-ttu-id="01271-116">Outra maneira de examinar isso é que uma raiz c \_ ainda existe, mas pode ser considerada na nuvem de rede, em oposição a um dos participantes.</span><span class="sxs-lookup"><span data-stu-id="01271-116">Another way to look at this is that a c\_root still exists, but can be considered to be in the network cloud as opposed to one of the participants.</span></span> <span data-ttu-id="01271-117">Como uma raiz de controle ainda existe, um plano de controle não-raiz também poderia ser considerado implicitamente enraizada.</span><span class="sxs-lookup"><span data-stu-id="01271-117">Because a control root still exists, a nonrooted control plane could also be considered to be implicitly rooted.</span></span> <span data-ttu-id="01271-118">Exemplos para esse tipo de esquemas MultiPoint com raiz implícita são:</span><span class="sxs-lookup"><span data-stu-id="01271-118">Examples for this kind of implicitly rooted multipoint schemes are:</span></span>

## <a name="data-transfer-in-the-data-plane"></a><span data-ttu-id="01271-122">Transferência de Dados no plano de dados</span><span class="sxs-lookup"><span data-stu-id="01271-122">Data Transfer in the Data Plane</span></span>

<span data-ttu-id="01271-123">No plano de dados, há dois tipos de estilos de transferência de dados:</span><span class="sxs-lookup"><span data-stu-id="01271-123">In the data plane, there are two types of data transfer styles:</span></span>

<span data-ttu-id="01271-126">Em um plano de dados com raiz, existe um participante especial chamado d \_ root.</span><span class="sxs-lookup"><span data-stu-id="01271-126">In a rooted data plane, a special participant called d\_root exists.</span></span> <span data-ttu-id="01271-127">A transferência de dados ocorre apenas entre a \_ raiz d e o restante dos membros dessa sessão do MultiPoint, sendo que cada uma delas é conhecida como \_ folha d.</span><span class="sxs-lookup"><span data-stu-id="01271-127">Data transfer only occurs between the d\_root and the rest of the members of this multipoint session, each of which are referred to as a d\_leaf.</span></span> <span data-ttu-id="01271-128">O tráfego pode ser unidirecional ou bidirecional.</span><span class="sxs-lookup"><span data-stu-id="01271-128">The traffic could be unidirectional or bidirectional.</span></span> <span data-ttu-id="01271-129">Os dados enviados da \_ raiz d são duplicados (se necessário) e entregues a cada \_ folha d, enquanto os dados de folhas d \_ vão apenas para a raiz d \_ .</span><span class="sxs-lookup"><span data-stu-id="01271-129">The data sent out from the d\_root is duplicated (if required) and delivered to every d\_leaf, while the data from d\_leafs only goes to the d\_root.</span></span> <span data-ttu-id="01271-130">No caso de um plano de dados com raiz, nenhum tráfego é permitido entre \_ folhas d.</span><span class="sxs-lookup"><span data-stu-id="01271-130">In the case of a rooted data plane, no traffic is allowed among d\_leafs.</span></span> <span data-ttu-id="01271-131">Um exemplo de um protocolo com raiz no plano de dados é ST-II.</span><span class="sxs-lookup"><span data-stu-id="01271-131">An example of a protocol that is rooted in the data plane is ST-II.</span></span>
