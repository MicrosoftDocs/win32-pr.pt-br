# <a name="secure-winsock-programming"></a>Programação de Winsock seguro

Veja a seguir um guia para proteger a programação do Windows Sockets. Ele foi projetado para fornecer uma compreensão da segurança do Winsock e das opções disponíveis para o desenvolvedor de aplicativo de rede seguro.

-   [Usando o \_ REUSEADDR e, portanto, \_ EXCLUSIVEADDRUSE](using-so-reuseaddr-and-so-exclusiveaddruse.md)
-   [Extensões do Winsock Secure Socket](winsock-secure-socket-extensions.md)

As comunicações usando soquetes também podem ser criptografadas usando os padrões de SSL/TLS usando o canal seguro, também conhecido como tecnologia Schannel. O Schannel é um SSP (provedor de suporte de segurança) que contém um conjunto de protocolos de segurança que fornecem autenticação de identidade e comunicação segura e privada por meio de criptografia. O Schannel é usado principalmente para aplicativos de Internet que exigem comunicações HTTP (Hypertext Transfer Protocol) seguras. Para obter mais informações, consulte [Secure Channel](../secauthn/secure-channel.md).
