<span data-ttu-id="1bd84-114">O *tamanho do pacote de mídia* reflete o tamanho máximo de pacote aceitável em qualquer mídia que o pacote atravessa para seu destino.</span><span class="sxs-lookup"><span data-stu-id="1bd84-114">*Media packet size* reflects the maximum packet size acceptable on any media the packet traverses to its destination.</span></span> <span data-ttu-id="1bd84-115">O tamanho do pacote varia entre mídias diferentes, como Ethernet e token ring.</span><span class="sxs-lookup"><span data-stu-id="1bd84-115">Packet size varies among differing media such as Ethernet and token ring.</span></span> <span data-ttu-id="1bd84-116">A quantidade de espaço de dados dentro de um pacote também pode variar em uma determinada mídia, dependendo da organização do cabeçalho do pacote.</span><span class="sxs-lookup"><span data-stu-id="1bd84-116">The amount of data space within a packet can also vary within a given media, depending on the packet header arrangement.</span></span> <span data-ttu-id="1bd84-117">Por exemplo, o tamanho efetivo dos dados de um pacote Ethernet varia dependendo se ele for do tipo Ethernet II, Ethernet 802,2 ou ajuste de Ethernet.</span><span class="sxs-lookup"><span data-stu-id="1bd84-117">For instance, the effective data size of an Ethernet packet varies depending on whether it is of the type Ethernet II, Ethernet 802.2, or Ethernet SNAP.</span></span>

<span data-ttu-id="1bd84-118">O *tamanho do pacote roteável* reflete o tamanho máximo do pacote que um roteador intermediário está disposto a encaminhar.</span><span class="sxs-lookup"><span data-stu-id="1bd84-118">*Routable packet size* reflects the maximum packet size an intermediate router is willing to forward.</span></span> <span data-ttu-id="1bd84-119">A maioria dos roteadores IPX são criados para rotear qualquer tamanho de datagrama, desde que permaneça dentro do tamanho da mídia da rede de envio e recebimento.</span><span class="sxs-lookup"><span data-stu-id="1bd84-119">Most IPX routers are built to route any size datagram as long as it remains within the media size of the sending and receiving network.</span></span> <span data-ttu-id="1bd84-120">No entanto, os roteadores mais antigos podem limitar o tamanho máximo do pacote a 576 bytes, incluindo os cabeçalhos de protocolo.</span><span class="sxs-lookup"><span data-stu-id="1bd84-120">However, older routers may limit maximum packet size to 576 bytes, including protocol headers.</span></span>

<span data-ttu-id="1bd84-121">O *tamanho do pacote de estação final* reflete o tamanho dos buffers de escuta que as estações finais lançaram para receber os pacotes de entrada.</span><span class="sxs-lookup"><span data-stu-id="1bd84-121">*End-station packet size* reflects the size of the listen buffers that end stations have posted to receive incoming packets.</span></span> <span data-ttu-id="1bd84-122">Mesmo quando os limites de mídia e roteador permitem um pacote, ele pode ser descartado pela estação final se o aplicativo de recebimento tiver postado um buffer menor.</span><span class="sxs-lookup"><span data-stu-id="1bd84-122">Even when the media and router limits allow a packet through, it may be discarded by the end station if the receiving application has posted a smaller buffer.</span></span> <span data-ttu-id="1bd84-123">Muitos aplicativos IPX/SPX tradicionais limitam o tamanho do buffer de recebimento, de modo que a parte de dados não deve ser maior que 512 ou 1024 bytes.</span><span class="sxs-lookup"><span data-stu-id="1bd84-123">Many traditional IPX/SPX applications limit receive buffer size such that the data portion must be no larger than 512 or 1024 bytes.</span></span>
