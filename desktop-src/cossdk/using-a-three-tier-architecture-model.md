# <a name="using-a-three-tier-architecture-model"></a><span data-ttu-id="e9fe1-103">Usando um modelo de arquitetura de Three-Tier</span><span class="sxs-lookup"><span data-stu-id="e9fe1-103">Using a Three-Tier Architecture Model</span></span>

<span data-ttu-id="e9fe1-104">O modelo de arquitetura de três camadas, que é a estrutura fundamental para o [modelo de design lógico](the-logical-model--application-definition-and-planning.md), segmenta os componentes de um aplicativo em três camadas de serviços.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-104">The three-tier architecture model, which is the fundamental framework for the [logical design model](the-logical-model--application-definition-and-planning.md), segments an application's components into three tiers of services.</span></span> <span data-ttu-id="e9fe1-105">Essas camadas não correspondem necessariamente a locais físicos em vários computadores em uma rede, mas sim a camadas lógicas do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-105">These tiers do not necessarily correspond to physical locations on various computers on a network, but rather to logical layers of the application.</span></span> <span data-ttu-id="e9fe1-106">A forma como as partes de um aplicativo são distribuídas em uma topologia física pode mudar, dependendo dos requisitos do sistema.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-106">How the pieces of an application are distributed in a physical topology can change, depending on the system requirements.</span></span>

<span data-ttu-id="e9fe1-107">Veja a seguir descrições breves dos serviços alocados para cada camada:</span><span class="sxs-lookup"><span data-stu-id="e9fe1-107">Following are brief descriptions of the services allocated to each tier:</span></span>

-   <span data-ttu-id="e9fe1-108">**A camada de apresentação, ou de serviços de usuário, fornece a um usuário acesso ao aplicativo.**</span><span class="sxs-lookup"><span data-stu-id="e9fe1-108">**The presentation tier, or user services layer, gives a user access to the application.**</span></span> <span data-ttu-id="e9fe1-109">Essa camada apresenta dados para o usuário e, opcionalmente, permite a manipulação de dados e a entrada de dados.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-109">This layer presents data to the user and optionally permits data manipulation and data entry.</span></span> <span data-ttu-id="e9fe1-110">Os dois tipos principais de interface do usuário para essa camada são o aplicativo tradicional e o aplicativo baseado na Web.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-110">The two main types of user interface for this layer are the traditional application and the Web-based application.</span></span> <span data-ttu-id="e9fe1-111">Agora, os aplicativos baseados na Web contêm a maioria dos recursos de manipulação de dados usados pelos aplicativos tradicionais.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-111">Web-based applications now often contain most of the data manipulation features that traditional applications use.</span></span> <span data-ttu-id="e9fe1-112">Isso é feito por meio do uso de fontes de dados HTML e do cliente dinâmicos e de cursores de dados.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-112">This is accomplished through use of Dynamic HTML and client-side data sources and data cursors.</span></span>

    > [!Note]  
    > <span data-ttu-id="e9fe1-113">Em um aplicativo de três camadas, o aplicativo do lado do cliente será skinnier do que um aplicativo cliente-servidor, pois ele não conterá os componentes de serviço agora localizados na camada intermediária.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-113">In a three-tiered application, the client-side application will be skinnier than a client-server application because it will not contain the service components now located in the middle tier.</span></span> <span data-ttu-id="e9fe1-114">Isso resulta em menos sobrecarga para o usuário, mas mais tráfego de rede para o sistema porque os componentes são distribuídos entre diferentes máquinas.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-114">This results in less overhead for the user, but more network traffic for the system because components are distributed among different machines.</span></span>

     

-   <span data-ttu-id="e9fe1-115">**A camada intermediária, ou de serviços comerciais, consiste em regras de negócios e dados.**</span><span class="sxs-lookup"><span data-stu-id="e9fe1-115">**The middle tier, or business services layer, consists of business and data rules.**</span></span> <span data-ttu-id="e9fe1-116">Também conhecida como a *camada de lógica de negócios*, a camada intermediária é onde os desenvolvedores de com+ podem resolver problemas de negócios críticos e atingir grandes vantagens de produtividade.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-116">Also referred to as the *business logic tier*, the middle tier is where COM+ developers can solve mission-critical business problems and achieve major productivity advantages.</span></span> <span data-ttu-id="e9fe1-117">Os componentes que compõem essa camada podem existir em um computador servidor, para auxiliar no compartilhamento de recursos.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-117">The components that make up this layer can exist on a server machine, to assist in resource sharing.</span></span> <span data-ttu-id="e9fe1-118">Esses componentes podem ser usados para impor regras de negócio, como algoritmos de negócios e regulamentos legais ou governamentais, além de regras de dados, que são projetadas para manter as estruturas de dados consistentes em um ou vários bancos de dado.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-118">These components can be used to enforce business rules, such as business algorithms and legal or governmental regulations, and data rules, which are designed to keep the data structures consistent within either specific or multiple databases.</span></span> <span data-ttu-id="e9fe1-119">Como esses componentes de camada intermediária não estão vinculados a um cliente específico, eles podem ser usados por todos os aplicativos e podem ser movidos para locais diferentes, pois o tempo de resposta e outras regras exigem.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-119">Because these middle-tier components are not tied to a specific client, they can be used by all applications and can be moved to different locations, as response time and other rules require.</span></span> <span data-ttu-id="e9fe1-120">Por exemplo, edições simples podem ser colocadas no lado do cliente para minimizar viagens de ida e volta da rede, ou as regras de dados podem ser colocadas em procedimentos armazenados.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-120">For example, simple edits can be placed on the client side to minimize network round-trips, or data rules can be placed in stored procedures.</span></span>

-   <span data-ttu-id="e9fe1-121">**A camada de dados, ou de serviços de dados, interage com os dados persistentes geralmente armazenados em um banco de dado ou em um armazenamento permanente.**</span><span class="sxs-lookup"><span data-stu-id="e9fe1-121">**The data tier, or data services layer, interacts with persistent data usually stored in a database or in permanent storage.**</span></span> <span data-ttu-id="e9fe1-122">Essa é a camada de acesso do DBMS real.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-122">This is the actual DBMS access layer.</span></span> <span data-ttu-id="e9fe1-123">Ele pode ser acessado por meio da camada de serviços corporativos e ocasionalmente pela camada de serviços do usuário.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-123">It can be accessed through the business services layer and on occasion by the user services layer.</span></span> <span data-ttu-id="e9fe1-124">Essa camada consiste em componentes de acesso a dados (em vez de conexões brutas de DBMS) para auxiliar no compartilhamento de recursos e permitir que os clientes sejam configurados sem instalar as bibliotecas DBMS e os drivers ODBC em cada cliente.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-124">This layer consists of data access components (rather than raw DBMS connections) to aid in resource sharing and to allow clients to be configured without installing the DBMS libraries and ODBC drivers on each client.</span></span>

<span data-ttu-id="e9fe1-125">Durante o ciclo de vida de um aplicativo, a abordagem de três camadas fornece benefícios como reutilização, flexibilidade, capacidade de gerenciamento, facilidade de manutenção e escalabilidade.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-125">During an application's life cycle, the three-tier approach provides benefits such as reusability, flexibility, manageability, maintainability, and scalability.</span></span> <span data-ttu-id="e9fe1-126">Você pode compartilhar e reutilizar os componentes e serviços criados por você e pode distribuí-los em uma rede de computadores, conforme necessário.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-126">You can share and reuse the components and services you create, and you can distribute them across a network of computers as needed.</span></span> <span data-ttu-id="e9fe1-127">Você pode dividir projetos grandes e complexos em projetos mais simples e atribuí-los a programadores ou equipes de programação diferentes.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-127">You can divide large and complex projects into simpler projects and assign them to different programmers or programming teams.</span></span> <span data-ttu-id="e9fe1-128">Você também pode implantar componentes e serviços em um servidor para ajudar a acompanhar as alterações e pode reimplantá-las como crescimento da base de usuários, dos dados e do volume de transações do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="e9fe1-128">You can also deploy components and services on a server to help keep up with changes, and you can redeploy them as growth of the application's user base, data, and transaction volume increases.</span></span>
