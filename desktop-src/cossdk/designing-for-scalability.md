# <a name="designing-for-scalability"></a><span data-ttu-id="260d7-103">Projetando para escalabilidade</span><span class="sxs-lookup"><span data-stu-id="260d7-103">Designing for Scalability</span></span>

<span data-ttu-id="260d7-104">A escalabilidade é a capacidade de um aplicativo de atender a uma carga adicional com um aumento linear no uso de recursos.</span><span class="sxs-lookup"><span data-stu-id="260d7-104">Scalability is the ability of an application to service an additional load with a linear increase in resource usage.</span></span> <span data-ttu-id="260d7-105">A escalabilidade é importante em qualquer aplicativo distribuído.</span><span class="sxs-lookup"><span data-stu-id="260d7-105">Scalability is important in any distributed application.</span></span> <span data-ttu-id="260d7-106">Os limites de escalabilidade geralmente são centralizados em relação ao uso de recursos e dependências inadvertidamente criadas no design do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="260d7-106">Limits to scalability usually center around resource use and dependencies inadvertently created in the application's design.</span></span>

<span data-ttu-id="260d7-107">A lista a seguir descreve problemas de escalabilidade e sugere soluções:</span><span class="sxs-lookup"><span data-stu-id="260d7-107">The following list describes scalability issues and suggests solutions:</span></span>

-   <span data-ttu-id="260d7-108">Recursos dentro do computador.</span><span class="sxs-lookup"><span data-stu-id="260d7-108">Intra-computer resources.</span></span> <span data-ttu-id="260d7-109">O número de threads e a memória disponível pode limitar a escalabilidade.</span><span class="sxs-lookup"><span data-stu-id="260d7-109">The number of threads and memory available can limit scalability.</span></span> <span data-ttu-id="260d7-110">Use um modelo de Threading que seja mais eficiente para seu aplicativo.</span><span class="sxs-lookup"><span data-stu-id="260d7-110">Use a threading model that is most efficient for your application.</span></span>
-   <span data-ttu-id="260d7-111">Recursos entre computadores.</span><span class="sxs-lookup"><span data-stu-id="260d7-111">Inter-computer resources.</span></span> <span data-ttu-id="260d7-112">O número de computadores disponíveis para distribuir a carga de trabalho do aplicativo pode afetar a escalabilidade.</span><span class="sxs-lookup"><span data-stu-id="260d7-112">The number of available computers to distribute the application workload can affect scalability.</span></span>
-   <span data-ttu-id="260d7-113">Afinidade de cliente.</span><span class="sxs-lookup"><span data-stu-id="260d7-113">Client affinity.</span></span> <span data-ttu-id="260d7-114">Duas situações de afinidade podem ser criadas inadvertidamente por um aplicativo: uma situação em que o aplicativo depende do estado dos dados que o cliente envia com sua solicitação; e uma situação em que o aplicativo requer um estado específico do cliente.</span><span class="sxs-lookup"><span data-stu-id="260d7-114">Two situations of affinity can be inadvertently created by an application: a situation where the application depends on state from data the client sends with its request; and a situation where the application requires a client-specific state.</span></span> <span data-ttu-id="260d7-115">Evite a criação de dependência de estado entre o cliente e o aplicativo.</span><span class="sxs-lookup"><span data-stu-id="260d7-115">Avoid designing state dependency between the client and the application.</span></span>
-   <span data-ttu-id="260d7-116">Afinidade de servidor.</span><span class="sxs-lookup"><span data-stu-id="260d7-116">Server affinity.</span></span> <span data-ttu-id="260d7-117">Um aplicativo COM+ pode limitar sua escalabilidade criando uma afinidade de servidor, onde o aplicativo depende de ir para um determinado computador servidor para obter informações.</span><span class="sxs-lookup"><span data-stu-id="260d7-117">A COM+ application can limit its scalability by creating a server affinity, where the application depends on going to a particular server computer for information.</span></span> <span data-ttu-id="260d7-118">Essa afinidade pode ocorrer com muitos aplicativos orientados a banco de dados.</span><span class="sxs-lookup"><span data-stu-id="260d7-118">This affinity can occur with many database-oriented applications.</span></span> <span data-ttu-id="260d7-119">A melhor maneira de evitar um afunilamento de afinidade de servidor é particionar dados em vários computadores de servidor.</span><span class="sxs-lookup"><span data-stu-id="260d7-119">The best way to avoid a server-affinity bottleneck is to partition data onto various server computers.</span></span> <span data-ttu-id="260d7-120">Por exemplo, divida os dados do cliente entre os servidores pela chave acessada com mais frequência ou alcance um banco de dado do cliente em vários servidores usando o sobrenome do cliente (por exemplo, server1: a-f, server2: g-m, Server3: n-z).</span><span class="sxs-lookup"><span data-stu-id="260d7-120">For example, divide customer data among servers by the most frequently accessed key, or span a customer database over several servers using the customer's last name (for example, Server1: a-f, Server2: g-m, Server3: n-z).</span></span>
    > [!Note]  
    > <span data-ttu-id="260d7-121">O particionamento de dados pode adicionar uma grande quantidade de complexidade à lógica de programação e deve ser feito somente depois que outras opções para aumentar a escalabilidade tiverem sido tentadas.</span><span class="sxs-lookup"><span data-stu-id="260d7-121">Data partitioning can add a great deal of complexity to the programming logic and should be done only after other options to increase scalability have been tried.</span></span>

     

-   <span data-ttu-id="260d7-122">Tempo de vida do objeto.</span><span class="sxs-lookup"><span data-stu-id="260d7-122">Object lifetime.</span></span> <span data-ttu-id="260d7-123">Para ser caros sejam gerenciados cuidadosamente.</span><span class="sxs-lookup"><span data-stu-id="260d7-125">It is important to make sure that the lifetimes of objects that hold onto expensive resources are licença para 20 conexões SQL, poderá ditar isso com a configuração Max pool.</span><span class="sxs-lookup"><span data-stu-id="260d7-127">And it's a natural way to govern connections: For example, if you have a license for 20 SQL connections, you can dictate that with the Max Pool setting.</span></span>
-   <span data-ttu-id="260d7-128">Agrupamento de componentes de aplicativos.</span><span class="sxs-lookup"><span data-stu-id="260d7-128">Application component grouping.</span></span> <span data-ttu-id="260d7-129">Para melhorar a escalabilidade de um aplicativo COM+, os componentes de camada intermediária devem ser divididos em serviços dependentes de tempo e independentes de tempo.</span><span class="sxs-lookup"><span data-stu-id="260d7-129">To enhance the scalability of a COM+ application, the middle-tier components should be divided into time-dependent and time-independent services.</span></span> <span data-ttu-id="260d7-130">Isso permite que você se concentre em possivelmente usando um serviço do Microsoft Windows para implementar uma ação de componente necessária.</span><span class="sxs-lookup"><span data-stu-id="260d7-130">This allows you to focus on possibly using a Microsoft Windows service to implement a required component action.</span></span> <span data-ttu-id="260d7-131">Por exemplo, você pode optar por usar um serviço como o [enfileiramento de mensagens](/previous-versions/windows/desktop/legacy/ms711472(v=vs.85)) ou [componentes em fila com+](com--queued-components.md) para lidar com tarefas assíncronas independentes de tempo.</span><span class="sxs-lookup"><span data-stu-id="260d7-131">For example, you could elect to use a service such as [Message Queuing](/previous-versions/windows/desktop/legacy/ms711472(v=vs.85)) or [COM+ queued components](com--queued-components.md) to handle time-independent, asynchronous tasks.</span></span>
