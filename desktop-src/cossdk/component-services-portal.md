# <a name="com-component-services"></a>COM+ (serviços de componentes)

## <a name="purpose"></a>Finalidade

O COM+ é uma evolução do Microsoft Component Object Model (COM) e do Microsoft Transaction Server (MTS). O COM+ baseia-se em e estende os aplicativos escritos usando COM, MTS e outras tecnologias baseadas em COM. O COM+ lida com muitas das tarefas de gerenciamento de recursos que você tinha que programar por conta própria, como a alocação e a segurança de threads. O COM+ também torna seus aplicativos mais escalonáveis fornecendo pooling de threads, pooling de objetos e ativação de objeto just-in-time. O COM+ também ajuda a proteger a integridade dos seus dados, fornecendo suporte a transações, mesmo se uma transação abranger vários bancos de dados em uma rede.

## <a name="where-applicable"></a>Quando aplicável

O COM+ pode ser usado para desenvolver aplicativos corporativos e de missão crítica e de toda a empresa para Windows.

Se você for um administrador de sistema, estará instalando, implantando e Configurando aplicativos COM+ e seus componentes. Se você for um programador de aplicativos, estará escrevendo componentes e integrando-os como aplicativos. Se você for um fornecedor de ferramentas, estará desenvolvendo ou modificando ferramentas para trabalhar no ambiente COM+.

## <a name="developer-audience"></a>Público de desenvolvedores

O COM+ foi projetado principalmente para desenvolvedores de Microsoft Visual C++ e do Microsoft Visual Basic.

## <a name="run-time-requirements"></a>Requisitos de tempo de execução

A versão 1,5 do COM+ está incluída no Windows a partir do Windows XP e do Windows Server 2003. A versão 1,0 do COM+ está incluída no Windows 2000.
