# <a name="com-component-services"></a>COM+ (Serviços de Componentes)

## <a name="purpose"></a>Finalidade

O COM+ é uma evolução do COM (Microsoft Component Object Model) e do MTS (Microsoft Transaction Server). O COM+ se baseia e estende aplicativos escritos usando COM, MTS e outras tecnologias baseadas em COM. O COM+ lida com muitas das tarefas de gerenciamento de recursos que você precisava programar por conta própria, como a alocação de threads e a segurança. O COM+ também torna seus aplicativos mais escalonáveis fornecendo pooling de threads, pooling de objetos e ativação de objeto Just-In-Time. O COM+ também ajuda a proteger a integridade de seus dados fornecendo suporte à transação, mesmo se uma transação abranger vários bancos de dados em uma rede.

## <a name="where-applicable"></a>Quando aplicável

O COM+ pode ser usado para desenvolver aplicativos distribuídos de toda a empresa, críticos e Windows.

Se você for um administrador do sistema, instalará, implantará e configurará aplicativos COM+ e seus componentes. Se você for um programador de aplicativos, escreverá componentes e os integrará como aplicativos. Se você for um fornecedor de ferramentas, desenvolverá ou modificará ferramentas para trabalhar no ambiente COM+.

## <a name="developer-audience"></a>Público de desenvolvedores

O COM+ foi projetado principalmente para desenvolvedores Microsoft Visual C++ microsoft Visual Basic microsoft.

## <a name="run-time-requirements"></a>Requisitos de tempo de execução

A versão COM+ 1.5 está incluída no Windows do Windows XP e Windows Server 2003. A versão COM+ 1.0 está incluída no Windows 2000.
