# <a name="security-boundaries"></a><span data-ttu-id="95fbf-103">Limites de segurança</span><span class="sxs-lookup"><span data-stu-id="95fbf-103">Security Boundaries</span></span>

<span data-ttu-id="95fbf-104">A segurança é verificada apenas nos limites do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="95fbf-104">Security is checked only at application boundaries.</span></span> <span data-ttu-id="95fbf-105">Ou seja, para dois componentes no mesmo aplicativo, quando um componente chama o outro, nenhuma verificação de segurança será feita.</span><span class="sxs-lookup"><span data-stu-id="95fbf-105">That is, for two components in the same application, when one component calls the other, no security check will be done.</span></span> <span data-ttu-id="95fbf-106">No entanto, se dois aplicativos compartilharem o mesmo processo e um componente em um chamar um componente no outro, uma verificação de segurança será feita porque um limite de aplicativo é ultrapassado.</span><span class="sxs-lookup"><span data-stu-id="95fbf-106">However, if two applications share the same process and a component in one calls a component in the other, a security check is done because an application boundary is crossed.</span></span> <span data-ttu-id="95fbf-107">Da mesma forma, se dois aplicativos residirem em processos de servidor diferentes e um componente no primeiro aplicativo chamar um componente no segundo aplicativo, uma verificação de segurança será feita.</span><span class="sxs-lookup"><span data-stu-id="95fbf-107">Likewise, if two applications reside in different server processes and a component in the first application calls a component in the second application, a security check is done.</span></span>

<span data-ttu-id="95fbf-108">Portanto, se você tiver dois componentes e quiser que as verificações de segurança sejam feitas quando um chamar o outro, você precisará colocar os componentes em aplicativos COM+ separados.</span><span class="sxs-lookup"><span data-stu-id="95fbf-108">Therefore, if you have two components and you want security checks to be done when one calls the other, you need to put the components in separate COM+ applications.</span></span>

<span data-ttu-id="95fbf-112">Determinar se uma verificação de segurança deve ser executada em uma chamada em um componente é baseada na propriedade de segurança no contexto de objeto criado quando o componente configurado é instanciado.</span><span class="sxs-lookup"><span data-stu-id="95fbf-112">Determining whether a security check must be carried out on a call into a component is based on the security property on the object context created when the configured component is instantiated.</span></span> <span data-ttu-id="95fbf-113">Para obter mais informações, consulte [propriedade de contexto de segurança](security-context-property.md).</span><span class="sxs-lookup"><span data-stu-id="95fbf-113">For more information, see [Security Context Property](security-context-property.md).</span></span>

<span data-ttu-id="95fbf-119">Para aplicativos de biblioteca COM+, você deve selecionar segurança em nível de componente se quiser usar funções.</span><span class="sxs-lookup"><span data-stu-id="95fbf-119">For COM+ library applications, you must select component-level security if you want to use roles.</span></span> <span data-ttu-id="95fbf-120">Os aplicativos de biblioteca não podem usar a segurança em nível de processo.</span><span class="sxs-lookup"><span data-stu-id="95fbf-120">Library applications cannot use process-level security.</span></span>

<span data-ttu-id="95fbf-121">Você deve selecionar a verificação de acesso no nível de componente se estiver usando a segurança programática baseada em função.</span><span class="sxs-lookup"><span data-stu-id="95fbf-121">You should select component-level access checking if you are using programmatic role-based security.</span></span> <span data-ttu-id="95fbf-122">As informações de contexto de chamada de segurança estão disponíveis somente quando a segurança em nível de componente está habilitada.</span><span class="sxs-lookup"><span data-stu-id="95fbf-122">Security call context information is available only when component-level security is enabled.</span></span> <span data-ttu-id="95fbf-123">Para obter mais informações, consulte [informações de contexto de chamada de segurança](security-call-context-information.md).</span><span class="sxs-lookup"><span data-stu-id="95fbf-123">For more information, see [Security Call Context Information](security-call-context-information.md).</span></span>

<span data-ttu-id="95fbf-124">Além disso, quando você seleciona a verificação de acesso no nível do componente, a propriedade de segurança será incluída no contexto do objeto.</span><span class="sxs-lookup"><span data-stu-id="95fbf-124">Additionally, when you select component-level access checking, the security property will be included on the object context.</span></span> <span data-ttu-id="95fbf-125">Isso significa que a configuração de segurança pode desempenhar uma função em como o objeto é ativado.</span><span class="sxs-lookup"><span data-stu-id="95fbf-125">This means that security configuration can play a role in how the object is activated.</span></span> <span data-ttu-id="95fbf-126">Para obter mais informações, consulte [propriedade de contexto de segurança](security-context-property.md).</span><span class="sxs-lookup"><span data-stu-id="95fbf-126">For more information, see [Security Context Property](security-context-property.md).</span></span>

<span data-ttu-id="95fbf-128">As verificações de nível de processo aplicam-se somente ao limite do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="95fbf-128">Process-level checks apply only to the application boundary.</span></span> <span data-ttu-id="95fbf-129">Ou seja, as funções que você definiu para todo o aplicativo COM+ determinarão quem recebe acesso a qualquer recurso dentro do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="95fbf-129">That is, the roles that you have defined for the whole COM+ application will determine who is granted access to any resource within the application.</span></span> <span data-ttu-id="95fbf-130">Nenhuma atribuição de função refinada se aplica.</span><span class="sxs-lookup"><span data-stu-id="95fbf-130">No finer-grained role assignments apply.</span></span> <span data-ttu-id="95fbf-131">Essencialmente, as funções são usadas para criar um descritor de segurança no qual qualquer chamada para os componentes do aplicativo é validada.</span><span class="sxs-lookup"><span data-stu-id="95fbf-131">Essentially, the roles are used to create a security descriptor against which any call into the application's components is validated.</span></span> <span data-ttu-id="95fbf-132">Nesse caso, você não desejaria construir uma política de autorização detalhada com várias funções.</span><span class="sxs-lookup"><span data-stu-id="95fbf-132">In this case, you would not want to construct a detailed authorization policy with multiple roles.</span></span> <span data-ttu-id="95fbf-133">O aplicativo usará um único descritor de segurança.</span><span class="sxs-lookup"><span data-stu-id="95fbf-133">The application will use a single security descriptor.</span></span>

<span data-ttu-id="95fbf-140">Além disso, a propriedade de segurança não será incluída no contexto do objeto.</span><span class="sxs-lookup"><span data-stu-id="95fbf-140">Additionally, the security property will not be included on the object context.</span></span> <span data-ttu-id="95fbf-141">Isso significa que, ao usar apenas verificações de acesso em nível de processo, a configuração de segurança nunca desempenhará uma função na forma como o objeto é ativado.</span><span class="sxs-lookup"><span data-stu-id="95fbf-141">This means that when using only process-level access checks, security configuration will never play a role in how the object is activated.</span></span> <span data-ttu-id="95fbf-142">Para obter mais informações, consulte [propriedade de contexto de segurança](security-context-property.md).</span><span class="sxs-lookup"><span data-stu-id="95fbf-142">For more information, see [Security Context Property](security-context-property.md).</span></span>
