<span data-ttu-id="2ba95-104">A API do servidor HTTP versão 2,0 fornece a capacidade de criar um serviço mais seguro e confiável isolando os processos de trabalho que estão atendendo às solicitações na fila de solicitações.</span><span class="sxs-lookup"><span data-stu-id="2ba95-104">The HTTP Server version 2.0 API provides the ability to build a safer, more reliable service by isolating worker processes that are servicing requests on the request queue.</span></span> <span data-ttu-id="2ba95-105">A fila de solicitações é criada e administrada por um processo de controlador ou criador que controla estritamente o acesso a ela.</span><span class="sxs-lookup"><span data-stu-id="2ba95-105">The request queue is created and administrated by a controller or creator process that strictly controls access to it.</span></span> <span data-ttu-id="2ba95-106">O processo do controlador inicia um ou mais processos de trabalho separados que executam e/s na fila de solicitações.</span><span class="sxs-lookup"><span data-stu-id="2ba95-106">The controller process launches one or more separate worker processes that perform I/O on the request queue.</span></span> <span data-ttu-id="2ba95-107">O processo do controlador é executado com privilégio administrativo e configura a fila de solicitações, enquanto o trabalho de menor privilégio processa o acesso e as solicitações de serviço da fila de solicitações.</span><span class="sxs-lookup"><span data-stu-id="2ba95-107">The controller process runs with administrative privilege and configures the request class="sxs-lookup"><span data-stu-id="2ba95-108">This architecture supports the policy of applications running under "least privilege" and reduces the possibility of security vulnerabilities <span data-ttu-id="2ba95-115">Os processos de trabalho individuais dentro do aplicativo são isolados de outros processos de trabalho, e a integridade de cada um dos processos de trabalho pode ser monitorada pelo processo do controlador.</span><span class="sxs-lookup"><span data-stu-id="2ba95-115">Individual worker processes within the application are isolated from other worker processes, and the health of each the worker processes can be monitored by the controller process.</span></span> <span data-ttu-id="2ba95-116">O processo do controlador é isolado dos processos de trabalho.</span><span class="sxs-lookup"><span data-stu-id="2ba95-116">The controller process is isolated from the worker processes.</span></span> <span data-ttu-id="2ba95-117">Os componentes da arquitetura HTTP são descritos abaixo:</span><span class="sxs-lookup"><span data-stu-id="2ba95-117">The components of the HTTP architecture are described below:</span></span>

-   <span data-ttu-id="2ba95-122">Processo de trabalho: os processos de trabalho, iniciados pelo processo do controlador, executam e/s na fila de solicitações associada às URLs que eles acessam.</span><span class="sxs-lookup"><span data-stu-id="2ba95-122">Worker Process: The worker processes, launched by the controller process, perform IO on the request queue that is associated with the URLs they service.</span></span> <span data-ttu-id="2ba95-123">O aplicativo Web recebe acesso à fila de solicitações pelo processo do controlador na ACL quando a fila de solicitações é criada.</span><span class="sxs-lookup"><span data-stu-id="2ba95-123">The web application is granted access to the request queue by the controller process in the ACL when the request queue is created.</span></span> <span data-ttu-id="2ba95-124">A menos que o aplicativo Web também seja o processo criador, ele não gerencia o serviço nem configura a fila de solicitações.</span><span class="sxs-lookup"><span data-stu-id="2ba95-124">Unless the web application is also the creator process, it does not manage the service or configure the request queue.</span></span> <span data-ttu-id="2ba95-125">O processo do controlador comunica o nome da fila de solicitações ao processo de trabalho e o processo de trabalho abre a fila de solicitações por nome.</span><span class="sxs-lookup"><span data-stu-id="2ba95-125">The controller process communicates the name of the request queue to the worker process and the worker process opens the request queue by name.</span></span> <span data-ttu-id="2ba95-126">Os processos de trabalho podem carregar aplicativos Web de terceiros sem introduzir vulnerabilidades de segurança em outras partes do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="2ba95-126">Worker processes can load third party web applications without introducing security vulnerabilities in other parts of the application.</span></span>

-   <span data-ttu-id="2ba95-129">Sessão de servidor: o processo do controlador normalmente cria e configura uma sessão de servidor único para o aplicativo.</span><span class="sxs-lookup"><span data-stu-id="2ba95-129">Server Session: The controller process typically creates and configures a single server session for the application.</span></span> <span data-ttu-id="2ba95-130">A sessão de servidor mantém as propriedades de configuração para todo o aplicativo.</span><span class="sxs-lookup"><span data-stu-id="2ba95-130">The server session maintains the configuration properties for the entire application.</span></span> <span data-ttu-id="2ba95-131">Os grupos de URLs são criados na sessão do servidor pelo processo do controlador.</span><span class="sxs-lookup"><span data-stu-id="2ba95-131">URL groups are created under the server session by the controller process.</span></span>
