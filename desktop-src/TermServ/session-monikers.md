# <a name="session-monikers"></a>Monikers de sessão

O COM distribuído (DCOM) permite a ativação de objeto por sessão usando um moniker de sessão fornecido pelo sistema. Para obter mais informações sobre como criar e usar monikers de sessão, incluindo um exemplo de código que demonstra como ativar um processo de servidor local, consulte os tópicos a seguir.

## <a name="in-this-section"></a>Nesta seção

<dl> <dt>

[Ativação de sessão para sessão com um moniker de sessão](session-to-session-activation-with-a-session-moniker.md)
</dt> <dd>

A ativação de sessão para sessão (também chamada de ativação entre sessões) permite que um processo de cliente inicie (ative) um processo de servidor local em uma sessão especificada.

</dd> <dt>

[Usando um moniker de sessão](using-a-session-moniker.md)
</dt> <dd>

A ativação de sessão para sessão permite que um processo de cliente ative um processo de servidor local em uma sessão especificada.

</dd> </dl>
