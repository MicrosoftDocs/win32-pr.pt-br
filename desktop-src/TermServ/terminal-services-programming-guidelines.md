# <a name="remote-desktop-services-programming-guidelines"></a>Serviços de Área de Trabalho Remota diretrizes de programação

A maioria dos aplicativos baseados em Windows de 32 ou 64 bits existentes é executado "como está" em um ambiente de Serviços de Área de Trabalho Remota (anteriormente conhecido como Serviços de Terminal). No entanto, alguns aplicativos funcionam corretamente e têm um bom Serviços de Área de Trabalho Remota ambiente, enquanto outros não. Os tópicos a seguir fornecem diretrizes para o desenvolvimento de aplicativos em Serviços de Área de Trabalho Remota ambiente.

## <a name="in-this-section"></a>Nesta seção

<dl> <dt>

[Diretrizes de vários usuários](multiple-user-guidelines.md)
</dt> <dd>

Diretrizes para o desenvolvimento de aplicativos para vários usuários em Serviços de Área de Trabalho Remota ambiente.

</dd> <dt>

[Diretrizes de desempenho](performance-guidelines.md)
</dt> <dd>

Diretrizes para o desenvolvimento de aplicativos que são bem-Serviços de Área de Trabalho Remota ambiente.

</dd> <dt>

[Diretrizes gerais de programação](general-programming-guidelines.md)
</dt> <dd>

Diretrizes para o desenvolvimento de aplicativos em Serviços de Área de Trabalho Remota ambiente.

</dd> </dl>

Muitas dessas diretrizes são boas práticas de programação que beneficiarão os aplicativos em execução em qualquer Windows ambiente. No entanto, algumas otimizações recomendadas, como limitar efeitos gráficos, são otimizações que você deseja apenas quando seu aplicativo está em execução em Serviços de Área de Trabalho Remota. Para ver um exemplo de código que mostra como detectar um ambiente Serviços de Área de Trabalho Remota, consulte [Detecting the Serviços de Área de Trabalho Remota Environment](detecting-the-terminal-services-environment.md).
