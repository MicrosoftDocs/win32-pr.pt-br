# <a name="remote-desktop-sessions"></a><span data-ttu-id="f7cb9-103">Sessões de Área de Trabalho Remota</span><span class="sxs-lookup"><span data-stu-id="f7cb9-103">Remote Desktop Sessions</span></span>

<span data-ttu-id="f7cb9-104">Quando um usuário faz logon em um computador com Serviços de Área de Trabalho Remota habilitado, uma sessão é iniciada para o usuário.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-104">When a user logs on to a Remote Desktop Services–enabled computer, a session is started for the user.</span></span> <span data-ttu-id="f7cb9-105">Cada sessão é identificada por uma ID de sessão exclusiva.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-105">Each session is identified by a unique session ID.</span></span> <span data-ttu-id="f7cb9-106">Como cada logon em um cliente de Conexão de Área de Trabalho Remota (RDC) recebe uma ID de sessão separada, a experiência do usuário é semelhante a ser conectada a vários computadores ao mesmo tempo; por exemplo, um computador do escritório e um computador doméstico.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-106">Because each logon to a Remote Desktop Connection (RDC) client receives a separate session ID, the user-experience is similar to being logged on to multiple computers at the same time; for example, an office computer and a home computer.</span></span>

<span data-ttu-id="f7cb9-107">Cada sessão de área de trabalho remota é associada a uma estação de janela interativa.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-107">Each remote desktop session is associated with an interactive window station.</span></span> <span data-ttu-id="f7cb9-108">O único nome de estação de janela com suporte para uma estação de janela interativa é "WinSta0"; Portanto, cada sessão é associada à sua própria estação de janela "WinSta0".</span><span class="sxs-lookup"><span data-stu-id="f7cb9-108">The only supported window station name for an interactive window station is "WinSta0"; therefore each session is associated with its own "WinSta0" window station.</span></span> <span data-ttu-id="f7cb9-109">Há três áreas de trabalho padrão para cada estação de janela: a área de trabalho do Winlogon, a área de trabalho de proteção de tela e a área de trabalho interativa.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-109">There are three standard desktops for each window station: the Winlogon desktop, the screen saver desktop, and the interactive desktop.</span></span>

<span data-ttu-id="f7cb9-110">O usuário associado à estação de janela interativa para uma sessão é conhecido como *usuário interativo*.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-110">The user associated with the interactive window station for a session is known as the *interactive user*.</span></span> <span data-ttu-id="f7cb9-111">Em um cliente Conexão de Área de Trabalho Remota (RDC), pode haver vários usuários interativos além do usuário interativo no console do Serviços de Área de Trabalho Remota.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-111">On a Remote Desktop Connection (RDC) client there can be multiple interactive users in addition to the interactive user on the Remote Desktop Services console.</span></span> <span data-ttu-id="f7cb9-112">Para recuperar o identificador da sessão atualmente anexada ao console, use a função [**WTSGetActiveConsoleSessionId**](/windows/desktop/api/Winbase/nf-winbase-wtsgetactiveconsolesessionid) .</span><span class="sxs-lookup"><span data-stu-id="f7cb9-112">To retrieve the identifier of the session currently attached to the console, use the [**WTSGetActiveConsoleSessionId**](/windows/desktop/api/Winbase/nf-winbase-wtsgetactiveconsolesessionid) function.</span></span>

<span data-ttu-id="f7cb9-113">Quando um usuário faz logoff de um cliente de Conexão de Área de Trabalho Remota (RDC), a sessão que o cliente tem no servidor Host da Sessão da Área de Trabalho Remota (Host da Sessão RD) (anteriormente conhecido como um servidor de terminal) é excluída e as estações de janela e áreas de trabalho associadas a essa sessão são removidas.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-113">When a user logs off from a Remote Desktop Connection (RDC) client, the session that the client has on the Remote Desktop Session Host (RD Session Host) server (formerly known as a terminal server) is deleted and the window stations and desktops associated with that session are removed.</span></span> <span data-ttu-id="f7cb9-114">No entanto, como a sessão de console do Serviços de Área de Trabalho Remota nunca é excluída, as estações de janela associadas à sessão do console não são excluídas.</span><span class="sxs-lookup"><span data-stu-id="f7cb9-114">However, because the Remote Desktop Services console session is never deleted, the window stations associated with the console session are not deleted.</span></span> <span data-ttu-id="f7cb9-115">Isso afeta a forma como os aplicativos se comportam em um ambiente de Serviços de Área de Trabalho Remota quando são configurados para execução no contexto de segurança do usuário interativo, também conhecido como o modo de ativação de objeto "RunAs Interactive User".</span><span class="sxs-lookup"><span data-stu-id="f7cb9-115">This affects how applications behave in a Remote Desktop Services environment when they are configured to run in the security context of the interactive user, also known as the "RunAs Interactive User" object activation mode.</span></span>

<span data-ttu-id="f7cb9-116">Para obter mais informações sobre como iniciar um processo de servidor local em uma sessão especificada, consulte [ativação de sessão para sessão com um moniker de sessão](session-to-session-activation-with-a-session-moniker.md) e [usando um moniker de sessão](using-a-session-moniker.md).</span><span class="sxs-lookup"><span data-stu-id="f7cb9-116">For more information about starting a local server process on a specified session, see [Session-to-Session Activation with a Session Moniker](session-to-session-activation-with-a-session-moniker.md) and [Using a Session Moniker](using-a-session-moniker.md).</span></span> <span data-ttu-id="f7cb9-117">Para obter mais informações sobre contextos de segurança, consulte [o contexto de segurança do cliente](/windows/desktop/SecAuthZ/the-client-security-context).</span><span class="sxs-lookup"><span data-stu-id="f7cb9-117">For more information about security contexts, see [The Client's Security Context](/windows/desktop/SecAuthZ/the-client-security-context).</span></span>
