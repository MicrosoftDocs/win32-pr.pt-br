# <a name="terminal-services-is-now-remote-desktop-services"></a>Os serviços de terminal agora estão Serviços de Área de Trabalho Remota

Os serviços de terminal foram renomeados Serviços de Área de Trabalho Remota.

No Windows Server 2008 R2, todos os serviços de função Serviços de Área de Trabalho Remota foram renomeados. A tabela a seguir lista o nome antigo e o novo nome de cada Serviços de Área de Trabalho Remota serviço de função.



| Nome anterior                                                   | Nome no Windows Server 2008 R2                                         |
|-----------------------------------------------------------------|------------------------------------------------------------------------|
| Serviços de Terminal <br/>                                   | Serviços da Área de Trabalho Remota<br/>                                     |
| Terminal Server<br/>                                      | Host de Sessão de Área de Trabalho Remota (Host de Sessão RD)<br/>               |
| Licenciamento de serviços de terminal (Licenciamento TS)<br/>          | Licenciamento de Área de Trabalho Remota (Licenciamento RD)<br/>                     |
| Gateway de Serviços de Terminal (Gateway TS)<br/>               | Gateway de Área de Trabalho Remota (Gateway RD)<br/>                         |
| Agente de Sessão dos Serviços de Terminal (Agente de Sessão TS)<br/> | Agente de Conexão de Área de Trabalho Remota (agente de conexão RD)<br/>     |
| Acesso via Web dos Serviços de Terminal (Acesso via Web TS)<br/>         | Acesso via Web à Área de Trabalho Remota (RD Acesso via Web)<br/>                  |
| Virtualização de serviços de terminal<br/>                     | Host de Virtualização de Área de Trabalho Remota (Host de Virtualização RD)<br/> |



 

Além disso, observe que um servidor com o serviço de função Host da Sessão RD instalado agora é chamado de servidor Host da Sessão RD, em vez de um servidor de terminal.
