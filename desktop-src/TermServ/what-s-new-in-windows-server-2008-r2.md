# <a name="whats-new-in-windows-server-2008-r2"></a>O que há de novo no Windows Server 2008 R2

O Windows Server 2008 R2 apresenta os seguintes novos elementos de programação para Serviços de Área de Trabalho Remota.



<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<tbody>
<tr class="odd">
<td>Os serviços de terminal agora estão Serviços de Área de Trabalho Remota<br/></td>
<td>Os serviços de terminal foram renomeados Serviços de Área de Trabalho Remota. Um servidor com o serviço de função Host da Sessão da Área de Trabalho Remota (Host da Sessão RD) instalado agora é chamado de servidor Host da Sessão RD, em vez de um servidor de terminal.<br/>
<ul>
<li><a href="terminal-services-is-now-remote-desktop-services.md">Os serviços de terminal agora estão Serviços de Área de Trabalho Remota</a></li>
</ul></td>
</tr>
<tr class="even">
<td>API de virtualização Área de Trabalho Remota<br/></td>
<td>A API de virtualização Área de Trabalho Remota fornece enumerações, interfaces e estruturas que você pode usar para criar plug-ins personalizados que substituem a lógica de redirecionamento padrão do Conexão de Área de Trabalho Remota Broker (agente de conexão RD). O agente de conexão RD (anteriormente conhecido como agente de Sessão TS) foi expandido para dar suporte a conexões com máquinas virtuais.<br/>
<ul>
<li><a href="using-the-remote-desktop-virtualization-api.md">Usando a API de virtualização Área de Trabalho Remota</a></li>
<li><a href="terminal-services-virtualization-api-reference.md">Referência de API de virtualização Área de Trabalho Remota</a></li>
</ul></td>
</tr>
<tr class="odd">
<td>Provedor de protocolo RDP<br/></td>
<td>A API do provedor de protocolo RDP pode ser usada para criar um protocolo que personaliza a interação entre o serviço Serviços de Área de Trabalho Remota e os clientes da área de trabalho.<br/>
<ul>
<li><a href="creating-a-custom-remote-protocol.md">Criando um provedor de protocolo RDP</a></li>
<li><a href="custom-remote-protocol-reference.md">Referência do provedor de protocolo RDP</a></li>
</ul></td>
</tr>
<tr class="even">
<td>API do AudioEndpoint Serviços de Área de Trabalho Remota<br/></td>
<td>O Serviços de Área de Trabalho Remota API AudioEndpoint dá suporte à enumeração, interfaces e estruturas para registro de ponto de extremidade de áudio e transporte de dados.<br/>
<ul>
<li><a href="terminal-services-audioendpoint-api-reference.md">Serviços de Área de Trabalho Remota referência da API AudioEndpoint</a></li>
</ul></td>
</tr>
<tr class="odd">
<td>API de Serviço Gerenciamento de Conexão de RemoteApp e Área de Trabalho<br/></td>
<td>A API do Serviço Gerenciamento de Conexão de RemoteApp e Área de Trabalho dá suporte a interfaces e estruturas que você pode usar para executar a filtragem personalizada de recursos e fornecer suporte para tipos de arquivos que não têm suporte nativo no Windows Server 2008 R2.<br/>
<ul>
<li><a href="centralized-publishing-api-reference.md">Referência da API do Serviço Gerenciamento de Conexão de RemoteApp e Área de Trabalho</a></li>
</ul></td>
</tr>
</tbody>
</table>
