# <a name="peer-to-peer"></a>Ponto a ponto

## <a name="purpose"></a>Finalidade

Tecnologias ponto a ponto são usadas para facilitar a comunicação e a colaboração em tempo real em redes distribuídas.

No modelo ponto a ponto, sem usar servidores da Internet, cada usuário do PC pode fazer o seguinte:

-   Dados do Exchange
-   Compartilhar recursos
-   Localizar outros usuários
-   Transmiti
-   Colabore diretamente em tempo real

Usando tecnologias ponto a ponto, os aplicativos que coordenam o uso de ciclos de CPU do computador e do armazenamento podem compartilhar recursos entre grupos pequenos ou grandes de computadores conectados à Internet.

## <a name="where-applicable"></a>Quando aplicável

Os desenvolvedores podem usar a infraestrutura de pares para criar uma ampla variedade de aplicativos distribuídos, ad hoc e ponto a ponto.

## <a name="developer-audience"></a>Público de desenvolvedores

Os desenvolvedores que usam a infraestrutura de pares devem estar familiarizados com os conceitos de programação de C. Os desenvolvedores que usam o provedor de namespace do Winsock do PNRP devem estar familiarizados com a API do Winsock.

## <a name="run-time-requirements"></a>Requisitos de tempo de execução

A infraestrutura par tem suporte no Windows Vista, no Windows XP com Service Pack 2 (SP2) e posterior, bem como no Advanced Networking Pack para Windows XP disponível para o Windows XP com Service Pack 1 (SP1). A infraestrutura ponto a ponto requer que o IPv6 seja instalado e iniciado para permitir que os aplicativos de rede de mesmo nível funcionem. O uso de colaboração ponto a ponto só tem suporte no Windows Vista.
