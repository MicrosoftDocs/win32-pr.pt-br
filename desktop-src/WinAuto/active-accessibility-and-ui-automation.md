# <a name="active-accessibility-and-ui-automation"></a>Acessibilidade Ativa e Automação da Interface do Usuário

Microsoft Active Accessibility é projetado para uso com Windows XP e sistemas operacionais anteriores. Os desenvolvedores de controles personalizados e aplicativos cliente de tecnologia acessíveis para Windows XP e Windows Vista devem considerar o uso do Microsoft [Automação da Interface do Usuário.](entry-uiauto-win32.md)

O Microsoft Automação da Interface do Usuário é um sistema completo que fornece informações mais precisas sobre a interface do usuário e oferece ao usuário mais capacidade de interagir com controles. Em particular, ele fornece suporte bastante aprimorado para texto.

Um conjunto de objetos de proxy Automação da Interface do Usuário suporte para controles padrão do Microsoft Win32 e Windows Forms. O suporte mais rico está disponível para controles especificamente escritos com Automação da Interface do Usuário em mente, incluindo elementos nativos do WPF (Windows Presentation Foundation), que não são diretamente Microsoft Active Accessibility. Controles personalizados que Automação da Interface do Usuário podem ser gravados em código gerenciado ou não gerenciado.

Microsoft Active Accessibility clientes têm acesso limitado, por meio de uma camada de ponte, a elementos de interface do usuário que só suportam Automação da Interface do Usuário. Para obter mais informações, consulte [Apêndice G: Acessibilidade Ativa ponte para Automação da Interface do Usuário](appendix-g--active-accessibility-bridge-to-ui-automation.md).

Para obter mais informações sobre as semelhanças e diferenças entre Microsoft Active Accessibility e Automação da Interface do Usuário, consulte [Microsoft Active Accessibility e Automação da Interface do Usuário comparados.](microsoft-active-accessibility-and-ui-automation-compared.md)
