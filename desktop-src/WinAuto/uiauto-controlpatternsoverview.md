# <a name="ui-automation-control-patterns-overview"></a><span data-ttu-id="48408-114">Visão Geral de Padrões de Controle de Automação de Interface de Usuário</span><span class="sxs-lookup"><span data-stu-id="48408-114">UI Automation Control Patterns Overview</span></span>

<span data-ttu-id="48408-115">Um *padrão de controle* é uma implementação de interface que expõe um aspecto específico da funcionalidade de um controle para aplicativos cliente de automação da interface do usuário da Microsoft.</span><span class="sxs-lookup"><span data-stu-id="48408-115">A *control pattern* is an interface implementation that exposes a particular aspect of a control's functionality to Microsoft UI Automation client applications.</span></span> <span data-ttu-id="48408-116">Os clientes usam as propriedades e os métodos expostos por meio de um padrão de controle para recuperar informações sobre um recurso específico do controle ou para manipular um aspecto específico do comportamento do controle.</span><span class="sxs-lookup"><span data-stu-id="48408-116">Clients use the properties and methods exposed through a control pattern to retrieve information about a particular capability of the control, or to manipulate a particular aspect of the control's behavior.</span></span> <span data-ttu-id="48408-117">Por exemplo, um controle que apresenta uma interface de tabela usa o padrão de controle de [grade](uiauto-implementinggrid.md) para expor o número de linhas e colunas na tabela, e para permitir que um cliente recupere itens da tabela.</span><span class="sxs-lookup"><span data-stu-id="48408-117">For example, a control that presents a tabular interface uses the [Grid](uiauto-implementinggrid.md) control pattern to expose the number of rows and columns in the table, and to enable a client to retrieve items from the table.</span></span>

<span data-ttu-id="48408-118">A automação da interface do usuário usa padrões de controle para representar comportamentos comuns de controle.</span><span class="sxs-lookup"><span data-stu-id="48408-118">UI Automation uses control patterns to represent common control behaviors.</span></span> <span data-ttu-id="48408-119">Por exemplo, você usa o padrão de controle [Invoke](uiauto-implementinginvoke.md) para controles que podem ser invocados, como botões e o padrão de controle [Scroll](uiauto-implementingscroll.md) para controles que têm barras de rolagem, como caixas de listagem, exibições de lista ou caixas de combinação.</span><span class="sxs-lookup"><span data-stu-id="48408-119">For example, you use the [Invoke](uiauto-implementinginvoke.md) control pattern for controls that can be invoked, such as buttons, and the [Scroll](uiauto-implementingscroll.md) control pattern for controls that have scroll bars, such as list boxes, list views, or combo boxes.</span></span> <span data-ttu-id="48408-120">Como cada padrão de controle representa uma funcionalidade separada, os padrões de controle podem ser combinados para descrever o conjunto completo de funcionalidades com suporte de um controle específico.</span><span class="sxs-lookup"><span data-stu-id="48408-120">Because each control pattern represents a separate functionality, control patterns can be combined to describe the full set of functionality supported by a particular control.</span></span>

> [!Note]  
> <span data-ttu-id="48408-121">Um controle de agregação é criado com controles filho que fornecem a interface do usuário para a funcionalidade que é exposta pelo pai, e o pai deve implementar todos os padrões de controle que são normalmente associados a seus controles filho.</span><span class="sxs-lookup"><span data-stu-id="48408-121">An aggregate control is built with child controls that provide the user interface for functionality that is exposed by the parent, and the parent should implement all control patterns that are typically associated with its child controls.</span></span> <span data-ttu-id="48408-122">Por sua vez, os mesmos padrões de controle não precisam ser implementados pelos controles filho.</span><span class="sxs-lookup"><span data-stu-id="48408-122">In turn, those same control patterns are not required to be implemented by the child controls.</span></span>
