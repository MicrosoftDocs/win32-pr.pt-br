# <a name="ui-automation-for-w3c-accessible-rich-internet-applications-specification"></a>Automação da Interface do Usuário especificação de aplicativos de Internet rich do W3C

Os sites aumentam seu utilitário com conteúdo dinâmico e controles avançados de interface do usuário com suporte por tecnologias como JavaScript assíncrono e XML (AJAX), DHTML e JavaScript. No entanto, as tecnologias adaptativas frequentemente não podem interagir com esses controles complexos nem expor conteúdo dinâmico aos usuários. Aplicativos de Internet Rich (ARIA) acessíveis é uma especificação técnica do W3C para desenvolver conteúdo da Web e aplicativos para que eles sejam acessíveis a pessoas com deficiências.

Para dar suporte à especificação ARIA, Automação da Interface do Usuário especificação do Microsoft Automação da Interface do Usuário permite que os desenvolvedores associem atributos Automação da Interface do Usuário AriaRole e AriaProperties a funções e estados do W3C ARIA ou propriedades. Isso ajuda os aplicativos de usuário, como o Microsoft Internet Explorer, a dar suporte ao modelo de objeto ARIA no contexto do Automação da Interface do Usuário, mantendo um modelo de objeto de acessibilidade de linha de base.

Este apêndice fornece o mapeamento sugerido de informações do W3C para Microsoft Active Accessibility especificações Automação da Interface do Usuário, aproveitando as duas novas Automação da Interface do Usuário propriedades.

Este tópico contém as seguintes seções:

-   [Função ARIA do W3C mapeada para Microsoft Active Accessibility e Automação da Interface do Usuário](#w3c-aria-role-mapped-to-microsoft-active-accessibility-and-ui-automation)
-   [Propriedades e estados do ARIA do W3C mapeados para Microsoft Active Accessibility e Automação da Interface do Usuário](#w3c-aria-states-and-properties-mapped-to-microsoft-active-accessibility-and-ui-automation)
-   [Considerações adicionais](#additional-considerations)

## <a name="w3c-aria-role-mapped-to-microsoft-active-accessibility-and-ui-automation"></a>Função ARIA do W3C mapeada para Microsoft Active Accessibility e Automação da Interface do Usuário

As funções do ARIA do W3C podem mapear para Microsoft Active Accessibility funções ou Automação da Interface do Usuário de controle. Com Automação da Interface do Usuário, a função original do ARIA do W3C também pode ser suportada pela propriedade AriaRole. Os agentes de usuário podem oferecer uma descrição localizada da função ARIA do W3C com a propriedade Automação da Interface do Usuário LocalizedControlType; no entanto, isso é opcional. Quando uma cadeia de caracteres localizada não for especificada, o sistema fornecerá a cadeia de caracteres LocalizedControlType padrão. Os agentes de usuário também podem oferecer funções secundárias na propriedade AriaRole usando o espaço como separador, conforme definido no padrão ARIA W3C.

## <a name="w3c-aria-states-and-properties-mapped-to-microsoft-active-accessibility-and-ui-automation"></a>Propriedades e estados do ARIA do W3C mapeados para Microsoft Active Accessibility e Automação da Interface do Usuário

Os estados e propriedades do ARIA do W3C são mapeados para várias Microsoft Active Accessibility e Automação da Interface do Usuário propriedades e funções. A maioria dos estados e propriedades do ARIA é suportada pela propriedade Automação da Interface do Usuário AriaProperties. As exceções são propriedades ARIA que têm referências de objeto (por exemplo, a propriedade ARIA descrita por ) e propriedades ARIA com suporte pelo modelo de objeto de acessibilidade. Por exemplo, espera-se que a propriedade ARIA activedescendent seja representada como o elemento com foco no modelo de objeto de acessibilidade. A propriedade ARIA owns também deve ser refletida no modelo de objeto de acessibilidade quando for especificada.

AriaProperties é uma propriedade de cadeia de caracteres que tem uma coleção de pares de nome/valor cujos delimitadores são o sinal de igual (=) e o ponto e vírgula (;), por exemplo, "checked=true;disabled=false". Uma faixa invertida ( ) é usada como um escape quando esses caracteres delimiter, ou uma faixa \\ invertida, aparecem nos valores. O código do provedor pode adicionar verificação para validar as propriedades originais do ARIA no Modelo de Objeto do Documento, mas não é necessário.

## <a name="additional-considerations"></a>Considerações adicionais

A automação da interface do usuário também oferece um modelo de objeto de texto simples com o padrão de controle de [texto](uiauto-implementingtextandtextrange.md) , que dá suporte à inserção de objetos em um objeto de documento. Isso permite que os agentes de usuários e aplicativos cliente tratem o conteúdo da Web como um documento HTML ou como uma interface do usuário de área de trabalho tradicional, dependendo dos cenários de usuários finais.
