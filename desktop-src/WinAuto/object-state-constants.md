# <a name="object-state-constants"></a>Constantes de estado do objeto

Este tópico descreve os valores constantes usados para descrever o estado dos objetos em uma interface do usuário do aplicativo. As constantes de estado são definidas em OleAcc. h.

Um objeto é associado a um ou mais desses valores de estado a qualquer momento. As seguintes constantes de estado do objeto não são usadas: alerta do sistema \_ \_ de estado \_ alto, alerta do sistema de estado \_ \_ \_ médio, alerta do sistema de estado \_ \_ \_ baixo e sistema de estado \_ \_ flutuante.

Os clientes recuperam o estado de um objeto chamando [**IAccessible:: get \_ accState**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-get_accstate), que retorna um inteiro que é uma combinação dos seguintes sinalizadores de bits. Os clientes chamam [**GetStateText**](/windows/desktop/api/Oleacc/nf-oleacc-getstatetexta) com o valor de estado para recuperar uma cadeia de caracteres localizada que descreve o estado do objeto.

Quando o estado de um objeto é alterado, os servidores devem chamar [**NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) com a constante de evento [**STATECHANGE do \_ objeto \_ de evento**](event-constants.md) . No entanto, os objetos com o sistema de estado \_ \_ invisível, estado do \_ sistema de Estados e constantes do estado do \_ \_ \_ objeto selecionado do sistema de Estado têm suas próprias constantes de evento. Para esses objetos, não use o **objeto de evento \_ \_ STATECHANGE**. Em vez disso, use a constante de evento individual.
